## Суть
n8n — платформа для создания автоматизаций и интеграций без кода. Видео объясняет основные концепции платформы: как работают ноды, триггеры, обработка данных и создание первого рабочего workflow для автоматизации бизнес-процессов.

## Бизнес-сценарий
Новички в автоматизации, которые хотят научиться создавать workflows в n8n для интеграции различных приложений (CRM, email, чаты) и автоматизации повторяющихся задач без написания кода.

## Алгоритм реализации
1. 1. Открыть n8n и создать новый workflow (File → New Workflow или кнопка +)
2. 2. Выбрать триггер для запуска автоматизации (Webhook, Schedule, Event trigger из раздела Trigger nodes)
3. 3. Добавить обработку данных: использовать Set node для преобразования данных, Function node для кастомной логики
4. 4. Подключить интеграции: выбрать нужный сервис (Gmail, Slack, Bitrix24 и т.д.) и настроить аутентификацию
5. 5. Запустить тест workflow кнопкой Execute Workflow, проверить результаты в панели Output
6. 6. Активировать workflow переключателем в верхнем меню (переводит из режима разработки в production)
7. 7. Дополнить workflow условиями (IF node) для разветвления логики в зависимости от данных
8. 8. Добавить уведомления или отправку данных на следующий шаг через Output nodes

## Связки инструментов
- Webhook → n8n триггер → обработка данных в Set/Function → интеграция (Gmail/Slack) → результат
- Schedule trigger → периодическая обработка → запрос данных по API → преобразование → отправка в целевой сервис
- Форма/вебхук → n8n → условная логика (IF) → несколько веток обработки → разные интеграции

## Конфигурация и параметры
- Основные элементы интерфейса n8n: левая панель с нодами (поиск по названию), центр — canvas для построения workflow, справа — параметры текущей ноды
- Триггеры находятся в разделе Trigger nodes (Schedule для периодических запусков, Webhook для событий, Email trigger для входящих писем)
- После добавления ноды нужно нажать на её значок для открытия параметров и заполнить необходимые поля
- Для многих интеграций требуется авторизация: нажать на кнопку параметра → выбрать Create New Credential → ввести API ключ или логин-пароль
- Режимы тестирования: Execute Workflow (одноразовый запуск) vs. Active режим (постоянное слушание триггера)
- Для просмотра данных между нодами: клик на линию связи покажет структуру JSON
- Экспортирование workflow: Menu → Export (сохранить в JSON для переноса на другой сервер)

## Ключевые инсайты
- n8n работает на основе JSON: все данные между нодами передаются в виде структурированного JSON, что позволяет легко трансформировать данные
- Ноды выполняются последовательно слева направо, каждая может работать с массивом объектов или одиночным объектом
- Set node используется для явного преобразования и маппинга полей, Function node — для сложной логики на JavaScript
- Webhook в n8n автоматически генерирует уникальный URL, который можно использовать как точку входа для внешних сервисов
- Условная логика (IF node) разветвляет workflow на две ветки (true/false) без выполнения лишних действий
- При ошибке в workflow: проверить JSON данные между нодами через Output панель, не запускать активный режим до тестирования
- Рекурсивные операции (Loop node) позволяют обработать множество элементов без отправки множества webhook запросов
- API ключи и пароли хранятся в Credentials (не видны в экспорте workflow) — использовать для безопасности
- n8n самостоятельный сервис, может быть установлен на собственный сервер (self-hosted) или использован облачный вариант
- Стоимость cloud n8n зависит от количества workflow executions в месяц, бесплатный план позволяет до 1000 executions

## Подводные камни
- Частая ошибка: забыть активировать workflow (он остаётся в режиме Edit, а не Active) — триггер не срабатывает
- При работе с Webhook: первый запрос нужно отправить ДО активации workflow иначе webhook не сможет распознать формат данных
- JSON маппинг чувствителен к регистру и синтаксису: 'email' != 'Email', неправильное имя поля приведёт к null
- Асинхронные операции в Function node требуют использования async/await, иначе workflow завершится раньше получения результата
- Limit в Loop node по умолчанию 10 итераций — нужно явно установить нужное значение при обработке больших списков
- Если workflow работает медленно: проверить количество API запросов, кэширование данных и использование batch операций
- При копировании workflow в другой account: все Credentials нужно пересоздать, они не экспортируются
- Webhook не должен содержать кириллицу в URL — использовать латиницу и дефисы для названий
- Таймауты по умолчанию 3600 сек — для долгих операций нужно увеличить в параметрах ноды
