## 💡 Что это даёт бизнесу
ИИ-агент принимает заявки на перевозку (звонок, мессенджер, email), автоматически подбирает транспорт и водителя, ведёт переписку с клиентом и подрядчиком без участия человека. Компания экономит 1–3 ставки диспетчера (от 80 000 руб./мес) и ускоряет обработку заявок с часов до минут. Снижается человеческий фактор: потерянные заявки, забытые звонки, ошибки в координации.

---

## ✅ Плюсы
- Острая боль: диспетчеры — дефицитный и дорогой персонал, текучка высокая
- Быстрый ROI: окупаемость за 1–2 месяца при замене даже одного сотрудника
- Масштабируемость: один агент обрабатывает 10x заявок без роста ФОТ

## ❌ Минусы / Риски
- Высокая вариативность бизнес-логики у каждого клиента — потребуется кастомизация под каждый проект
- Интеграция с 1С/TMS/парком транспорта может сильно усложнить разработку и удлинить проект

## 🧰 Артефакты для реализации

**Инструменты:** n8n (оркестрация агента), Bitrix24 / amoCRM (хранение заявок и клиентов), Voiceflow или Vapi (голосовой агент), Telegram Bot API (приём заявок)
**API:** OpenAI API / Claude API (обработка логики и переписки), Vapi.ai API (голосовые звонки), Google Maps API (маршруты и расчёт расстояний), WhatsApp Business API / Telegram Bot API

## 🚀 Шаги запуска
1. Шаг 1: Кастдев — интервью с 3–5 транспортными/логистическими компаниями, фиксация типовых сценариев заявок и болей диспетчера
2. Шаг 2: MVP — агент принимает заявку через Telegram/WhatsApp, парсит детали (откуда, куда, груз, дата), сохраняет в CRM и отвечает клиенту
3. Шаг 3: Расширение — подключение голосового агента (Vapi), логика подбора транспорта/водителя из базы, уведомления исполнителям
4. Шаг 4: Упаковка в тиражируемый продукт с шаблонами n8n и инструкцией по адаптации под клиента
