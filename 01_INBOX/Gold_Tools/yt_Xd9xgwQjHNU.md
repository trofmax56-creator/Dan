## Суть
Обучающий курс по разработке AI-агентов с использованием инструментов Google (Gemini API, Google AI). Курс фокусируется на "vibe coding" подходе — интуитивной разработке с использованием генеративного ИИ для ускорения создания автоматизированных решений и агентов обработки данных.

## Бизнес-сценарий
Разработчики, автоматизаторы и предприниматели изучают создание AI-агентов через практические задачи. Курс обучает интеграции Google AI (Gemini) в рабочие процессы для автоматизации бизнес-задач, обработки данных и создания самостоятельных систем с применением prompt engineering и API.

## Алгоритм реализации
1. 1. Вводная часть: изучение концепции 'vibe coding' и отличий от традиционного программирования. Понимание что такое AI-агенты и как они работают
2. 2. Установка и настройка Google AI/Gemini API. Получение API ключей, настройка окружения разработки, подключение к Google AI Studio
3. 3. Основы работы с Gemini API: создание первого запроса к модели, понимание параметров температуры, max_tokens, система промптов
4. 4. Разработка простых AI-агентов: создание функций для обработки пользовательского ввода, интеграция с различными сервисами (API, базы данных)
5. 5. Продвинутые техники: создание многошаговых рабочих процессов, цепочки вызовов функций (function calling), обработка состояния агента
6. 6. Интеграция с внешними сервисами: подключение к CRM, email сервисам, базам данных через API
7. 7. Тестирование и отладка: проверка корректности работы агентов, обработка ошибок, логирование
8. 8. Практический проект: создание полнофункционального AI-агента для решения реальной бизнес-задачи
9. 9. Деплой и производство: развёртывание агентов в боевой окружение, масштабирование, мониторинг

## Технический стек
- Google Gemini API (LLM модель)
- Google AI Studio / Google AI SDK
- Python 3.8+ (основной язык разработки)
- REST API для интеграции сервисов
- Google Cloud Platform (GCP) для хостинга
- Node.js (альтернатива для JavaScript разработчиков)
- Function calling механизм Gemini
- Prompt Engineering для оптимизации запросов
- JSON для структурирования данных
- HTTP/HTTPS для сетевых запросов
- Webhook для получения событий
- Docker для контейнеризации

## Связки инструментов
- Webhook/Event → Google Cloud Functions → Gemini API
- User Input → Gemini API (prompt) → Function Calling → External API → Response
- Database → Gemini API (RAG контекст) → AI Agent → CRM/Service Integration
- Real-time Events → Google Pub/Sub → AI Agent → Multi-step Workflow
- Frontend/Client → REST API (Flask/FastAPI) → Gemini API → Backend Services

## Конфигурация и параметры
- API ключ Google Gemini: получение из Google AI Studio (можно использовать бесплатный API ключ для обучения)
- Модель: использование Gemini 1.5 Pro или Gemini 2.0 (последние версии с лучшей поддержкой функций)
- Параметры запроса: temperature (0-1, обычно 0.7-0.9 для vibe coding), max_tokens (зависит от задачи), top_p
- System Prompt: определение роли и поведения AI-агента в начале сессии
- Function Calling Schema: JSON описание функций которые может вызывать агент (name, description, parameters)
- Safety Settings: настройка фильтров контента (BLOCK_NONE для обучения, BLOCK_SOME для production)
- Timeout: установка лимита времени на ответ (обычно 30-60 секунд)
- Retry Policy: количество повторных попыток при ошибке API (обычно 3-5)
- Rate Limiting: лимит запросов в минуту (бесплатный тариф обычно 60 RPM)
- Context Window: размер исторических данных передаваемых в модель (Gemini поддерживает 1М+ токенов)
