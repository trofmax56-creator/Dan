## 1. Название
**AI-Аналитик** — Читатель графиков и таблиц из CRM

## 2. Суть
Claude получает скриншоты дашбордов, выгрузки из 1С/МойСклад или фото таблиц, распознаёт цифры, находит аномалии и отправляет руководителю еженедельный текстовый разбор «что случилось и что делать» — без аналитика в штате.

## 3. Бизнес-сценарий
ДО: Директор розничной сети Михаил каждый понедельник просит бухгалтера «сделать сводку по продажам». Бухгалтер тратит 2 часа, делает Excel с цифрами без выводов. Михаил смотрит на таблицу и не понимает почему в пятницу провал — звонит управляющему, тот не знает. Решение принимается через 3 дня. → ИИ делает: n8n в 8:00 понедельника автоматически выгружает отчёт из МойСклад за неделю (JSON с данными по выручке, товарам, точкам), передаёт в Claude. Claude находит: «Выручка -23% vs прошлая неделя. Причина: товар-локомотив "Кофе Arabica 1кг" отсутствовал на складе с четверга (0 продаж пт-вс = -41 000 руб.). Рекомендация: установить минимальный остаток 50 ед. с авто-заказом поставщику.» → Клиент получает: Михаил читает в Telegram готовый вывод в 8:05. Бухгалтер не тратит 2 часа. Проблема обнаружена и решение предложено до открытия магазина.

## 4. Логика
```
n8n cron (пн 8:00) → выгрузка МойСклад/1С API → Claude анализирует данные + находит аномалии → текстовый дайджест → Telegram руководителю
```

## 5. Технический стек
- n8n
- claude-sonnet-4-6
- МойСклад API
- 1С (REST или выгрузка)
- Telegram Bot API

## 6. Связки инструментов
```
n8n Cron (8:00 пн) → МойСклад REST API (выгрузка продаж за 7 дней) → Claude API (анализ аномалий, сравнение с предыдущим периодом) → Telegram Bot (отправка дайджеста директору)
```

## 7. Алгоритм реализации
1. Настроить n8n Cron на понедельник 8:00; HTTP-запрос к МойСклад REST API: выгрузить продажи по товарам, точкам, менеджерам за 7 и 14 дней (для сравнения) в JSON
2. Если данные из 1С — настроить выгрузку через REST-интерфейс 1С или регулярный экспорт CSV в папку на сервере с чтением n8n; изображения/скриншоты — передать в Claude vision через base64
3. Передать оба JSON (текущая и прошлая неделя) в Claude с промптом: найди топ-3 аномалии (падение >15%, новые лидеры, нулевые продажи), сформулируй причину и одно конкретное действие для каждой — ответ в формате Telegram-сообщения без таблиц
4. n8n отправляет готовый текст директору в Telegram; дополнительно — отдельное сообщение в чат команды если найдена критическая аномалия (падение >30% или складской ноль по топ-10 товарам)
5. Логировать все аномалии в Google Sheets для накопления истории; раз в месяц Claude делает сравнительный разбор трендов за 4 недели

## 8. Почему купят (ROI)
Боль: руководитель МСБ принимает решения на основе устаревших данных — бухгалтер готовит отчёт 2–4 часа, за это время теряются продажи. Аналитик в штате — 80 000–120 000 руб./мес ФОТ, и всё равно не работает в 8 утра в понедельник. Один пропущенный «стоп-продажа» популярного товара на выходных = 30 000–80 000 руб. упущенной выручки. Крючок: «Данные без вывода — это шум. AI-Аналитик превращает цифры в решение до того, как вы выпили кофе.»
