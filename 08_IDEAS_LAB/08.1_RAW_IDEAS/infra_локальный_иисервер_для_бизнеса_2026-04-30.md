# 🗂️ Паспорт идеи: Локальный ИИ-сервер для бизнеса

**Источник:** [host ALL your AI locally](https://youtube.com/watch?v=Wjrdr0NU4Sk) — NetworkChuck (2024-05-03)
**Категория:** Инфраструктура
**Заменяет:** —

---

## 💡 Что это даёт бизнесу
Клиент получает собственный ИИ-сервер (Ollama + открытые модели) без утечки данных в облако и без абонентской платы OpenAI. Все корпоративные данные остаются внутри периметра компании, что критично для юридических, медицинских и финансовых компаний. Студия настраивает, интегрирует с CRM и обучает команду — разово.

---

## ✅ Плюсы
- Нет ежемесячных платежей за API — окупается за 3-6 месяцев vs OpenAI/Claude
- Полное соответствие требованиям 152-ФЗ и корпоративной ИБ-политике
- Можно дообучать модель на данных клиента (fine-tuning) без передачи данных третьим лицам

## ❌ Минусы / Риски
- Требует покупки сервера с GPU (от 150 000 руб.) — высокий порог входа для малого бизнеса
- Открытые модели (Llama 3, Mistral) заметно уступают GPT-4o по качеству — клиент может разочароваться

---

## 💰 Экономика

| Параметр | Значение |
|---|---|
| Стоимость разработки | 60 000 – 120 000 руб. (настройка Ollama, интеграция с n8n/Bitrix24, UI через Open WebUI) |
| Инфраструктура/мес | 0 – 5 000 руб./мес (электричество + аренда сервера или собственное железо) |
| Поддержка/мес | 10 000 – 20 000 руб./мес (обновление моделей, мониторинг, доработки) |
| **Цена для клиента** | **150 000 – 280 000 руб. (под ключ: сервер + настройка + интеграция + обучение)** |
| Потенциал выручки Студии | 3-5 клиентов × 200 000 руб. = 600 000 – 1 000 000 руб. за 6 мес + рекуррентная поддержка 50 000 – 100 000 руб./мес |

## 🧰 Артефакты для реализации

**Инструменты:** Ollama, Open WebUI, n8n, LM Studio, Docker
**API:** Ollama REST API (совместим с OpenAI API формат), Hugging Face Hub API (загрузка моделей)

**Репозитории:**
- ollama/ollama (github.com/ollama/ollama)
- open-webui/open-webui (github.com/open-webui/open-webui)

**Документация:**
- ollama.com/docs
- docs.openwebui.com

## 🚀 Шаги запуска
1. Шаг 1: Аудит клиента — объём данных, чувствительность, текущие расходы на OpenAI/облако, наличие сервера или бюджет на железо
2. Шаг 2: Подбор и развёртывание модели (Llama 3.1 8B/70B, Mistral, Qwen2.5) через Ollama на сервере клиента или арендованном VPS с GPU
3. Шаг 3: Интеграция с рабочими инструментами клиента — n8n-воркфлоу, Bitrix24 через вебхуки, Open WebUI как корпоративный чат
4. Шаг 4: Передача, обучение сотрудников, SLA-договор на поддержку и обновление моделей
