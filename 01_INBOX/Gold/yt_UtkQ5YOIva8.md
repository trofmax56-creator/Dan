## Суть
Построение полнофункциональной системы автоматизированных AI-звонков потенциальным клиентам (лидам) с помощью n8n. Система позволяет запустить бота для автоматического холодного обзвона с целью генерации продаж без прямого участия человека.

## Бизнес-сценарий
SaaS-компании, агентства и фрилансеры, которые хотят масштабировать outbound-продажи. Система берёт базу контактов (номера телефонов, имена лидов), автоматически совершает голосовые вызовы через AI, ведёт диалог, собирает информацию о заинтересованности и сохраняет результаты в CRM.

## Алгоритм реализации
1. Шаг 1: Подготовка базы данных лидов — создание таблицы с контактной информацией (номер телефона, имя, компания, email) в Google Sheets или подключение к внешней БД через API
2. Шаг 2: Настройка триггера в n8n — создание вебхука или расписания (Cron) для периодического запуска workflow в определённое время (например, каждый час)
3. Шаг 3: Получение данных лидов — использование узла HTTP Request или встроенного коннектора Google Sheets для извлечения списка контактов, которые ещё не обзвонены
4. Шаг 4: Интеграция с сервисом телефонии (Twilio, Voicebase, Synthesia) — настройка API-запроса с параметрами: номер вызываемого абонента, текст приветствия, сценарий диалога
5. Шаг 5: AI-генерация речи и обработка ответов — использование Claude API (или GPT) для генерации естественного ответа бота на основе реплик клиента, передача текста в Text-to-Speech сервис
6. Шаг 6: Запись результатов разговора — сохранение статуса вызова (поднял/не поднял трубку), транскрипции диалога, оценки интереса в Google Sheets или amoCRM
7. Шаг 7: Логирование ошибок и уведомления — настройка обработки исключений (ошибка подключения, timeout), отправка уведомлений в Slack или на email при проблемах

## Технический стек
- n8n (платформа для workflow-автоматизации)
- Twilio/Voicebase/Synthesia (API для совершения звонков и TTS)
- Claude API или OpenAI GPT (для генерации диалога)
- Google Sheets API (для хранения и синхронизации базы лидов)
- amoCRM или другой CRM (API для сохранения результатов)
- Google Text-to-Speech API (синтез речи)
- Slack API или SendGrid (уведомления)

## Связки инструментов
- Webhook/Cron Trigger → n8n HTTP Request → Twilio API (инициация вызова)
- Twilio Webhook (ответ клиента) → Claude API (обработка текста) → Google TTS (синтез ответа)
- Результат разговора → Google Sheets или amoCRM API (сохранение)

## Конфигурация и параметры
- Twilio: Account SID, Auth Token, номер отправителя (+1XXXXXXXXXX), параметр URL для вебхука обратного вызова
- n8n HTTP Request: метод POST, URL эндпоинта Twilio (/Calls), заголовки с авторизацией (Basic Auth), параметры тела: To (номер), From, Url (параметр для IVR-скрипта)
- Claude API: модель claude-3-5-sonnet или claude-3-opus, температура 0.7, max_tokens 500, system prompt с инструкциями о сценарии продаж
- Google Sheets: диапазон (Sheet1!A2:E), фильтр на пустой столбец 'Status' для выбора необработанных лидов
- Webhook ответ от Twilio: параметры SpeechResult (распознанный текст), CallSid, CallStatus
- Условие для повторного обзвона: if CallStatus == 'no-answer' → добавить в очередь через 2 часа

## Ключевые инсайты
- AI-сценарий должен быть сжатым и действенным (не более 30 сек на первое обращение) — иначе клиент повесит трубку. Сценарий лучше хранить в отдельной таблице, чтобы можно было быстро менять без редактирования workflow
- Используй фильтрацию на уровне n8n (Set node с условием) для исключения уже обзванных номеров и невалидных телефонов — это сэкономит API-квоту Twilio и Claude
- Для обработки натурального диалога лучше использовать voice-over модель Claude (claude-3-5-sonnet) или встроить LangChain с памятью разговора (ConversationBufferMemory) — тогда бот будет помнить контекст
- Время ожидания ответа клиента (timeout) должно быть 3-5 сек, иначе Twilio автоматически завершит вызов. Установи это в параметре MaxSpeechTime в XML-скрипте
- Логируй все вызовы в отдельную таблицу с полями: CallSid, LeadName, PhoneNumber, StartTime, Duration, TranscriptText, LeadQuality (1-5), NextAction — это поможет анализировать качество и ROI
- Для масштабирования (100+ вызовов в день) используй функцию Loop в n8n или батчинг: разбивай базу лидов на порции по 5-10, запускай паралельно через Split in batches
- Убедись, что сценарий соответствует местному законодательству (в РФ нужны документы о согласии на обзвон, в США — compliance с TCPA)
- Количество одновременных вызовов ограничено Twilio тарифом — в free tier только 1 вызов в секунду, поэтому для production используй Pro план
