### 1. Описание идеи

Компании ведут сделки в Bitrix24, но финансовый учёт — в Финтабло. Данные переносятся вручную: менеджер закрыл сделку → бухгалтер вводит поступление в Финтабло. Интеграция исключает этот процесс: оплата сделки в Bitrix24 автоматически создаёт платёж в Финтабло с правильной статьёй, проектом и контрагентом.

Результат: финансовый учёт ведётся в реальном времени без участия менеджеров.

### 2. Технический стек

- Bitrix24 REST API (`crm.deal.list`, `crm.deal.update`, Webhook на смену статуса)
- Финтабло REST API (создание платежей, контрагентов, проектов)
- OAuth2 для Финтабло
- n8n или Make.com — оркестратор (альтернатива: официальное приложение из маркетплейса Bitrix24)

### 3. Пошаговый план внедрения

**Вариант А — Официальное приложение (проще)**
```
Bitrix24 Маркетплейс → поиск "Финтабло"
→ Установить приложение → Авторизоваться в Финтабло
→ Настроить маппинг:
   Сделка "Оплачено" → Платёж в Финтабло
   Сумма сделки → Сумма платежа
   Контрагент CRM → Контрагент Финтабло (по ИНН или названию)
→ Выбрать статью ДДС (доход от продаж)
```

**Вариант Б — n8n (гибче)**
```
Шаг 1: Bitrix24 Webhook → n8n
  Событие: ONCRMDEALUPDATE (смена статуса на "Оплачено")

Шаг 2: n8n HTTP Request → Bitrix24 API
  GET /rest/crm.deal.get?id={{deal_id}}
  Получить: OPPORTUNITY, CONTACT_ID, COMPANY_ID, UF_INVOICE_NUM

Шаг 3: n8n HTTP Request → Финтабло API
  POST /api/v1/payments
  Body:
  {
    "amount": {{deal.OPPORTUNITY}},
    "date": "{{today}}",
    "counterparty_inn": "{{contact.INN}}",
    "category": "Выручка от продаж",
    "project": "{{deal.UF_PROJECT}}",
    "comment": "Сделка #{{deal.ID}} — {{deal.TITLE}}"
  }

Шаг 4: n8n → Bitrix24
  POST /rest/crm.deal.update
  Добавить поле UF_FINTABLO_ID = {{payment.id}} (для обратной ссылки)
```

**Шаг 4 — Обратная синхронизация**
```
Финтабло Webhook (платёж отменён) → n8n
  → Bitrix24: поставить задачу менеджеру "Уточнить статус оплаты"
```

### 4. Сценарии использования

| Событие в Bitrix24 | Действие в Финтабло |
|---|---|
| Сделка → «Оплачено» | Создать входящий платёж |
| Сделка → «Возврат» | Создать исходящий платёж (возврат) |
| Новый контрагент | Создать контрагента с ИНН |
| Выставлен счёт | Создать плановый платёж (в ожидании) |

### 5. Нюансы

- Для работы Финтабло API нужен тариф не ниже «Бизнес»
- ИНН контрагента должен быть заполнен в Bitrix24 — иначе создастся дубль в Финтабло
- Не все поля Финтабло доступны через API — проверить документацию текущей версии
- При больших объёмах (100+ сделок/день) — использовать n8n очереди, не Webhook напрямую
