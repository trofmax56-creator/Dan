**🛠 Как я победил вечно зависающий деплой (промпт для Claude Code)**

Друзья, вот честно — задолбался! 

Последние пару недель каждый деплой на сервер превращался в лотерею: то ли зависнет на середине и я буду тупо смотреть в терминал, то ли пройдёт. Это было как у собаки в анекдоте — на столько больно, чтобы скулить, но не настолько, чтобы слезть с гвоздя.

**Проблема:** SSH на Windows + Claude Code = зависание при heredoc-скриптах (`ssh host bash <<EOF`). Claude ждёт вывод бесконечно, деплой **__висит__**, я матерюсь.

Сегодня терпение кончилось. Разобрался, решил и делюсь с вами готовым промптом, который **раз и навсегда** закрывает эту боль.

**Что добавить в ****CLAUDE.md**** (или любую инструкцию для Claude Code):**

```SSH Anti-Hang (обязательно для Windows + Claude Code)

SSH на Windows может зависать при heredoc-скриптах.
Claude Code ждёт вывод бесконечно → деплой "висит".

Правила для всех SSH-вызовов в скриптах:

1. Всегда используй SSH-опции против зависания:
   ```bash
   SSH_OPTS="-T -o ConnectTimeout=10 -o ServerAliveInterval=15 -o ServerAliveCountMax=3 -o BatchMode=yes"
   ssh $SSH_OPTS "$HOST" bash <<REMOTE
     # команды
   REMOTE
   ```

2. **Добавляй глобальный таймаут-watchdog в длинные скрипты:**
   ```bash
   DEPLOY_TIMEOUT=300
   ( sleep $DEPLOY_TIMEOUT && echo "TIMEOUT" && kill -TERM $$ ) &
   TIMEOUT_PID=$!
   trap "kill $TIMEOUT_PID 2>/dev/null || true" EXIT
   ```

3. **Используй tar pipe вместо scp для загрузки файлов:**
   ```bash
   # Плохо: 30с (отдельное SSH-соединение на каждый файл/папку)
   scp -rq src/ host:/path/
   
   # Хорошо: 6с (один поток)
   tar czf - src prisma | ssh $SSH_OPTS host "tar xzf - -C /path/"
   ```

4. **Консолидируй SSH-вызовы** — каждый `ssh` это ~3с хендшейк на Windows. 
   Вместо 5 отдельных `ssh` для проверок — один `ssh` с одним скриптом.
``````


**Что это даёт:**
✅ Деплой больше не виснет
✅ Скорость загрузки файлов x5 (tar pipe вместо scp)
✅ Автоматический timeout, если что-то пошло не так
✅ Claude Code понимает, когда надо **__отпустить__** процесс

Проверено на себе. Работает как часы.
