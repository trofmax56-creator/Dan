# 🗂️ Паспорт идеи: Автоматизация отчётности отдела продаж

**Источник:** [Кейс. Автоматизация отчёности для отдела продаж в Битрикс24](https://youtube.com/watch?v=9MyTOQn0UIM) — Автоматизация рутины! (2024-04-11)
**Категория:** Автоматизация отдела
**Заменяет:** Аналитик/руководитель отдела продаж в части ручного сбора отчётов

---

## 💡 Что это даёт бизнесу
Система автоматически собирает данные из Битрикс24 и формирует ежедневные/еженедельные отчёты по сделкам, воронке и KPI менеджеров без участия человека. Руководитель получает готовую аналитику в мессенджер или email по расписанию. Исключается ручной экспорт данных, ошибки и задержки в принятии решений.

---

## ✅ Плюсы
- Высокий спрос: почти каждый клиент на Битрикс24 страдает от ручной отчётности
- Быстро внедряется: Битрикс24 имеет готовое API и REST-хуки, типовое решение реализуется за 1–2 недели
- Легко тиражируется: шаблон отчётов адаптируется под разные ниши (недвижимость, авто, услуги) с минимальными доработками

## ❌ Минусы / Риски
- Низкий средний чек: клиенты воспринимают отчётность как 'простую задачу' и готовы платить мало — нужно грамотно упаковывать ценность
- Битрикс24 регулярно меняет API и логику отчётов — требуется периодическая поддержка и обновления

## 🧰 Артефакты для реализации

**Инструменты:** Битрикс24 REST API, n8n или Make.com (автоматизация сбора и отправки), Google Sheets / Notion (хранение и визуализация), Telegram Bot API (доставка отчётов)
**API:** Битрикс24 REST API (crm.deal.list, crm.activity.list, user.get), Telegram Bot API, Google Sheets API

**Репозитории:**
- github.com/n8n-io/n8n — самохостинг автоматизации
- Поиск по 'bitrix24 report automation n8n' на GitHub

**Документация:**
- https://dev.1c-bitrix.ru/rest_help/
- https://docs.n8n.io/integrations/builtin/app-nodes/n8n-nodes-base.bitrix24/

## 🚀 Шаги запуска
1. Шаг 1: Провести аудит — какие отчёты нужны клиенту (воронка, конверсии, активность менеджеров, план/факт), определить периодичность и получателей
2. Шаг 2: Настроить n8n/Make.com сценарии: запросы к Битрикс24 API по расписанию, фильтрация и агрегация данных по нужным срезам
3. Шаг 3: Подключить хранилище (Google Sheets/Notion) для истории данных и настроить автоматическую отправку сводки в Telegram или email
4. Шаг 4: Тестирование на реальных данных клиента, обучение руководителя, оформление документации и передача на поддержку
