# Пост: Managed Agents vs самосборные связки

---

Anthropic выкатили Claude Managed Agents — агенты без собственного сервера, с готовым управлением памятью, инструментами и цепочкой задач.

Уже подключены: Notion, Asana, Rakuten.
Паттерн: Opus думает → Sonnet исполняет.

Звучит красиво. Разберём, когда это работает, а когда — нет.

---

**Managed Agents — ваш вариант, если:**
- Нет DevOps-ресурса внутри
- Используете стандартные инструменты (Notion, Asana, Google Workspace)
- Нужно запустить быстро, не думая про инфраструктуру

**Самосборная архитектура — ваш вариант, если:**
- CRM — Bitrix24, RetailCRM, МойСклад (не в официальном списке Managed)
- Нужна кастомная бизнес-логика: «лид с суммой > 500к → сразу в отдел B2B»
- Хотите портируемость: сегодня Bitrix24, завтра другая система — без переделки с нуля

---

**Для российского рынка вывод простой:**

Managed Agents пока не знают, что такое Bitrix24 REST API, вебхуки МойСклад или воронки amoCRM.

Пока — самосборная архитектура гибче:

```
Claude Code (claude-sonnet-4-6)
  + MCP-сервер Bitrix24
  + MCP-сервер МойСклад
  + n8n как оркестратор
  + Telegram как интерфейс
```

Это не «хардкорный» стек. Собирается за неделю, работает стабильно, не зависит от того, добавит ли Anthropic Битрикс в Managed.

---

Когда Managed Agents появятся с нативным Bitrix24 — пересмотрю. Слежу.

А пока: MCP + Claude Code + n8n = рабочая связка прямо сейчас.
