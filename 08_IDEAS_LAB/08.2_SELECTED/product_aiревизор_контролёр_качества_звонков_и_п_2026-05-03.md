## 1. Название
**AI-Ревизор** — Контролёр качества звонков и переписки

## 2. Суть
Claude прослушивает 100% звонков и читает все чаты менеджеров, выставляет оценку по чеклисту, фиксирует нарушения скрипта и отправляет РОПу дайджест с конкретными ошибками — вместо ОКК из двух человек.

## 3. Бизнес-сценарий
ДО: РОП Андрей раз в неделю вручную прослушивает 10–15 звонков из 200 — остальные 185 никто не слышит. Менеджер Сергей не дожимает клиента на этапе возражений, теряет 3–4 сделки в месяц на 180 000 руб. Андрей узнаёт об этом через месяц случайно. → ИИ делает: Каждый завершённый звонок автоматически транскрибируется через UIS/МТС Exolve, Claude анализирует текст по 12-пунктовому чеклису скрипта: «Поприветствовал? Выявил потребность? Назвал цену с обоснованием? Взял следующий шаг?» Выставляет оценку 1–10, цитирует конкретную фразу-ошибку. → Клиент получает: Каждое утро в 9:00 РОП получает в Telegram сводку: «Вчера 23 звонка. Средний балл: 6.2. Критично: Сергей 3 раза не взял следующий шаг. Слушать: [ссылка, 2:34].» Два сотрудника ОКК — минус 160 000 руб./мес ФОТ.

## 4. Логика
```
Завершённый звонок в UIS → webhook в n8n → транскрипция (Whisper API / UIS STT) → Claude оценивает по чеклисту → результат в Bitrix24 (карточка сделки) + дайджест РОПу в Telegram
```

## 5. Технический стек
- n8n
- claude-sonnet-4-6
- UIS или МТС Exolve
- Whisper API
- Bitrix24
- Telegram Bot API

## 6. Связки инструментов
```
UIS webhook (call_end) → n8n → Whisper STT → Claude API (оценка по чеклисту) → Bitrix24 REST (запись в сделку) → Telegram Bot (дайджест РОПу в 9:00)
```

## 7. Алгоритм реализации
1. Подключить UIS/МТС Exolve webhook на событие call_end: n8n получает ссылку на аудиофайл звонка и ID менеджера
2. n8n скачивает аудио, отправляет в Whisper API (openai или self-hosted) — получает транскрипт; при записи > 30 мин бить на чанки по 10 мин
3. Передать транскрипт в Claude с промптом: оцени по 12 критериям чеклиска (передаётся как системный промпт), верни JSON {score: 1-10, violations: [], positive: [], critical_quote: ''}
4. Записать результат оценки в поле карточки сделки Bitrix24 через REST API; если score < 5 — немедленное уведомление РОПу в Telegram с цитатой
5. n8n cron 9:00 каждый день: собирает все оценки за вчера, Claude генерирует текстовый дайджест на 5–7 строк, отправляет РОПу в Telegram

## 8. Почему купят (ROI)
Боль: 90% звонков не прослушивается никогда — менеджеры годами повторяют одни и те же ошибки, а РОП узнаёт о проблеме только когда план не выполнен. Штат ОКК из 2 человек = 160 000 руб./мес ФОТ. Пропущенный дожим на 3–4 сделки/мес при среднем чеке 50 000 руб. = 200 000 руб. потерь ежемесячно. Крючок: «Один менеджер с плохим скриптом стоит дороже, чем внедрение AI-Ревизора на год вперёд.»
