## 1. Название
**AI-Квалиметр** — Автоскоринг входящих лидов

## 2. Суть
Агент перехватывает каждый новый лид в CRM, задаёт уточняющие вопросы через WhatsApp, скорит по 5 критериям BANT и автоматически распределяет: горячих — старшему менеджеру, холодных — в nurture-цепочку.

## 3. Бизнес-сценарий
ДО: Менеджер получает 80 заявок в месяц с сайта. Половина — студенты, случайные люди, «просто посмотреть». Тратит 3 часа в день на обзвон всех подряд, к горячим добирается через 2 дня — они уже ушли. ИИ делает: заявка упала в Bitrix24 — через 2 минуты клиент получает WhatsApp: «Привет, [Имя]! Это [Компания]. Вы оставили заявку — подскажите, для какого проекта нужна автоматизация?» Клиент отвечает, Claude анализирует ответы на соответствие 5 критериям (бюджет, сроки, полномочия, потребность, масштаб), выставляет скор 1–10. Скор ≥ 7 → задача старшему менеджеру через 15 минут. Скор < 4 → автоматическая серия из 3 писем через неделю. Клиент получает: быстрый персональный контакт, ощущение внимания.

## 4. Логика
```
Bitrix24 Webhook (новый лид) → n8n → Wazzup (серия вопросов) → Claude (BANT-скоринг по ответам) → Bitrix24 API (обновление скора + назначение ответственного) → Telegram (алерт РОПу при скоре ≥ 7)
```


## 7. Алгоритм реализации
1. Настроить Bitrix24 Webhook на событие crm.lead.add: передавать поля name, phone, source, utm_source в n8n
2. Через Wazzup API отправить первое квалификационное сообщение клиенту в течение 2 минут после создания лида (шаблон: приветствие + 1 открытый вопрос о задаче)
3. Собирать ответы клиента через Wazzup Webhook в n8n, хранить диалог в переменной сессии (до 5 обменов сообщениями или 30 минут молчания)
4. После завершения диалога передать весь тред в Claude с промптом: оценить по 5 критериям BANT (Budget/Authority/Need/Timeline + Company Size), вернуть JSON со скором 1–10 и обоснованием каждого критерия
5. Записать скор и аргументы в кастомные поля лида Bitrix24 через PATCH /rest/crm.lead.update; при скоре ≥ 7 назначить ответственным старшего менеджера, при скоре < 4 — перевести в статус «Nurture»
6. Уведомить РОПа в Telegram при скоре ≥ 7: «🔥 Горячий лид: [Имя], скор [X]/10. Бюджет: [Y], срок: [Z]. Диалог: [ссылка на лид в Bitrix24]»
7. Запустить n8n Schedule-триггер: через 7 дней проверить лиды со статусом Nurture и отправить follow-up через Wazzup

## 8. Почему купят (ROI)
Менеджер тратит 40 минут на квалификацию одного лида по телефону. При 80 лидах в месяц — 53 часа = 1,3 рабочие недели. Из них 40–50% лидов изначально нецелевые. AI-Квалиметр отсекает мусор за 3 минуты переписки, менеджер тратит время только на лиды со скором ≥ 7. Крючок: «Пусть ИИ занимается анкетированием — менеджер должен закрывать сделки, а не играть в колл-центр.»
