### 1. Описание идеи

Компании ведут продажи в Bitrix24 и электронный документооборот в СБИС. Менеджер закрывает сделку в CRM → бухгалтер вручную создаёт счёт и акт в СБИС. Интеграция автоматизирует этот переход: смена статуса сделки в Bitrix24 запускает создание документа в СБИС с данными контрагента из CRM.

### 2. Технический стек

- Bitrix24 REST API (`crm.deal.get`, `crm.contact.get`, Webhook на `ONCRMDEALUPDATE`)
- СБИС API v1 (REST, документация: https://sbis.ru/help/integration/api)
- Bearer Token для СБИС (получить в Личном кабинете)
- n8n / Make.com — оркестратор
- Python 3 (альтернатива для скрипта)

### 3. Пошаговый план внедрения

**Шаг 1 — Получить доступы**
```
Bitrix24: Администратор → Интеграции → REST API → создать приложение
  → Скопировать токен и REST endpoint

СБИС: Личный кабинет → Интеграции → API → получить Token
  Базовый URL: https://online.sbis.ru/service/
```

**Шаг 2 — Webhook Bitrix24 на смену статуса сделки**
```
Bitrix24 → Администратор → Исходящие вебхуки
→ Событие: ONCRMDEALUPDATE
→ URL: ваш n8n Webhook URL
→ Фильтр: STATUS_ID = 'WON' (сделка выиграна)
```

**Шаг 3 — n8n: получить данные сделки и контрагента**
```javascript
// HTTP Request → Bitrix24
GET /rest/crm.deal.get?id={{deal_id}}
// Получить: OPPORTUNITY, CONTACT_ID, TITLE

GET /rest/crm.contact.get?id={{contact_id}}
// Получить: INN (ИНН), NAME, ADDRESS
```

**Шаг 4 — Создать документ в СБИС**
```javascript
// HTTP Request → СБИС API
POST https://online.sbis.ru/service/
Headers:
  X-SBISSessionID: {{sbis_token}}
  Content-Type: application/json
Body:
{
  "jsonrpc": "2.0",
  "method": "СБИС.СоздатьДокумент",
  "params": {
    "Документ": {
      "Тип": "СчётФактура",
      "Контрагент": {
        "ИНН": "{{contact.INN}}",
        "Название": "{{contact.NAME}}"
      },
      "Сумма": {{deal.OPPORTUNITY}},
      "Основание": "Договор по сделке #{{deal.ID}}"
    }
  }
}
```

**Шаг 5 — Записать ID документа СБИС обратно в Bitrix24**
```
HTTP Request → Bitrix24 crm.deal.update
  UF_SBIS_DOC_ID = {{sbis_response.result.Идентификатор}}
```

### 4. Сценарии использования

| Событие в Bitrix24 | Документ в СБИС |
|---|---|
| Сделка «Выиграна» | Счёт-фактура + Акт выполненных работ |
| Создан контрагент | Синхронизация реквизитов |
| Обновлён ИНН | Обновление контрагента в СБИС |
| Поступление оплаты | Платёжное поручение |

### 5. Нюансы

- СБИС API использует JSON-RPC, не REST — нужно корректно формировать запросы
- ИНН — главный ключ синхронизации. Без ИНН в Bitrix24 создастся дубль контрагента
- СБИС требует квалифицированную электронную подпись (КЭП) для подписания документов — API её не заменяет
- Для облачного СБИС — проверить тариф, API доступен не на всех

### 6. Оценка эффективности

- Сложность: 4/5 (СБИС API специфичен)
