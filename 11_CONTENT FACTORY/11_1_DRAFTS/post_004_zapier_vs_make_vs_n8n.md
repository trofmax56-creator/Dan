# Пост: Zapier vs Make vs n8n — честное сравнение

---

Один вопрос задают чаще всего перед первым проектом: «Что выбрать для автоматизации?»

Вот мой ответ после работы с тремя инструментами в десятках проектов.

---

**Zapier — если нужно запустить сегодня**

5000+ готовых интеграций. Ничего не надо настраивать. Подключил сервисы — работает.

Ограничение: за каждую сложность платишь дополнительно. Условная логика, циклы, большой объём — тариф растёт быстро.

Берите, если: небольшой бизнес, простые процессы, нужен результат за час.

---

**Make — если процесс сложный, но без кода**

Визуальный редактор позволяет строить многоуровневые сценарии с ветвлениями, фильтрами, агрегаторами. Это не Zapier — здесь видно весь поток данных на одном экране.

Ограничение: self-hosted нет, данные уходят в облако Make.

Берите, если: средний бизнес, сложная логика маршрутизации, нужна наглядность.

---

**n8n — если нужен контроль**

Open-source. Self-hosted — данные остаются у вас. Код внутри нод. Claude/GPT подключаются напрямую через HTTP Request без дополнительных плагинов.

Ограничение: порог входа выше. Первые 3 дня — настройка среды.

Берите, если: работаете с чувствительными данными (CRM клиентов, финансы), нужны AI-агенты, важна цена при масштабе.

---

**Мой выбор для CRM-проектов — n8n.**

Причина одна: Bitrix24 + amoCRM + 1С + AI в одной схеме — это не 3 простых шага. n8n позволяет построить это без ограничений тарифного плана.
