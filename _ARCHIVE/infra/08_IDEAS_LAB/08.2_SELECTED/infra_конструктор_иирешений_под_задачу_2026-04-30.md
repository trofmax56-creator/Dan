# 🗂️ Паспорт идеи: Конструктор ИИ-решений под задачу

**Источник:** [ИИ-АГЕНТ, ИИ-АССИСТЕНТ, БОТ - что выбрать под свою задачу ( разбор на примерах )](https://youtube.com/watch?v=MneQa2ZtnNo) — Web3nity (2026-03-27)

**Заменяет:** —

## 💡 Что это даёт бизнесу
Клиент получает чёткую классификацию своей задачи (бот / ассистент / агент) и готовое техническое решение без лишних затрат на переделки. Студия продаёт не абстрактный «ИИ», а конкретный продукт с понятной архитектурой и бюджетом. Снижается риск провала проекта из-за неправильно выбранного подхода.

---

## ✅ Плюсы
- Универсальный методологический фреймворк — продаётся как консалтинг или упаковывается в лид-магнит для привлечения клиентов
- Снижает стоимость пресейла: стандартизированный опросник за 30 минут заменяет 2-3 созвона
- Высокий спрос: запрос 'сколько стоит ИИ агент' — горячий коммерческий интент, аудитория готова платить

## ❌ Минусы / Риски
- Видео носит образовательный характер без готового кода или инструментов — всё придётся создавать с нуля
- Конкуренция высокая: подобные фреймворки уже предлагают крупные интеграторы, нужна уникальная упаковка

## 🧰 Артефакты для реализации

**Инструменты:** n8n (автоматизация квалификации заявок), Typeform / Tally (опросник для классификации задачи), Notion (база шаблонов архитектур), Claude API (генерация рекомендаций по опроснику)
**API:** Claude API (Anthropic), OpenAI API, Bitrix24 REST API, amoCRM API

**Репозитории:**

## 🚀 Шаги запуска
1. Шаг 1: Разработать опросник (10-15 вопросов) для классификации задачи клиента на типы: простой бот / ИИ-ассистент / ИИ-агент
2. Шаг 2: Создать 3 шаблонных архитектуры с описанием стека, сроков и бюджетов — оформить как PDF-презентацию и Notion-базу
3. Шаг 3: Автоматизировать квалификацию: клиент заполняет форму → n8n + Claude генерирует рекомендацию → менеджер получает готовый бриф
4. Шаг 4: Упаковать в лид-магнит ('Пройди тест и узнай какой ИИ нужен твоему бизнесу') → запустить трафик через Telegram-канал и контекст
