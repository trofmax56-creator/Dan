## Суть
Разработка и настройка Telegram-бота на платформе n8n с интеграцией ИИ-моделей для обработки сообщений, получения данных из внешних источников и организации диалога с пользователем.

## Бизнес-сценарий
Создание интеллектуального Telegram-бота, который умеет вести диалог с пользователями через языковую модель, извлекать информацию из внешних API/баз данных и предоставлять персонализированные ответы на основе контекста разговора.

## Алгоритм реализации
1. Шаг 1: Создание нового Workflow в n8n и инициализация Telegram-триггера. Добавить ноду 'Telegram Trigger' с указанием токена бота (получить у @BotFather в Telegram) и установкой вебхука для получения входящих сообщений
2. Шаг 2: Обработка входящего сообщения. Добавить ноду 'Set' для фильтрации входящих данных: извлечь текст сообщения (message.text), ID чата (message.chat.id) и ID пользователя, проверить наличие текста в сообщении
3. Шаг 3: Интеграция языковой модели (Claude/ChatGPT/Gemini). Добавить ноду с API-запросом к выбранной LLM, передать текст пользователя и историю диалога в параметр prompt, получить ответ с выполненным действием
4. Шаг 4: Запрос данных из внешних источников по результатам ответа модели. Если LLM указала необходимость получить информацию, выполнить HTTP-запрос к API (например, базе данных или сервису), передать параметры из ответа модели, получить результат
5. Шаг 5: Формирование финального ответа. Добавить ноду 'Set' для подготовки сообщения с результатом, объединить ответ от LLM с полученными данными в читаемом формате
6. Шаг 6: Отправка сообщения в Telegram. Использовать ноду 'Telegram Send Message' с параметрами: chat_id (из входящего сообщения), text (финальный ответ), указать параметр parse_mode='HTML' или 'Markdown' для форматирования
7. Шаг 7: Обработка ошибок и логирование. Добавить веточку 'Error' для перехвата сбоев на любом этапе, логировать проблемы, отправлять пользователю уведомление об ошибке через Telegram

## Технический стек
- n8n (платформа автоматизации)
- Telegram Bot API
- Claude API / ChatGPT API / Gemini API (языковая модель)
- HTTP Request ноды в n8n
- Webhook для получения обновлений от Telegram
- Set/Merge нода для трансформации данных
- JavaScript код для обработки JSON
- Внешние API для получения данных (опционально: CRM, базы данных, поисковые сервисы)
- JSON-парсер для обработки ответов от API

## Связки инструментов
- Telegram API (входящие сообщения) → n8n Webhook → Telegram Trigger ноде
- Telegram Trigger → HTTP Request ноду с Claude/ChatGPT API
- Claude/ChatGPT API (ответ с указанием нужного действия) → Conditional Branch ноде
- Conditional Branch (если нужны данные) → HTTP Request ноду к внешнему API
- Внешний API (результат) → Set ноде для форматирования → Telegram Send Message ноде → Telegram API (исходящее сообщение)

## Конфигурация и параметры
- Telegram Trigger: поле 'Authorization' — токен бота (формат: '123456:ABCDefghijklmnopqrst-uvwxyz123456', получить от @BotFather), поле 'Webhook URL' — автоматически генерируется n8n
- HTTP Request (для LLM): Method='POST', URL='https://api.anthropic.com/v1/messages' (для Claude), Headers: {'Authorization': 'Bearer YOUR_API_KEY', 'content-type': 'application/json'}, Body: {'model': 'claude-3-sonnet-20240229', 'max_tokens': 1024, 'messages': [{'role': 'user', 'content': 'входящее сообщение'}]}
- Set ноде (обработка ответа): создать переменные для хранения текста, ID чата, истории диалога. Пример: { chatId: '{{ $node.Telegram_Trigger.json.message.chat.id }}', userMessage: '{{ $node.Telegram_Trigger.json.message.text }}', timestamp: '{{ new Date().toISOString() }}'}
- Telegram Send Message: параметр 'chatId' = ID из входящего сообщения, параметр 'text' = сформированное сообщение с ответом, параметр 'parseMode' = 'HTML' или 'Markdown', опционально 'replyToMessageId' для цепочек сообщений
- Условное ветвление: если в ответе от LLM указано, что нужны данные из внешнего API, добавить условие на наличие флага или ключевого слова в ответе
- HTTP Request (к внешнему API): Method='GET' или 'POST' в зависимости от API, параметры передать из ответа LLM, обработать результат через JSON.parse()
- Error Handler: добавить на каждый критический шаг, параметр 'Continue on Error' = true, отправлять в Telegram сообщение вида 'Произошла ошибка: {{ $error.message }}'
