# 🗂️ Паспорт идеи: RAG-база знаний компании на LLM

**Источник:** [ВСЯ БАЗА для работы с ИИ за 10 минут (Агенты, LLM, RAG, Токены)](https://youtube.com/watch?v=YoHuVfr_rjk) — Dmitry Slepov (2025-10-02)
**Категория:** Автоматизация отдела
**Заменяет:** Менеджер по внутренним знаниям / часть функций техподдержки и HR

---

## 💡 Что это даёт бизнесу
Сотрудники получают мгновенные ответы на вопросы по регламентам, продуктам и процессам компании без поиска по папкам и мессенджерам. Новые сотрудники адаптируются в 2–3 раза быстрее, снижается нагрузка на руководителей и HR. Клиентский сервис отвечает точнее и быстрее, опираясь на актуальную базу данных компании.

---

## ✅ Плюсы
- Высокий спрос: почти каждый МСБ страдает от хаоса в знаниях и долгой адаптации новичков
- Легко упаковать в типовой продукт: сбор документов → индексация → чат-интерфейс → интеграция в Bitrix24/Telegram
- Повторяющиеся продажи: обновление базы, расширение источников, поддержка — ежемесячная выручка

## ❌ Минусы / Риски
- Клиенты часто не готовы структурировать документы — большая часть работы ложится на Студию на этапе сбора данных
- Качество ответов сильно зависит от качества исходных документов клиента, что создаёт репутационные риски

## 🧰 Артефакты для реализации

**Инструменты:** n8n (оркестрация пайплайна RAG), Qdrant / Chroma (векторная база данных), LangChain / LlamaIndex (RAG-фреймворк), Telegram Bot API или Bitrix24 Open Channels (интерфейс)
**API:** OpenAI API (GPT-4o / text-embedding-3-small), Anthropic Claude API (альтернатива), Yandex GPT API (если нужен российский стек)

**Репозитории:**
- github.com/hwchase17/langchain
- github.com/run-llama/llama_index
- github.com/qdrant/qdrant

**Документация:**
- docs.llamaindex.ai
- python.langchain.com/docs/use_cases/question_answering
- qdrant.tech/documentation

## 🚀 Шаги запуска
1. Шаг 1: Аудит документов клиента — регламенты, инструкции, FAQ, скрипты продаж; структурирование в единый формат (PDF/DOCX → текст)
2. Шаг 2: Настройка пайплайна индексации в n8n: загрузка документов → чанкинг → эмбеддинги → запись в Qdrant
3. Шаг 3: Разработка RAG-агента (LlamaIndex или LangChain) с промптом под роль компании; подключение чат-интерфейса (Telegram-бот или виджет в Bitrix24)
4. Шаг 4: Тестирование на реальных вопросах сотрудников, настройка обновления базы по расписанию, передача клиенту + обучение администратора
