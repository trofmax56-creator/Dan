## 1. Название
**AI-Мозг** — База знаний компании с поиском

## 2. Суть
Claude превращает разрозненные регламенты, скрипты и инструкции компании в живую базу знаний с семантическим поиском — сотрудник задаёт вопрос в Telegram-боте и получает точный ответ со ссылкой на источник за 5 секунд вместо поиска по папкам 20 минут.

## 3. Бизнес-сценарий
ДО: Новый менеджер Катя спрашивает: «Как мы оформляем возврат, если клиент оплатил через ЮКассу, но товар пришёл повреждённым?» Коллеги пишут ей в личку, кто-то скидывает PDF 2021 года, кто-то говорит «спроси у Марины». Марина в отпуске. ИИ делает: Катя пишет в корпоративный Telegram-бот тот же вопрос. Через 4 секунды бот отвечает: «По регламенту возвратов (обновлён 14.03.2025, раздел 3.2): при повреждении при доставке — оформляем акт через форму [ссылка], возврат на карту через ЮКассу в течение 3 рабочих дней. Ответственный: логист Дмитрий.» Катя решает вопрос сама. Марина отдыхает. Клиент получает ответ быстро. Клиент получает: онбординг новых сотрудников ускоряется с 2 недель до 3 дней.

## 4. Логика
```
Вопрос в Telegram-боте → n8n → Claude (семантический поиск по векторной базе документов) → Telegram Bot API (ответ с цитатой и источником)
```

## 5. Технический стек
- n8n
- claude-opus-4-7
- PostgreSQL + pgvector
- Telegram Bot API
- Google Drive API

## 6. Связки инструментов
```
Google Drive / Notion (источники документов) → n8n (парсинг и чанкинг) → PostgreSQL с pgvector (векторное хранилище) → Claude (семантический поиск + генерация ответа) → Telegram Bot API (ответ пользователю)
```

## 7. Алгоритм реализации
1. Собрать все документы компании (регламенты, скрипты, инструкции, FAQ) из Google Drive через API, разбить на чанки по 500 токенов с сохранением метаданных (название документа, дата, раздел)
2. Векторизовать чанки через Claude Embeddings API и сохранить в PostgreSQL с расширением pgvector; настроить автообновление при изменении файла в Google Drive (Webhook на Drive Activity API)
3. Создать Telegram-бота через BotFather, подключить к n8n через Webhook; при входящем сообщении выполнять косинусный поиск по pgvector (топ-5 релевантных чанков)
4. Передать найденные чанки в Claude с промптом: «Отвечай строго на основе этих документов, указывай название источника и дату; если ответа нет — честно сообщи об этом»
5. Логировать все вопросы без ответа в отдельную Google Sheet для еженедельного пополнения базы знаний; отправлять недельный дайджест «топ-10 вопросов без ответа» HR-менеджеру

## 8. Почему купят (ROI)
Сотрудник тратит в среднем 2,5 часа в день на поиск информации (данные McKinsey). При штате 15 человек это 37,5 человеко-часов/день = 750 часов/месяц = ~375 000 руб./мес потерянного рабочего времени. AI-Мозг сокращает поиск до 5 секунд. Плюс: каждый уволившийся сотрудник уносит экспертизу — AI-Мозг делает знания бессмертными. Крючок: «Марина уволилась, но всё что она знала — осталось в компании.»
