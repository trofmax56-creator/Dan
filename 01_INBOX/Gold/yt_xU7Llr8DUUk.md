## Суть
Использование Claude Code и Obsidian вместе для замены традиционного RAG (Retrieval-Augmented Generation) подхода одним оптимизированным промптом. Автоматизация работы с локальными заметками через Claude AI для анализа и обработки информации без необходимости сложной конвейерной архитектуры RAG.

## Бизнес-сценарий
Разработчики, исследователи и специалисты по информационному анализу используют комбинацию Claude Code (функция выполнения кода в Claude) и Obsidian (приложение для управления заметками) для автоматизации обработки локальной базы знаний. Система читает заметки Obsidian, обрабатывает их через Claude с предварительно подготовленным промптом и генерирует результаты без использования традиционного RAG pipeline с embedding и vector database.

## Алгоритм реализации
1. Шаг 1: Подготовка базы знаний в Obsidian — организация заметок в структурированную иерархию с метаданными (tags, frontmatter) и экспорт в доступный формат (Markdown или JSON)
2. Шаг 2: Разработка master-промпта для Claude — создание системного промпта, который описывает роль Claude, контекст использования, требования к выходным данным и правила обработки информации
3. Шаг 3: Загрузка содержимого Obsidian в контекст Claude — использование Claude Code для чтения файлов заметок (с помощью встроенной функции file reading) или передача содержимого через API
4. Шаг 4: Выполнение анализа через Claude Code — передача загруженного контента в Claude с master-промптом, выполнение требуемой обработки (синтез, категоризация, ответы на вопросы)
5. Шаг 5: Обработка результатов и экспорт — Claude Code форматирует результаты и сохраняет их обратно в Obsidian или экспортирует в необходимый формат (Markdown, PDF, JSON)
6. Шаг 6: Автоматизация через интеграцию — при необходимости настраивается запуск через API или webhook для регулярной обработки или реактивного выполнения при изменении заметок

## Технический стек
- Claude AI (основной LLM)
- Claude Code (функция выполнения кода в Claude API)
- Obsidian (платформа управления заметками)
- Obsidian API / File System API (доступ к заметкам)
- Python или Node.js (опционально для локального скрипта)
- Claude API (anthropic/messages endpoint)
- Markdown / JSON (форматы данных)
- File I/O операции (чтение/запись файлов)

## Связки инструментов
- Obsidian (папка заметок) → Claude Code → Claude API → Результаты обратно в Obsidian
- File System → Python/Node.js скрипт → Claude API with file context → Output в Markdown/JSON
- Webhook/Trigger → Claude Code execution → Obsidian notes processing → Claude analysis → Export

## Конфигурация и параметры
- Master-промпт (system prompt): детальное описание роли Claude, контекста и требований
- Контекстное окно Claude: использование максимального размера контекста (200K tokens) для загрузки всей базы знаний
- Формат входных данных: Markdown файлы из папки Obsidian с метаданными в frontmatter (YAML)
- Структура вывода: JSON с полями для результатов анализа или Markdown с категоризацией
- Функция чтения файлов в Claude Code: использование встроенного file reading или API для доступа к содержимому
- Параметры Claude API вызова: model='claude-3-5-sonnet' или выше, temperature в зависимости от задачи (0.3-0.7), max_tokens для контроля размера ответа

## Ключевые инсайты
- Идея Карпати: вместо сложной RAG архитектуры (embedding + vector search + retrieval) использовать контекстное окно Claude (до 200K tokens) для загрузки всего контента и обработки через один оптимизированный промпт
- Claude Code позволяет автоматически читать локальные файлы Obsidian и передавать их в Claude API без необходимости ручной подготовки
- Master-промпт с точными инструкциями эффективнее, чем traditional RAG, для задач, где объём данных умещается в контекст (small-to-medium knowledge bases)
- Obsidian служит не только хранилищем, но и интерфейсом для управления версионированием и историей изменений заметок
- Экономия затрат: отсутствие необходимости в отдельной базе данных embeddings, vector database (Pinecone, Weaviate) и сложной инфраструктуре
- Простота воспроизведения: вся логика находится в одном промпте, легко модифицируется и отлаживается без переобучения моделей
- Преимущество для приватности: данные остаются локально в Obsidian, передаются в Claude только при явном запросе, не индексируются внешними сервисами

## Подводные камни
- Ограничение контекста: подход работает только если вся база знаний умещается в контекстное окно Claude (≤200K tokens). Для больших корпусов нужна фильтрация или segmentation
- Задержка обработки: загрузка всего контента в каждый запрос медленнее, чем vector search в RAG, но быстрее, чем fine-tuning
- Качество промпта критично: если master-промпт написан плохо, результаты будут низкого качества. Требуется тестирование и итерация
- Управление зависимостями: если заметки в Obsidian ссылаются друг на друга (wiki links), нужно убедиться, что Claude Code корректно разрешает и включает эти зависимости
- Версионирование данных: Obsidian не ведёт встроенное версионирование. Если нужна история изменений, требуется настроить Git integration или backup
- Масштабируемость: при росте базы знаний до 500K+ tokens подход становится дорогим (стоимость API возрастает), может быть нужна гибридная архитектура
- Форматирование: некорректно подготовленный Markdown или нарушенная структура YAML frontmatter приведут к ошибкам парсинга в Claude Code
