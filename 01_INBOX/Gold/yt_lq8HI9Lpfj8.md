## Суть
Демонстрация M&G CRM — AI-powered системы автоматизации на базе N8N, которая интегрирует Chatwoot и WhatsApp для управления бизнес-коммуникациями. Система автоматизирует процессы управления сообщениями, обработку запросов клиентов и маршрутизацию чатов с использованием искусственного интеллекта.

## Бизнес-сценарий
Бизнесы используют M&G CRM для автоматизации управления входящими сообщениями в WhatsApp через Chatwoot. Система обрабатывает клиентские запросы, классифицирует их, генерирует AI-ответы и автоматически маршрутизирует чаты между агентами или создаёт задачи в CRM на основе содержимого сообщений.

## Алгоритм реализации
1. 1. Webhook-триггер получает входящее сообщение из Chatwoot при новом сообщении клиента в WhatsApp
2. 2. N8N парсит тело webhook-запроса, извлекая текст сообщения, ID чата, ID контакта, имя клиента и другие метаданные
3. 3. Текст сообщения отправляется в Claude AI API (или аналогичный LLM) с системным промптом для классификации и генерации ответа
4. 4. AI возвращает структурированный результат: классификацию (тип запроса), приоритет, рекомендуемый ответ, требуется ли созвон
5. 5. На основе AI-результатов N8N выполняет условную логику: либо отправляет автоматический ответ через Chatwoot API, либо назначает чат агенту, либо создаёт задачу в CRM
6. 6. Система логирует все операции и обновляет метаданные чата (тэги, приоритет, статус) в Chatwoot

## Технический стек
- N8N — платформа для построения workflow автоматизации
- Chatwoot — омниканальная платформа для управления клиентским общением
- WhatsApp (интеграция через Chatwoot) — канал доставки сообщений
- Claude AI API (или ChatGPT/Gemini) — LLM для обработки и анализа текста
- Webhook — для получения событий из Chatwoot в N8N
- REST API — для отправки ответов и обновления данных в Chatwoot
- JSON — формат данных для передачи между сервисами

## Связки инструментов
- Входящее сообщение WhatsApp → Chatwoot (платформа) → Webhook → N8N
- N8N → Claude/ChatGPT API → AI-анализ текста и классификация
- N8N → Chatwoot API → Отправка ответа / Назначение агента / Создание задачи
- N8N → внутренняя база данных (опционально) → логирование и аналитика

## Конфигурация и параметры
- Webhook ноде: HTTP Method = POST, Path = /chatwoot-webhook, Body содержит JSON с сообщением
- Chatwoot webhook payload включает: message.content (текст), conversation.id, contact.name, contact.phone_number, conversation.status
- Claude API ноде: Model = claude-3-sonnet (или claude-3-opus), Temperature = 0.7, Max tokens = 1000
- System prompt для Claude: инструкции по классификации типов запросов (Sales, Support, Complaint, Question), приоритизации, генерации ответа
- Условная логика (If ноде): IF classification === 'Complaint' AND priority === 'High' THEN assign_to_agent ELSE send_auto_response
- Chatwoot API endpoint для отправки ответа: POST /api/v1/conversations/{conversation_id}/messages с параметрами: content (текст ответа), message_type (outgoing), private (boolean)
- Переменные окружения: CHATWOOT_API_KEY, CHATWOOT_BASE_URL, CLAUDE_API_KEY, WHATSAPP_ACCOUNT_ID

## Ключевые инсайты
- Использование Chatwoot Webhook позволяет в реальном времени реагировать на входящие сообщения, без необходимости постоянного polling API
- Claude/ChatGPT API обеспечивает контекстный анализ сообщений на естественном языке, что невозможно с простыми regex-правилами
- Структурированный AI output (JSON с classification, priority, suggested_response) позволяет N8N принимать автоматические решения: отправлять ответ, назначать агента или создавать задачу
- Обязательная валидация входящих данных: проверка наличия message.content, обработка спецсимволов, экранирование перед отправкой в API
- Асинхронная обработка с таймаутом (~5-10 сек на Claude API) требует корректной обработки ошибок и fallback логики (например, отправка стандартного ответа при сбое AI)
- Логирование всех действий (какой ответ был отправлен, кому назначен чат, какой статус чата) критично для аудита и отладки
- Интеграция с WhatsApp через Chatwoot автоматически обрабатывает форматирование сообщений, медиа, доставку квитанций

## Подводные камни
- Chatwoot Webhook может отправлять дублирующиеся события — добавьте проверку message_id в базе для избежания повторной обработки
- Claude API имеет rate limit (обычно 100k tokens/месяц на стартовом плане) — мониторьте использование и добавьте queue для массовых запросов
- Если Chatwoot работает локально или за NAT, webhook должен быть доступен через публичный URL (используйте ngrok, tunnel или облачный хостинг N8N)
- Неправильное форматирование JSON в System Prompt может привести к невалидным AI ответам — используйте строгие инструкции и примеры в prompt
- WhatsApp имеет собственные ограничения на форматирование (максимум 4096 символов, нет HTML) — не отправляйте неотфильтрованный AI output
- Если чат уже назначен агенту, автоматическая отправка AI ответа может перекрыть работу агента — добавьте проверку conversation.status и assignment_status
- Обработка специальных символов: эмодзи, кириллица, URL могут вызвать ошибки при парсинге JSON — явно кодируйте в UTF-8
