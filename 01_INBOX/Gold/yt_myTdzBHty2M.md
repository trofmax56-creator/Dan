## Суть
Интеграция системы управления запасами 1С:УТ, интернет-магазина 1С-Битрикс и API платформ ABCP и Laximo.OEM с CRM Битрикс24 для синхронизации каталога автозапчастей, управления заказами и контактами клиентов в едином информационном пространстве.

## Бизнес-сценарий
Компания, производящая или продающая автозапчасти, использует несколько отдельных систем: 1С:УТ (управление товарами и складом), 1С-Битрикс (веб-магазин), ABCP и Laximo.OEM (каталоги поставщиков OEM-запчастей), и Битрикс24 (CRM для управления клиентами). Задача: синхронизировать данные между всеми системами, чтобы актуальная информация о наличии, цене и характеристиках запчастей была доступна везде, заказы автоматически попадали в CRM, а история контактов единая.

## Алгоритм реализации
1. Шаг 1: Установка и настройка модулей интеграции в 1С-Битрикс для подключения к REST API ABCP и Laximo.OEM. Выбор модулей синхронизации товаров и каталогов, настройка расписания обновления (например, каждые 30 минут или 1 час). Указание API-ключей и эндпоинтов платформ.
2. Шаг 2: Создание правил маппинга (соответствия полей) между структурой товаров в 1С:УТ и структурой CRM Битрикс24. Настройка трансформации данных: артикул товара → ID товара в Битрикс24, цена 1С → цена в CRM, наличие на складе 1С → поле Количество в карточке товара.
3. Шаг 3: Разработка webhook-обработчика для отправки данных из 1С-Битрикс в Битрикс24. При создании заказа в магазине (статус «Новый заказ») автоматически создается сделка в CRM с реквизитами покупателя, суммой и списком товаров.
4. Шаг 4: Синхронизация справочника контактов: при заказе в магазине → новый контакт или обновление существующего контакта в Битрикс24 (по email или телефону). Привязка к компании, если клиент юридическое лицо.
5. Шаг 5: Настройка двусторонней синхронизации в Битрикс24: при изменении статуса сделки (например, «Оплачено») → автоматическое изменение статуса заказа в 1С-Битрикс (например, «Ожидание отправки»). Использование REST API Битрикс24 и встроенных автоматизаций (боты, триггеры).
6. Шаг 6: Интеграция каталогов ABCP и Laximo.OEM в 1С-Битрикс: периодическое получение данных о доступности запчастей по VIN или артикулу, обновление цены в зависимости от поставщика, выбор наиболее выгодного предложения. Настройка логики выбора источника товара (свой склад или поставщик).
7. Шаг 7: Обработка ошибок и логирование: создание служебной таблицы ошибок синхронизации, уведомление менеджеров в Битрикс24 при сбое, повторная отправка данных. Мониторинг статуса API и доступности серверов интегрируемых систем.

## Технический стек
- 1С:УТ (1С Управление торговлей и складом)
- 1С-Битрикс (CMS/платформа интернет-магазина)
- Битрикс24 (CRM и платформа для управления компанией)
- ABCP (API для синхронизации каталога запчастей и поиска по VIN)
- Laximo.OEM (OEM-каталог с REST API для интеграции)
- REST API (для обмена данными между системами)
- Webhooks (для триггерных событий)
- PHP (язык программирования для модулей 1С-Битрикс)
- MySQL/SQL (база данных синхронизации)
- cron-tasks или встроенные агенты 1С-Битрикс (для периодической синхронизации)

## Связки инструментов
- 1С:УТ (экспорт товаров) → REST API модуля синхронизации в 1С-Битрикс → Битрикс24 REST API
- 1С-Битрикс (заказ товара) → Webhook → Битрикс24 (создание сделки и контакта)
- ABCP REST API → Parser модуль 1С-Битрикс → обновление каталога и цен
- Laximo.OEM API → модуль синхронизации → 1С-Битрикс каталог
- Битрикс24 REST API → обновление статуса заказа → 1С-Битрикс API → 1С:УТ

## Конфигурация и параметры
- Модули 1С-Битрикс: Интеграция с ABCP (REST, API-ключ в настройках модуля), Интеграция с Laximo (REST, основной URL эндпоинта)
- Параметры синхронизации товаров: интервал обновления (рекомендуется 30-60 минут), селектор категорий для синхронизации (все или выбранные), поле для хранения внешних ID (код поставщика)
- Маппинг полей в Битрикс24: UF_AUTO_SKU (артикул товара) ← код товара 1С, UF_PRICE (цена) ← цена продажи 1С, UF_STOCK (наличие) ← остаток на складе, UF_SUPPLIER (поставщик) ← источник товара
- Настройка webhook в 1С-Битрикс: URL вебхука Битрикс24 REST API (вида https://b24-xxxxx.bitrix24.ru/rest/1/xxxxxxxxxxxx/...), событие срабатывания (OnSaleOrderAdd, OnSaleOrderPayment)
- REST API ключи: Битрикс24 (webhook ID и код авторизации), ABCP (API-ключ в личном кабинете), Laximo (токен доступа)
- Условия для создания сделки в CRM: сумма заказа > минимум, статус заказа = 'Новый', клиент зарегистрирован или email указан
- Поля для синхронизации контактов: Фамилия, Имя, email, телефон, адрес доставки, компания (если B2B)
