## 1. Название
**AI-Прораб** — Автоматизация задач через Telegram-команды

## 2. Суть
Claude-агент, подключённый к Telegram, принимает голосовые и текстовые команды руководителя, сам ставит задачи в Bitrix24, создаёт напоминания, формирует отчёты и управляет расписанием — без открытия CRM и почты.

## 3. Бизнес-сценарий
ДО: Директор малого производства едет в машине, вспоминает: «Надо поставить Артёму задачу по отгрузке, согласовать счёт с бухгалтером и узнать статус трёх сделок.» Он ничего не записывает — забудет. Или останавливается и 10 минут тыкает в CRM. ИИ делает: директор говорит голосовое в Telegram: «Артёму задачу — проверить отгрузку по заказу 4521 до пятницы, напомни мне в четверг в 10 утра.» Claude расшифровывает голосовое, создаёт задачу в Bitrix24 на Артёма со сроком, ставит напоминание директору в календарь. Следующее голосовое: «Что у нас с тремя крупными сделками?» — Claude тянет данные из CRM и отвечает: «Сделка Иванов — в стадии КП, последний контакт 5 дней назад. Петров — счёт выставлен, оплата не поступила. Сидоров — закрыта, 250 000 руб.» Клиент получает: управление бизнесом голосом из любой точки, ноль потерянных задач.

## 4. Логика
```
Голосовое/текст в Telegram → Яндекс SpeechKit (транскрипция голоса) → n8n → Claude (извлечение намерения + параметров) → Bitrix24 REST API (задача/событие/запрос данных) → Telegram Bot (подтверждение)
```

## 5. Технический стек

## 6. Связки инструментов
```
Telegram Bot (голос/текст) → n8n Webhook → Яндекс SpeechKit (если голосовое) → Claude (классификация намерения: создать задачу / получить отчёт / поставить напоминание / найти сделку) → Bitrix24 REST API → Google Calendar → Telegram Bot (подтверждение с деталями)
```

## 7. Алгоритм реализации
1. Создать Telegram-бота с белым списком ID (только директор и топ-менеджеры), подключить к n8n через Webhook; голосовые сообщения отправлять в Яндекс SpeechKit для транскрипции
2. Написать системный промпт Claude с классификатором намерений на 6 типов: создать_задачу, поставить_напоминание, получить_отчёт, найти_сделку, обновить_статус, ответить_на_вопрос — на выходе получать структурированный JSON с типом и параметрами
3. Реализовать в n8n роутер по типу намерения: для создать_задачу — вызов Bitrix24 tasks.task.add с исполнителем, сроком и описанием; для получить_отчёт — запрос crm.deal.list с фильтрами и форматирование ответа
4. Настроить двусторонний диалог: если Claude не смог извлечь обязательный параметр (например, исполнителя задачи), бот задаёт уточняющий вопрос вместо того, чтобы создавать задачу с пустыми полями
5. Добавить еженедельный автоотчёт: каждую пятницу в 17:00 n8n автоматически запрашивает из Bitrix24 статус всех открытых сделок и задач, Claude формирует резюме на 5–7 строк, бот отправляет директору

## 8. Почему купят (ROI)
Руководитель МСБ тратит в среднем 2–3 часа в день на операционное управление через CRM и мессенджеры. 60% задач теряется, не будучи зафиксированными. AI-Прораб превращает любую мысль в задачу за 10 секунд, не требуя открывать ноутбук. При стоимости времени директора 3 000 руб./час экономия 1,5 часа/день = 90 000 руб./мес. Крючок: «Ваша CRM должна слушать вас, а не наоборот.»
