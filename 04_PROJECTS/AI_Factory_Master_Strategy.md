## ЭТАП 0 — СБОР СЫРЬЯ

### Два ручных потока (раз в 2 недели, локально)

**Поток А — Telegram:**
1. `python parser.py` — собирает посты из 27 каналов (Telethon, бесплатно)
2. `python processor.py` — разветвляет по двум потокам (без API):
   - **GOLD_CRM** → `01_INBOX/Gold/` — CRM, n8n, интеграции, AI для бизнеса
   - **GOLD_TOOLS** → `01_INBOX/Gold_Tools/` — Cursor, LangGraph, Emergent, vibe coding, гайды
   - **TRASH** → игнорируется

**Логика разветвления processor.py:**
```
Есть CRM-якорь (bitrix/amocrm/n8n/crm/webhook)?
  └─ да → GOLD_CRM → 01_INBOX/Gold/
Только инструментальные слова (cursor/langgraph/emergent/dify)?
  └─ да → GOLD_TOOLS → 01_INBOX/Gold_Tools/
Ничего не совпало → TRASH
```

**Поток Б — YouTube RSS:**
1. `python youtube_parser.py` — парсит RSS 10 каналов (бесплатно)
2. Результат: новые файлы в `00_RAW/YouTube/`

### Поток через GitHub Actions (вручную)

**YouTube API:**
1. GitHub → Actions → YouTube Discovery → Run workflow
2. `yt_discovery.py` → YouTube Data API → `00_RAW/YouTube/`
3. `yt_processor_2.py` → Claude Haiku → `01_INBOX/Gold/yt_*.md`
4. `git pull` локально

### Поиск новых TG-каналов (по необходимости)
```
python 02_TOOLS/Scripts/tg_discovery_v2.py
```
Граф-обход по упоминаниям в `00_RAW/Telegram/`. Результат → `tg_discovered_v2.md`.
Новые каналы добавляются вручную в `parser.py`.

### Технический стек сбора
- YouTube RSS (бесплатно, без ключей)
- YouTube Data API v3 (10k запросов/день, бесплатно)
- Telegram MTProto / Telethon (`dan_session` — только локально)
- Claude Haiku — классификация YT (дёшево)
- GitHub Actions — облачный запуск yt_discovery

## ЭТАП 1 — КЛАССИФИКАЦИЯ И СКОРИНГ

### Demand Matrix
| Критерий | Вопрос | Диапазон |
|---|---|---|
| **Pain** | Клиент теряет деньги/время прямо сейчас? | 1–10 |
| **Dev** | Собрать на n8n + Claude за 1–2 недели? | 1–10 (инверсия) |
| **Profit** | Продать за ≥50к разово или ≥15к/мес? | 1–10 |
| **ИТОГ** | Pain + (10 − Dev) + Profit | max 30 |

### Роутинг по score
```
ИТОГ > 24  → 05_BIZ_RECIPES/                   (в производство)
ИТОГ 20–24 → 08_IDEAS_LAB/08.2_SELECTED/       (следующий спринт)
ИТОГ 18–19 → 08_IDEAS_LAB/08.1_RAW_IDEAS/      (идея, не срочно)
ИТОГ < 18  → пропускаем
```

### Gold Synthesizer v2.0 (авто, каждое воскресенье)
- Читает только `01_INBOX/Gold/` (Gold_Tools не трогает — API не расходуется)
- Берёт последние 40 файлов (BATCH_SIZE=40)

## ЭТАП 2 — УПАКОВКА ПРОДУКТА (BIZ_RECIPE)

### Стандарт продукта v3.0 (9 пунктов)
1. **Название** — `AI-[Роль]` + расшифровка в скобках (пример: «AI-Сито» (Умный Квалификатор))
2. **Суть** — механика решения (1–2 предложения)
3. **Бизнес-сценарий** — живая сцена ДО → ИИ делает → клиент получает
4. **Логика** — Trigger → Process → Action
5. **Технический стек** — инструменты (РФ-адаптация: Wazzup, ЮKassa, Dadata)
6. **Связки инструментов** — как сервисы общаются (n8n как оркестратор)
7. **Алгоритм реализации** — 3–5 шагов, конкретные глаголы + числа
8. **Почему купят (ROI)** — боль + цифра + крючок-инсайт
9. **Экономика** — разработка / инфра / цена клиенту

### Векторы (из crm-ai-ideator.md)
- **Qualifier** — ускорение обработки входящего лида
- **Reanimator** — работа с базой отказников
- **Efficiency** — автоматизация рутины (КП, сметы, документы)

### Целевые ценники
| Продукт | Внедрение | Поддержка/мес | Срок |
|---|---|---|---|
| Простая интеграция (1 workflow) | 30–50к руб. | 10к руб. | 1 нед. |
| AI-агент для CRM | 80–150к руб. | 20к руб. | 2 нед. |
| AI-Sales Booster | 150–250к руб. | 30к руб. | 3–4 нед. |
| Завод ИИ-решений под ключ | 300–500к руб. | 50к руб. | 2–3 мес. |

## ЭТАП 3 — РАЗРАБОТКА (Цех Сборки)

### Алгоритм
1. Берём BIZ_RECIPE с тегом `#action_needed`
2. Разворачиваем self-hosted n8n (Docker на VPS)
3. Строим workflow по алгоритму из карточки
4. Тестируем на реальных данных клиента
5. Экспортируем JSON в репо `Dan` (библиотека чертежей)
6. Меняем тег: `#action_needed` → `#done`

### Стек разработки
```
Trigger (Webhook / Cron / CRM Event)
  → n8n (оркестратор)
  → Claude claude-sonnet-4-6 (сложный анализ)
  → Claude claude-haiku-4-5 (рутина, мониторинг)
  → Bitrix24 / amoCRM REST API
  → Wazzup (WhatsApp)
  → Telegram Bot API
  → PostgreSQL / Supabase (хранение состояния)
```

### Цех №5 — Инженерный Hub
Для сложных мультиагентных решений (материалы в `01_INBOX/Gold_Tools/`):
- **LangGraph / CrewAI** — агентные цепочки
- **RAG на локальных данных** — векторные БД (Chroma, Qdrant)
- **Ollama / DeepSeek** — self-hosted LLM для клиентов с требованием безопасности
- **Cursor / Windsurf** — vibe coding для быстрой сборки прототипов
- **Docker** — изолированные контейнеры на сервере клиента

## ЭТАП 4 — МАРКЕТИНГ (Схема "Якорь + Эхо")

### Алгоритм
1. Берём реализованный кейс из `05_BIZ_RECIPES/` (#done)
2. **Якорь** — лонгрид на Habr (технари) или VC.ru (бизнес)
3. **Эхо** — адаптации: Telegram, TenChat, ВК, Яндекс Дзен
4. Опционально: HeyGen → Reels с цифровым аватаром

### Ритм публикаций
```
Понедельник: Habr или VC.ru (якорь)
Среда:       Telegram (инсайт из кейса)
Пятница:     TenChat + ВК (адаптации)
```
