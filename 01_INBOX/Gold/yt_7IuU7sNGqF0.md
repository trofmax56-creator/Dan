## Суть
Подробное руководство по интеграции n8n с Zoho CRM через API для автоматизации работы с лидами и бизнес-процессами без написания кода. Система позволяет автоматически синхронизировать данные, обрабатывать входящие лиды и запускать кастомные workflow-ы в Zoho CRM через визуальный конструктор n8n.

## Бизнес-сценарий
Маркетологи, менеджеры по продажам и админы Zoho CRM, которые хотят автоматизировать создание лидов, синхронизацию контактов между системами, отправку уведомлений и запуск действий в CRM на основе внешних триггеров (веб-хуки, расписание, события из других приложений).

## Технический стек
- n8n - платформа для бизнес-автоматизации
- Zoho CRM API - REST API для управления контактами и лидами
- OAuth 2.0 - протокол авторизации
- Zoho Developer Console - для создания приложений и получения ключей доступа
- HTTP/REST запросы
- JSON для структурирования данных
- Webhook - для приема данных из внешних источников
- Cron expressions - для расписания выполнения

## Связки инструментов
- Webhook (внешний источник) → n8n Trigger → Zoho CRM Node → Zoho CRM Database
- HTTP Request → n8n → OAuth Authentication → Zoho CRM API → Создание/Обновление записей
- Timer/Scheduler → n8n → Fetch Data from Zoho → Transform Data → Send Email/Webhook

## Конфигурация и параметры
- OAuth Client ID и Client Secret из Zoho Developer Console
- Выбор модуля Zoho CRM: Leads, Contacts, Accounts, Deals, Tasks и т.д.
- Маппирование полей: Email → email, First Name → first_name, Last Name → last_name, Company → account_name, Phone → phone и т.д.
- Настройка условий: IF module == 'Leads' THEN create in Leads else in Contacts
- Параметры HTTP запроса: Method (GET/POST/PUT), Headers (Authorization: Zoho-oauthtoken TOKEN), Body с JSON-данными
- Указание временного пояса для Scheduler нод
- Формат даты: YYYY-MM-DD для совместимости с Zoho CRM API

## Ключевые инсайты
- n8n имеет встроенную ноду для Zoho CRM, что значительно упрощает интеграцию по сравнению с использованием сырых HTTP-запросов
- OAuth-токен в Zoho действует 1 час, но n8n автоматически обновляет refresh-token, поэтому интеграция не требует ручного переавторизации
- При работе с большим объемом лидов (>100 в день) рекомендуется использовать batch-обработку для оптимизации производительности
- Webhook-триггеры в n8n требуют наличия готового URL для принятия данных из внешних источников - убедитесь что firewall не блокирует входящие запросы
- Zoho CRM API имеет rate limit 5000 запросов в час, при превышении используйте очередь (queue) в n8n для регулирования нагрузки
- При маппировании полей обратите внимание на обязательные поля в Zoho (Last_Name для лидов), их отсутствие приведет к ошибке создания записи
- Для тестирования используйте фиктивный вебхук через webhook.site - это позволит отловить формат данных из внешнего источника перед интеграцией

## Подводные камни
- Частая ошибка: забывают указать Last Name при создании Leads - поле обязательное в Zoho CRM, без него запрос вернет 400 ошибку
- OAuth-токен требует правильно указанного Redirect URI при создании приложения - если он не совпадает с тем что использует n8n, авторизация не будет работать
- Если workflow зависит от действия пользователя в Zoho (например, изменение статуса), убедитесь что в Zoho включены webhooks для этого модуля - по умолчанию они отключены
- n8n требует активного интернет-соединения даже для локального развертывания при работе с облачными API
- Для batch-обработки более 100 записей нужно использовать Loop ноду с функцией wait для соблюдения rate limits API
- Если данные не попадают в Zoho, проверьте логи n8n для увидения точного сообщения об ошибке - обычно это проблемы с маппированием полей или валидацией данных
