# 🗂️ Паспорт идеи: RAG-бот на документах компании

**Источник:** [RAG простыми словами: как научить LLM работать с файлами](https://youtube.com/watch?v=3k8rGsE4L0M) — Alexander K (2025-12-14)
**Категория:** Замена должности
**Заменяет:** Секретарь / специалист поддержки / HR по внутренним регламентам

---

## 💡 Что это даёт бизнесу
Сотрудники и клиенты получают мгновенные ответы из внутренней базы знаний (регламенты, прайсы, договоры, FAQ) без обращения к живому специалисту. Компания сокращает нагрузку на поддержку и HR на 40–70%. Система обновляется по мере загрузки новых документов без переобучения модели.

---

## ✅ Плюсы
- Быстрый старт: MVP за 1–2 недели на готовых фреймворках (LangChain, LlamaIndex)
- Понятная ценность для клиента — легко считается ROI через часы сэкономленного труда
- Масштабируется на любой объём документов без роста стоимости поддержки

## ❌ Минусы / Риски
- Качество ответов сильно зависит от качества и структуры исходных документов клиента
- Клиент боится отдавать внутренние документы в облако — нужно предлагать локальный деплой или объяснять безопасность

## 🧰 Артефакты для реализации

**Инструменты:** LangChain / LlamaIndex — RAG-фреймворки, Chroma / Qdrant / Weaviate — векторные базы данных, Ollama — локальный запуск LLM, n8n / Make.com — интеграция с CRM и мессенджерами, immers.cloud — облачный GPU-хостинг
**API:** OpenAI API (GPT-4o / text-embedding-3-small), Anthropic Claude API, Yandex GPT API (для российских клиентов с требованием локализации)

## 🚀 Шаги запуска
1. Шаг 1: Собрать документы клиента (PDF, DOCX, таблицы), очистить и разбить на чанки через LangChain TextSplitter
2. Шаг 2: Создать векторные эмбеддинги и загрузить в Chroma или Qdrant на VPS / immers.cloud
3. Шаг 3: Подключить LLM (OpenAI или локальная через Ollama), настроить retrieval-chain с промптом под специфику клиента
4. Шаг 4: Интегрировать чат-интерфейс в Telegram-бот, Bitrix24 или amoCRM через n8n/Make.com, провести тест на реальных вопросах
