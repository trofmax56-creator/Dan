## Суть
Автоматизация сбора заказов из WhatsApp/Telegram напрямую в Google Sheets без ручного ввода данных. При получении заказа в мессенджере информация автоматически парсится и добавляется в электронную таблицу за счёт использования n8n как оркестратора между платформами обмена сообщениями и облачным хранилищем данных.

## Бизнес-сценарий
Владельцы малого бизнеса, магазины электроники или розничной торговли, которые принимают заказы через WhatsApp или Telegram, и нуждаются в автоматическом логировании всех поступивших заказов в Google Sheets для учёта, анализа и управления инвентарем. Позволяет избежать ручного переписания данных и минимизировать ошибки.

## Алгоритм реализации
1. Шаг 1: Создание триггера в n8n — привязка Webhook или прямого подключения к WhatsApp API для отслеживания входящих сообщений с заказами. Выбор тригера 'Webhook' или 'WhatsApp Trigger Node' в зависимости от версии n8n.
2. Шаг 2: Парсинг текста заказа — использование узла JavaScript или regex для извлечения структурированных данных из свободного текста сообщения (товар, количество, цена, контакты покупателя). Применение регулярных выражений для разделения информации по категориям.
3. Шаг 3: Форматирование данных — создание объекта данных с полями: Дата заказа, Имя покупателя, Номер телефона, Товар, Количество, Сумма, Статус заказа. Установка текущего времени через функцию NOW() или динамическое заполнение полей.
4. Шаг 4: Интеграция с Google Sheets через API или встроенный Google Sheets Node в n8n. Настройка авторизации через OAuth2, выбор конкретной таблицы и листа для добавления строки. Указание диапазона ячеек или использование режима 'Append' для добавления новой строки.
5. Шаг 5: Добавление строки в таблицу — отправка сформированных данных в Google Sheets через узел 'Google Sheets' с параметром 'Append Row' или 'Update' в зависимости от требуемого действия. Проверка успешного добавления через лог выполнения.
6. Шаг 6: Опциональная отправка подтверждения — использование узла 'WhatsApp Send Message' или 'Telegram Send Message' для отправки покупателю подтверждения получения заказа с автоматически назначенным номером или статусом из Google Sheets.

## Технический стек
- n8n (open-source или cloud версия)
- WhatsApp API / Telegram Bot API
- Google Sheets API v4
- OAuth2 для авторизации
- Webhook для получения входящих сообщений
- JavaScript/Node.js для парсинга текста
- Regular Expressions (RegEx) для извлечения данных

## Связки инструментов
- WhatsApp Webhook → n8n WhatsApp Node → JavaScript Parser → Google Sheets API → Append Row
- Telegram Bot Webhook → n8n Telegram Trigger → Data Extraction → Google Sheets Update
- Incoming Message → Pattern Matching → Structured Data Object → Sheets Append

## Конфигурация и параметры
- Google Sheets Node: авторизация через OAuth2, выбор Spreadsheet ID из URL таблицы (формат: https://docs.google.com/spreadsheets/d/{SPREADSHEET_ID}/edit)
- Webhook Mode: установка публичного URL для получения POST запросов от WhatsApp/Telegram серверов
- Trigger Node: выбор события 'message' или 'order' в зависимости от типа платформы
- Data Mapping: указание соответствия полей входящего сообщения (message.text, message.from) с колонками Google Sheets (Товар, Покупатель, Номер телефона и т.д.)
- Append Row Setting: режим добавления строки в конец листа, без перезаписи существующих данных
- Regex Pattern: для парсинга заказа можно использовать паттерны типа /товар:\s*(.+?)(?:кол|количество)/i для извлечения названия товара
- Timestamp Field: автоматическое добавление текущего времени получения заказа через функцию `new Date().toLocaleString()` или встроенный узел Date

## Ключевые инсайты
- n8n позволяет создавать сложные многошаговые workflows без кодирования благодаря visual builder с предзаполненными узлами для популярных сервисов
- Использование Webhook требует настройки публичного URL (может быть сгенерирован n8n автоматически) — это получаемый мессенджером адрес для отправки данных о входящих сообщениях
- Google Sheets имеет ограничение на количество запросов в минуту (примерно 60 запросов/минуту для стандартного аккаунта) — при высокой нагрузке следует добавить очередь (Queue) в n8n
- Парсинг свободного текста из мессенджера может быть неточным — рекомендуется использовать структурированный формат заказов (например, меню кнопок в Telegram или Quick Replies в WhatsApp) или добавить AI-парсинг через Claude/ChatGPT API
- Авторизация Google Sheets через OAuth2 требует добавления n8n в список разрешённых приложений в Google Account — процесс автоматизирован в UI n8n, но может потребовать подтверждения доступа
- Для отправки подтверждения покупателю в WhatsApp используется Message Template API (требуется одобрение от Meta) — упрощённый способ: отправить текстовое сообщение через n8n WhatsApp Node
- При интеграции с несколькими WhatsApp номерами или Telegram ботами используйте условные узлы (If/Switch) в n8n для маршрутизации заказов в разные листы Google Sheets
- Для отладки используйте функцию 'Test' в каждом узле n8n и проверяйте вывод данных в JSON формате перед отправкой в Google Sheets
