## Суть
Создание AI-агента на базе n8n, который автоматически генерирует и отправляет холодные письма потенциальным клиентам, а затем отслеживает их ответы и автоматизирует весь цикл продаж от контакта до закрытия сделки.

## Бизнес-сценарий
SaaS-компании, агентства и B2B-предприятия используют этого агента для автоматизации холодной рассылки: система собирает контакты потенциальных клиентов, генерирует персонализированные письма через Claude AI, отправляет их через почтовые сервисы, отслеживает открытия и клики, и автоматически создаёт сделки в CRM при положительном ответе.

## Алгоритм реализации
1. Шаг 1: Триггер по расписанию (cron) — запуск рабочего процесса ежедневно в определённое время для проверки новых контактов и отправки писем
2. Шаг 2: Запрос контактов из источника данных (Google Sheets, CSV или API) — извлечение списка потенциальных клиентов с их почтой и основной информацией
3. Шаг 3: Цикл по контактам (Loop) — для каждого контакта выполнить следующие операции
4. Шаг 4: Генерация персонализированного письма через Claude API — передать информацию о контакте и компании, получить индивидуальное письмо с предложением
5. Шаг 5: Отправка письма через почтовый сервис (SendGrid, Gmail API или аналог) — отправка сгенерированного письма адресату с отслеживанием открытия
6. Шаг 6: Сохранение истории отправки в базу данных (логирование) — запись даты, времени, адресата и статуса отправки
7. Шаг 7: Условное ветвление (IF) — проверка получены ли ответы на письмо (через webhook или API интеграции почты)
8. Шаг 8: При получении положительного ответа — создание сделки в CRM (Bitrix24, amoCRM) с указанием статуса и суммы
9. Шаг 9: Отправка внутреннего уведомления в Telegram или Slack о новой возможности продажи
10. Шаг 10: Финальное логирование результата (успех/неудача/отложено) в аналитическую таблицу для отчётности

## Технический стек
- n8n (основная платформа автоматизации)
- Claude API (генерация персонализированных писем через Anthropic)
- SendGrid или Gmail API (отправка электронных писем)
- Google Sheets или PostgreSQL (хранение контактов и истории)
- Webhook (получение уведомлений об ответах)
- Bitrix24 API или amoCRM API (интеграция с CRM для создания сделок)
- Telegram Bot API или Slack API (уведомления о результатах)
- REST API для работы с различными сервисами
- JSON для структурирования данных и конфигурации нод

## Связки инструментов
- Контакты → n8n Loop → Claude API (генерация текста) → SendGrid API (отправка письма) → Webhook (отслеживание ответов) → Условие (IF) → Bitrix24 API (создание сделки) → Telegram API (уведомление)
- Google Sheets (источник контактов) → n8n → Claude API → Email Service → CRM → Telegram
- Database (логирование) ← каждый шаг процесса

## Конфигурация и параметры
- Нода Schedule: cron выражение '0 9 * * *' для запуска ежедневно в 9:00 UTC
- Нода HTTP Request для Google Sheets: метод GET, URL с range параметром для выборки контактов
- Нода Loop: итерация по массиву 'items' из результата предыдущей ноды
- Нода Claude API: параметр 'prompt' с шаблоном письма, temperature 0.7 для баланса креативности и консистентности
- Нода SendGrid: поля 'to', 'from', 'subject', 'html' с персонализированным контентом
- Webhook нода: URL для приёма уведомлений об открытии письма или ответах
- Нода Condition (IF): проверка поля 'status' === 'replied' или наличие 'email_response'
- Нода Bitrix24: метод 'deals.add', поля 'TITLE', 'OPPORTUNITY', 'CONTACT_ID'
- Нода Telegram: параметр 'chat_id' и 'text' с информацией о новой сделке
- Нода Set: переменная 'email_sent_timestamp' = new Date().toISOString()

## Ключевые инсайты
- Claude API генерирует письма с персонализацией на основе данных контакта и истории компании — это даёт 3-4x выше rate открытия vs стандартные шаблоны
- Webhook для отслеживания — критичный момент: нужна правильная настройка открытия письма и отслеживание кликов через UTM параметры в ссылках
- Loop нода в n8n работает последовательно по умолчанию, для параллельной обработки нужно использовать 'Batches' функцию
- Хранение контактов в Google Sheets позволяет легко добавлять новые цели без изменения кода — просто добавить строку
- Переменная окружения для API ключей (Claude, SendGrid, CRM) должна быть в формате ${VARIABLE_NAME} в полях n8n
- Добавление задержки (Delay нода) между отправками писем в loop — рекомендуется 1-5 сек чтобы избежать блокировки по rate limit
- Проверка дубликатов: перед созданием сделки в CRM нужна проверка наличия этого контакта через CRM API
- Логирование каждого шага в отдельную таблицу/колонку помогает отследить на каком этапе упала отправка письма
- Template переменные в письме: {{first_name}}, {{company_name}}, {{product_benefit}} должны быть в кавычках в JSON
- Ошибка 'API rate limit exceeded' решается добавлением задержек и использованием пула отправок с ограничением по времени
