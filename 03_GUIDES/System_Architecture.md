# 🏛 Архитектура Системы "Dan | Второй Мозг"

## 📂 Описание папок и процессов

### 1. 📥 00_RAW (Сырые данные)
- **Цель:** Сбор всей входящей информации. Сюда падают все результаты парсинга (parser.py, youtube_parser.py).
- **Статус:** Зона сырого потока, не для финального чтения.

### 2. 📨 01_INBOX (Входящие)
- **01_INBOX/Gold:** Место для хранения "золотых" статей, отобранных из 00_RAW. Основной источник аналитики.
- **01_INBOX/Digest:** Папка для хранения итоговых аналитических дайджестов.
- **processed_log.md:** Файл-лог для исключения дубликатов при анализе.

### 3. 🛠️ 02_TOOLS (Инструменты)
- Промпты, скрипты, настройки системы.

### 4. 📂 04_PROJECTS (Проекты)
- Активные рабочие проекты.

### 5. 💡 05_BIZ_RECIPES (Бизнес-рецепты)
- Библиотека готовых решений (n8n + AI + CRM).

### 6. 🏗️ 07_CONSTRUCTION (Стройка)
- Инженерные расчеты и справочники (GIP_Handbook).

## ⚙️ Конвейер знаний: Инструкция для Claude

### Шаг 1: Фильтрация (Из 00_RAW в 01_INBOX/Gold)
Промпт: "Просканируй 00_RAW. Полезные файлы (связки ИИ+CRM, гайды, новые модели, n8n схемы) перемести в 01_INBOX/Gold. Треш удали."

### Шаг 2: Аналитика (Дайджест из Gold в Digest)
**Алгоритм Данилы:**
1. Взять новые файлы из `01_INBOX/Gold` (свериться с `processed_log.md`).
2. Проанализировать основные мысли, выделить самое главное: новые функции, схемы автоматизации, связки.
3. **ОБЯЗАТЕЛЬНО:** К каждому пункту прикреплять прямую ссылку на статью (внутренние [[название]] или внешние URL).
4. Результат: `01_INBOX/Digest/Digest_YYYY_MM_DD.md`.
5. Обновить `processed_log.md`.

### Шаг 3: Переработка (Из 01_INBOX в 05_BIZ_RECIPES)
Промпт: "Возьми инновацию из Gold и оформи её как бизнес-рецепт в папке 05_BIZ_RECIPES по шаблону T_Recipe."
