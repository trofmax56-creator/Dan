### 1. Описание идеи

Компании, которые продают онлайн-курсы через Геткурс и ведут клиентскую базу в Bitrix24, сталкиваются с ручным переносом данных между системами. Vakas-tools — no-code платформа, которая автоматически синхронизирует контакты между Bitrix24 и Геткурс: при создании сделки в CRM клиент автоматически попадает на курс, и наоборот — прохождение курса обновляет статус в CRM.

Применение: медицинские клиники с обучением пациентов, образовательные компании, онлайн-школы с CRM.

### 2. Технический стек

- Vakas-tools (https://vakas-tools.ru) — платформа интеграции, no-code
- Bitrix24 REST API — источник данных о контактах и сделках
- Геткурс REST API — получение и обновление статусов учеников
- Webhook Bitrix24 — триггер при создании/изменении контакта или сделки
- OAuth2 / API-ключи обоих сервисов

### 3. Пошаговый план внедрения

**Шаг 1 — Регистрация в Vakas-tools**
```
https://vakas-tools.ru → Регистрация → Подключить Bitrix24
→ Получить API-ключ Bitrix24 (Администратор → Интеграции → REST API)
→ Вставить в Vakas-tools → Авторизовать
```

**Шаг 2 — Подключение Геткурс**
```
Геткурс → Настройки → Интеграция → API
→ Скопировать API ключ и URL школы
→ Вставить в Vakas-tools → Авторизовать
```

**Шаг 3 — Создание правила синхронизации**
```
Vakas-tools → Создать правило:
ТРИГГЕР:  Bitrix24 → новая сделка со статусом "Оплачено"
ДЕЙСТВИЕ: Геткурс → добавить пользователя на группу (курс)
МАППИНГ:  Email (Bitrix24 contact.EMAIL) = Email (Геткурс user.email)
```
⚠️ Критично: Email должен совпадать в обоих системах — это ключ идентификации.

**Шаг 4 — Обратная синхронизация (опционально)**
```
ТРИГГЕР:  Геткурс → пользователь завершил курс (статус: "Выпускник")
ДЕЙСТВИЕ: Bitrix24 → обновить поле контакта UF_COURSE_STATUS = "Выпускник"
          Bitrix24 → создать задачу менеджеру "Предложить следующий курс"
```

**Шаг 5 — Тест**
- Создать тестовую сделку в Bitrix24 с email test@test.ru
- Убедиться что user появился в Геткурс в нужной группе
- Проверить логи Vakas-tools на ошибки маппинга

### 4. Сценарии использования

| Ситуация | Triгер | Действие |
|---|---|---|
| Оплата курса | Сделка "Оплачено" в Bitrix24 | Добавить на курс в Геткурс |
| Завершение курса | Статус "Выпускник" в Геткурс | Задача менеджеру в Bitrix24 |
| Новая регистрация | Форма на сайте → Геткурс | Создать контакт в Bitrix24 |
| Истёк доступ к курсу | Триггер из Геткурс | Письмо с продлением из Bitrix24 |

### 5. Нюансы

- Email — единственный надёжный идентификатор между системами. Если email разный → дубли
- Vakas-tools платная: от ₽1 500/мес. При большом объёме (1000+ контактов) — окупается за 1 день
- Для Геткурс нужен тариф с API (не базовый)
- Геткурс API ограничивает запросы: не более 60/мин. При массовой загрузке — добавить паузы
