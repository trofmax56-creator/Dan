## 💡 Что это даёт бизнесу
Автоматическое построение и оптимизация маршрутов через Google Maps API + n8n без участия логиста. Система сама распределяет заказы по водителям, считает время в пути и уведомляет клиентов о статусе доставки. Сокращает ручной труд диспетчера на 60–80% и снижает пробег на 15–25%.

---

## ✅ Плюсы
- Google Maps API доступен в России через прокси/VPN или замена на Яндекс.Карты API — решение адаптируемо
- n8n — уже в стеке Студии, минимальный порог входа для команды
- Высокий спрос у курьерских служб, интернет-магазинов, сервисных компаний с выездными сотрудниками

## ❌ Минусы / Риски
- Google Maps API официально ограничен в РФ — нужна замена на Яндекс.Карты или 2GIS API, что добавляет время на адаптацию
- Интеграция с учётными системами клиента (1С, Bitrix24, WMS) может сильно увеличить scope и сроки

## 🚀 Шаги запуска
1. Шаг 1: Определить источник заказов клиента (CRM, таблица, WMS) и подключить его как триггер в n8n
2. Шаг 2: Настроить HTTP-запросы к Яндекс.Карты или 2GIS API для геокодинга адресов и построения маршрутов
3. Шаг 3: Реализовать логику распределения заказов по водителям (по зонам, загрузке, времени) в n8n workflow
4. Шаг 4: Подключить уведомления водителям (Telegram) и клиентам (SMS/WhatsApp) + запись результатов в CRM
