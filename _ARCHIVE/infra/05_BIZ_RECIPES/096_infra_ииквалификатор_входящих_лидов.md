# 🗂️ Паспорт идеи: ИИ-квалификатор входящих лидов

**Источник:** [Кейс. Квалификация клиентов: смотри как за тебя работает  и продает AI ассистент](https://youtube.com/watch?v=R4WiPdNClVc) — Ольга Омелюсик | Хватит блогить! Делай бизнес! (2025-04-09)
**Категория:** Замена должности
**Заменяет:** Менеджер по первичной квалификации лидов / SDR

---

## 💡 Что это даёт бизнесу
ИИ-ассистент автоматически обрабатывает входящие заявки в мессенджерах или на сайте, задаёт квалифицирующие вопросы (бюджет, срок, потребность, ЛПР) и передаёт в CRM уже сегментированных лидов с заполненными полями. Менеджер подключается только к тёплым и горячим клиентам, экономя до 60% времени отдела продаж. Особенно ценно для бизнесов с большим потоком заявок (50+ в месяц) — интернет-магазины, недвижимость, медклиники, образование.

---

## ✅ Плюсы
- Работает 24/7 без выходных — не пропускает ни одну заявку вне рабочих часов
- Стандартизирует качество квалификации: убирает человеческий фактор и забывчивость менеджеров
- Быстро окупается: замещает 0.5–1 ставку SDR (30–60 тыс. руб./мес.) при стоимости поддержки 5–10 тыс. руб./мес.

## ❌ Минусы / Риски
- Требует тщательной настройки сценариев под нишу клиента — универсального шаблона нет
- Клиенты со сложным B2B-циклом могут негативно реагировать на бота вместо живого менеджера на старте

## 🧰 Артефакты для реализации

**Инструменты:** n8n (оркестрация сценариев), amoCRM / Bitrix24 (запись лида и постановка задачи), Telegram Bot API / WhatsApp Business API (точка входа), Make.com (альтернатива n8n для быстрого старта)
**API:** Claude API (Anthropic) — основная LLM для диалога, OpenAI API — альтернатива, amoCRM REST API / Bitrix24 REST API, Telegram Bot API, WhatsApp Cloud API (Meta)

## 🚀 Шаги запуска
1. Шаг 1: Провести аудит текущей воронки клиента — выявить точки входа лидов (сайт, Telegram, WhatsApp, VK) и существующие квалифицирующие вопросы менеджеров
2. Шаг 2: Разработать сценарий диалога (5–8 вопросов), логику ветвления по ответам и критерии сегментации (горячий / тёплый / нецелевой)
3. Шаг 3: Собрать бота в n8n или Make.com: интеграция мессенджера → LLM (Claude) → парсинг ответов → запись в CRM с тегами и заполненными полями + уведомление менеджеру
4. Шаг 4: Провести тестирование на реальных лидах (2 недели), откалибровать промпты, настроить edge-cases (нет ответа, агрессия, нецелевой запрос), сдать клиенту с инструкцией
