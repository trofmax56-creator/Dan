# 🗂️ Паспорт идеи: ИИ-ассистент в Claude Projects

**Источник:** [How to Use Claude Cowork Better Than 99% of People (Full Guide)](https://youtube.com/watch?v=HTu1OGWAn5w) — Ben AI (2026-01-31)
**Категория:** Инфраструктура
**Заменяет:** —

---

## 💡 Что это даёт бизнесу
Клиент получает структурированную систему работы с Claude через Projects и кастомные инструкции, что позволяет многократно переиспользовать контекст без потерь. Сокращается время на повторяющиеся задачи (брифинг, шаблоны, промпты) за счёт готовых рабочих пространств. Подходит как внутренний инструмент для команды клиента или как база для построения ИИ-агентов поверх Claude API.

---

## ✅ Плюсы
- Низкий порог входа — не требует разработки, только грамотная настройка Projects и системных промптов
- Универсально для любого бизнеса: юристы, маркетинг, HR, продажи — переиспользуется как шаблон
- Легко масштабировать: продаётся как пакет настройки + обучение команды клиента

## ❌ Минусы / Риски
- Видео носит обучающий/гайдовый характер без демонстрации конкретного бизнес-кейса — ценность для Студии косвенная
- Низкий барьер копирования: клиент после обучения может настроить сам, без Студии

| Потенциал выручки Студии | За 6 месяцев при продаже 8–12 клиентам: 480 000 – 960 000 руб. (разовые внедрения) + 40 000 – 120 000 руб./мес на поддержке |

## 🧰 Артефакты для реализации

**Инструменты:** Claude.ai Projects, Claude API, Notion (для хранения промптов), n8n (если интеграция с CRM)
**API:** Anthropic Claude API, Anthropic Messages API

**Репозитории:**
- Репозиторий не упомянут — видео формата гайд без открытого кода

**Документация:**
- https://docs.anthropic.com/en/docs/build-with-claude/projects
- https://docs.anthropic.com/en/docs/system-prompts

## 🚀 Шаги запуска
1. Шаг 1: Выбрать нишу клиента (юристы / HR / продажи) и описать 5–7 повторяющихся задач, которые решает сотрудник вручную
2. Шаг 2: Создать Claude Project под нишу — загрузить базовые документы, прописать системный промпт с ролью, ограничениями и форматом вывода
3. Шаг 3: Собрать библиотеку из 10–15 кастомных промптов под типовые задачи клиента, упаковать в Notion или Google Docs
4. Шаг 4: Провести воркшоп для команды клиента (2–3 часа), сдать в эксплуатацию, предложить поддержку по подписке
