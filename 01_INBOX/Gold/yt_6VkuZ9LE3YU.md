## Суть
Автоматическое обнаружение and обработка ответов "Out of Office" (вне офиса) из поступающих писем через интеграцию Smartlead с Make.com, чтобы не тратить время на неактивные контакты.

## Бизнес-сценарий
Email-маркетологи, холодные продажи специалисты и sales команды, которые используют Smartlead для кампаний, получают множество автоматических ответов "Out of Office" и нуждаются в автоматическом обнаружении таких писем для переклассификации лидов или остановки кампании для конкретного контакта.

## Алгоритм реализации
1. Шаг 1: Настройка вебхука в Smartlead — в разделе интеграции создать webhook, который будет отправлять данные о входящих ответах при получении письма
2. Шаг 2: Создание сценария в Make.com — добавить триггер на получение вебхука от Smartlead с информацией о письме (Subject, Body, From адрес контакта)
3. Шаг 3: Парсинг текста письма — использовать модуль для анализа тела письма и проверки наличия ключевых слов Out of Office (out of office, out of the office, currently out, away, на отдыхе и т.д.)
4. Шаг 4: Условие IF — если обнаружены ключевые слова Out of Office, выполнить действие; иначе пропустить дальнейшие шаги
5. Шаг 5: Обновление статуса в Smartlead — отправить запрос через API Smartlead для обновления статуса лида (например, на 'Out of Office Detected' или паузу кампании для этого контакта)
6. Шаг 6: Логирование/Уведомление — отправить уведомление (например, через Slack или email) о найденном ответе Out of Office для мониторинга

## Технический стек
- Smartlead — platform для холодных писем с API и webhook поддержкой
- Make.com (Integromat) — no-code платформа для автоматизации workflows
- Webhook — для получения real-time событий от Smartlead
- HTTP модуль в Make.com — для отправки PUT/PATCH запросов в Smartlead API
- Text Parser/Regex модуль — для анализа содержимого письма
- Array/Iterator модуль (опционально) — если нужно обработать несколько писем
- Slack API (опционально) — для отправки уведомлений
- JSON — формат данных для API запросов и ответов

## Связки инструментов
- Webhook (Smartlead) → Make.com HTTP Trigger
- Make.com HTTP Trigger → Text Parser/Search (поиск ключевых слов)
- Text Parser → IF Condition (логика определения Out of Office)
- IF True → Smartlead API (HTTP PATCH/PUT) для обновления статуса лида
- IF True → Slack/Email модуль для уведомления
- Smartlead API → Make.com HTTP Request для получения/обновления данных лида

## Конфигурация и параметры
- Smartlead Webhook URL — скопировать из интеграций Smartlead и добавить в Make.com как триггер
- HTTP Headers — Authorization: Bearer {SMARTLEAD_API_KEY} для аутентификации в Smartlead API
- Ключевые слова для поиска: 'out of office', 'out of the office', 'currently out', 'away from desk', 'return', 'автоответчик' и похожие варианты
- API Endpoint Smartlead для обновления лида: /api/v1/lead/{lead_id} (точный endpoint нужно уточнить в документации Smartlead)
- Метод запроса: PATCH или PUT для обновления статуса
- Payload для обновления: { 'status': 'out_of_office_detected' } или аналогичная структура
- Поле для поиска в письме: письмо должно содержать Subject и Body — искать ключевые слова в обоих полях
- Статус коды ответа: 200 OK для успешного обновления, 4xx для ошибок валидации

## Ключевые инсайты
- Использование регулярных выражений (regex) позволяет ловить различные варианты Out of Office сообщений на разных языках и форматах
- Необходимо проверять не только Subject строку, но и Body письма, так как Out of Office ответ часто содержит детали в теле письма
- Smartlead API требует API Key, который нужно хранить в переменных окружения Make.com для безопасности
- Можно настроить разные действия в зависимости от типа Out of Office (временный или постоянный статус) — парсить даты возврата если они указаны
- Вебхук должен быть защищен — рекомендуется использовать签名验证 (signature verification) для подтверждения, что запрос действительно от Smartlead
- Важно исключить false positives — например, слово 'office' может появиться в других контекстах, поэтому использовать точный поиск фраз
- Rate limiting — учитывать лимиты API Smartlead при обновлении большого количества лидов одновременно
- Логирование всех срабатываний автоматизации критично для аудита и отладки — сохранять информацию о каждом обнаруженном Out of Office ответе
