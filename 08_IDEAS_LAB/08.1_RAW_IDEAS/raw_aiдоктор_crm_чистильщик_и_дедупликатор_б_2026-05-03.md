## 1. Название
**AI-Доктор CRM** — Чистильщик и дедупликатор базы

## 2. Суть
Еженощный n8n-агент сканирует Битрикс24 или amoCRM, находит дубли контактов по телефону/email, сливает карточки, нормализует форматы данных и присылает утром отчёт: «Найдено 34 дубля, исправлено 28, требует ручной проверки 6».

## 3. Бизнес-сценарий
ДО: В Битрикс24 компании 4 800 контактов. Треть — дубли, потому что менеджеры создают новую карточку вместо поиска. Маркетолог запускает рассылку — 600 человек получают письмо дважды. Клиент пишет: «Вы мне уже второй раз отправляете одно и то же» — репутация подмочена. РОП заказал «почистить базу» — менеджер убил на это 2 рабочих дня и всё равно пропустил половину. ИИ ДЕЛАЕТ: В 03:00 n8n запускает сканирование, Claude-агент сравнивает все контакты по нечёткому совпадению (Иванов И.И. = Иванов Иван = +79161234567), автоматически сливает карточки, оставляя более полную. КЛИЕНТ ПОЛУЧАЕТ: утром в Telegram отчёт «Сегодня ночью найдено 34 дубля, автоматически слито 28. Прикладываю 6 спорных — нужно ваше решение» со ссылками на карточки.

## 4. Логика
```
n8n Schedule (03:00 ежедневно) → Bitrix24 REST API (выгрузка всех контактов) → Claude API (нечёткое сравнение + решение о слиянии) → Bitrix24 REST API (merge/update) → Telegram-отчёт
```

## 5. Технический стек
- n8n
- claude-3-5-sonnet
- Bitrix24 REST API
- Telegram Bot API
- Dadata API

## 6. Связки инструментов
```
n8n Schedule Node → Bitrix24 crm.contact.list (батчами по 50) → Function Node (группировка по телефону/email) → Claude API (анализ спорных совпадений) → Bitrix24 crm.contact.merge → Telegram отчёт
```

## 7. Алгоритм реализации
1. Настроить n8n Schedule Node на запуск в 03:00 ежедневно; выгрузить все контакты из Bitrix24 через crm.contact.list батчами по 50 записей с полями: ID, NAME, PHONE, EMAIL, DATE_CREATE, ASSIGNED_BY_ID
2. В Function Node сгруппировать контакты по нормализованному телефону (убрать +7/8, пробелы, скобки) и email (toLowerCase); группы размером > 1 — кандидаты на слияние (обычно 15-40% базы)
3. Передать спорные группы (где имена различаются более чем на 30%) в Claude с запросом: 'Это один человек или разные? Верни JSON: {merge: true/false, master_id: ID_более_полной_карточки}'
4. Для групп с merge:true вызвать Bitrix24 crm.contact.merge, указав master_id и список slave_id; для спорных — создать задачу в Bitrix24 ответственному менеджеру
5. Нормализовать оставшиеся данные через Dadata API: привести телефоны к формату +7XXXXXXXXXX, email к нижнему регистру; утром отправить в Telegram сводный отчёт с числами и ссылками на спорные случаи

## 8. Почему купят (ROI)
По данным Experian, 22% CRM-баз имеют дубли, у компаний без контроля качества — до 40%. Дубли — это: двойные рассылки (жалобы клиентов), неверная аналитика воронки, конкуренция менеджеров за одного клиента, срыв сделок. Ручная чистка базы в 5 000 контактов — 2-3 дня работы (~24 000 руб. ФОТ). AI-Доктор делает это за ночь и повторяет каждый день бесплатно. Крючок: «Грязная CRM — это не проблема данных, это потерянные деньги.»
