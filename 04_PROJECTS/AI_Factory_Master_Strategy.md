## ЭТАП 0 — СБОР СЫРЬЯ

### Три параллельных потока

**Поток А — Telegram (раз в 2 недели, локально):**
1. `python parser_deep.py` — собирает посты из 21 канала
2. `python processor.py` — классифицирует GOLD/TRASH по ключевым словам
3. Результат: новые файлы в `01_INBOX/Gold/`

**Поток Б — YouTube-каналы (вручную, GitHub Actions):**
1. GitHub → Actions → YouTube Discovery → Run workflow
2. `youtube_parser.py` → RSS 10 каналов → `00_RAW/YouTube/`
3. `yt_processor_2.py` → Claude Haiku → `01_INBOX/Gold/yt_*.md`
4. `git pull` локально

**Поток В — Infra Discovery (авто, каждый понедельник):**
1. GitHub Actions запускает `infra_discovery.py` автоматически
2. 60 хирургических запросов: замена должностей, автоматизация отделов, ROI-кейсы
3. YouTube API → `00_RAW/Infra/YYYY-MM-DD_infra_raw.json`
4. `infra_processor.py` → Claude Sonnet → паспорта идей → `05_BIZ_RECIPES/` + `08_IDEAS_LAB/`
5. Дайджест → `01_INBOX/Gold/infra_digest_YYYY-MM-DD.md`

### Технический стек сбора
- YouTube Data API v3 (10k запросов/день, бесплатно)
- Telegram MTProto / telethon (`dan_session` — только локально)
- Claude claude-haiku-4-5 — классификация YT (дёшево)
- Claude claude-sonnet-4-6 — паспорта идей (качественно)
- GitHub Actions — облачный запуск без участия Максима

## ЭТАП 1 — КЛАССИФИКАЦИЯ И СКОРИНГ

### Demand Matrix
| Критерий | Вопрос | Диапазон |
|---|---|---|
| **Pain** | Клиент теряет деньги/время прямо сейчас? | 1–10 |
| **Dev** | Собрать на n8n + Claude за 1–2 недели? | 1–10 (инверсия) |
| **Profit** | Продать за ≥50к разово или ≥15к/мес? | 1–10 |
| **ИТОГ** | Pain + (10 − Dev) + Profit | max 30 |

### Роутинг по score
```
ИТОГ > 24  → 05_BIZ_RECIPES/         (в производство)
ИТОГ 20–24 → 08_IDEAS_LAB/08.2_SELECTED/  (следующий спринт)
ИТОГ 18–19 → 08_IDEAS_LAB/08.1_RAW_IDEAS/ (идея, не срочно)
ИТОГ < 18  → frontmatter: status: archive
```

### Два автоматических потока скоринга

**infra_processor** (1 видео = 1 паспорт идеи):
- Читает каждое видео из `00_RAW/Infra/`
- Claude анализирует: что заменяет, экономика, артефакты, шаги
- Сохраняет паспорт с полным описанием стека и цен

**gold_synthesizer** (N Gold-файлов = 1 синтезированный продукт):
- Берёт последние 40 Gold-файлов
- Claude синтезирует продукты в формате vern_ideas (9 пунктов)
- Бизнес-сценарий, стек, алгоритм, ROI, цена для клиента

## ЭТАП 2 — УПАКОВКА ПРОДУКТА (BIZ_RECIPE)

### Стандарт продукта (9 пунктов)
Каждый BIZ_RECIPE содержит:
1. **Суть** — механика решения (1–2 предложения)
2. **Бизнес-сценарий** — конкретная ситуация клиента
3. **Логика** — Trigger → Process → Action
4. **Технический стек** — инструменты с версиями
5. **Связки инструментов** — как сервисы общаются
6. **Алгоритм реализации** — шаги для n8n-инженера
7. **Почему купят (ROI)** — экономия ФОТ в рублях
8. **Экономика** — разработка / инфра / цена клиенту
9. **Источник** — ссылка на Gold-файл

### Векторы (из crm-ai-ideator.md)
- **Qualifier** — ускорение обработки входящего лида
- **Reanimator** — работа с базой отказников
- **Efficiency** — автоматизация рутины (КП, сметы, документы)

### Целевые ценники
| Продукт | Внедрение | Поддержка/мес | Срок |
|---|---|---|---|
| Простая интеграция (1 workflow) | 30–50к руб. | 10к руб. | 1 нед. |
| AI-агент для CRM | 80–150к руб. | 20к руб. | 2 нед. |
| AI-Sales Booster | 150–250к руб. | 30к руб. | 3–4 нед. |
| Завод ИИ-решений под ключ | 300–500к руб. | 50к руб. | 2–3 мес. |

## ЭТАП 3 — РАЗРАБОТКА (Цех Сборки)

### Алгоритм
1. Берём BIZ_RECIPE с тегом `#action_needed`
2. Разворачиваем self-hosted n8n (Docker на VPS)
3. Строим workflow по алгоритму из карточки
4. Тестируем на реальных данных клиента
5. Экспортируем JSON в репо `Dan` (библиотека чертежей)
6. Меняем тег: `#action_needed` → `#done`

### Стек разработки
```
Trigger (Webhook / Cron / CRM Event)
  → n8n (оркестратор)
  → Claude claude-sonnet-4-6 (сложный анализ)
  → Claude claude-haiku-4-5 (рутина, мониторинг)
  → Bitrix24 / amoCRM REST API
  → Wazzup (WhatsApp)
  → Telegram Bot API
  → PostgreSQL / Supabase (хранение состояния)
```

### Цех №5 — Инженерный Hub (новый)
Для сложных мультиагентных решений:
- **LangGraph / CrewAI** — агентные цепочки
- **RAG на локальных данных** — векторные БД (Chroma, Qdrant)
- **Ollama / DeepSeek** — self-hosted LLM для клиентов с требованием безопасности
- **Docker** — изолированные контейнеры на сервере клиента

## ЭТАП 4 — МАРКЕТИНГ (Схема "Якорь + Эхо")

### Алгоритм
1. Берём реализованный кейс из `05_BIZ_RECIPES/` (#done)
2. **Якорь** — лонгрид на Habr (технари) или VC.ru (бизнес)
3. **Эхо** — адаптации: Telegram, TenChat, ВК, Яндекс Дзен
4. Опционально: HeyGen → Reels с цифровым аватаром

### Ритм публикаций
```
Понедельник: Habr или VC.ru (якорь)
Среда:       Telegram (инсайт из кейса)
Пятница:     TenChat + ВК (адаптации)
```
