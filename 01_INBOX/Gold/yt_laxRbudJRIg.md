## Технический стек
- Open-source LLM (Ollama, LLaMA 2, Mistral)
- LangChain, CrewAI или AutoGPT для создания AI-агентов
- n8n или Zapier-альтернатива для workflow-автоматизации
- Docker и Docker Compose для контейнеризации
- Python с библиотеками requests, FastAPI для разработки кастомных интеграций
- PostgreSQL или SQLite для хранения данных
- Nginx для reverse proxy и балансировки нагрузки
- GitHub Actions для CI/CD
- Grafana или ELK Stack для мониторинга

## Связки инструментов
- LLM (Ollama/LLaMA) → LangChain/CrewAI → n8n → API интеграции (CRM/Email/Database)
- Webhook триггер → AI-агент (обработка запроса) → REST API внешних сервисов
- Docker контейнер → Kubernetes/Docker Swarm → Мониторинг и логирование

## Конфигурация и параметры
- Выбор открытого LLM модели (Mistral 7B, LLaMA 2 13B или меньше для экономии ресурсов)
- Развёртывание Ollama с выбранной моделью на локальной машине или сервере
- Настройка API endpoint для доступа к LLM (обычно http://localhost:11434 для Ollama)
- Создание prompt templates для каждого AI-агента с системным контекстом и инструкциями
- Конфигурация env переменных для API ключей интеграций (если требуются)
- Установка и настройка n8n на Docker с переменной N8N_HOST и портом 5678
- Создание workflows в n8n с использованием HTTP Request нод для интеграции с внешними API
- Настройка аутентификации и авторизации для защиты workflow endpoints

## Ключевые инсайты
- 1. Открытые LLM модели (Mistral, LLaMA) требуют значительно меньше вычислительных ресурсов, чем GPT-4, но достаточны для типовых автоматизаций
- 2. Использование Ollama позволяет запустить LLM локально без облачных сервисов и экономит месячно от $20 до $500+ в зависимости от использования
- 3. LangChain предоставляет удобные абстракции для создания многошаговых AI-агентов с памятью и цепочкой рассуждений
- 4. n8n как open-source альтернатива Zapier позволяет избежать ежемесячных подписок и получить полный контроль над workflow логикой
- 5. Комбинация локального LLM + n8n может заменить 3-5 дорогих SaaS-решений (Zapier, ChatGPT Plus, специализированные инструменты)
- 6. Критическое значение имеет выбор модели с оптимальным соотношением размера и производительности (7B параметров достаточно для большинства задач)
- 7. Использование Docker обеспечивает портативность и простоту развёртывания на различных хостингах без переустановки
- 8. Для production использования необходим мониторинг latency, error rates и возможность автоматического масштабирования

## Подводные камни
- Модели Ollama требуют GPU для приемлемой скорости обработки; CPU вариант работает крайне медленно (миллиарды операций)
- LLM галлюцинирует и может давать неправильные ответы даже на простые задачи; требуется валидация output и retry логика
- Открытые модели менее компетентны в английском и практически не работают на русском; требуется fine-tuning или более крупная модель
- n8n workflows имеют лимиты на количество executions и require.txt зависимостей; масштабирование требует upgrade плана
- Latency открытых LLM (2-10 секунд на один запрос) может быть неприемлем для real-time применений (chat, instant notifications)
- API ключи и sensitive data должны хранится в .env файлах, а не в коде; требуется шифрование и access control
- Интеграция с Legacy системами через API может быть затруднена отсутствием документации или rate limiting
- Бесплатные облачные серверы (Heroku, Replit) имеют ограничения по ресурсам и могут непредвиденно отключать приложения
