## 1. Название
**AI-Ревизор** — Контроллёр недовыставления услуг

## 2. Суть
Ежедневный n8n-агент сравнивает оказанные услуги из МИС или ERP с выставленными счетами в Битрикс24/1С, находит расхождения с помощью Claude и присылает руководителю отчёт: «Сегодня не выставлено услуг на 47 000 руб — вот список и ответственные».

## 3. Бизнес-сценарий
ДО: Владелец медицинской клиники подозревает, что врачи «забывают» выставлять услуги. Говорит администратору — та пожимает плечами. Вызывает бухгалтера — та делает выборку вручную за 4 часа и находит расхождение на 180 000 руб только за последний месяц. Это систематическая проблема, но контролировать каждый день некому. ИИ ДЕЛАЕТ: В 21:00 n8n автоматически выгружает из МИС список всех приёмов за день, из системы оплат — список выставленных счетов, Claude сопоставляет и находит: «Врач Козлова: 3 процедуры ЭКГ оказаны, 1 выставлена. Недовыставлено: 2 × 1 500 руб. = 3 000 руб.» КЛИЕНТ ПОЛУЧАЕТ: каждое утро в Telegram: «Вчера найдено недовыставлений на 12 400 руб. у 3 специалистов. Задачи на проверку созданы в Битрикс24».

## 4. Логика
```
n8n Schedule (21:00) → МИС API / CSV-экспорт (оказанные услуги) → Битрикс24 API (выставленные счета) → Claude API (сопоставление и анализ расхождений) → Bitrix24 (задачи ответственным) → Telegram-отчёт руководителю
```

## 5. Технический стек
- n8n
- claude-3-5-sonnet
- Bitrix24 REST API
- Google Sheets API
- Telegram Bot API

## 6. Связки инструментов
```
n8n Schedule → Google Sheets / МИС CSV (услуги) + Bitrix24 crm.deal.list (счета) → Function Node (JOIN по дате+врач+пациент) → Claude API (анализ расхождений) → Bitrix24 tasks.task.add → Telegram sendMessage
```

## 7. Алгоритм реализации
1. Настроить ежедневный импорт данных об оказанных услугах: для МИС с API — HTTP Request в n8n, для МИС без API — парсинг CSV-выгрузки из Google Drive (врач кладёт файл вечером); поля: дата, врач, пациент, код услуги, цена прайса
2. Выгрузить из Битрикс24 через crm.deal.list все закрытые сделки/счета за тот же день; нормализовать ФИО пациентов через Function Node (убрать отчество, привести к нижнему регистру для сравнения)
3. Передать два нормализованных списка в Claude с задачей: 'Найди услуги из Списка А, которых нет в Списке Б. Верни JSON: [{doctor, patient, service, price, reason}]'; Claude справляется с нечёткими совпадениями имён и синонимами услуг
4. Для каждого расхождения создать задачу в Битрикс24 через tasks.task.add: ответственный = старший врач или администратор, дедлайн = следующее утро, описание = детали расхождения с суммой
5. Отправить руководителю Telegram-сообщение с агрегированной сводкой: топ-3 специалиста по сумме недовыставлений, общая сумма за день и нарастающим итогом за месяц

## 8. Почему купят (ROI)
Медицинские клиники теряют от 5 до 15% выручки на недовыставлении — это не злой умысел, это забывчивость и спешка. Клиника с оборотом 2 000 000 руб/мес при потерях 8% = 160 000 руб/мес = 1 920 000 руб/год уходит в никуда. ФОТ контролёра, который мог бы это отслеживать вручную — 60 000-80 000 руб/мес. AI-Ревизор стоит в 10 раз дешевле и не пропускает ни одного случая. Крючок: «Каждый день без контроля — это деньги, которые вы уже подарили.»
