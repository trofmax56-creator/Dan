## Конфигурация и параметры
- Проблемы с перформансом при большом количестве узлов в workflow
- Ограничения на количество параллельных выполнений
- Проблемы с документацией по интеграциям
- Недостаточность встроенных коннекторов для популярных сервисов
- Сложность настройки кастомных интеграций через HTTP запросы
- Высокие затраты на масштабирование (цена за выполнения)
- Ограничение на размер данных, обрабатываемых в одном workflow

## Ключевые инсайты
- n8n имеет высокую стоимость за скейлирование при растущем объёме автоматизаций
- Интерфейс и документация n8n усложняют разработку сложных workflows
- Перформанс платформы деградирует при работе с большими объёмами данных
- Make.com предоставляет лучший баланс функциональности и цены для многих сценариев
- Для специфичных задач эффективнее использовать кастомный код на Python или JavaScript
- n8n хорошо подходит только для простых, часто используемых интеграций
- Альтернативы предлагают более удобный UI для управления сложными workflows
- Поддержка и сообщество Make.com и Zapier более активны и отзывчивы
- Миграция с n8n на альтернативы требует переработки workflows, но экономит средства

## Подводные камни
- Цена за выполнения (executions) в n8n растёт экспоненциально при масштабировании
- Бесплатный тариф n8n имеет серьёзные ограничения на функциональность
- Миграция существующих workflows между платформами требует полной переработки
- Документация некоторых интеграций в n8n устаревает быстро
- Кастомные узлы (custom nodes) требуют JavaScript-знаний и сложны в настройке
- Workflow с большим количеством условных ветвлений становятся неуправляемыми
- Нет встроенной версионности и истории изменений workflows на бесплатном плане
- Инстансы n8n требуют регулярного обслуживания при self-hosted версии
