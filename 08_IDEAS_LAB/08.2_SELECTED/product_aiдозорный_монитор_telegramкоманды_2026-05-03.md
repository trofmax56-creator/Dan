## 1. Название
**AI-Дозорный** — Монитор Telegram-команды

## 2. Суть
Читает корпоративные Telegram-чаты и каналы, каждые 4 часа формирует для руководителя сжатый дайджест: кто что сделал, какие задачи зависли, какие клиенты ждут ответа — без ручного скроллинга.

## 3. Бизнес-сценарий
ДО: Владелец стройкомпании состоит в 12 рабочих Telegram-чатах. Утром 200+ сообщений — листает 40 минут, половину всё равно пропускает. Клиент написал про срыв поставки — увидел через 6 часов, скандал. ИИ делает: в 9:00

## 4. Логика
```
Cron (каждые 4 ч) → Telegram Bot читает историю чатов → Claude классифицирует сообщения по типам → дайджест-сообщение руководителю в личный чат
```

## 5. Технический стек
- n8n
- claude-sonnet-4-5
- Telegram Bot API
- Telegram MTProto (Telethon/Pyrogram на VPS)

## 6. Связки инструментов
```
n8n Cron → Pyrogram (чтение истории чатов) → Claude (классификация: вопрос/решение/конфликт/клиент) → n8n формирует Markdown-дайджест → Telegram Bot отправляет руководителю
```

## 7. Алгоритм реализации
1. Развернуть Pyrogram-скрипт на VPS, авторизовать корпоративный аккаунт, передать список из 5–15 чатов для мониторинга
2. Настроить n8n Cron каждые 4 часа: забирать сообщения за период через Pyrogram API, фильтровать системные и медиа-сообщения
3. Claude получает пакет до 200 сообщений и классифицирует каждое по тегам: [вопрос без ответа] [решение принято] [клиент упомянут] [конфликт] [задача завершена]
4. n8n агрегирует теги, вычисляет сообщения без ответа дольше 2 часов — эти выносятся в блок «Требует реакции» первым пунктом дайджеста
5. Отправить структурированный дайджест руководителю в Telegram с кнопками «Ответить» (deep link в нужный чат) и «Закрыть» (пометить как обработано)

## 8. Почему купят (ROI)
Руководитель МСБ тратит 40–60 минут в день на мониторинг чатов (≈ 15 000 руб./мес своего времени) и всё равно пропускает критичные сообщения — средняя цена одного пропущенного клиентского запроса от 5 000 до 50 000 руб. потерянной сделки. AI-Дозорный даёт полную картину за 2 минуты трижды в день. Крючок: «Руководитель должен управлять командой, а не читать переписку.»
