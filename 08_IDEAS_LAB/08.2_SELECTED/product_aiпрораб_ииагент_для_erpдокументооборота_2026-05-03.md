## 1. Название
**AI-Прораб** — ИИ-Агент для ERP/Документооборота

## 2. Суть
ИИ-агент автоматически обрабатывает входящие документы (счета, накладные, акты) из почты и мессенджеров: распознаёт реквизиты, проверяет контрагента через Dadata, создаёт записи в 1С/МойСклад и уведомляет бухгалтера только если нужно подтверждение — всё остальное проводится без участия человека.

## 3. Бизнес-сценарий
ДО: бухгалтер Людмила каждый день получает 15–20 документов на почте и в WhatsApp от поставщиков — сканы накладных, PDF-счетов, фото актов. Каждый нужно открыть, вручную переписать реквизиты в 1С, проверить ИНН, создать контрагента если нового. 3–4 часа ежедневно на чистый ввод данных. ИИ делает: поставщик присылает скан накладной в WhatsApp → n8n перехватывает файл → Claude Vision извлекает: наименование контрагента, ИНН, номер документа, сумму, позиции → Dadata проверяет ИНН → если контрагент известен, создаёт приходный ордер в МойСклад автоматически → Людмила получает Telegram-уведомление: «Проведена накладная №847 от ООО Техснаб на 84 300 руб. — проверь и подтверди». Клиент получает: бухгалтер тратит 20 минут вместо 4 часов и только на контроль, а не на ввод.

## 4. Логика
```
Email IMAP / Wazzup Webhook (входящий документ/файл) → n8n → Claude Vision (извлечение реквизитов из скана) → Dadata API (валидация контрагента) → МойСклад/1С REST API (создание документа) → Telegram уведомление бухгалтеру
```

## 6. Связки инструментов
```
Email IMAP + Wazzup Webhook → n8n → Claude Vision API (OCR + структурирование) → Dadata API → МойСклад REST API (createInvoice) → Telegram Bot (алерт бухгалтеру)
```

## 7. Алгоритм реализации
1. Подключить два источника документов: IMAP-триггер на почтовый ящик бухгалтерии (вложения PDF/JPG) и Wazzup webhook для WhatsApp-файлов — объединить в единый поток n8n
2. Передать файл в Claude Vision с промптом: «Извлеки из документа в JSON: тип_документа, контрагент_название, ИНН, КПП, номер_документа, дата, сумма_без_НДС, НДС, позиции[] — если поле не читается, верни null»
3. Валидировать ИНН через Dadata API — получить официальное название, адрес, статус компании (действующая/ликвидированная); при ИНН с ошибкой — отправить на ручную проверку
4. Создать документ в МойСклад через REST API: POST /api/remap/1.2/entity/supply (приёмка) или /invoicein (счёт) с заполненными полями из Claude-ответа; при успехе вернуть ID документа
5. Отправить Telegram-уведомление бухгалтеру с кратким резюме: контрагент, сумма, тип документа, ссылка на созданный документ в МойСклад — одно нажатие «Подтвердить» или «Вернуть на доработку»

## 8. Почему купят (ROI)
Бухгалтер тратит 3–4 часа ежедневно на ручной ввод первичных документов — при ставке 80 000 руб./мес это 30 000–40 000 руб. в месяц на механическую работу, которую делает человек вместо машины. Ошибки при ручном вводе (неверный ИНН, сумма, дата) стоят компании штрафов и переделок. Крючок: «Бухгалтер нужен для учёта, а не для переписывания цифр».
