# 🗂️ Паспорт идеи: Локальный ИИ-агент на Ollama

**Источник:** [Local OpenClaw &amp; Ollama in 27 minutes](https://youtube.com/watch?v=n2a1FfqjHcU) — Keith AI (2026-03-10)
**Категория:** Инфраструктура

---

## 💡 Что это даёт бизнесу
Клиент получает приватную ИИ-инфраструктуру без утечки данных в облако — критично для юристов, медицины, финансов. Все модели работают на сервере клиента, данные не покидают контур компании. Снижает зависимость от OpenAI и валютных платежей.

---

## ✅ Плюсы
- Полная конфиденциальность данных — закрывает compliance-требования для регулируемых отраслей
- Нет ежемесячных платежей за API OpenAI — экономия от 30 000 руб./мес при активном использовании
- Работает без интернета — подходит для изолированных производственных сетей

## ❌ Минусы / Риски
- Требует мощного железа у клиента (GPU от 8 ГБ VRAM) — не каждый МСБ готов вложиться
- Качество локальных моделей (Llama, Mistral) заметно ниже GPT-4 — клиент должен понимать компромисс

## 🧰 Артефакты для реализации

**Инструменты:** Ollama, Docker, Open WebUI (бывш. OpenClaw), n8n, Nginx reverse proxy
**API:** Ollama REST API (localhost:11434), OpenAI-compatible API endpoint Ollama

**Репозитории:**
- ollama/ollama (github.com/ollama/ollama)
- open-webui/open-webui (github.com/open-webui/open-webui)

**Документация:**
- https://ollama.com/docs
- https://docs.openwebui.com

## 🚀 Шаги запуска
1. Шаг 1: Аудит железа клиента — GPU, RAM, ОС; подбор подходящей open-source модели (Llama 3, Mistral, Qwen)
2. Шаг 2: Развернуть Ollama + Open WebUI в Docker на сервере клиента, настроить reverse proxy и авторизацию
3. Шаг 3: Подключить Ollama API к n8n/Make — собрать 2-3 автоматизации под задачи клиента (саппорт-бот, суммаризация документов, генерация ответов)
4. Шаг 4: Обучить команду клиента, задокументировать, настроить мониторинг и план обновления моделей
