# 🗂️ Паспорт идеи: Self-hosted LLM для малого бизнеса

**Источник:** [Self-Hosted AI That&#39;s Actually Useful](https://youtube.com/watch?v=GrLpdfhTwLg) — Techno Tim (2024-07-05)
**Категория:** Инфраструктура
**Заменяет:** —

---

## 💡 Что это даёт бизнесу
Клиент получает приватный ИИ-ассистент внутри своей инфраструктуры без утечки данных в OpenAI/облака. Подходит для компаний с NDA, медицины, юристов, бухгалтеров — где нельзя отправлять данные наружу. Студия продаёт «под ключ»: сервер + Ollama/Open WebUI + интеграция с CRM.

---

## ✅ Плюсы
- Высокий спрос среди юристов, медклиник, господрядчиков — боль конфиденциальности реальная
- После установки клиент платит только за железо/электричество, нет ежемесячных API-платежей OpenAI
- Студия продаёт повторно: донастройка моделей, интеграция с Bitrix24/amoCRM, обучение персонала

## ❌ Минусы / Риски
- Локальные модели (Llama 3, Mistral) заметно слабее GPT-4o — клиент может разочароваться в качестве ответов
- Нужен приличный сервер (GPU от 12 ГБ VRAM) — это либо CapEx клиента ~150–300к, либо барьер для входа

## 🧰 Артефакты для реализации

**Инструменты:** Ollama, Open WebUI, LiteLLM, n8n, Docker
**API:** Ollama REST API, OpenAI-compatible API (через LiteLLM), Bitrix24 REST API

## 🚀 Шаги запуска
1. Шаг 1: Квалификация клиента — есть ли реальная боль конфиденциальности и бюджет на железо (или аренду GPU-сервера)
2. Шаг 2: Разворачиваем стек: сервер Ubuntu + Docker + Ollama + Open WebUI, выбираем модель под задачу (Llama 3.1 8B / Mistral 7B / Qwen2.5)
3. Шаг 3: Интегрируем с CRM клиента через n8n или Make.com — чат-бот внутри Bitrix24/amoCRM, автоответы, суммаризация
4. Шаг 4: Сдача, обучение сотрудников, подписание договора на поддержку 10–20к/мес
