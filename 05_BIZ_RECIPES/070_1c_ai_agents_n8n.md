### 1. Описание идеи

1С хранит все критические данные бизнеса (заказы, контрагенты, товары, финансы), но не умеет «думать». n8n забирает данные из 1С через REST API, передаёт их AI-агенту (Claude/GPT), который принимает решения и инициирует следующие действия: обновляет записи, отправляет уведомления, создаёт задачи в CRM.

Пример: каждое утро агент берёт дебиторскую задолженность из 1С → анализирует через Claude → отправляет менеджерам персональные списки клиентов для обзвона в Telegram.

### 2. Технический стек

- 1С (REST API — включить через Администратор → Интернет-сервисы)
- n8n (self-hosted предпочтительно — данные 1С конфиденциальны)
- Claude API (`claude-haiku-4-5-20251001` для анализа)
- HTTP Request ноды n8n
- Telegram Bot API (уведомления)
- Bitrix24 / amoCRM API (создание задач)

### 3. Пошаговый план внедрения

**Шаг 1 — Включить REST API в 1С**
```
1С Конфигуратор → Администрирование → Публикация HTTP-сервисов
→ Включить HTTP-сервисы
→ URL: http://your-1c-server:8080/base/hs/

# Проверить доступность:
curl -u login:password http://your-server:8080/base/hs/products/list
```

**Шаг 2 — Подключить 1С в n8n**
```
n8n → HTTP Request нода:
URL:     http://your-1c-server:8080/base/hs/{{endpoint}}
Method:  GET / POST
Auth:    Basic Auth (логин и пароль 1С)
Headers: Content-Type: application/json
```

**Шаг 3 — Построить AI-агент (пример: дебиторка)**
```javascript
// Workflow: Schedule Trigger (каждое утро 8:00)
// → HTTP Request → 1С API (получить должников)
// → Code Node (JS): структурировать данные
// → HTTP Request → Claude API (анализ)
// → Telegram → персональные сообщения менеджерам

// Claude промпт:
const prompt = `
Данные дебиторской задолженности:
${JSON.stringify(debtors, null, 2)}

Для каждого менеджера составь список клиентов для приоритетного обзвона.
Критерии приоритета: сумма долга > 50000 руб ИЛИ просрочка > 14 дней.
Формат: JSON { manager_name, clients: [{name, debt, days_overdue, action}] }
`;
```

**Шаг 4 — Обратная запись в 1С**
```javascript
// После звонка менеджер нажимает кнопку в Telegram
// → n8n ловит callback
// → HTTP Request → 1С API: обновить статус задолженности
// POST http://1c-server:8080/base/hs/debt/update
// Body: { client_id: "xxx", status: "called", comment: "Обещал оплатить 05.05" }
```

### 4. Сценарии использования

| Данные из 1С | AI-агент делает | Куда |
|---|---|---|
| Дебиторская задолженность | Список для обзвона по менеджерам | Telegram |
| Низкий остаток товара | Черновик заказа поставщику | Email / 1С |
| Просроченные договоры | Уведомления юристу | Telegram / задача в CRM |
| Новые заказы | Подтверждение + маршрутизация | Bitrix24 сделка |
| Аномалии в продажах | Алерт руководителю | Telegram |

### 5. Нюансы

- 1С HTTP-сервисы нужно явно разработать в конфигурации (требует 1С-программиста или готового расширения)
- Для облачной 1С (1С:Fresh) — API может быть ограничен тарифом
- Данные 1С конфиденциальны → использовать n8n self-hosted, не cloud
- Claude Haiku достаточно для большинства аналитических задач. Opus использовать только для сложных выводов

### 6. Оценка эффективности

- Сложность: 4/5 (требует настройки API 1С)
- Время внедрения: 5–10 дней
- ROI: автоматизация дебиторки в среднем снижает просрочку на 20–30% → прямой денежный эффект для клиента
