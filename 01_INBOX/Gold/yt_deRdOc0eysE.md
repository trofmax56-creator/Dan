## Суть
Автоматизация отслеживает все онлайн-покупки через ИИ, анализирует письма с квитанциями и категоризирует расходы. Система автоматически извлекает данные о покупках из email и записывает их в CRM или базу данных для управления личными финансами.

## Бизнес-сценарий
Пользователь получает письма с подтверждением покупок от различных интернет-магазинов. Автоматизация перехватывает эти письма, ИИ парсит информацию (название товара, сумму, дату, продавца), категоризирует расходы и синхронизирует в CRM или трекер расходов. Позволяет вести полный учет всех покупок без ручной работы.

## Алгоритм реализации
1. Шаг 1: Настройка триггера на получение email. Создайте в n8n/Make ноду Gmail Watch Email или эквивалент с фильтром по ключевым словам ('order', 'receipt', 'purchase', 'заказ', 'квитанция')
2. Шаг 2: Извлечение содержимого письма. Используйте ноду Extract Email Content для получения темы письма, отправителя, html-тела письма с информацией о покупке
3. Шаг 3: Отправка содержимого в Claude API или ChatGPT. Создайте промпт-инструкцию для ИИ: 'Из текста письма извлеки: название товара, цену, дату покупки, магазин, валюту, количество. Верни результат JSON'
4. Шаг 4: Парсинг JSON ответа от ИИ. Ноду помещайте результат через JSON.parse() и сохраняйте в переменные: product_name, price, date, store, currency, quantity
5. Шаг 5: Определение категории расходов. Используйте условную логику (switch-ноду) или отправьте второй промпт ИИ с вопросом 'К какой категории относится покупка: еда, одежда, электроника, развлечение, прочее?' на основе названия товара
6. Шаг 6: Запись в базу данных. Используйте ноду Airtable, Google Sheets или амоCRM для создания записи с полями: дата, товар, цена, категория, магазин, статус 'учтено'
7. Шаг 7: Уведомление пользователя (опционально). Отправьте SMS или Telegram сообщение: 'Покупка учтена: [название] на сумму [цена] [валюта]', чтобы пользователь видел что система работает

## Технический стек
- n8n или Make.com (платформа автоматизации)
- Gmail API или встроенный триггер Email Watch
- Claude API (claude-3-sonnet или claude-3-opus) для парсинга текста
- Альтернатива: ChatGPT API (gpt-4-turbo или gpt-3.5-turbo)
- Airtable API или Google Sheets API для хранения данных
- амоCRM API (если используется CRM вместо таблицы)
- JSON парсер/обработчик
- HTTP Request ноды для API вызовов
- Условная логика (IF, Switch ноды)
- Опционально: Telegram Bot API или Twilio для уведомлений

## Связки инструментов
- Email триггер (Gmail Watch) → HTTP Request (Claude API) → JSON Parse → Conditional (категоризация) → Airtable/Google Sheets Create Row
- Альтернативно: Email → ChatGPT API → Parsing → амоCRM API
- Уведомления: Automation → Telegram/SMS API → User

## Конфигурация и параметры
- Gmail API: требуется OAuth2 авторизация, scope: https://www.googleapis.com/auth/gmail.readonly
- Claude API endpoint: https://api.anthropic.com/v1/messages, метод POST, header Authorization: Bearer sk-...
- Промпт для Claude: 'Analyze this purchase receipt email and extract in JSON format: {product_name, price, currency, date, store, quantity}'
- Airtable: Base ID и Table Name из URL, поле для каждого столбца: 'Date', 'Product', 'Price', 'Category', 'Store'
- Google Sheets: Sheet ID из URL (между /d/ и /edit), Name диапазона типа 'A1:F1000'
- Email фильтр: ищем в теме письма ключевые слова order, receipt, purchase, confirmation, invoice
- Условие для категоризации: IF product_name содержит ['книга', 'kindle'] → 'Книги'; IF содержит ['рубашка', 'платье', 'обувь'] → 'Одежда' и т.д.
- Уведомление Telegram: требуется Bot Token, Chat ID, format сообщения: 'Покупка учтена: 📦 {product_name} - {price} {currency} от {store}'

## Ключевые инсайты
- ИИ способен парсить текстовые письма с точностью 95-98% без дополнительной обработки HTML, достаточно отправить plain text
- Используйте более дешевый Claude 3.5 Sonnet вместо GPT-4 для экономии: стоит в 3-5 раз дешевле при аналогичной точности парсинга
- Добавьте retry-логику: если ИИ не смог парсить (вернул ошибку) — пошлите письмо в архив, чтобы обработать вручную
- Категоризация работает лучше если в промпте передать примеры: 'Amazon Basics USB cable → электроника, Starbucks receipt → еда'
- Обучение: сохраняйте исходное письмо вместе с распарсенными данными для дальнейшего анализа траты и анализа трендов
- Интеграция с амоCRM: создавайте контакт магазина автоматически если его нет, привязывайте к каждой покупке для истории покупателя
- Ограничение: транзакции без четкой суммы в письме требуют ручной верификации — пошлите такие в отдельный очередь для review
