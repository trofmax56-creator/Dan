## 1. Название
**AI-Архивариус** — База Знаний с RAG

## 2. Суть
ИИ-агент на базе RAG превращает внутренние документы компании (прайсы, регламенты, скрипты, FAQ) в интерактивного корпоративного эксперта: менеджеры и клиенты задают вопросы в чат и получают точные ответы из реальных данных компании — без галлюцинаций и без поиска по папкам.

## 3. Бизнес-сценарий
ДО: новый менеджер Дима спрашивает коллег «Какой срок поставки для клиентов из Сибири при заказе от 500 кг?» — три человека отвечают по-разному, он идёт искать в папке «Документы 2023», тратит 20 минут, находит устаревший файл. ИИ делает: Дима пишет в корпоративный Telegram-бот: «Срок поставки Сибирь от 500 кг?» — RAG-агент за 3 секунды находит актуальный регламент поставок, извлекает нужный пункт и отвечает: «По регламенту логистики v2.4 (обновлён 15.03.2025): Сибирский ФО, от 500 кг — 7 рабочих дней, перевозчик СДЭК, базовая ставка 85 руб./кг». Клиент получает: менеджер отвечает клиенту точно и уверенно за 5 минут вместо часового поиска.

## 4. Логика
```
Telegram/WhatsApp вопрос → n8n Webhook → Supabase pgvector (векторный поиск по корпусу документов) → Claude (генерация ответа с контекстом из RAG) → ответ в мессенджер + лог запроса
```

## 5. Технический стек
- n8n
- claude-sonnet-4-5
- Supabase (pgvector)
- Telegram Bot API
- Wazzup

## 6. Связки инструментов
```
Документы PDF/DOCX → n8n парсинг → Supabase embeddings → Telegram Webhook → n8n → Supabase vector search → Claude API (RAG prompt) → Telegram sendMessage
```

## 7. Алгоритм реализации
1. Собрать корпус документов клиента (PDF, DOCX, Google Docs), настроить n8n-пайплайн для парсинга и чанкинга текстов по 512 токенов с перекрытием 50 токенов
2. Создать Supabase-базу с расширением pgvector, сгенерировать embeddings через Claude или OpenAI Embeddings API и загрузить 100% документов (типичный корпус МСБ — 50–200 документов)
3. Настроить Telegram-бота через BotFather, подключить к n8n webhook, реализовать поиск топ-5 релевантных чанков по косинусному сходству при каждом вопросе
4. Написать Claude-промпт в режиме RAG: «Отвечай ТОЛЬКО на основе предоставленных фрагментов документов, если ответа нет — честно скажи об этом, указывай источник»
5. Добавить логирование всех вопросов и ответов в Google Sheets для еженедельного аудита качества и пополнения базы документов

## 8. Почему купят (ROI)
Компания из 10 менеджеров тратит в среднем 45 минут в день каждый на поиск внутренней информации — 10 × 45 мин × 22 дня = 165 часов/мес = ~82 000 руб. ФОТ впустую. Новый сотрудник первые 2 месяца работает на 40% мощности из-за незнания регламентов — RAG сокращает адаптацию с 2 месяцев до 2 недель. Крючок: «Ваши документы уже содержат все ответы. Просто никто не может их найти».
