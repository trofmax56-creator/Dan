# ⚙️ Техническая схема n8n: AI-Sales Booster (Этап 1)

## 1. Входные данные (Webhook)
### А. Телефония (МТС/UIS)
- **Метод:** POST
- **URL:** `[Твой_URL_n8n]/webhook/booster-call-sync`
- **Логика:** Принимает JSON об окончании звонка. Извлекает `recording_url` и `phone`.

### Б. Мессенджеры (Wazzup)
- **Метод:** POST
- **URL:** `[Твой_URL_n8n]/webhook/booster-chat-sync`
- **Логика:** Принимает входящие сообщения. Извлекает `text` и `chat_id` (телефон).

---

## 2. Транскрибация (Speech-to-Text)
- **Узел:** HTTP Request
- **API:** OpenAI Whisper (или аналог)
- **Параметры:**
  - `file`: `{{ $node["Webhook"].json["recording_url"] }}`
  - `model`: `whisper-1`

---

## 3. Мозги системы (Claude 3.5 Sonnet)
- **Узел:** AI Agent / Advanced AI
- **Системный промпт (Утвержден):**
> "Инструкция: Ты — ИИ-ассистент по квалификации лидов. Проанализируй текст разговора/переписки. 
> Твоя цель — извлечь данные в формате JSON:
> 1. `name`: Имя клиента (если есть).
> 2. `phone`: Номер телефона.
> 3. `budget`: Примерный бюджет сделки (числом).
> 4. `pain`: Главная проблема/запрос клиента.
> 5. `deadline`: Когда клиенту нужно решение.
> 
> Если данных нет, в поле пиши 'null'. 
> Если данных недостаточно для квалификации, в поле `next_question` напиши вежливый вопрос, который менеджер (или я) должен задать следующим."

---

## 4. Интеграция с CRM (Bitrix24 / amoCRM)
- **Шаг 1:** Поиск контакта по `{{ phone }}`.
- **Шаг 2 (IF):** Если найден -> Обновить сделку. Если НЕТ -> Создать контакт + Создать сделку.
- **Шаг 3:** Записать данные в доп. поля:
  - `Бюджет (AI)` -> `{{ budget }}`
  - `Боль клиента (AI)` -> `{{ pain }}`
  - `Срок (AI)` -> `{{ deadline }}`
  - `Тег` -> `AI_BOOSTER_QUALIFIED`
