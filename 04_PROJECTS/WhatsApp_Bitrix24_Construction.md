# 🏗️ WhatsApp-бот + Bitrix24: автоматизация заявок в строительстве

> WhatsApp Business API + n8n + Bitrix24 → клиент пишет в WA, данные автоматически создают сделку в CRM, менеджер получает уведомление.

## Боль клиента (строительные компании)

- Заявки приходят в WhatsApp, менеджер вручную переносит в CRM
- Повторные продажи (новые объекты, допработы) теряются — нет автоматического касания
- Клиент ждёт ответа — менеджер занят, статус заявки неизвестен

## Решение

Двусторонняя интеграция WhatsApp ↔ Bitrix24:
1. Клиент пишет → бот принимает (имя, телефон, описание объекта)
2. n8n создаёт/обновляет контакт и сделку в Bitrix24
3. Менеджер получает уведомление с ссылкой на сделку
4. После завершения проекта: автокасание через 6 мес. (новые предложения)

## Технический стек

```
WhatsApp Business API (Webhook)
  → n8n Webhook Trigger
  → Code Node: парсинг имени, телефона, описания из сообщения
  → HTTP Request: Bitrix24 crm.contact.list (поиск по телефону)
    → EXISTS → crm.deal.update (обновить)
    → NOT EXISTS → crm.contact.add + crm.deal.add (создать)
  → HTTP Request: Bitrix24 im.notify.system.add (уведомить менеджера)
  → HTTP Request: WhatsApp API (подтверждение клиенту)
  → Schedule (через 6 мес.): автокасание с новым предложением
```

**Критический момент — маппинг полей Bitrix24:**
```
UF_CRM_PHONE → телефон клиента (для поиска дублей)
UF_CRM_DEAL_STAGE → этап воронки при создании
```
