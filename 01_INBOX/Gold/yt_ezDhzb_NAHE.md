## Суть
Автоматизация синхронизации лидов из SmartLead в Attio через Make.com при отсутствии контактной информации. Система автоматически обрабатывает неполные данные о лидах и передает их в CRM Attio для дальнейшей работы.

## Бизнес-сценарий
Sales-команды и маркетологи используют SmartLead для генерации лидов, но часто получают контакты без полной информации (например, только имя и компания). Автоматизация передает эти неполные данные в Attio, где система может дальше их обогащать или отправлять для ручной обработки, экономя время на ручном копировании и валидации.

## Алгоритм реализации
1. 1. Триггер в Make.com: Webhook или регулярный чек SmartLead API для получения новых лидов
2. 2. Фильтр данных: Проверка наличия или отсутствия контактной информации (email, phone) через условный блок If/Else
3. 3. Обогащение данных: Маппинг полей SmartLead (имя, компания, позиция) на поля Attio
4. 4. Создание лида в Attio: HTTP-запрос POST к Attio API с преобразованными данными
5. 5. Обработка ошибок: Логирование случаев с отсутствующей информацией, отправка уведомлений или сохранение в резервный лист
6. 6. Завершение цикла: Отмечание лида как обработанного в SmartLead или архивирование

## Технический стек
- Make.com (основная платформа автоматизации)
- SmartLead API (источник лидов)
- Attio API (целевая CRM)
- Webhooks (для триггеров)
- HTTP-модули Make.com (GET/POST запросы)
- JSON-трансформация данных
- Условные блоки If/Else
- Google Sheets (опционально, для логирования)

## Связки инструментов
- SmartLead API → Make.com Webhook
- Make.com HTTP Module → Attio API
- Make.com Router → If/Else (контроль потока по наличию данных)
- Attio Webhook ← Make.com (обратная связь о создании)

## Конфигурация и параметры
- API ключи: SmartLead API key, Attio API key, Attio Workspace ID
- Endpoint SmartLead: https://api.smartlead.ai/v1/leads или аналогичный
- Endpoint Attio: https://api.attio.com/v2/records/contacts
- Поля для маппинга: first_name, last_name, company_name, job_title, email (опционально), phone (опционально)
- Статус лида при создании в Attio: 'New', 'No Contact Info', или кастомный статус
- Параметр фильтра: IS_EMPTY(email) OR IS_EMPTY(phone) для определения неполных данных
- Заголовки HTTP: Content-Type: application/json, Authorization: Bearer [TOKEN]

## Ключевые инсайты
- 1. Обработка неполных данных: Система не отклоняет лидов без контактов, а создает их с пометкой 'No contact info given' для последующего обогащения
- 2. Маппинг полей: SmartLead передает данные в определённом формате, необходимо правильно преобразовать в структуру Attio
- 3. Webhook vs Polling: Make.com может использовать webhook для мгновенной обработки или регулярный polling каждые N минут
- 4. Условная логика: Важно разделить процесс для лидов С и БЕЗ контактной информации — разные действия следующие
- 5. API Rate Limits: SmartLead и Attio имеют ограничения по количеству запросов, нужна правильная настройка задержек
- 6. Обработка ошибок API: 401 (неавторизованный), 403 (запрещено), 429 (слишком много запросов), 500 (ошибка сервера) требуют разных обработчиков
- 7. Формат дат: Убедиться, что даты приходят в одном формате (ISO 8601 рекомендуется)
- 8. Дублирование: Проверить через unique ID или email, чтобы не создавать дубли в Attio

## Подводные камни
- ⚠️ Отсутствие контактной информации может привести к трудности идентификации лида в Attio — используйте комбинацию company_name + job_title как уникальный ключ
- ⚠️ Поля в API SmartLead могут отличаться от документации в зависимости от версии и подписки — всегда проверяйте реальный формат ответа
- ⚠️ Attio требует обязательное наличие некоторых полей (например, workspace_id), иначе запрос вернёт 400 Bad Request
- ⚠️ Если email в SmartLead содержит опечатку или невалидный формат, Attio может отклонить создание контакта — нужна валидация
- ⚠️ Webhook SmartLead может не срабатывать, если не настроена URL-адрес с https и правильный port (обычно 443)
- ⚠️ Rate limiting: если обрабатывать более 100 лидов в минуту, Make.com может добавить задержки — учитывайте в плане
- ⚠️ Поле 'No contact info given' может быть не стандартное для Attio — нужно создать кастомное поле предварительно или использовать статус
- ⚠️ При обновлении маршрутов или ключей API в Make.com прошлые выполнения могут заависнуть — всегда останавливайте сценарий перед изменениями
