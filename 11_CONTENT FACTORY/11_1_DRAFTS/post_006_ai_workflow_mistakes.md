# Пост: 7 ошибок которые убивают AI-автоматизации

---

Запустили AI-воркфлоу, работал неделю — и сломался.

Разбираю ошибки, которые встречаю в 80% проектов.

---

**1. Не валидировать JSON из LLM**

Claude вернул не JSON — workflow упал. Добавляйте try/catch и проверку структуры перед обработкой. LLM иногда «фантазирует» форматы.

**2. Игнорировать rate limits API**

Отправили 200 запросов за минуту — сервис заблокировал. Решение: очередь + exponential backoff retry. В n8n это нода Wait + Loop.

**3. Нет обработки пустых ответов**

AI вернул пустую строку — следующая нода упала на `null.field`. Проверяйте: если ответ пустой — workflow должен завершиться корректно, не с ошибкой.

**4. Весь контекст в одном промпте**

Промпт на 10 000 токенов = медленно + дорого + Claude «теряет» конец. Разбивайте на шаги: сначала классификация → потом детальный анализ.

**5. Нет логирования**

Воркфлоу отработал — что именно он сделал? Если нет Google Sheets с логами, отладка займёт в 5 раз больше времени.

**6. Забыть про часовые пояса**

Cron `0 9 * * *` — это UTC. В Москве это полдень. Проверяйте timezone при настройке расписания.

**7. Self-hosted n8n без бэкапа**

Сервер упал — все workflows потеряны. Настройте экспорт workflows в Git или S3 раз в день. Это 10 минут настройки.

---

Самая дорогая ошибка из практики: забыли про rate limit, Claude сгенерировал 5000 запросов за час, счёт за API — $180 за одну ночь.
