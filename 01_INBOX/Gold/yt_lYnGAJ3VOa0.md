## Суть
Клонирование и кастомизация готовой AI lead automation workflow в n8n для масштабирования маркетинговых кампаний. Позволяет быстро создавать несколько аналогичных автоматизаций с разными параметрами без необходимости начинать с нуля.

## Бизнес-сценарий
Маркетологи, lead generation агентства и B2B компании используют эту автоматизацию для масштабирования процесса генерации и квалификации лидов с помощью AI. Система обрабатывает входящие данные лидов, передает их в Claude для анализа, и сохраняет результаты в CRM.

## Алгоритм реализации
1. Шаг 1: Откройте готовую workflow в n8n и нажмите кнопку 'Clone' для дублирования конфигурации
2. Шаг 2: Переименуйте клонированную workflow на новое имя, соответствующее новой маркетинговой кампании или источнику лидов
3. Шаг 3: Обновите триггер (webhook или HTTP-endpoint) с новым URL для получения данных из нового источника (API партнера, форма на сайте и т.д.)
4. Шаг 4: Модифицируйте промпт для Claude API согласно специфике новой целевой аудитории или критериям квалификации лидов
5. Шаг 5: Переконфигурируйте интеграцию с CRM (Bitrix24, amoCRM, Pipedrive) - обновите токены/API ключи и маппинг полей для корректного сохранения лидов
6. Шаг 6: Протестируйте workflow отправкой тестового вебхука с примером данных лида
7. Шаг 7: Запустите в продакшене и мониторьте логи для отслеживания обработки лидов

## Технический стек
- n8n (платформа для workflow automation)
- Claude API (Anthropic) - для AI анализа лидов
- Webhook для получения данных
- REST API для интеграции с внешними сервисами
- Bitrix24 API / amoCRM API / Pipedrive API (для CRM)
- JSON парсер в n8n
- HTTP Request ноды для API вызовов
- Set/Edit Fields ноды для маппинга данных

## Связки инструментов
- Webhook в n8n ← получает данные лида → HTTP Request (Claude API) ← отправляет промпт для анализа → Claude API ← возвращает оценку/классификацию → Set Fields ← форматирует данные → HTTP Request (CRM API) ← сохраняет лида в CRM

## Конфигурация и параметры
- Триггер: Webhook с методом POST, параметры тела: email, name, phone, company, leadSource
- Claude API ноды: Model - claude-3-5-sonnet (или claude-3-opus), Temperature - от 0 до 1 (в зависимости от задачи), Max tokens - 1000-2000
- Системный промпт: 'You are a lead qualification expert. Analyze the provided lead and classify as hot/warm/cold based on criteria...'
- CRM интеграция: Поля маппинга - name → Имя, email → Email, phone → Телефон, qualityScore → Кастомное поле Оценка Качества
- Переменные окружения: API_KEY для Claude, CRM_API_KEY, WEBHOOK_URL
- Условие ветвления: IF qualityScore >= 70 THEN сохранить в high-priority, иначе в general queue

## Ключевые инсайты
- При клонировании workflow все внутренние параметры и промпты копируются автоматически - нужно изменить только внешние API ключи и триггер URL
- Для разных источников лидов используйте одну базовую workflow и просто обновляйте вебхук URL - это экономит время на разработку
- Claude API дешевле (5-10x) чем GPT-4, но сравним по качеству анализа лидов - оптимален для lead qualification
- Маппинг полей в CRM должен быть идентичным между всеми клонированными workflow для унификации данных
- Используйте переменные в n8n вместо hardcode значений - так проще масштабировать и обновлять конфигурацию
- Тестируйте каждую клонированную workflow с реальными примерами данных перед запуском в продакшене
- Добавьте логирование ошибок (Slack уведомления) для отслеживания падения workflow при массовой обработке лидов

## Подводные камни
- При клонировании workflow не забудьте обновить API ключи - используя старые ключи приведет к ошибкам авторизации
- Если использовать одинаковый вебхук URL для нескольких клонов - все они будут срабатывать на одно событие и произойдет дублирование лидов
- Claude API имеет rate limit (requests/minute в зависимости от плана) - при массовой обработке лидов может возникнуть throttling
- Маппинг полей в разных CRM работает по-разному (в Bitrix нужен поле ID, в amoCRM нужна структура массива) - внимательно читайте документацию
- Промпт для Claude должен быть четким и структурированным (JSON format output) для корректного парсинга результата в n8n
- Если вебхук падает - workflow может терять данные, нужна система очереди (Redis, очередь в CRM) для надежности
- Не изменяйте структуру данных между триггером и Claude API - это нарушит парсинг и может привести к ошибкам анализа
