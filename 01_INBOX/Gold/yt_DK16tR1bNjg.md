## Суть
Интеграция Claude Code и NoteBookLM для создания автоматизированных навыков (Skills) — система объединяет возможности кодирования Claude с аналитическими возможностями NoteBookLM для генерации и обработки сложных рабочих процессов с минимальным ручным кодированием.

## Бизнес-сценарий
Разработчики и специалисты по автоматизации используют интеграцию Claude Code + NoteBookLM для создания специализированных Skills (навыков) — автономных модулей для обработки данных, генерации контента, анализа информации и автоматизации рутинных операций в бизнес-процессах, сокращая время разработки и улучшая качество результатов.

## Алгоритм реализации
1. 1. Подготовка исходных данных и документации в NoteBookLM — загрузка текстов, документов, инструкций, которые послужат основой для генерации Skills
2. 2. Активация Claude Code в NoteBookLM — включение функции генерации кода на основе загруженных материалов для автоматического создания шаблонов Skills
3. 3. Генерация первичного кода через Claude — система анализирует загруженные материалы и генерирует код на Python/JavaScript для базовой функциональности
4. 4. Доработка и тестирование сгенерированного кода — редактирование параметров, переменных, логики условий в Claude Code, тестирование на примерах из документации
5. 5. Экспорт готового навыка (Skill) и интеграция в workflow — сохранение финального кода, экспорт в необходимом формате (API, функция, модуль) и подключение к автоматизационной платформе (n8n, Make, Zapier)
6. 6. Монитор и итерация — отслеживание работы Skills в реальных условиях, логирование результатов, обновление логики на основе полученных данных в NoteBookLM

## Технический стек
- Claude Code (встроенный редактор кода в интерфейсе Claude)
- NoteBookLM (Google — платформа для анализа и синтеза документов)
- Python 3.8+
- JavaScript/Node.js
- API Claude (claude-3-sonnet, claude-3.5-sonnet)
- REST API (для экспорта и интеграции)
- Платформы автоматизации: n8n, Make.com, Zapier
- Webhook (для получения данных)
- JSON (формат обмена данными)
- Git/GitHub (версионирование кода Skills)

## Связки инструментов
- Документация → NoteBookLM → Claude Code → Генерация функции → Экспорт JSON → n8n Webhook → API обработка → Возврат результата
- Исходные материалы (PDF/Docx) → NoteBookLM Audio Overview → Claude Code (с контекстом из материалов) → Готовый Skill
- Skill (сгенерированный код) → API обертка → Интеграция в Make/Zapier → Автоматический запуск по триггерам

## Конфигурация и параметры
- Активация Claude Code в NoteBookLM: включить опцию 'Code Generation' в настройках проекта
- Параметры Claude API: model='claude-3.5-sonnet', temperature=0.5-0.7 (для креативного кода с предсказуемостью), max_tokens=2000-4000
- NoteBookLM Project Settings: язык по умолчанию (русский/английский), формат экспорта (JSON/Python/JavaScript)
- Webhook endpoint для получения запросов: /api/v1/skill-execute с методом POST
- Заголовки запроса: Content-Type: application/json, Authorization: Bearer {API_KEY}
- Поля Request Body: input_data (объект с параметрами), skill_id (идентификатор навыка), context (дополнительный контекст)
- Response format: {status: 'success', result: {...}, execution_time: ms}
- Переменные окружения: CLAUDE_API_KEY, NOTEBOOKLM_PROJECT_ID, SKILL_DATABASE_URL

## Ключевые инсайты
- Claude Code автоматически генерирует 70-80% рабочего кода на основе контекста из загруженных документов в NoteBookLM, сокращая время разработки Skills с недель до часов
- NoteBookLM Audio Overviews (автоматические видеоуроки) из исходных материалов помогают Claude лучше понять требования — фиксируйте интонацию и порядок информации в исходном контексте
- Skills (сгенерированные навыки) могут быть упакованы как отдельные callable функции и переиспользоваться в разных workflows без изменения кода
- Критичный момент: указывать в исходных документах для NoteBookLM не только ЧТО должен делать код, но и ПРИМЕРЫ входных данных и ожидаемых выходов — это резко повышает качество генерации
- Для сложных Skill'ов с множеством условий используйте несколько итераций: сначала основной функционал, потом edge cases, потом оптимизация — каждый раз добавляя контекст в NoteBookLM
- API интеграция сгенерированных Skills: используйте OpenAI-совместимый формат для максимальной совместимости с платформами автоматизации
- Версионирование Skills критично — сохраняйте каждую итерацию с комментариями в Git, указывайте версию Skill в metadata для отката при необходимости
- Мониторинг: логируйте все вызовы Skills с timestamp, input, output, время выполнения — это позволит Claude автоматически выявлять проблемы при переобучении
