## Суть
На примере реального кейса показано внедрение ИИ-агента на базе n8n в компанию. Агент автоматизирует рутинные бизнес-процессы, обеспечивая экономию времени сотрудников и улучшение качества обработки данных.

## Бизнес-сценарий
Компания внедряет ИИ-агента для автоматизации обработки входящих заявок/запросов, анализа данных и генерации ответов. Агент принимает информацию из различных источников (почта, формы, чаты), обрабатывает её через Claude/GPT и отправляет результаты в CRM или другие системы.

## Алгоритм реализации
1. 1. Агент получает входящие данные через вебхук или триггер (новая заявка, письмо, сообщение)
2. 2. Данные передаются в n8n с указанием типа задачи и контекста
3. 3. n8n отправляет информацию в API ИИ-модели (Claude/ChatGPT/Gemini) с подготовленным промптом
4. 4. ИИ анализирует входные данные и генерирует рекомендацию/ответ
5. 5. Результат обрабатывается условной логикой (IF/SWITCH) для определения действия
6. 6. Данные сохраняются в базу/CRM с присвоением статуса
7. 7. Отправляется уведомление ответственному или автоматический ответ клиенту

## Технический стек
- n8n (платформа оркестрации)
- Claude API (или ChatGPT/Gemini)
- Webhook/HTTP-триггеры
- REST API интеграции
- PostgreSQL/MongoDB (опционально для истории)
- Email-интеграция (SMTP/IMAP)
- CRM API (Bitrix24/amoCRM/HubSpot)
- JSON парсер
- Условная логика (IF/SWITCH ноды)

## Связки инструментов
- Webhook/HTTP Request → n8n Workflow
- HTTP Request ноды n8n → Claude API/ChatGPT API
- Claude API → JSON парсер → Condition ноды
- Condition → Split логика на разные действия
- Разные ветки → Email/CRM API/Database Update
- Результат → Webhook обратный или уведомление

## Конфигурация и параметры
- Вебхук URL: настройка триггера на получение POST-запроса с JSON-данными
- Headers: Content-Type: application/json, Authorization: Bearer [TOKEN]
- Claude API key в переменных окружения n8n
- Промпт-инструкция: система указывает роль, формат ответа, стиль общения
- API endpoint модели: https://api.anthropic.com/v1/messages (для Claude)
- Параметры запроса: model, max_tokens, temperature, system prompt
- Маппинг полей: входящие данные → переменные в промпте → вывод в структурированный JSON
- CRM field mapping: id заявки, статус, комментарий ИИ → поля CRM
- Условия маршрутизации: если скор > 0.8 → автоматически, иначе → в очередь для человека

## Ключевые инсайты
- 1. ИИ-агент на n8n экономит 3-5 часов в день на рутину без потери качества
- 2. Критически важно правильно сформулировать система-промпт — от этого зависит 80% качества ответов
- 3. Для каждого типа задачи нужен свой раздел в промпте (разная логика для разных сценариев)
- 4. Confidence score/скор уверенности от ИИ позволяет автоматически отделять простые случаи от сложных
- 5. Error handling обязателен: ловить timeout API, некорректный JSON, недоступность сервисов
- 6. Rate limiting API может быть узким местом — обычно 1-100 RPM в зависимости от плана
- 7. История всех обращений и действий ИИ должна логироваться для аудита и улучшения
- 8. Стоимость: Claude API ~0.003 USD за 1K tokens input, реальные затраты 5-20 USD/день на 100-500 заявок
- 9. Холодный старт: агент учится первые 2-3 недели, затем точность стабилизируется
- 10. Интеграция с Telegram/Slack допускает рассылку уведомлений об исключениях в реальном времени

## Подводные камни
- ⚠️ Забывают про timeout на API запросы — коннектор висит 30+ сек, нужно ставить лимит 15-20 сек
- ⚠️ Промпт должен быть на том же языке, что и входящие данные, иначе качество упадёт на 30-40%
- ⚠️ Не обрабатывают случаи, когда API вернул пустой ответ или некорректный JSON — workflow падает
- ⚠️ Забывают про рейт-лимиты API: если прийдёт 1000 заявок в секунду, то большая часть упадёт
- ⚠️ Стоимость может быстро расти: 1 запрос = 1-5к tokens = 0.002-0.01 USD, при 1000 запросов в день это 2-10 USD/день
- ⚠️ История диалогов должна удаляться после 30 дней (GDPR), но многие забывают про cleanup job
- ⚠️ Переполнение контекста: если попытаться передать в ИИ историю диалога из 100+ сообщений, то ответ может быть медленным
- ⚠️ Отсутствие fallback'а: если ИИ не справился, нужно четкий план что делать дальше (перенаправить человеку, отправить стандартный ответ и т.д.)
