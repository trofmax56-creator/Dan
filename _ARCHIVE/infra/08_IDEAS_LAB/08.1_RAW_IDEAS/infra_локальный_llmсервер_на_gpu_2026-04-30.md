# 🗂️ Паспорт идеи: Локальный LLM-сервер на GPU

**Источник:** [Local Ai Server Setup Guides Proxmox 9 - vLLM in LXC w/ GPU Passthrough](https://youtube.com/watch?v=APBpBFZIVEk) — Digital Spaceport (2025-08-20)

---

## 💡 Что это даёт бизнесу
Клиент получает собственный LLM-сервер на базе vLLM, работающий на локальном железе без передачи данных в облако — критично для юридических, медицинских и финансовых компаний с NDA. Стоимость инференса падает в 10–30 раз по сравнению с OpenAI API при высоких нагрузках. Студия настраивает инфраструктуру под ключ: Proxmox + LXC + GPU passthrough + vLLM endpoint.

---

## ✅ Плюсы
- Полная конфиденциальность данных — данные не покидают контур компании
- Фиксированные расходы на инфраструктуру вместо переменных облачных счетов
- Совместимость vLLM с OpenAI API — существующие интеграции (n8n, Make, Bitrix24) работают без переписывания

## ❌ Минусы / Риски
- Требует наличия у клиента сервера с GPU (NVIDIA RTX 3090/4090 или A-серия) — не у всех МСБ есть железо
- Высокий порог входа по DevOps-компетенциям: Proxmox, LXC, CUDA — нужен специалист или обучение клиента

## 🚀 Шаги запуска
1. Шаг 1: Аудит железа клиента — GPU (VRAM ≥ 24 GB), RAM, хранилище; подбор модели (Mistral, Llama 3, Qwen)
2. Шаг 2: Установка Proxmox 9, создание LXC-контейнера с GPU passthrough, проброс CUDA-драйверов
3. Шаг 3: Развёртывание vLLM в контейнере, настройка OpenAI-совместимого endpoint, тестирование нагрузки
4. Шаг 4: Интеграция endpoint в существующие системы клиента (n8n / Make / Bitrix24), настройка мониторинга и документация
