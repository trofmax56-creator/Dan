# 🗂️ Паспорт идеи: ИИ-юрист проверки договоров

**Источник:** [Ваши юристы все еще читают?](https://youtube.com/watch?v=-UFnDLtktsY) — Приток про ИИ (2026-01-25)
**Категория:** Замена должности
**Заменяет:** Юрист на первичной проверке договоров / помощник юриста

---

## 💡 Что это даёт бизнесу
Система автоматически анализирует входящие договоры, выделяет рискованные пункты, несоответствия и подозрительные формулировки за минуты вместо часов. Клиент получает структурированный отчёт с цветовой маркировкой рисков и конкретными рекомендациями по правке. Сокращает время согласования договоров на 70–85% и снижает зависимость от штатного юриста.

---

## ✅ Плюсы
- Острая боль: каждый договор вручную — дорого и медленно, особенно для МСБ без штатного юриста
- Легко упаковать в продукт с понятным ROI: экономия 1 юриста = 80–150 тыс. руб./мес
- Повторяющийся спрос — договоры есть у всех: стройка, аренда, поставки, IT-аутсорс

## ❌ Минусы / Риски
- Юридическая ответственность: клиент может обвинить Студию при пропуске реального риска — нужен дисклеймер и ограничение ответственности
- Требует доработки под российское право (ГК РФ, специфика 44-ФЗ, 223-ФЗ) — готовые западные решения типа LegalOn не адаптированы

## 🧰 Артефакты для реализации

**Инструменты:** n8n (оркестрация пайплайна обработки документов), Claude 3.5 Sonnet / GPT-4o (анализ текста договора), LangChain / LlamaIndex (работа с длинными документами через RAG), Bitrix24 / amoCRM (интеграция для уведомлений и хранения результатов)
**API:** Anthropic Claude API (claude.ai/api), OpenAI API (platform.openai.com), Unstructured.io API (парсинг PDF/DOCX договоров)

**Репозитории:**
- github.com/langchain-ai/langchain (документообработка)
- github.com/run-llama/llama_index (RAG для длинных договоров)

**Документация:**
- docs.anthropic.com — работа с длинными документами
- unstructured-io.github.io/unstructured — извлечение текста из PDF
- docs.n8n.io — HTTP-ноды для интеграции
