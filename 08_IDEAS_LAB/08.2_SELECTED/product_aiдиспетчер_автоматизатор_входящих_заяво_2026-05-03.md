## 1. Название
**AI-Диспетчер** — Автоматизатор входящих заявок

## 2. Суть
Принимает заявки из WhatsApp, Telegram и форм сайта, классифицирует по типу и срочности, создаёт задачи в Bitrix24/amoCRM с нужным ответственным и дедлайном — без участия администратора.

## 3. Бизнес-сценарий
ДО: В оконную компанию «СтройПласт» за день приходит 40+ сообщений: «Хочу замер», «Сколько стоит балкон?», «Когда приедет монтажник?», «Рекламация — щель в окне». Администратор Оля вручную сортирует, создаёт задачи, распределяет. Опаздывает, путается. Клиент, который написал «щель в окне», ждёт ответа 3 часа — пишет отзыв в 2ГИС. → ИИ делает: Клиент пишет в WhatsApp: «У меня продувает окно, поставили месяц назад». Claude классифицирует: тип=рекламация, приоритет=высокий, департамент=сервис. n8n создаёт задачу в Bitrix24 с дедлайном 2 часа, назначает сервисного инженера, отправляет клиенту: «Приняли заявку, сервисный инженер свяжется с вами в течение 2 часов». → Клиент получает: Оля занимается только сложными случаями. Время первого ответа — с 15 минут до 30 секунд. Ни одна рекламация не теряется.

## 4. Логика
```
Входящее сообщение Wazzup/Telegram → n8n webhook → Claude (классификация: тип, приоритет, отдел) → Bitrix24 REST (создание задачи + назначение) → Wazzup (автоответ клиенту)
```

## 5. Технический стек
- n8n
- claude-sonnet-4-6
- Wazzup
- Telegram Bot API
- Bitrix24 или amoCRM

## 6. Связки инструментов
```
Wazzup webhook (входящее WA/TG) → n8n → Claude API (извлечь: тип заявки, приоритет, контакт, суть) → Bitrix24 REST createTask + assignUser → Wazzup API (отправить автоответ клиенту)
```

## 7. Алгоритм реализации
1. Подключить Wazzup к n8n через webhook; настроить фильтр: только входящие сообщения, не от своих номеров; дедупликация по chat_id за 5 минут
2. Передать текст сообщения в Claude с системным промптом: классифицируй по 5 типам (новая заявка / вопрос цены / рекламация / статус заказа / прочее), определи приоритет (1-3), извлеки имя клиента и суть одной фразой — верни JSON
3. По типу и приоритету n8n выбирает ответственного из справочника (настраивается клиентом в Google Sheets — источник правды); рекламации и приоритет 1 — немедленное уведомление руководителю в Telegram
4. Создать задачу в Bitrix24 через REST API с полями: заголовок (суть от Claude), ответственный, дедлайн (тип→дедлайн: рекламация=2ч, замер=24ч, цена=1ч), ссылка на чат Wazzup
5. Отправить клиенту автоответ через Wazzup с шаблоном под каждый тип: для рекламации — «Передали в сервис, ответим за 2 часа», для замера — «Запишем вас на удобное время, сейчас уточним детали»

## 8. Почему купят (ROI)
Боль: администратор на 40+ входящих в день тратит 3–4 часа только на сортировку и создание задач — это 60 000–80 000 руб./мес ФОТ на ручную маршрутизацию. Пропущенная рекламация = 1 плохой отзыв в 2ГИС = минус 15–20 потенциальных клиентов (средний чек 35 000 руб.) = 525 000 руб. потенциальных потерь. Крючок: «Клиент написал в 23:47 — AI-Диспетчер ответил в 23:47. Конкурент ответит завтра в 9:30.»
