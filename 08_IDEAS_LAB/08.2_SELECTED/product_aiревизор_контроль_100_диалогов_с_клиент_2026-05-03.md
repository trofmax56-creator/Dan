## 1. Название
**AI-Ревизор** — Контроль 100% Диалогов с Клиентами

## 2. Суть
Claude прослушивает и читает 100% переписок и звонков менеджеров, выставляет оценку по чек-листу компании, выявляет потерянные возражения и нарушения скрипта — РОП перестаёт тратить 3 часа в день на выборочный контроль и получает сводку утром.

## 3. Бизнес-сценарий
ДО: РОП Сергей каждое утро открывает amoCRM и вручную слушает 5–7 звонков из 40 за вчера. На остальные 33 времени нет. В конце месяца выясняется, что менеджер Коля три недели не называл цену первым — клиенты уходили «подумать» и не возвращались. Это 8 потерянных сделок по 35 000 руб. = 280 000 руб. ИИ ДЕЛАЕТ: после каждого завершённого звонка UIS автоматически отправляет запись в n8n. Claude транскрибирует, сверяет с чек-листом из 12 пунктов и через 4 минуты пишет РОПу: «Звонок Коли 14:23. Оценка 4/10. Нарушения: не выявил бюджет (п.3), не закрыл на следующий шаг (п.11). Цитата клиента: "Мне надо подумать" — Коля ответил 

## 4. Логика
```
Звонок завершён в UIS → webhook в n8n → транскрибация (Deepgram/МТС Exolve) → Claude (оценка по 12-пунктовому чек-листу) → запись в amoCRM/Bitrix24 (поле «Оценка ОКК») → Telegram-уведомление РОПу при оценке < 5 → ежедневный дайджест 08:00
```

## 6. Связки инструментов
```
UIS Webhook (звонок завершён) → n8n → Deepgram (транскрибация, ≤ 30 мин запись) → Claude (scoring по чек-листу клиента, JSON с оценками по 12 пунктам) → amoCRM REST API (запись оценки в сделку) → Telegram Bot (уведомление РОПу мгновенно при score < 5, дайджест в 08:00 через cron)
```

## 7. Алгоритм реализации
1. Подключить UIS или МТС Exolve к n8n через webhook: событие «звонок завершён» передаёт ссылку на запись, длительность, ID менеджера и ID сделки в amoCRM
2. n8n скачивает аудиофайл и отправляет в Deepgram API: получить транскрипт с разметкой спикеров (менеджер / клиент) в течение 60–90 секунд
3. Claude-промпт получает транскрипт + чек-лист компании (12 пунктов, загружается один раз при настройке) → возвращает JSON: {score: 7, violations: ['п.3', 'п.11'], citations: ['...'], summary: '...'}
4. Записать результат в amoCRM через REST API: кастомное поле «Оценка ОКК» в карточке сделки, тег «нарушение-скрипта» при score < 5, примечание с цитатами
5. Cron в 08:00 — n8n агрегирует все оценки за вчера: формирует топ-3 нарушения команды, средний балл, менеджеры с score < 5 и отправляет дайджест РОПу в Telegram

## 8. Почему купят (ROI)
Штат ОКК из 2 аналитиков — это 140 000–180 000 руб./мес ФОТ, и они всё равно слушают лишь 15–20% звонков. AI-Ревизор слушает 100% за 4 000 руб./мес инфраструктуры. Одна найденная системная ошибка скрипта (как в примере с Колей) возвращает 200 000–400 000 руб. потерянной выручки в месяц. Крючок: «ОКК из двух человек стоит 180 000 руб. в месяц и слышит каждый пятый звонок. ИИ слышит все — за 4 000.»
