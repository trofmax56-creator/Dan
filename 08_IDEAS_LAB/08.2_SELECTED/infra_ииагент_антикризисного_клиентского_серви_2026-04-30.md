# 🗂️ Паспорт идеи: ИИ-агент антикризисного клиентского сервиса

**Источник:** [Как ИИ-агент превращает токсичных клиентов в лояльных: кейс с реальными «спасибо» в чатах](https://youtube.com/watch?v=yGnY-QgNd04) — ИИ-агенты для бизнеса. DeGood. (2025-08-28)
**Категория:** Замена должности
**Заменяет:** Менеджер по работе с конфликтными/токсичными клиентами, оператор поддержки

---

## 💡 Что это даёт бизнесу
ИИ-агент обрабатывает агрессивных и недовольных клиентов 24/7 без эмоционального выгорания, снижает отток клиентов через деэскалацию конфликтов. Бизнес сокращает затраты на стрессоустойчивых менеджеров и получает измеримый рост лояльности (NPS) без найма дорогого персонала.

---

## ✅ Плюсы
- Устраняет человеческий фактор: агент не грубит, не срывается, не уходит на больничный
- Высокий pain у МСБ — текучка в поддержке колоссальная, обучение стоит дорого
- Легко упаковывается в типовой продукт для e-commerce, клиник, сервисных компаний

## ❌ Минусы / Риски
- Сложные юридические или эмоциональные кейсы всё равно требуют эскалации на человека — нужна чёткая логика передачи
- Клиенты МСБ боятся 'робота в поддержке' — нужен этап убеждения и пилот с реальными диалогами

## 🧰 Артефакты для реализации

**Инструменты:** n8n (оркестрация сценариев деэскалации), Claude API (генерация эмпатичных ответов), amoCRM / Bitrix24 (интеграция с историей клиента), Telegram Bot API / WhatsApp Business API (каналы общения)
**API:** Anthropic Claude API (claude-3-5-sonnet), OpenAI API (альтернатива), WhatsApp Business Cloud API, Telegram Bot API, Bitrix24 REST API / amoCRM API

**Репозитории:**
- n8n-io/n8n (github.com/n8n-io/n8n)
- Поиск по: 'customer support ai agent n8n template'

**Документация:**
- docs.anthropic.com — Claude system prompt для деэскалации
- docs.n8n.io — Webhook + AI Agent node

## 🚀 Шаги запуска
1. Шаг 1: Собрать 30-50 реальных диалогов конфликтных ситуаций у клиента → разметить сценарии (возврат, грубость, угрозы, претензии к качеству)
2. Шаг 2: Написать system prompt для Claude с ролью 'заботливый, но твёрдый менеджер' + правила эскалации на человека при триггерах
3. Шаг 3: Собрать в n8n пайплайн: входящее сообщение → обогащение контекстом из CRM → генерация ответа → логирование → отправка
4. Шаг 4: Пилот 2 недели на реальном трафике, замер метрик (процент урегулированных без эскалации, оценки клиентов), презентация ROI
