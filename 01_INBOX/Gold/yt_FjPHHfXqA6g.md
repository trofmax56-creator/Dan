## Суть
Настройка AI-агентов на основе Claude, Code Interpreter и Gemini с оптимизацией затрат на API. Раскрывается процесс конфигурации агентов для автоматизации бизнес-процессов с использованием бесплатных и низкозатратных опций.

## Бизнес-сценарий
Разработчики и бизнес-автоматизаторы используют AI-агентов для автоматизации задач обработки данных, генерации контента и интеграции с внешними системами (CRM, Bitrix24, amoCRM) с минимизацией расходов на API-запросы через правильную настройку параметров и выбор оптимальных моделей.

## Алгоритм реализации
1. 1. Выбор платформы для развертывания AI-агента (n8n, Make.com, Zapier или самостоятельный сервер)
2. 2. Подключение API ключей Claude, Gemini и Code Interpreter к платформе автоматизации
3. 3. Конфигурация параметров модели: temperature, max_tokens, system prompt для оптимизации стоимости
4. 4. Создание цепочки обработки: входные данные → агент Claude/Gemini → Code Interpreter (опционально) → выходные данные
5. 5. Настройка условного маршрутизирования: выбор модели/инструмента в зависимости от сложности задачи для снижения расходов
6. 6. Интеграция с CRM системами через API и вебхуки для автоматического создания задач/сделок
7. 7. Логирование и мониторинг затрат на API-запросы для отслеживания ROI
8. 8. Тестирование на малых объёмах данных перед масштабированием на продакшн

## Связки инструментов
- Webhook → n8n → Claude API → JSON обработка → Bitrix24 API → создание сделки
- Email триггер → Make.com → Gemini API → Code Interpreter → обработка вложений → amoCRM API → добавление контакта
- Входящий API запрос → n8n → выбор модели (Claude/Gemini) → обработка → кэширование результатов → выходной JSON

## Конфигурация и параметры
- Модель Claude: выбрать между claude-3-opus (мощный, дорогой) vs claude-3-haiku (быстрый, дешёвый) в зависимости от задачи
- Temperature: 0.0-0.3 для детерминированных задач (классификация, парсинг), 0.7-1.0 для творческих
- Max_tokens: устанавливать ровно столько, сколько нужно для ответа, минимум расходов
- System prompt: краткий, конкретный инструкции на русском или английском
- Gemini: бесплатный tier имеет лимит 60 запросов в минуту
- Code Interpreter timeout: 30-60 секунд, зависит от сложности кода
- Кэширование промптов в Claude API: использовать prompt caching для повторяющихся системных инструкций (экономия 90% токенов)
- Batch API для Claude: обработка 10,000+ запросов за ночь со скидкой 50%

## Ключевые инсайты
- 1. Claude 3 Haiku в 10 раз дешевле Opus при приемлемом качестве для 80% задач классификации и парсинга
- 2. Prompt caching в Claude экономит 90% затрат на повторяющиеся большие системные инструкции (лучше для batch-обработки)
- 3. Условное маршрутизирование: простые задачи → Gemini (бесплатный tier), сложные → Claude, очень простые → regex/парсинг
- 4. Code Interpreter в некоторых случаях выполняет задачу быстрее одного LLM-запроса (экономия токенов)
- 5. Кэширование результатов в n8n/Make: если одна задача повторяется, второй раз идёт из кэша вместо API
- 6. Batch API для Claude: если можно дождаться результатов 12-24 часа, экономия 50% стоимости
- 7. Использование меньших моделей (Gemini 1.5 Flash, Claude Haiku) как первого фильтра перед Opus
- 8. Rate limiting в n8n: 1-2 запроса в секунду вместо параллельных снижает затраты на перебор вариантов
- 9. Мониторинг через логирование каждого запроса: стоимость, задержка, токены in/out для анализа неэффективных операций
- 10. Fallback логика: если Claude недоступен, автоматический переход на Gemini или другую модель без потери данных
