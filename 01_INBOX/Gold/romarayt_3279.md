**4 ошибки Claude Code, из-за которых он работает хуже, чем может**

Claude Code может быть почти полноценным разработчиком. Но только если не давать ему работать “как попало”.

В новом видео разобрал 4 правила, которые помогают:
- не давать Claude угадывать вместо уточнений  
- не получать 200 строк кода вместо 10  
- не ломать части проекта, которые ты не просил трогать  
- экономить токены и не объяснять одно и то же по 10 раз  
- правильно внедрить правила в Claude MD файл  

Плюс показал готовый бесплатный skill с GitHub и объяснил, как добавить его в свой проект, чтобы Claude Code работал аккуратнее и стабильнее.
