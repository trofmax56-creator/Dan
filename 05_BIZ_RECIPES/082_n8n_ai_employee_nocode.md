### 1. Описание идеи

«AI-сотрудник» — универсальный агент на n8n, который принимает задачи через Telegram/форму, выполняет их (пишет текст, анализирует данные, обновляет CRM, отправляет письма) и отчитывается о результате. Это не чат-бот — это полноценный рабочий агент с инструментами (tools).

Подходит как «первый AI-сотрудник» для малого бизнеса, который не хочет нанимать отдельных специалистов для каждой задачи.

### 2. Технический стек

- n8n AI Agent нода (встроена в n8n v1.x)
- Claude API (основная модель) — рекомендуется `claude-sonnet-4-6`
- Инструменты агента: Bitrix24 Tool, Google Sheets Tool, Gmail Tool, Telegram Tool
- Telegram Bot — интерфейс постановки задач
- n8n Memory — контекст между сессиями

### 3. Пошаговый план внедрения

**Шаг 1 — Создать AI Agent в n8n**
```
n8n → New Workflow → Add Node → AI Agent
→ Выбрать Chat Model: HTTP Request к Claude API
→ Добавить Memory: Window Buffer Memory (последние 10 сообщений)
→ Добавить Tools (инструменты)
```

**Шаг 2 — Подключить инструменты (Tools)**
```javascript
// Tool 1: Создать задачу в Bitrix24
{
  name: "create_task",
  description: "Создать задачу в Bitrix24 с заголовком, описанием и ответственным",
  parameters: { title: string, description: string, responsible_id: number }
  // action: HTTP Request → POST /rest/tasks.task.add
}

// Tool 2: Отправить письмо
{
  name: "send_email",
  description: "Отправить email на указанный адрес",
  parameters: { to: string, subject: string, body: string }
  // action: Gmail нода → Send Email
}

// Tool 3: Записать в Google Sheets
{
  name: "log_to_sheets",
  description: "Добавить запись в таблицу учёта",
  parameters: { sheet: string, data: object }
  // action: Google Sheets → Append Row
}

// Tool 4: Поиск в CRM
{
  name: "find_client",
  description: "Найти клиента в Bitrix24 по имени или телефону",
  parameters: { query: string }
  // action: HTTP Request → crm.contact.list
}
```

**Шаг 3 — Системный промпт агента**
```
Ты — AI-сотрудник компании [Название]. Работаешь через Telegram.

Твои возможности:
1. Создавать задачи в Bitrix24
2. Отправлять письма клиентам
3. Искать клиентов в CRM
4. Записывать данные в таблицы

Всегда:
- Уточняй детали, если задание неясно
- Подтверждай выполненные действия
- При ошибке — сообщай что именно не получилось

Стиль: профессиональный, краткий, без лишних слов.
```

**Шаг 4 — Интерфейс через Telegram**
```
Telegram Trigger → AI Agent → Telegram Send Message

Примеры команд пользователя:
"Создай задачу: позвонить Иванову Ивану, срок — завтра"
"Найди клиента ООО Ромашка и напиши мне его телефон"
"Отправь письмо на info@company.ru: подтверждение встречи 15 мая"
```

**Шаг 5 — Мониторинг и логирование**
```
После каждого действия агент пишет в Google Sheets:
timestamp | user | task | tool_used | result | duration_sec
→ Еженедельный дайджест: что сделал агент за неделю
```

### 4. Сценарии использования

| Задача | Tool | Пример команды |
|---|---|---|
| Поставить задачу | create_task | «Поставь задачу Пете: подготовить КП к пятнице» |
| Найти клиента | find_client | «Найди ООО Альфа, скажи телефон» |
| Написать письмо | send_email | «Напиши письмо с благодарностью для ivanov@mail.ru» |
| Записать результат | log_to_sheets | «Запиши: встреча с Петровым прошла успешно» |

### 5. Нюансы

- n8n AI Agent нода — встроена в n8n 1.x. Для более старых версий: конструировать вручную через Code + HTTP
- Claude Sonnet справляется с выбором инструментов лучше Haiku — здесь экономить не стоит
- Количество инструментов: 4–8 оптимум. Больше → агент путается в выборе
- Логировать ВСЁ с самого начала — клиент через месяц спросит статистику
