# 🗂️ Паспорт идеи: ИИ-юрист: проверка договоров

**Источник:** [ИИ проверит договор за минуту: риски и правки](https://youtube.com/watch?v=aRHu1PRUXjs) — Anton Larichev (2026-02-09)
**Категория:** Замена должности
**Заменяет:** Юрист на первичной проверке договоров / юридический ассистент

---

## 💡 Что это даёт бизнесу
Клиент загружает договор и за 1–2 минуты получает структурированный отчёт: риски, невыгодные условия, рекомендуемые правки — без найма юриста. Экономит 5 000–30 000 руб. за каждую проверку внешним юристом. Подходит для МСБ, который подписывает 10–50 договоров в месяц (поставщики, клиенты, аренда).

---

## ✅ Плюсы
- Острая и понятная боль: каждый бизнес подписывает договоры, но не держит штатного юриста
- Быстрый WOW-эффект на демо — результат виден за 60 секунд, легко продавать
- Легко упаковать в повторяющуюся подписку: клиент платит ежемесячно за неограниченные проверки

## ❌ Минусы / Риски
- Юридическая ответственность: клиент может принять ИИ-вывод за полноценную юридическую консультацию и потом предъявить претензии — нужен чёткий дисклеймер
- Качество сильно зависит от промпта и модели: сложные многостраничные договоры с отсылками к законодательству требуют точной настройки и тестирования

## 🧰 Артефакты для реализации

**Инструменты:** n8n (оркестрация: приём файла → парсинг → запрос к LLM → отчёт), Make.com (альтернатива n8n для простых сценариев), Telegram Bot или веб-форма (интерфейс загрузки), Google Docs / Notion (выдача отчёта клиенту)
**API:** Anthropic Claude API (claude-3-5-sonnet) — лучший для длинных документов, OpenAI GPT-4o API — альтернатива, PDF parsing: PyMuPDF или Adobe PDF Extract API

**Репозитории:**
- github.com/antonlarichev — проверить наличие репо по видео
- github.com/topics/contract-analysis-llm

**Документация:**

## 🚀 Шаги запуска
1. Шаг 1: Определить типовые виды договоров целевого клиента (поставка, аренда, услуги, NDA) — под каждый написать специализированный промпт с чеклистом рисков
2. Шаг 2: Собрать пайплайн в n8n: Telegram/веб-форма → загрузка PDF → парсинг текста → Claude API → форматированный Markdown-отчёт → отправка клиенту
3. Шаг 3: Протестировать на 20+ реальных договорах разных типов, откалибровать промпты, добавить дисклеймер об ответственности
4. Шаг 4: Упаковать в коммерческое предложение для бухгалтерий, малых ООО, агентств — продавать через демо-проверку первого договора бесплатно
