## 1. Название
**AI-Летопись** — База знаний компании с голосовым вводом

## 2. Суть
Сотрудник надиктовывает голосом или пишет в Telegram вопрос/ответ/инструкцию — Claude структурирует, тегирует и кладёт в базу знаний; новые сотрудники и менеджеры задают вопросы боту и получают ответ из накопленной базы за 10 секунд вместо 40 минут поиска.

## 3. Бизнес-сценарий
ДО: В строительной компании уволился Иван — единственный, кто знал, как оформлять акты для проблемного подрядчика. Теперь каждый новый менеджер звонит старым коллегам, тратит 2–3 часа на поиск информации, делает ошибки. Руководитель злится: 'Почему опять то же самое?' ИИ делает: Директор пишет в корпоративный Telegram-бот: 'Как оформлять акт если подрядчик задержал сроки?' Бот отвечает через 8 секунд выжимкой из 3 релевантных инструкций с источниками. Параллельно любой сотрудник может надиктовать голосовое сообщение 'сегодня решили делать вот так' — Claude транскрибирует, структурирует и добавляет запись в базу с тегом и датой. Клиент получает: живая база, которая растёт каждый день, и новый сотрудник выходит на самостоятельную работу за 3 дня вместо 3 недель.

## 4. Логика
```
Telegram (вопрос/голос) → n8n Webhook → Claude (транскрипция + RAG-поиск по Notion/Google Docs) → ответ в Telegram; новая запись → Claude (структурирование) → Notion API (сохранение)
```

## 5. Технический стек
- n8n
- claude-sonnet-4-5
- Telegram Bot API
- Notion API
- OpenAI Whisper (транскрипция)

## 6. Связки инструментов
```
Telegram Bot → n8n Webhook → Whisper API (голос→текст) → Claude (RAG-запрос к Notion) → Notion API (чтение/запись) → Telegram Bot (ответ)
```

## 7. Алгоритм реализации
1. Создать Notion-базу с шаблоном: заголовок, теги (отдел/тип), содержание, дата, автор — импортировать существующие регламенты клиента (обычно 20–50 документов) через Notion API
2. Настроить Telegram-бота с двумя режимами: 'вопрос' (поиск в базе) и 'добавить' (новая запись) — переключение командами /ask и /add
3. При режиме /ask: Claude получает вопрос + все записи из релевантных разделов Notion (через API-поиск по тегам), формирует ответ с цитатами и ссылками на источники
4. При режиме /add или голосовом сообщении: Whisper транскрибирует аудио, Claude извлекает суть, присваивает теги из фиксированного списка (согласовать с клиентом, обычно 8–12 тегов), создаёт запись в Notion
5. Еженедельный дайджест в Telegram-канал команды: 'На этой неделе добавлено 7 записей, топ-3 вопроса' — отправляется автоматически каждую пятницу в 17:00

## 8. Почему купят (ROI)
Компания 20+ человек теряет минимум 5 часов/сотрудника в месяц на поиск информации и переобучение — это 100 часов × 300 руб./час = 30 000 руб. чистых потерь ежемесячно, не считая ошибок из-за устаревших данных. При текучке 2 человека/год стоимость адаптации новичка (3 недели × 50 000 руб./мес) = 37 500 руб. Крючок: 'Знания компании перестают уходить вместе с сотрудниками.'
