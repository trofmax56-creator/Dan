## 1. Название
**AI-Архивариус** — Живая База Знаний Компании

## 2. Суть
Автоматически собирает переписку, голосовые сообщения и документы из рабочих Telegram-каналов, транскрибирует, структурирует через Claude и складывает в единую базу знаний — сотрудники перестают искать «где это было написано».

## 3. Бизнес-сценарий
ДО: Новый менеджер по продажам спрашивает в общем чате: «А как мы работаем с возвратами?» Коллеги молчат — кто-то в отпуске, кто-то занят. РОП через 2 часа находит нужный регламент в переписке за март и пересылает вручную. ИИ ДЕЛАЕТ: Telegram Exporter каждые 6 часов выгружает все сообщения из рабочих каналов, Claude транскрибирует голосовые и классифицирует контент по 8 категориям (регламенты, кейсы, цены, возражения и др.), складывает в Obsidian/Notion с тегами и датами. КЛИЕНТ ПОЛУЧАЕТ: менеджер пишет боту: «Как работаем с возвратами?» — через 15 секунд получает выжимку из трёх актуальных источников с датами и ссылками на оригиналы.

## 4. Логика
```
Telegram каналы (cron 6ч) → Telegram Exporter → n8n (транскрибация Deepgram) → Claude (классификация + тэгирование) → Notion/Obsidian API (запись) → Telegram-бот (поиск по запросу сотрудника)
```

```
Telegram Exporter → n8n Webhook → Deepgram (транскрибация голосовых) → Claude (классификация по 8 категориям) → Notion REST API (структурированная запись) → Telegram-бот (RAG-поиск по запросу)
```

## 7. Алгоритм реализации
1. Развернуть Telegram Exporter на сервере клиента, добавить 3–10 рабочих каналов, настроить cron-выгрузку каждые 6 часов в формате .md
2. В n8n создать workflow: входящий .md файл → HTTP-запрос к Deepgram для транскрибации голосовых вложений (≤ 10 мин на файл)
3. Claude-промпт классифицирует каждый блок текста по 8 категориям (регламент / цена / кейс / возражение / задача / анонс / мусор / прочее) и генерирует 3–5 тегов
4. Отфильтрованный контент (без категории 'мусор') записывается в Notion через REST API: база «Корпоративная память», поля: дата, источник, категория, теги, текст, автор
5. Telegram-бот принимает вопрос сотрудника → n8n делает semantic search по Notion (фильтр по тегам + full-text) → Claude формирует ответ со ссылками на 2–3 источника, уведомление в чат ≤ 20 секунд

## 8. Почему купят (ROI)
Компания в 15 человек теряет в среднем 45 минут в день на поиск информации в переписке — это 7 500 руб./день при ФОТ 200 000 руб./мес на одного сотрудника, или 165 000 руб./мес потерянного рабочего времени. При онбординге каждый новый сотрудник «переоткрывает велосипед» 2–3 недели. Крючок: «Корпоративная память не должна умирать вместе с сотрудником, который ушёл в отпуск.»
