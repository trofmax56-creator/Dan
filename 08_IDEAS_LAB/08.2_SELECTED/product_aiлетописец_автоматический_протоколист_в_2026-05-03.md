## 1. Название
**AI-Летописец** — Автоматический Протоколист Встреч

## 2. Суть
Агент принимает аудиозапись или транскрипт планёрки/переговоров, извлекает задачи с ответственными и дедлайнами, создаёт задачи в Битрикс24 и рассылает протокол участникам в Telegram — руководитель перестаёт быть секретарём на своих же встречах.

## 3. Бизнес-сценарий
ДО: Директор Михаил проводит 3 планёрки в неделю по 1 часу. После каждой 20-30 минут тратит на то, чтобы написать протокол, разослать задачи в мессенджер и создать их в Битрикс24. Половина задач всё равно теряется, потому что не все их занесли. ИИ делает: Михаил нажимает «Запись» в Telegram-боте прямо перед планёркой. После встречи пересылает запись боту. Claude транскрибирует, вычленяет формат «[Задача] → [Кто отвечает] → [Срок]», создаёт задачи в Битрикс24 на каждого ответственного и отправляет в общий чат готовый протокол: «Итоги планёрки 14.05: ✅ Иван — подготовить КП для Газпрома до 16.05 ✅ Марина — согласовать договор до 17.05». Михаил получает: 30 минут жизни обратно после каждой встречи.

## 4. Логика
```
Аудиофайл в Telegram-бот → n8n → SaluteSpeech (транскрипция) → Claude (извлечение задач: кто/что/когда) → Bitrix24 REST API (создание задач на ответственных) → Telegram (рассылка протокола участникам)
```

## 5. Технический стек
- n8n
- claude-sonnet-4-6
- SaluteSpeech API
- Bitrix24
- Telegram Bot API

## 6. Связки инструментов
```
Telegram Bot (приём аудио) → n8n → SaluteSpeech (транскрипция MP3/OGG) → Claude (структурирование: задача + ответственный + дедлайн) → Bitrix24 REST API TASK.ADD → Telegram Bot (протокол в группу)
```

## 7. Алгоритм реализации
1. Создать Telegram-бота, принимающего голосовые сообщения и аудиофайлы до 50 МБ; в n8n настроить триггер на входящий файл от авторизованных пользователей (белый список ID)
2. Отправить аудио в SaluteSpeech (СберДевайсы, РФ API) для транскрипции с разделением по спикерам (diarization), получить текст с меткой времени каждой реплики
3. Передать транскрипт в Claude с промптом: «Извлеки все задачи в формате JSON: task (текст задачи), responsible (имя из текста), deadline (дата или "не указан"), priority (высокий/средний/низкий)»
4. Для каждого элемента JSON создать задачу в Bitrix24 через REST API TASK.ADD с полями: название, ответственный (матч по имени к пользователям портала), дедлайн, постановщик = директор
5. Сформировать текстовый протокол и отправить в указанный Telegram-чат через бота; при наличии задач с меткой «не указан» — запросить у постановщика уточнение дедлайна отдельным сообщением

## 8. Почему купят (ROI)
Директор, проводящий 3 планёрки в неделю, тратит 1,5-2 часа в неделю только на рутину после встреч — 6-8 часов в месяц рабочего времени топ-менеджера по ставке 150 000+ руб./мес = 15 000-20 000 руб./мес стоимость этой рутины. Плюс задачи без фиксации теряются: по статистике до 30% договорённостей на устных встречах не выполняются из-за отсутствия записи. Крючок: «Планёрка без протокола — это просто разговоры.»
