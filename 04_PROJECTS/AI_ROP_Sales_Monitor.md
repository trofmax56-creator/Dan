# 🤖 AI-РОП: Агент, который управляет отделом продаж 24/7

> n8n + CRM API + Claude → автоматический анализ воронки, выявление проблем и рекомендации — без живого руководителя отдела продаж.

---

## Боль клиента

- РОП стоит ₽100–200k/мес и работает 8 часов в день, 5 дней в неделю
- Сделки «зависают» в воронке неделями — никто не замечает вовремя
- Руководитель не видит реальную картину по каждому менеджеру без ручного анализа CRM
- Отчёты составляются вручную и часто опаздывают или искажаются
- Прогноз закрытия месяца — интуиция, не данные

## Решение

AI-агент на n8n каждые 2 часа / каждое утро в 9:00:
1. Забирает все активные сделки из CRM (amoCRM / Bitrix24)
2. Считает KPI по каждому менеджеру: % выполнения плана, скорость закрытия, средний чек
3. Передаёт данные в Claude для анализа и приоритизации проблем
4. Отправляет отчёт в Telegram/Slack с конкретными сделками и рекомендациями
5. Логирует историю в Google Sheets для динамики

## Технический стек

```
Schedule Trigger (cron) → n8n
  → HTTP Request: CRM API (amoCRM /api/v4/leads или Bitrix24 crm.deal.list)
  → Code Node: нормализация данных, расчёт KPI (JS)
  → HTTP Request: Claude API (анализ, топ-5 проблем, JSON ответ)
  → Switch: критичность (HIGH / MEDIUM / LOW)
  → Telegram Bot / Slack: форматированный отчёт
  → Google Sheets: логирование
```

**CRM → n8n подключение:**
- amoCRM: Bearer токен в HTTP headers, endpoint `/api/v4/leads?with=contacts,loss_reason&filter[status]=142&limit=250`
- Bitrix24: `crm.deal.list` через REST API, фильтр по стадии и ответственному

**Claude промпт (System):**
```
Ты — аналитик отдела продаж. Получаешь JSON со сделками.
Найди ТОП-5 критических проблем. Для каждой укажи:
- issue_type (сделка зависла / менеджер отстаёт / прогноз провала)
- severity (high / medium)  
- deal_id и manager_name
- action_required (конкретное действие)
Верни только валидный JSON массив.
```

**Расписание (cron):**
- `0 9 * * MON-FRI` — утренний отчёт в 9:00
- `0 */2 * * *` — мониторинг каждые 2 часа (опционально)

## Алгоритм внедрения (5 дней)

### День 1: Подключение CRM
- [ ] Получить API-ключ amoCRM / токен Bitrix24
- [ ] Создать n8n HTTP Request → проверить возврат сделок
- [ ] Убедиться что в ответе есть: id, name, status, responsible_user, price, created_at, updated_at

### День 2: Расчёт KPI
- [ ] Code Node (JS): группировка сделок по менеджеру
- [ ] Расчёт: % выполнения плана, кол-во сделок > 30 дней, средний чек
- [ ] Пороги: сделка > 30 дней в статусе = HIGH, 15–30 дней = MEDIUM

### День 3: Интеграция Claude
- [ ] HTTP Request к Claude API (`claude-haiku-4-5-20251001` для экономии)
- [ ] Протестировать промпт на реальных данных
- [ ] Парсинг JSON ответа → Switch по severity

### День 4: Уведомления
- [ ] Telegram Bot: отформатированное сообщение с emoji и списком проблем
- [ ] Google Sheets: запись даты, кол-ва критических проблем, прогноза
- [ ] Тест запуска вручную (Execute Workflow)

### День 5: Расписание + Сдача

## Ключевые инсайты

- Агент работает **24/7 без усталости и эмоций** — объективнее любого РОПа
- Интеграция с историей взаимодействия (последний звонок, задачи) — РОП часто забывает это проверить
- **Автоматический прогноз месяца** по текущей траектории: если отстаём на 15% — это видно за 2 недели
- JSON-структура ответа Claude открывает расширение: автосоздание задач менеджерам прямо в CRM
- История в Google Sheets = метрика качества самого агента (можно проверить рекомендации через неделю)
