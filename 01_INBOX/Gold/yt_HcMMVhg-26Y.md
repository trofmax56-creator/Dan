## Суть
Автоматизация квалификации лидов в amoCRM через n8n, который интегрирует ChatGPT для анализа и классификации потенциальных клиентов без участия человека. Система автоматически определяет качество лида, присваивает оценку и выполняет необходимые действия в CRM.

## Бизнес-сценарий
Менеджеры и владельцы компаний, работающие в amoCRM, автоматизируют процесс первичной квалификации входящих лидов через интеллектуального ИИ-менеджера. Система обрабатывает информацию о новых контактах (имя, контактные данные, информация о проекте, бюджет), анализирует её через ChatGPT и автоматически устанавливает статусы, теги и приоритеты в amoCRM.

## Алгоритм реализации
1. 1. Новый лид поступает в amoCRM (триггер на создание контакта/сделки)
2. 2. n8n ловит вебхук из amoCRM с информацией о лиде
3. 3. Данные о лиде форматируются и передаются в ChatGPT через API
4. 4. ChatGPT анализирует лида по определённым критериям (бюджет, соответствие ЦА, готовность к покупке)
5. 5. ИИ возвращает структурированный результат (оценка качества, рекомендуемый статус, теги)
6. 6. n8n парсит ответ ChatGPT и обновляет запись о лиде в amoCRM
7. 7. Автоматически устанавливаются: статус этапа, теги, приоритет, назначение менеджера
8. 8. Логирование всех операций для аудита и анализа эффективности

## Технический стек

- ChatGPT API (анализ и классификация)
- amoCRM API (интеграция CRM)

- Webhook для триггеров

## Связки инструментов
- amoCRM (входящий вебхук) → n8n → ChatGPT API → n8n → amoCRM API (обновление лида)
- Вебхук → HTTP Request (ChatGPT) → JSON Parsin → amoCRM Update Contact/Lead

## Конфигурация и параметры
- Триггер: Webhook из amoCRM на событие 'Создан контакт' или 'Создана сделка'
- URL вебхука: генерируется n8n и копируется в amoCRM
- Параметры передачи: ID контакта, имя, email, телефон, описание, привязанные сделки
- API ключ ChatGPT: указывается в credentials n8n
- Prompt для ChatGPT: определение критериев анализа лида (соответствие целевой аудитории, бюджет, срок реализации)
- Маппинг ответов ChatGPT на действия в amoCRM: холодный лид → статус 'Контакт', теплый → 'Квалифицированный', горячий → 'Готов к продаже'
- Поля обновления в amoCRM: custom_field (оценка от 1-10), статусы в воронке, теги для фильтрации
- Опция retry при ошибке API
- Логирование в JSON или webhook для аналитики

## Ключевые инсайты
- ChatGPT анализирует лида не только по явным данным, но и по контексту письма/звонка - может определить заинтересованность по интонации
- Система экономит 15-30 минут на квалификацию одного лида, переводя менеджеров на работу только с горячими контактами
- Важно составить point проактивный Prompt для ChatGPT - в нём указать все критерии вашего идеального клиента (ЦА, бюджет, индустрия)
- Теги в amoCRM можно использовать для дальнейшей автоматизации: холодные лиды идут в nurture последовательность, горячие - сразу менеджеру
- API лимиты ChatGPT: нужно учитывать стоимость (примерно $0.001-0.002 на лида), при большом объёме использовать батчинг
- Можно добавить условие на отсеивание спама или явно неподходящих лидов ещё до отправки в ChatGPT для экономии
- Система работает асинхронно - лид получает обновления в течение 3-5 секунд после создания
- В n8n используются встроенные функции для парсинга JSON и логики условных переходов (Switch node)

## Подводные камни
- Вебхук из amoCRM может срабатывать дважды при быстром создании лида - нужен deduplicate или проверка timestamp
- ChatGPT иногда возвращает ответ в неправильном JSON формате - обязательно добавить Try-Catch обработку ошибок
- amoCRM API имеет rate limit - при большом потоке лидов может быть задержка, используйте очередь в n8n
- Чувствительность ИИ к формулировке prompt - нужна постоянная оптимизация и A/B тестирование
- Если в лиде нет достаточно данных (пусто поле description), ChatGPT вернёт неполный анализ - добавьте валидацию входных данных
- Интеграция требует наличия API ключей и доступа - убедитесь в правах на изменение контактов в amoCRM
- Стоимость ChatGPT API может неожиданно возрасти при неправильной настройке длины prompt или количества запросов
