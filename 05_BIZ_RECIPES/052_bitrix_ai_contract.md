### 1. Описание идеи
При закрытии этапа сделки в Битриксе n8n перехватывает переписку (активности, письма), передаёт её в Claude и получает готовый текст договора. Документ автоматически прикрепляется к карточке. Менеджер не тратит время на рутинное заполнение.

### 2. Технический стек
- AI: Claude (claude-sonnet-4-6)
- CRM/ERP: Bitrix24 (облако)
- Middleware: n8n (self-hosted или cloud)

### 3. Пошаговый план внедрения
1. В Bitrix24 создать исходящий вебхук на событие `CRM_DEAL_UPDATE` (стадия = "Договор")
2. В n8n создать Webhook-триггер, принять `deal_id`
3. HTTP Request → `crm.deal.get` + `crm.activity.list` — вытянуть данные сделки и переписку
4. Claude API node: передать контекст переписки + системный промпт (см. блок ниже)
5. HTTP Request → `crm.deal.update` (поле `COMMENTS`) или загрузить файл через `disk.file.upload` + прикрепить к сделке
6. Уведомление ответственному в Bitrix24 через `im.message.add`

### 4. Промпты и Код
```
SYSTEM:
Ты — юридический ассистент. На основе переписки сгенерируй договор оказания услуг.
Структура: преамбула, предмет договора, стоимость и порядок оплаты, сроки, реквизиты сторон (оставь плейсхолдеры [[РЕКВИЗИТЫ_КЛИЕНТА]]).
Формат: markdown. Язык: русский.

USER:
Переписка по сделке:
{activities_text}

Данные сделки: клиент — {company_name}, сумма — {deal_sum} руб., менеджер — {assigned_name}.
```

```json
// n8n HTTP Request — загрузка файла в Bitrix24
{
  "method": "POST",
  "url": "https://YOUR_DOMAIN.bitrix24.ru/rest/1/TOKEN/disk.folder.uploadfile",
  "body": {
    "id": "FOLDER_ID",
    "data": { "NAME": "Договор_{{ $json.deal_id }}.md" },
    "fileContent": "{{ $json.contract_text | base64 }}"
  }
}
```
