## 1. Название
**AI-Проксёр** — РФ-шлюз для ИИ в n8n

## 2. Суть
Готовый инфраструктурный модуль: Cloudflare Worker или личный прокси-сервер, который позволяет n8n-сценариям клиента работать с Claude и Gemini API без VPN, без блокировок и без потери скорости — за 15 минут установки.

## 3. Бизнес-сценарий
ДО: Клиент звонит и говорит: «Мы всё настроили в n8n, но Claude API не отвечает — пишет 403, наверное, блокировка». Вся автоматизация стоит. Разработчик тратит 2–4 часа на поиск причины и ручную настройку прокси. ИИ делает: клиент получает готовый Cloudflare Worker URL и инструкцию на 3 шага: вставь URL в n8n-ноду вместо стандартного endpoint, сохрани, запусти тест. Клиент получает: все ИИ-сценарии работают без VPN, Worker пропускает 100 000 запросов/сутки бесплатно, при превышении — автоматический фолбэк на личный Worker клиента.

## 4. Логика
```
n8n HTTP-нода (запрос к Claude/Gemini) → Cloudflare Worker (РФ-совместимый прокси) → Anthropic/Google API → ответ в n8n
```

## 5. Технический стек
- n8n
- Cloudflare Workers
- claude-sonnet-4-7
- Gemini API
- GitHub (хранение конфига Worker)

## 6. Связки инструментов
```
n8n Credentials (Base URL = Worker URL) → Cloudflare Worker (proxy rewrite headers) → Anthropic API / Google AI API → JSON-ответ обратно в n8n
```

## 7. Алгоритм реализации
1. Развернуть Cloudflare Worker из готового шаблона (wrangler deploy): worker перехватывает запросы на /claude/* и /gemini/*, переписывает Host-заголовок, проксирует на официальные API-эндпоинты
2. Добавить rate-limit логику в Worker: если дневной лимит 100 000 запросов превышен — автоматический редирект на резервный Worker клиента (URL передаётся через переменную среды FALLBACK_URL)
3. В n8n заменить стандартный Base URL в Credentials: для Claude — https://api.anthropic.com → https://{worker}.workers.dev/claude, для Gemini — аналогично; проверить тестовым HTTP-запросом статус 200
4. Настроить мониторинг: n8n Cron каждые 6 часов пингует Worker health-check эндпоинт, при статусе != 200 — Telegram-алерт разработчику студии
5. Подготовить клиентскую инструкцию (PDF, 1 страница): 3 шага самостоятельного развёртывания личного Worker через Cloudflare Dashboard без терминала — для случая когда общий Worker перегружен

## 8. Почему купят (ROI)
Каждый раз когда n8n-автоматизация падает из-за блокировки API — клиент теряет лиды, заказы, данные. Один простой в работает стоит бизнесу от 5 000 до 50 000 руб. упущенной выручки. AI-Проксёр стоит 15 000 руб. разово и решает проблему навсегда. Это не технический продукт — это страховка от простоя. Крючок: «VPN для бизнеса умирает в 2 часа ночи. Worker — никогда».
