## 1. Название
**AI-Воронщик** — Автопродавец в Telegram

## 2. Суть
Telegram-бот на n8n + Claude ведёт диалог с потенциальным клиентом, квалифицирует его по сценарию продаж, заполняет карточку в CRM и передаёт менеджеру уже «тёплого» лида с историей переписки и скором готовности к покупке.

## 3. Бизнес-сценарий
ДО: Клиент пишет в Telegram-канал компании: «Хочу автоматизацию, сколько стоит?» Менеджер отвечает через 3 часа (был на встрече), клиент уже смотрит конкурентов. ИИ ДЕЛАЕТ: Бот мгновенно отвечает: «Привет! Расскажите — какой процесс хотите автоматизировать?» Клиент: «Обработку заявок с сайта». Бот: «Отлично! А сколько заявок в месяц примерно приходит?» — «Около 200». Бот: «Понял! И уже есть CRM или пока в таблицах?» — после 4 вопросов Claude присваивает скор 4/5, создаёт карточку в Битрикс24. КЛИЕНТ ПОЛУЧАЕТ: сообщение «Передаю вас Алексею, он свяжется в течение 30 минут» — и менеджер получает уведомление с готовым досье: имя, потребность, бюджет, размер компании, скор.

## 4. Логика
```
Telegram Message → n8n Telegram Trigger → Claude API (диалог + квалификация с памятью сессии) → Google Sheets (лог) → Bitrix24 REST API (создание лида) → Telegram → менеджер
```

## 5. Технический стек
- n8n
- claude-3-5-sonnet
- Telegram Bot API
- Bitrix24 REST API
- Google Sheets API

## 6. Связки инструментов
```
Telegram Webhook → n8n → Claude API с системным промптом (роль: менеджер по продажам, сценарий: 4-6 вопросов квалификации) → Session Storage в Google Sheets → по завершении: Bitrix24 REST API создание лида → Telegram sendMessage менеджеру
```

## 7. Алгоритм реализации
1. Создать Telegram-бота через BotFather, подключить Webhook к n8n; хранить chat_id + историю диалога в Google Sheets (строка = сессия, колонки = сообщения, до 10 реплик)
2. Написать системный промпт Claude: роль менеджера по продажам компании, задача — узнать 5 параметров (сфера бизнеса, проблема, размер команды, бюджет, срок) за не более 6 вопросов, после — вернуть JSON с квалификацией
3. Реализовать логику завершения: если Claude вернул {qualified: true} — запустить ветку создания лида; если {qualified: false, reason: 'не целевой'} — отправить вежливый отказ с рекомендацией
4. Создать лид в Bitrix24 через POST /crm.lead.add с полями: ФИО, телефон/username, SOURCE_ID=TELEGRAM, комментарий = полный транскрипт диалога, кастомные поля скора и бюджета
5. Отправить менеджеру Telegram-уведомление с кнопками: [📞 Позвонить] [💬 Написать] [📋 Открыть в CRM] — deeplink на сделку в Битрикс24

## 8. Почему купят (ROI)
Среднее время ответа менеджера на входящий запрос в мессенджере — 2,5 часа. За это время клиент получает 3-4 предложения от конкурентов. Компания с 150 лидами/мес теряет до 40% из них только из-за скорости ответа — это 60 потенциальных сделок. При среднем чеке 100 000 руб. — 6 000 000 руб. упущенной выручки в год. Крючок: «Пока менеджер на обеде — ИИ уже квалифицировал и прогрел.»
