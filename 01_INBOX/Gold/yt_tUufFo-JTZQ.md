## Суть
Полный обучающий курс по платформе n8n для создания автоматизаций и ИИ-агентов без написания кода. Курс охватывает разбор всех нативных узлов n8n, архитектуру workflows и интеграцию с внешними сервисами для автоматизации бизнес-процессов.

## Бизнес-сценарий
Для бизнес-аналитиков, операционистов и менеджеров, которые хотят автоматизировать повторяющиеся бизнес-процессы (обработка документов, управление данными, интеграция CRM, автоматизация коммуникаций) без навыков программирования, используя low-code платформу n8n.

## Алгоритм реализации
1. 1. Введение в n8n: регистрация на platform n8n, создание нового workflow, знакомство с интерфейсом
2. 2. Разбор нативных узлов n8n: Trigger-узлы (Webhook, Schedule, Manual), Action-узлы (HTTP Request, Set, Merge, Function), Data-узлы (Split, Group By, Aggregate), Logic-узлы (If, Switch, Loop)
3. 3. Настройка триггеров: создание Webhook для получения входящих данных или использование Schedule для периодического запуска workflows по расписанию
4. 4. Обработка данных: использование узлов Set, Merge, Split для трансформации и структурирования данных, применение Map функций для манипуляции элементами массивов
5. 5. Интеграция с внешними сервисами: настройка HTTP Request узлов для подключения к API третьих сервисов (Bitrix24, amoCRM, Google Sheets), использование нативных интеграционных узлов платформы
6. 6. Подключение ИИ: интеграция с Claude API, ChatGPT, Gemini для создания ИИ-агентов, обработка текста и данных через AI models
7. 7. Условная логика: использование If/Switch узлов для ветвления workflows в зависимости от значений данных
8. 8. Тестирование и отладка: запуск workflows вручную, проверка логов, отладка data flow между узлами

## Технический стек
- n8n (самохостируемая или облачная версия)

- Claude API (Anthropic) для ИИ-обработки

- Google Gemini API для альтернативного ИИ

- amoCRM API/интеграция
- Google Sheets API
- Set узел (для переменных и структурирования данных)
- Merge узел (объединение нескольких потоков данных)
- Split узел (разделение массивов)
- Function узел (для кастомных трансформаций при необходимости)
- If/Switch узлы (условная логика)
- Loop узел (итерация по элементам)

## Связки инструментов
- Webhook → n8n workflow (триггер входящих данных)
- Schedule node → n8n (периодический запуск)
- HTTP Request → внешний API (интеграция сервисов)
- n8n nodes → Claude/ChatGPT API (ИИ-обработка)
- Set/Merge/Split узлы → трансформация данных → Action узлы
- If/Switch logic → условное ветвление workflow → разные Action узлы
- Loop → HTTP Request (массовая обработка данных)
- Webhook ← n8n webhook response (возврат результата)

## Конфигурация и параметры
- Создание нового workflow: нажать 'New', выбрать 'Workflow', дать имя
- Добавление Webhook триггера: добавить узел 'Webhook', выбрать метод (POST/GET/PUT), скопировать URL вебхука
- Добавление Schedule триггера: узел 'Schedule', настроить cron expression или выбрать из предустановок (каждый день в 9:00, каждый понедельник и т.д.)
- HTTP Request узел: поле 'URL' - вставить endpoint API, 'Method' - выбрать HTTP метод, 'Authentication' - настроить ключи (API key, Basic Auth, Bearer token)
- Set узел: поле 'Key' - название переменной, 'Value' - значение или выражение из предыдущих узлов (используется {{ }} синтаксис для ссылок на данные)
- If узел: 'Condition' - выбрать поле для проверки, 'Operation' - equals/contains/greater than и т.д., 'Value' - значение для сравнения
- Function узел: JavaScript код для кастомной логики, доступ к входящим данным через $input.all() или $input.first()
- Merge узел: настройка способа слияния (append, merge objects, combine), выбор полей для merge
- Split узел: 'splitIn' параметр - указать поле с массивом для разделения, каждый элемент будет обработан отдельно
- Loop узел: выбрать количество итераций или использовать данные из предыдущих узлов
