# 🗂️ Паспорт идеи: AI-агент на n8n с Claude

**Источник:** [I&#39;ll never use n8n the same......](https://youtube.com/watch?v=s96JeuuwLzc) — NetworkChuck (2025-12-10)
**Категория:** Автоматизация отдела
**Заменяет:** Операционный менеджер / ассистент по обработке данных и коммуникациям

---

## 💡 Что это даёт бизнесу
Клиент получает self-hosted n8n-агента на базе Claude, который автономно обрабатывает входящие запросы, документы и задачи без участия человека. Агент интегрируется с CRM, почтой, мессенджерами и внутренними системами, сокращая ручной труд на 60–80%. Полный контроль над данными — всё крутится на собственном сервере клиента.

---

## ✅ Плюсы
- Self-hosted: данные не уходят в облако — критично для юридических, финансовых и медицинских компаний
- n8n + Claude = мощный стек, который уже знаком аудитории NetworkChuck — спрос на интеграторов есть
- Масштабируемо: один шаблон агента продаётся разным нишам (юристы, HR, продажи)

## ❌ Минусы / Риски
- Видео — маркетинговый обзор Hostinger, глубина технических деталей минимальна, придётся строить архитектуру самостоятельно
- Self-hosting требует DevOps-компетенций при сопровождении — возможны проблемы у клиентов без IT-отдела

## 🚀 Шаги запуска
1. Шаг 1: Развернуть n8n на VPS через Docker, настроить HTTPS и базовую безопасность
2. Шаг 2: Подключить Claude API через n8n AI Agent node, настроить системный промпт под нишу клиента (юристы / HR / продажи)
3. Шаг 3: Интегрировать агента с каналами клиента — Telegram, email, Bitrix24/amoCRM
4. Шаг 4: Передать клиенту, обучить, подключить абонемент на поддержку и доработки
