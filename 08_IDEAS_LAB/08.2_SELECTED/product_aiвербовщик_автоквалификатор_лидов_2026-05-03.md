## 1. Название
**AI-Вербовщик** — Автоквалификатор Лидов

## 2. Суть
Claude ведёт первичный диалог с новым лидом в WhatsApp по заданному сценарию: задаёт квалификационные вопросы, определяет бюджет, срочность и потребность, присваивает скор и передаёт горячего лида менеджеру с заполненной карточкой в CRM — менеджер подключается только к квалифицированным контактам.

## 3. Бизнес-сценарий
ДО: менеджер Марина обрабатывает 80 входящих в месяц, тратит 30–40 минут на каждый первичный контакт, из которых 60% оказываются нецелевыми — «просто посмотреть» или бюджет в 10 раз меньше минимального. 48 часов её рабочего времени уходит на разговоры, которые никогда не станут сделкой. ИИ делает: лид пишет в WhatsApp «Хочу заказать сайт» → ИИ отвечает: «Привет! Расскажите, для какого бизнеса сайт и какой бюджет рассматриваете?» → после 3–5 вопросов Claude присваивает скор: бюджет (1–3), срочность (1–3), ЛПР (1–2) → если сумма ≥ 6/8 — создаёт сделку в Битрикс24 со статусом «Горячий» и уведомляет Марину: «Новый горячий лид: Алексей, агентство недвижимости, бюджет 150–200 тыс., нужно до конца месяца, он ЛПР». Клиент получает: Марина общается только с теми, кто реально купит.

## 4. Логика
```
Wazzup Webhook (входящее WA) → n8n (сессия диалога) → Claude (квалификационный скрипт, 5 вопросов) → скоринг JSON → IF скор ≥ 6 → Bitrix24 crm.deal.add (статус Горячий) + уведомление менеджеру; IF скор < 4 → автонурturing-сообщение
```

## 5. Технический стек
- n8n
- claude-sonnet-4-5
- Wazzup
- Bitrix24
- PostgreSQL (сессии диалогов)

## 6. Связки инструментов
```
Wazzup Webhook → n8n (хранение контекста сессии в PostgreSQL) → Claude API (multi-turn диалог) → Scoring JSON → Bitrix24 crm.deal.add + Wazzup sendMessage
```

## 7. Алгоритм реализации
1. Настроить Wazzup webhook в n8n, реализовать хранение контекста диалога в PostgreSQL по chat_id — агент помнит предыдущие ответы в рамках сессии 24 часа
2. Написать Claude system-промпт с квалификационным скриптом: 5 обязательных вопросов (ниша, задача, бюджет, срок, ЛПР ли собеседник), стиль — живой, без роботизированности
3. Реализовать скоринг: Claude возвращает JSON {budget: 1-3, urgency: 1-3, decision_maker: 0-2, total: X}, n8n обрабатывает через IF-нodu — 3 ветки: горячий (≥6), тёплый (4–5), холодный (≤3)
4. Для горячих: создать сделку в Битрикс24 через crm.deal.add с заполненными полями из диалога + отправить менеджеру Telegram-уведомление с резюме за 30 секунд
5. Для холодных: отправить через Wazzup nurturing-сообщение с полезным контентом и добавить в список для повторного касания через 30 дней

## 8. Почему купят (ROI)
Менеджер тратит 40 мин на квалификацию одного лида вручную — при 80 лидах/мес = 53 часа = 1,5 рабочие недели на чистый отсев. 60% лидов нецелевые — менеджер теряет 32 часа на разговоры без результата. AI-Вербовщик сокращает это до 0 часов менеджера на нецелевых. Крючок: «Менеджер должен продавать, а не анкетировать».
