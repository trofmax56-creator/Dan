### 1. Описание идеи

Обычный ChatGPT/Claude не знает внутренних документов компании: регламентов, прайсов, кейсов, технической документации. RAG (Retrieval-Augmented Generation) решает это: документы компании индексируются в векторную БД, при вопросе — система находит релевантный фрагмент и передаёт его в LLM как контекст. AI отвечает строго на основе реальных данных.

### 2. Технический стек

- n8n — оркестратор (AI Agent нода с RAG)
- Embeddings API: OpenAI `text-embedding-ada-002` или Claude Embeddings
- Векторная БД: Pinecone / Qdrant (self-hosted) / n8n Postgres Vector Store
- Claude API — генерация ответа
- Google Drive / Notion / SharePoint — источники документов
- Make.com — альтернатива n8n

### 3. Пошаговый план внедрения

**Шаг 1 — Собрать и подготовить документы**
```
Источники для индексирования:
✓ FAQ и регламенты (PDF, DOCX)
✓ Прайс-лист (Google Sheets → экспорт в текст)
✓ Кейсы и портфолио (Notion pages)
✓ Техническая документация (Confluence)

Правило: 1 документ = 1 тема. Смешанные документы → разбить.
```

**Шаг 2 — Индексирование (однократно)**
```javascript
// n8n: для каждого документа:
// 1. Разбить на чанки (500-1000 символов с перекрытием 100)
// 2. Получить embedding
// 3. Сохранить в векторную БД

// HTTP Request → OpenAI Embeddings
POST https://api.openai.com/v1/embeddings
Body: { "model": "text-embedding-ada-002", "input": "{{chunk_text}}" }
// → получить vector[1536]

// Pinecone: сохранить вектор + метаданные
POST https://{{pinecone_host}}/vectors/upsert
Body: {
  "vectors": [{
    "id": "{{doc_id}}_{{chunk_idx}}",
    "values": {{embedding}},
    "metadata": { "text": "{{chunk_text}}", "source": "{{filename}}", "date": "{{doc_date}}" }
  }]
}
```

**Шаг 3 — Поиск при вопросе пользователя**
```javascript
// Получить embedding вопроса
const q_embedding = await openai.embed(user_question)

// Найти топ-3 похожих чанка
const results = await pinecone.query({
  vector: q_embedding,
  topK: 3,
  includeMetadata: true
})

const context = results.matches.map(m => m.metadata.text).join('\n\n')
```

**Шаг 4 — Ответ через Claude с контекстом**
```javascript
const answer = await claude({
  system: `Ты — ассистент компании [Название].
  Отвечай ТОЛЬКО на основе предоставленных данных.
  Если информации нет в данных — скажи "Не нашёл в базе знаний".
  Не придумывай данные.`,
  user: `Данные из базы знаний:\n${context}\n\nВопрос: ${user_question}`
})
```

**Шаг 5 — Интерфейс (Telegram / Web)**
```
Telegram Bot (рецепт 069) → вопрос → RAG → ответ
ИЛИ
n8n Form → вопрос → RAG → ответ в браузере
```

### 4. Сценарии использования

| База знаний | Кто использует | Пример вопроса |
|---|---|---|
| Регламенты HR | Сотрудники | «Сколько дней отпуска положено?» |
| Прайс-лист + кейсы | Менеджеры продаж | «Какова стоимость X для банков?» |
| Тех. документация | Поддержка | «Как настроить интеграцию с Y?» |
| Договоры | Юрист / менеджер | «Каковы условия расторжения договора №123?» |

### 5. Нюансы

- Качество RAG = качество подготовки документов. Плохо структурированный PDF → плохие ответы
- Размер чанка: 500–800 символов + перекрытие 100 — оптимум для большинства документов
- Pinecone бесплатный план: 100k векторов — достаточно для ~500 документов
- Для русского языка: multilingual-e5-large embeddings работают лучше ada-002

### 6. Оценка эффективности

- Сложность: 3/5 (сложность в подготовке документов)
- Время внедрения: 5–10 дней
- ROI: HR-специалист тратит 30–60 мин/день на ответы на типовые вопросы → ноль после внедрения
