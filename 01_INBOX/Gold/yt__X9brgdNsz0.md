## Суть
Интеграция 1С УНФ (Универсальный Новый Формат) с Битрикс24 для синхронизации данных контрагентов. Использование приложения "Полезные реквизиты контрагентов" позволяет автоматически передавать информацию из учетной системы в CRM, обеспечивая актуальность клиентской базы и упрощение работы с контрагентами.

## Бизнес-сценарий
Синхронизация справочной информации и реквизитов контрагентов между 1С УНФ (система учёта и финансов) и Битрикс24 (CRM-система). Используется для малых и средних предприятий, требующих актуальности данных контрагентов в CRM без ручного внесения информации. Обрабатывает данные о клиентах: реквизиты, контактные данные, статусы, платежную информацию.

## Алгоритм реализации
1. Шаг 1: Установка приложения 'Полезные реквизиты контрагентов' в Битрикс24 из маркетплейса или загрузка готового решения
2. Шаг 2: Настройка подключения к 1С УНФ - указание адреса сервера, учетных данных для доступа (логин/пароль)
3. Шаг 3: Конфигурирование правил синхронизации - выбор полей контрагентов, которые должны синхронизироваться между системами
4. Шаг 4: Установка расписания обновления данных (периодичность синхронизации: каждый час, ежедневно и т.д.)
5. Шаг 5: Тестирование интеграции - создание тестового контрагента в 1С и проверка его появления в Битрикс24
6. Шаг 6: Запуск полной синхронизации существующей базы контрагентов из 1С в Битрикс24
7. Шаг 7: Мониторинг логов синхронизации и обработка ошибок при несовпадении данных

## Технический стек
- 1С:Бухгалтерия 3.0 (УНФ - Универсальный Новый Формат)
- Битрикс24 (CRM)
- Приложение 'Полезные реквизиты контрагентов'
- REST API Битрикс24
- HTTP-интеграция 1С
- JSON для обмена данными
- REST API 1С (если используется облачное решение)
- Веб-хуки Битрикс24 для обновления информации в реальном времени

## Связки инструментов
- 1С (экспорт данных контрагентов) → HTTP-запрос → API Битрикс24 → приложение 'Полезные реквизиты контрагентов' → обновление контактов CRM
- Битрикс24 (создание/изменение контрагента) → Webhook → 1С УНФ (обновление справочника)

## Конфигурация и параметры
- Параметр 'Адрес сервера 1С': формат http://[IP-адрес]:8080/
- Учетные данные: логин и пароль пользователя 1С с правами доступа к справочникам
- Поле 'Периодичность синхронизации': значения - ежечасно, ежедневно, еженедельно
- Выбор полей для синхронизации: Наименование, ИНН, КПП, ОГРН, адрес, телефоны, E-mail
- Параметр 'Уникальный идентификатор': использование ИНН как ключа для привязки контрагентов
- API-ключ Битрикс24: получение из настроек приложения
- Параметр 'Действие при отсутствии контрагента в 1С': создать новый / пропустить
- Настройка маппинга полей (соответствие названий полей между системами)

## Ключевые инсайты
- Использование ИНН как уникального идентификатора позволяет избежать дублирования контрагентов при синхронизации
- Приложение 'Полезные реквизиты контрагентов' автоматически дополняет контакты в Битрикс24 информацией о счетах-фактурах и платежах из 1С
- Синхронизация может быть как односторонней (из 1С в Битрикс24), так и двусторонней в зависимости от настроек
- Для облачного Битрикс24 используется REST API с токен-аутентификацией, для коробочной версии - может потребоваться доп. настройка firewall
- При синхронизации больших объемов данных (>10000 контрагентов) рекомендуется разделить импорт на этапы и мониторить нагрузку на серверы
- Типичная ошибка: несовпадение формата телефонов между системами - 1С использует один формат, Битрикс24 может требовать другой
- Важно регулярно проверять логи синхронизации и настроить уведомления об ошибках
- Приложение позволяет добавить в карточку контрагента дополнительные реквизиты: расчетные счета, банковские реквизиты, руководителей компании

## Подводные камни
- Необходимо убедиться, что сервер 1С доступен из интернета или находится в одной сети с Битрикс24 (для коробочной версии)
- При изменении полей синхронизации уже существующие контакты в Битрикс24 не обновляются автоматически - требуется переустановка приложения или ручное обновление
- ИНН должен быть уникальным и корректно заполнен в обеих системах, иначе произойдет дублирование
- Если в 1С удален контрагент, то в Битрикс24 его контакт остается - требуется настройка дополнительной логики для удаления
- Разница в часовых поясах серверов может привести к ошибкам при синхронизации по расписанию
- API Битрикс24 имеет лимиты на количество запросов - при большом объеме данных может потребоваться увеличение плана подписки
- При двусторонней синхронизации возможна потеря данных, если не настроить правильно приоритет обновления (какая система является источником истины)
