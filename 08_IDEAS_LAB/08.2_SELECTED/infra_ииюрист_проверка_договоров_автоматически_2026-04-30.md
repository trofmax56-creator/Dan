## 💡 Что это даёт бизнесу
Клиент загружает договор — система за 1–2 минуты выявляет риски, невыгодные условия, противоречия и отсутствующие пункты. Экономит 2–4 часа работы юриста на каждый договор. Малый бизнес получает юридическую защиту без штатного юриста за фиксированную стоимость.

---

## ✅ Плюсы
- Острая боль у МСБ: юрист дорог, а договоры проверять нужно постоянно
- Быстрая окупаемость для клиента: 1 сэкономленный спор покрывает стоимость системы
- Легко упаковать в SaaS или разовый пакет под CRM (Bitrix24/amoCRM)

## ❌ Минусы / Риски
- Юридическая ответственность: клиент может предъявить претензии если ИИ пропустит критичный риск — нужен дисклеймер
- Высокая конкуренция: Inspecto, LegalTech-стартапы, GPT-обёртки уже существуют — нужна ниша или интеграция с CRM как УТП

## 🧰 Артефакты для реализации

**Инструменты:** n8n / Make.com — оркестрация пайплайна, Claude API (Anthropic) — анализ и структурированный вывод рисков, Bitrix24 / amoCRM — интеграция как виджет в карточку сделки, Telegram Bot — альтернативный интерфейс загрузки договора
**API:** Anthropic Claude API (claude-3-5-sonnet), OpenAI GPT-4o API, Битрикс24 REST API, Google Drive API (загрузка документов)

## 🚀 Шаги запуска
1. Шаг 1: Провести 5 интервью с клиентами (юристы, собственники МСБ) — выявить топ-5 типов договоров и болей
2. Шаг 2: Собрать MVP на n8n + Claude API: загрузка PDF/DOCX → парсинг текста → промпт-анализ → отчёт с рисками в PDF/Google Doc
3. Шаг 3: Упаковать в виджет Bitrix24 — кнопка 'Проверить договор' в карточке сделки, результат сохраняется в комментарии
4. Шаг 4: Продать 3–5 пилотам по 80–120к, собрать кейсы, запустить пакетное предложение для юридических и риелторских агентств
