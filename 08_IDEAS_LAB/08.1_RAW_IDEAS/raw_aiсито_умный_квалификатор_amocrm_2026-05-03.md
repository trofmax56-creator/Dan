## 1. Название
**AI-Сито** — Умный Квалификатор amoCRM

## 2. Суть
ИИ-агент перехватывает входящего лида в amoCRM, за 3 минуты проводит квалификацию по 5 критериям через Claude и автоматически расставляет теги, статус, приоритет и назначает менеджера без единого клика человека.

## 3. Бизнес-сценарий
ДО: Новый лид упал в amoCRM в 23:47. Менеджер увидит его утром, потратит 20 минут на выяснение «а что вам нужно?» и пропишет в карточку от руки. Из 80 лидов в месяц — 30 теряются в очереди. ИИ ДЕЛАЕТ: amoCRM стреляет вебхук в n8n. Claude читает имя, телефон, источник, сообщение из формы и за 40 секунд возвращает JSON: {score: 4, segment: 'горячий', budget: 'от 200 тыс', tag: 'автоматизация-продаж', manager: 'Иванов'}. КЛИЕНТ ПОЛУЧАЕТ: утром менеджер открывает amoCRM — карточка уже заполнена, лид помечен «горячий 🔥», назначен ответственный, поставлена задача «Позвонить до 10:00». Клиент из формы думал: «Напишу, посмотрим» — а ему уже звонят в 9:05.

## 4. Логика
```
amoCRM Webhook (новый лид) → n8n → Claude API (квалификация по промпту) → JSON-парсинг → amoCRM REST API (обновление полей + назначение менеджера) → Telegram-уведомление РОПу
```

## 5. Технический стек
- n8n
- claude-3-5-haiku
- amoCRM API
- Telegram Bot API

## 6. Связки инструментов
```
amoCRM Webhook → n8n Webhook Node → Claude API (системный промпт квалификации) → Function Node (парсинг JSON) → amoCRM REST API (PATCH /leads/{id}) → Telegram sendMessage (РОП)
```

## 7. Алгоритм реализации
1. Создать вебхук в amoCRM на событие 'Создана сделка', направить POST на n8n endpoint
2. Собрать в Function Node пакет данных: имя, телефон, email, источник, текст формы, UTM-метки — передать в Claude с промптом квалификации (5 критериев: бюджет, срок, ЛПР, потребность, соответствие ЦА)
3. Claude возвращает строгий JSON за 1 API-вызов: score 1-5, сегмент, предполагаемый бюджет, рекомендуемый тег, ID менеджера
4. Обновить сделку в amoCRM через PATCH: записать score в кастомное поле, установить тег, назначить ответственного, сдвинуть на нужный этап воронки
5. Отправить Telegram-уведомление РОПу если score ≥ 4 — с кнопкой быстрого звонка, если score ≤ 2 — переместить в воронку 'Дозрев' и поставить задачу через 14 дней

## 8. Почему купят (ROI)
Менеджер тратит в среднем 25 мин на ручную квалификацию и заполнение карточки. 80 лидов/мес × 25 мин = 33 часа = 4 рабочих дня впустую. При ФОТ менеджера 80 000 руб/мес — это 16 000 руб потерянного времени ежемесячно только на ввод данных, плюс ещё 20-30% лидов теряются пока лежат необработанными. Крючок: «Менеджер продаёт, а не анкетирует.»
