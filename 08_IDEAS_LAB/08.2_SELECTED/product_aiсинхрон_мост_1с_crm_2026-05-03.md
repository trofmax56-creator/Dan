## 1. Название
**AI-Синхрон** — Мост 1С ↔ CRM

## 2. Суть
n8n-агент синхронизирует данные между 1С/МойСклад и Битрикс24/amoCRM в реальном времени: новые контрагенты, счета, статусы оплат и остатки на складе автоматически появляются в нужной системе без ручного переноса и дублирования.

## 3. Бизнес-сценарий
ДО: менеджер Оля закрыла сделку в Битрикс24, но в 1С контрагента ещё нет — бухгалтер Петрович вручную забивает реквизиты, звонит Оле уточнить ИНН, теряет полчаса. Счёт выставлен в 1С — Оля не знает, оплачено ли, звонит бухгалтеру ещё раз. ИИ делает: сделка переходит в статус «Договор подписан» в Битрикс24 → n8n через 30 секунд создаёт контрагента в 1С с ИНН из сделки (проверен через Dadata), выставляет счёт. Как только бухгалтер проводит оплату в 1С — сделка в Битрикс24 автоматически переходит в «Оплачено», менеджер получает уведомление. Клиент получает: менеджер и бухгалтер работают в своих системах и не мешают друг другу — данные синхронизируются сами.

## 4. Логика
```
Bitrix24 webhook (смена статуса сделки) → n8n → Dadata (валидация ИНН) → 1С REST API (создать контрагента/счёт) → обратный webhook от 1С → n8n → Bitrix24 REST API (обновить статус сделки)
```

```
Bitrix24 Webhook → n8n → Dadata API (валидация) → 1С HTTP-сервис → МойСклад REST API → обратно Bitrix24 crm.deal.update
```

## 7. Алгоритм реализации
1. Настроить webhook в Битрикс24 на события: смена стадии сделки, создание контакта/компании — передавать в n8n все поля включая ИНН
2. Подключить Dadata API для автоматической проверки и обогащения реквизитов контрагента по ИНН (название, адрес, КПП, ОГРН) перед созданием в 1С
3. Реализовать n8n → 1С через HTTP-сервис (публикация REST на сервере 1С): создание контрагента, выставление счёта, получение статуса оплаты — 3 отдельных эндпоинта
4. Настроить обратную синхронизацию: 1С отправляет webhook в n8n при проведении оплаты → n8n обновляет стадию сделки в Битрикс24 и отправляет уведомление менеджеру в Telegram
5. Добавить дедупликацию по ИНН — перед созданием проверять наличие контрагента в обеих системах, при дубле — обновлять, не создавать нового

## 8. Почему купят (ROI)
Компания с оборотом 50 сделок/мес тратит: менеджер 20 мин × 50 = 16 часов + бухгалтер 30 мин × 50 = 25 часов = 41 час/мес на ручной перенос данных между системами. При ставках 80 000 и 70 000 руб./мес — это 27 000 руб. прямых потерь ФОТ ежемесячно. Плюс ошибки при ручном вводе реквизитов приводят к возврату документов и задержке оплат на 5–15 дней. Крючок: «Когда 1С и CRM не разговаривают — платит ваш бухгалтер своим временем».
