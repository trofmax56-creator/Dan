## Суть
Создание системы из 5 AI-агентов в n8n для автоматизации работы отдела продаж и контента. Агенты используют LLM (Claude/ChatGPT) для самостоятельного принятия решений, работают с CRM и обрабатывают данные клиентов, генерируя продажи и контент без ручного участия.

## Бизнес-сценарий
Компании с отделами продаж и маркетинга автоматизируют: 1) квалификацию лидов через AI-анализ заявок, 2) генерацию персонализированных писем и контента, 3) управление воронкой продаж в CRM (Bitrix24/amoCRM), 4) обработку заказов и коммуникацию с клиентами. Агенты работают 24/7, экономя 30-60% времени менеджеров.

## Алгоритм реализации
1. Шаг 1: Создание первого AI-агента квалификации лидов. Настройка триггера вебхука в n8n для получения новых заявок из формы сайта. Добавление ноды HTTP Request для обращения к Claude API (модель claude-3-sonnet) с параметром max_tokens: 1024 и температурой 0.7. Агент получает данные о контактах клиента и анализирует качество лида по критериям: бюджет, сроки, соответствие услугам.
2. Шаг 2: Обработка результатов анализа в условной ноде If. Лиды с оценкой выше 7/10 направляются в поток горячих клиентов (тег 'hot_lead' в Bitrix24), ниже 7 — в холодные (тег 'cold_lead'). Настройка ноды Bitrix24 с параметрами: contact_name, email, phone, score (оценка агента), tags (теги).
3. Шаг 3: Создание агента для генерации персонализированного контента. Использование Claude API с системным промптом на русском языке (значение поля 'system': 'Ты опытный copywriter. Генерируй email для продажи B2B услуг'). Входные данные: компания, должность контакта, предыдущие покупки. Выход: готовое письмо в формате Markdown.
4. Шаг 4: Интеграция с CRM (Bitrix24 или amoCRM). Настройка ноды для создания задачи менеджеру с текстом письма, полученным от агента. Параметры: deal_id, assignee_id, description (содержит письмо от AI). Установка напоминания через 24 часа на отправку письма.
5. Шаг 5: Создание агента обработки возражений. Триггер — получение письма от клиента (через Email или Webhook). Агент анализирует текст, извлекает возражение (цена, конкурент, сроки), генерирует ответ на возражение. Параметр n_completion_tokens: 800. Результат отправляется менеджеру на проверку или сразу клиенту.
6. Шаг 6: Создание агента для анализа и управления воронкой продаж. Получение данных из CRM API (список сделок с их статусами). Агент анализирует каждую сделку: дата последнего контакта, статус, сумма. Рекомендует: перемещение в следующий этап, отправку follow-up письма, архивирование неактивных. Результаты логируются в таблицу (Google Sheets или базу данных).
7. Шаг 7: Интеграция сроков выполнения и расписания. Использование ноды Schedule (типы: every day at 09:00 для утренней квалификации, every 8 hours для проверки воронки). Параметр timezone: Europe/Moscow. Условие: выполнять только в рабочие дни (Пн-Пт).
8. Шаг 8: Настройка логирования и мониторинга. Добавление ноды для сохранения логов (JSON формат) в Google Sheets или базу данных с колонками: timestamp, agent_name, input_data, output_result, error_message. Использование Google Sheets API v4 или Postgresql.
9. Шаг 9: Создание пятого агента для интеграции социальных сетей. Получение комментариев/сообщений из Instagram/Facebook (через Webhook или API). Агент анализирует тональность, определяет намерение (вопрос, жалоба, интерес). Генерирует ответ, согласованный с тоном бренда. Отправляет через API соцсети или создаёт задачу менеджеру.
10. Шаг 10: Финальная настройка и тестирование. Проверка всех вебхуков (ping тесты). Тестирование с реальными данными небольшого объёма (10-20 контактов). Мониторинг использования API (claude-3-sonnet стоит ~$0.003 за 1k tokens на вход). Настройка алертов при ошибке (Email уведомление менеджеру или лог в Slack).

## Технический стек
- n8n (платформа оркестрации workflow)
- Claude API 3 (claude-3-sonnet или claude-3-opus для сложных анализов)
- Bitrix24 API или amoCRM API (управление контактами и сделками)
- HTTP Request ноды (для обращения к LLM)
- Webhook ноды (приём данных из форм, email, социальных сетей)
- If ноды (условная логика маршрутизации лидов)
- Schedule ноды (расписание запуска агентов)
- Google Sheets API v4 (логирование результатов)
- Email ноды или SMTP (отправка писем, уведомлений)
- Function ноды (обработка JSON, парсинг ответов от LLM)
- Set переменные (сохранение состояния между шагами)
- Slack API (опционально, для алертов)
- PostgreSQL или MongoDB (опционально, вместо Google Sheets для логов)
