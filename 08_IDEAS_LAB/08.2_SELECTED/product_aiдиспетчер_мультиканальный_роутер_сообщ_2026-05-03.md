## 1. Название
**AI-Диспетчер** — Мультиканальный роутер сообщений

## 2. Суть
AI-агент получает входящие обращения из WhatsApp, Telegram и веб-формы одновременно, квалифицирует их через Claude, создаёт контакты и задачи в Bitrix24 и маршрутизирует к нужному менеджеру — без единого ручного клика.

## 3. Бизнес-сценарий
ДО: Клиент написал в WhatsApp в 22:00 «Нужен ваш продукт срочно, перезвоните». Менеджер увидел утром, клиент уже купил у конкурента. Параллельно тот же клиент написал в Telegram — там вообще никто не смотрит. ИИ делает: Wazzup передаёт сообщение в n8n, Claude читает контекст и определяет: новый клиент, горячий запрос, категория «оборудование». Автоматически создаётся контакт в Bitrix24, ставится задача менеджеру отдела оборудования с дедлайном 9:00, в WhatsApp уходит автоответ: «Привет! Мы получили ваш запрос, менеджер свяжется с вами в 9:00. Пока расскажите: какой объём нужен?» Клиент получает: ощущение что компания работает 24/7 и не теряет обращения.

## 4. Логика
```
Wazzup/Telegram Bot/Web Form Webhook → n8n Router → Claude (классификация + квалификация) → Bitrix24 REST API (контакт + задача + сделка) → Wazzup/Telegram (автоответ клиенту) → Telegram Bot (уведомление менеджеру)
```


## 7. Алгоритм реализации
1. Подключить Wazzup к n8n через webhook: настроить приём входящих сообщений WhatsApp с полями phone, message, timestamp, chat_id
2. Настроить параллельный триггер Telegram Bot webhook и парсинг веб-формы (HTTP POST) в единый унифицированный payload
3. Передать текст сообщения в Claude с системным промптом: определить тип обращения (новый/существующий клиент), категорию продукта (5 категорий), горячесть (шкала 1–5), извлечь имя и телефон если есть
4. По результату Claude: через Switch-ноду направить в нужный сценарий — горячий лид (скор 4–5) → немедленное создание сделки + SMS менеджеру; холодный (1–3) → только контакт + задача на утро
5. Создать контакт в Bitrix24 через POST /rest/crm.contact.add с источником канала (WhatsApp/Telegram/Web) в кастомном поле
6. Отправить автоответ клиенту через Wazzup API в течение 30 секунд с персонализированным текстом от Claude (учитывает время суток и тему обращения)
7. Уведомить ответственного менеджера в Telegram: «🔥 Новый горячий лид [имя/телефон], категория [X], написал: [цитата]»

## 8. Почему купят (ROI)
Российский МСБ теряет в среднем 23% лидов из-за ответа дольше 1 часа (данные AmoCRM 2024). При среднем чеке 50 000 руб. и 40 лидах в месяц — это 460 000 руб. упущенной выручки ежемесячно. Один пропущенный WhatsApp в 22:00 = потерянный клиент. Крючок: «Конкуренты спят — ваш AI-Диспетчер работает.»
