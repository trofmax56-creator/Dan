## 1. Название
**AI-Некромант** — Реаниматор Мёртвой Базы

## 2. Суть
Агент проходит по базе клиентов, которые не покупали более 90 дней, сегментирует их по истории покупок и причине молчания, генерирует персонализированное сообщение для каждого сегмента и рассылает через WhatsApp — без участия менеджера, с передачей горячих ответов в CRM.

## 3. Бизнес-сценарий
ДО: В Битрикс24 у оптового поставщика стройматериалов 1 200 клиентов, из которых 400 не покупали больше 4 месяцев. Менеджеры говорят «мы им звонили, они не берут трубку» и больше не трогают. Это 400 × 45 000 руб. средний чек = 18 млн руб. замороженной выручки. ИИ делает: выгружает сегмент «молчуны 90+», смотрит последний купленный товар и сумму, генерирует сообщение: «Сергей, добрый день! Вы у нас последний раз брали профлист в сентябре. Сейчас у нас новая поставка по цене на 8% ниже — хотите актуальный прайс?» Рассылает по 20 сообщений в час, горячие ответы («да, пришлите», «когда доставка?») автоматически меняют статус в Битрикс24 и назначают задачу менеджеру. Клиент получает: личное сообщение, а не спам-рассылку.

## 4. Логика
```
Cron (понедельник 10:00) → Bitrix24 (выгрузка сделок без активности 90+ дней) → n8n → Claude (сегментация + генерация персонального текста) → Wazzup (рассылка 20 сообщ/час) → входящий ответ → n8n → Bitrix24 (смена статуса + задача менеджеру)
```

## 5. Технический стек
- n8n
- claude-sonnet-4-6
- Wazzup
- Bitrix24
- Telegram Bot API

## 6. Связки инструментов
```
Bitrix24 REST API (фильтр сделок) → n8n → Claude (3 сегмента × уникальный промпт) → Wazzup REST API (рассылка с throttle 20/час) → Wazzup webhook (входящий ответ) → Bitrix24 REST API (статус + задача)
```

## 7. Алгоритм реализации
1. Настроить Cron-триггер в n8n (каждый понедельник 10:00), делать GET-запрос к Bitrix24 CRM.DEAL.LIST с фильтром: последняя активность > 90 дней, статус не «Закрыта проигрышем»
2. Сегментировать базу на 3 группы через n8n Code-ноду: A — покупали на 50 000+ руб., B — 10 000–50 000 руб., C — до 10 000 руб.; передавать в Claude сегмент, последний товар, дату последней покупки
3. Claude генерирует персонализированный текст WhatsApp-сообщения (≤ 300 символов) для каждого контакта с обращением по имени и релевантным оффером по сегменту
4. Отправлять через Wazzup с throttle строго 20 сообщений в час, логировать статус отправки (доставлено/прочитано) в Google Sheets
5. При входящем ответе от реанимированного клиента: автоматически менять статус сделки на «Тёплый повторный», создавать задачу менеджеру «Перезвонить в течение 2 часов» с текстом ответа клиента

## 8. Почему купят (ROI)
Стоимость возврата «спящего» клиента в 7-10 раз ниже привлечения нового лида через рекламу. База из 400 молчунов при конверсии реанимации 5% = 20 повторных сделок. При среднем чеке 45 000 руб. — 900 000 руб. выручки с одного запуска. Крючок: «Ваши деньги уже в базе — их просто никто не попросил вернуться.»
