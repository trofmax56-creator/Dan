## 1. Название
**AI-Некромант** — Реаниматор Мёртвой Базы

## 2. Суть
n8n-агент выгружает из CRM все сделки со статусом 'закрыто/отказ' за последние 6-24 месяца, Claude сегментирует базу по причинам отказа и генерирует персонализированные реактивационные сообщения, которые рассылаются через WhatsApp порциями по 30 штук в день.

## 3. Бизнес-сценарий
ДО: В Битрикс24 накопилось 800 «мёртвых» сделок за 2 года. Все со статусом «Не дозвонились», «Дорого», «Не сейчас». Менеджеры про них забыли. Это 800 тёплых контактов, которые уже знают компанию, но деньги на их повторный прогрев не тратятся. ИИ делает: агент выгружает все закрытые сделки, Claude сортирует: 120 — «дорого», 200 — «не вовремя», 150 — «не дозвонились». Для каждой группы — своё сообщение. Клиент из группы «дорого» получает в WhatsApp: «Антон, добрый день! Полгода назад вы интересовались автоматизацией отдела продаж. Мы запустили тариф Старт от 50 тыс. — специально для компаний, кто хочет начать без большого бюджета. Хотите посмотрим, что войдёт?» Из 800 контактов 40-60 отвечают — это готовые тёплые лиды из уже оплаченного трафика. Клиент получает: новые сделки без затрат на рекламу.

## 4. Логика
```
Cron (1 раз в неделю) → Bitrix24 API (закрытые сделки за период) → Claude (сегментация по причине отказа + генерация персонального сообщения) → n8n Queue (30 сообщений/день, задержка 20 мин между отправками) → Wazzup (WhatsApp рассылка) → при ответе: новая сделка в Bitrix24 + Telegram менеджеру
```

## 5. Технический стек
- n8n
- claude-3-5-sonnet
- Bitrix24 REST API
- Wazzup
- Telegram Bot API
- Google Sheets (статус реактивации)

## 6. Связки инструментов
```
n8n Cron (понедельник 10:00) → Bitrix24 crm.deal.list (filter: STAGE_ID=LOSE, DATE_MODIFY > 6 месяцев назад) → Claude (сегментация + текст сообщения под причину отказа) → n8n Queue 30 шт/день → Wazzup sendMessage → Wazzup Webhook (ответы) → Claude (анализ intent) → Bitrix24 crm.deal.add (при интересе) → Telegram уведомление
```

## 7. Алгоритм реализации
1. Выгрузить из Битрикс24 через crm.deal.list все сделки: STAGE_ID в [LOSE, WON_THEN_CANCELED], DATE_MODIFY за последние 6-24 месяца, включая поля: контакт, телефон, причина_проигрыша, сумма, менеджер — до 1000 записей
2. Через Claude сегментировать базу: определить причину отказа из поля 'причина проигрыша' или комментария → категория {price | timing | no_contact | competitor | no_need}; вернуть массив с категорией для каждой сделки
3. Для каждой категории Claude генерирует сообщение: учитывает категорию, имя клиента, название компании, продукт из старой сделки, текущий повод (новый тариф/сезон/кейс); temperature = 0.9 для разнообразия; длина ≤ 600 символов
4. Загрузить все сообщения в n8n Queue; отправлять через Wazzup по 30 штук в день с задержкой 20 минут между сообщениями (избегать блокировки WhatsApp); фиксировать статус в Google Sheets
5. При входящем ответе через Wazzup Webhook — Claude определяет intent (interested/not_interested/question); при interested создать новую сделку в Bitrix24 со ссылкой на старую и уведомить менеджера в Telegram с историей контакта

## 8. Почему купят (ROI)
Стоимость привлечения нового лида через рекламу — 3 000-15 000 руб. В мёртвой базе на 500 контактов лежит потенциал на 1,5-7,5 млн руб. уже оплаченного трафика. Реактивация обходится в 10-20 раз дешевле нового лида — клиент уже знает компанию. При конверсии 5% из 500 базы = 25 новых лидов из воздуха. Крючок: «Золото лежит в вашей мёртвой базе — нужно только его поднять».
