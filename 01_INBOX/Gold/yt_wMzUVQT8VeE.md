## Суть
AI-агент на платформе n8n, который автоматизирует контроль и мониторинг продаж лучше, чем традиционный РОП (Рулевой Отдела Продаж). Агент анализирует метрики продаж в реальном времени, выявляет проблемы и рекомендует действия для оптимизации работы отдела продаж.

## Бизнес-сценарий
Менеджеры по продажам и руководители отделов продаж используют этот агент для мониторинга KPI, анализа воронки продаж, контроля выполнения плана, идентификации проблемных сделок и получения автоматических рекомендаций по улучшению результатов продаж.

## Алгоритм реализации
1. 1. Агент получает данные о продажах из CRM-системы (амoCRM, Bitrix24 или другой) через API на регулярной основе (по расписанию каждый час/день)
2. 2. Структурирование и нормализация данных: извлечение KPI (количество сделок, сумма, этап воронки, прогноз)
3. 3. Передача структурированных данных в LLM (Claude, ChatGPT или другую языковую модель) через API
4. 4. LLM анализирует метрики и вычисляет показатели: процент выполнения плана, скорость закрытия сделок, среднее значение контракта, прогноз месячного результата
5. 5. На основе анализа модель генерирует обоснованные выводы и рекомендации (например: 'сделка XX дней в статусе Y - требуется звонок', 'менеджер ZZZ отстает от плана на 30%')
6. 6. Формирование отчета и отправка уведомлений в Slack, Email или Telegram руководителям с приоритизацией по критичности

## Технический стек
- n8n — платформа для создания workflow и автоматизации
- Claude API / ChatGPT API / Google Gemini — LLM для анализа данных и генерации рекомендаций
- амoCRM API / Bitrix24 API — интеграция с CRM для получения данных о сделках и менеджерах
- Webhook — триггер для запуска агента по расписанию или событиям
- HTTP Request ноды — для обращения к API
- JSON парсер — для структурирования данных
- Slack API / Telegram Bot API — для отправки уведомлений
- Spreadsheet (Google Sheets) — для логирования данных и истории анализов

## Связки инструментов
- Webhook (расписание cron) → n8n HTTP Request → амoCRM API /api/v4/leads/list (получить сделки с фильтром по статусу, менеджеру, дате изменения)
- амoCRM → Трансформация данных (JS код для расчета KPI) → Claude API (с System Prompt для анализа)
- Claude API → Парсинг JSON ответа → Логирование в Google Sheets
- Выявленные проблемы → Slack Notification с форматированием сообщения
- Альтернативный путь: Bitrix24 API /batchapi.json → Fetch contacts, deals, activities

## Конфигурация и параметры
- амoCRM: настройка API ключа в интеграциях, выбор полей для экспорта (статус сделки, контрольная дата, сумма, этап воронки, ответственный менеджер)
- n8n Schedule Trigger: cron выражение '0 9 * * MON-FRI' (каждый рабочий день в 9 утра) или '0 */2 * * *' (каждые 2 часа)
- HTTP Request к амoCRM: метод GET, headers с Authorization Bearer token
- Параметры запроса: filter[status][0]=142 (статус 'в работе'), limit=250, with=contacts,loss_reason
- Claude API prompt: 'Analyze these sales metrics and identify top 5 critical issues with specific deal IDs and manager names. Format as JSON with fields: issue_type, severity (high/medium), deal_id, manager_name, action_required'
- Slack channel: выбрать канал #sales-monitoring или #sales-alerts, использовать UID канала для надежности
- Google Sheets: столбцы - дата_проверки, критические_проблемы, количество_менеджеров_отстающих, прогноз_месячного_результата, статус_план

## Ключевые инсайты
- AI-агент работает 24/7 без усталости и эмоций, что дает объективный анализ в отличие от РОПа, который может пропустить проблемы
- Система выявляет не только текущие проблемы, но и предсказывает результаты на месяц на основе текущей траектории (например, если прогноз < план на 15%, это критично)
- Интеграция с амoCRM/Bitrix24 позволяет анализировать историю взаимодействия (дата последнего звонка, задачи, письма) — РОП часто забывает проверить эти данные
- Автоматические уведомления в Slack в 9 утра экономят 30-60 минут ежедневно на ручное составление отчетов для руководства
- Система может выявить низкопроизводительных менеджеров раньше — по скорости закрытия сделок, не только по выполнению плана
- JSON структура ответа от Claude позволяет легко автоматизировать дальше (например, создание автоматических задач менеджерам через амoCRM API)
- Настройка пороговых значений (например, если сделка > 30 дней в статусе 'переговоры' → HIGH, если 15-30 дней → MEDIUM) помогает избежать шума
- История в Google Sheets создает метрики для оценки самого агента — можно увидеть, насколько хороши его рекомендации через неделю/месяц
