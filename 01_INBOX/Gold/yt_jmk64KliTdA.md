## Суть
Автоматизация синхронизации контактов из SmartLead в Attio CRM через Make.com с корректировкой категории персоны для устранения неправильной категоризации при импорте данных.

## Бизнес-сценарий
Агентства по аутсорсингу и B2B-компании используют SmartLead для сбора потенциальных клиентов (лидов), но стандартная интеграция в Attio CRM неправильно категоризирует персон. Решение автоматически преобразует категорию персоны в Attio из неправильного значения в правильное при каждом импорте нового контакта.

## Алгоритм реализации
1. Шаг 1: Настройка триггера в Make.com — создать вебхук, который будет слушать новые контакты из SmartLead или использовать готовый коннектор SmartLead с фильтром на новых лидов
2. Шаг 2: Получение данных лида — извлечь все необходимые поля: имя, email, компания, должность, категория персоны из SmartLead через API
3. Шаг 3: Проверка категории персоны — добавить условие (IF) для проверки текущей категории, если она совпадает с 'Wrong person' или неправильным значением, то перейти к следующему шагу
4. Шаг 4: Преобразование данных — создать новый объект с корректной категорией персоны (например, 'Prospect', 'Decision Maker', 'Contact' — в зависимости от логики бизнеса)
5. Шаг 5: Обновление контакта в Attio CRM — использовать действие 'Update Person' в коннекторе Attio с передачей исправленной категории и других полей лида
6. Шаг 6: Логирование результата — добавить сообщение об успешном обновлении в лог Make.com для отслеживания синхронизованных контактов

## Связки инструментов
- SmartLead (новый лид) → Make.com вебхук
- Make.com вебхук → API SmartLead (получение полных данных лида)
- Make.com условие (IF) → проверка категории персоны
- Make.com трансформация → исправление значения категории
- Make.com → API Attio (создание/обновление персоны)
- Attio CRM → сохранение контакта с правильной категорией

## Конфигурация и параметры
- Триггер: 'SmartLead' модуль или вебхук с событием 'New Lead'
- Поля SmartLead: firstName, lastName, email, company, jobTitle, personCategory (исходное значение)
- Условие IF: проверка {{smartlead.personCategory}} на совпадение с 'Wrong person'
- Действие трансформации: используется функция или маппинг {{if(smartlead.personCategory == 'Wrong person'; 'Prospect'; smartlead.personCategory)}} или похожая логика
- Целевое поле Attio: 'Person Category' или 'category' (в зависимости от версии API)
- Параметры Attio API: метод PUT/PATCH к эндпоинту /persons/{id} или POST к /persons с указанием всех необходимых полей
- Обязательные поля для Attio: name, email, category (исправленная)
- Опциональные поля: company, jobTitle, source (например, 'SmartLead')

## Ключевые инсайты
- 1. Проблема категоризации персон часто возникает при интеграции между платформами из-за несоответствия значений — используйте условия IF для нормализации данных
- 2. SmartLead и Attio имеют разные наборы категорий персон — всегда создавайте маппинг (словарь соответствия) между системами
- 3. Используйте вебхуки вместо регулярных проверок (polling) для снижения потребления операций в Make.com и ускорения синхронизации
- 4. Тестируйте автоматизацию на пилотном наборе контактов перед запуском на всю базу, чтобы избежать массовой переписи неправильных данных
- 5. Добавьте фильтр в начале сценария для исключения дублей и тестовых контактов (например, email, содержащие '@test' или '@example.com')
- 6. Используйте функции Make.com для парсинга сложных полей: {{split()}}, {{trim()}}, {{lower()}} для нормализации текста перед отправкой в Attio
- 7. Не забудьте настроить обработку ошибок (Error Handling) в случае, если Attio API вернёт 400 или 404 статус
- 8. Логируйте все транзакции: создавайте записи в Google Sheets или отправляйте уведомления в Slack при ошибках синхронизации
- 9. Установите правильные таймауты и retry-логику для API-запросов (обычно 3-5 попыток с экспоненциальной задержкой)
- 10. Рассмотрите добавление поля 'syncedAt' в Attio для отслеживания, когда был последний импорт из SmartLead

## Подводные камни
- ⚠ Частая ошибка: неправильное имя поля категории персоны в API Attio — всегда проверяйте документацию API или используйте 'Test' в Make.com для вывода структуры ответа
- ⚠ SmartLead может отправлять категорию как 'Wrong person', 'WRONG_PERSON' или 'wrong-person' — используйте функцию {{lower()}} и обработку разных форматов
- ⚠ Ограничение API: Attio может иметь лимит 300 запросов в минуту — добавьте задержки (delay) между обновлениями, если обрабатываете большие объёмы
- ⚠ Если лид уже существует в Attio, API может вернуть конфликт (409) или обновить только определённые поля — проверьте, нужно ли использовать 'upsert' вместо update
- ⚠ Недостаток информации в видео: не указаны точные названия полей и значений в Attio — может потребоваться дополнительное тестирование и корректировка логики условия
