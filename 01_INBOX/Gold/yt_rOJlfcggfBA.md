## Суть
Разработка WhatsApp-бота для строительной компании с интеграцией в Битрикс 24 для автоматизации обработки заявок, управления контактами и реализации стратегии повторных продаж через чатбота.

## Бизнес-сценарий
Строительная компания использует WhatsApp-бот для: приёма заявок от клиентов, автоматической синхронизации данных в CRM Битрикс 24, управления контактами, отправки уведомлений о новых заявках менеджерам и автоматизации процесса повторных продаж (уведомления о готовности объектов, новые предложения).

## Алгоритм реализации
1. 1. Клиент отправляет сообщение в WhatsApp боту (название, телефон, описание проекта)
2. 2. WhatsApp API передаёт данные на вебхук платформы автоматизации (n8n/Make/Zapier)
3. 3. Платформа парсит полученные данные и форматирует в структурированный формат
4. 4. Автоматическая отправка данных в Битрикс 24 API: создание контакта или обновление существующего
5. 5. В Битрикс 24 создаётся новая сделка (Deal) в статусе 'Новая заявка' или 'На рассмотрении'
6. 6. Система отправляет уведомление менеджеру в WhatsApp или внутренний чат Битрикс 24
7. 7. При изменении статуса сделки в Битрикс 24 (например 'Завершена') — бот автоматически отправляет сообщение клиенту о готовности объекта
8. 8. Бот получает от Битрикс 24 список контактов и на основе критериев отправляет предложения повторных продаж (персонализированные
9. 9. Система ведёт историю взаимодействий в Битрикс 24 (все сообщения логируются в активность сделки)
10. 10. Аналитика: отчёты о количестве заявок, конверсии и эффективности бота через встроенные метрики Битрикс 24

## Связки инструментов
- WhatsApp API → n8n Webhook → JSON Parse → Битрикс 24 REST API (создание контакта)
- Битрикс 24 API → n8n → WhatsApp API (отправка уведомлений менеджеру)
- Битрикс 24 Webhook (смена статуса сделки) → n8n → WhatsApp API (уведомление клиенту)
- Битрикс 24 REST (получение списка контактов) → n8n (фильтрация по критериям) → WhatsApp API (рассылка предложений повторных продаж)

## Конфигурация и параметры
- WhatsApp Business Phone Number ID (из Meta Business Manager)
- Битрикс 24 REST API Key (Webhook URL и токен)
- n8n Webhook URL для приёма сообщений от WhatsApp
- ID поля в Битрикс 24 для синхронизации (PHONE, NAME, COMMENTS и т.д.)
- Статусы сделок в Битрикс 24 (константы для условных блоков)
- Шаблоны сообщений для WhatsApp (переменные типа {{customer_name}}, {{project_status}})
- Таймауты на повторную попытку отправки (retry logic)
- Фильтры для выбора контактов на повторные продажи (по дате, статусу, сумме)

## Ключевые инсайты
- Используется двусторонняя синхронизация: данные из WhatsApp → Битрикс 24, и обратно Битрикс 24 → WhatsApp через вебхуки
- Критический момент: правильная маппировка полей Битрикс 24 (UF_CRM_ID) — без этого контакты не свяжутся с существующими
- WhatsApp Business API требует верификации номера и одобрения шаблонов сообщений Meta (не всё можно отправить)
- Повторные продажи основаны на анализе истории взаимодействий: бот отправляет предложения только тем, кто ранее взаимодействовал
- Задержки в Битрикс 24 API могут быть до 2-3 секунд — нужны асинхронные проверки статуса
- Обработка ошибок: если контакт дублируется, нужна деду-плирование по номеру телефона перед отправкой в Битрикс 24
- Логирование всех запросов и ответов критично для отладки — сохранять полные JSON в n8n/Make логи
- Стоимость WhatsApp Business API: $0.0015 за входящее сообщение + стоимость шаблонных сообщений, нужно учитывать в бюджете

## Подводные камни
- WhatsApp Business API: есть лимиты на рассылку — нельзя отправлять более 1000 сообщений в день на один номер без проверки
- Битрикс 24: при создании контакта через API, если номер телефона уже существует, может создаться дубль вместо обновления
- n8n: при большом количестве запросов может быть rate limiting — нужно добавить task.wait() между отправками
- Шаблонные сообщения WhatsApp одобряются 24-48 часов — нельзя сразу после регистрации отправлять рассылки
- Интеграция с Битрикс 24: REST API работает только с публичными IP (localhost не подойдёт)
- Webhook от Битрикс 24 может быть нестабилен — рекомендуется polling (опрос) каждые 5 минут вместо real-time events
- При отправке больших текстов в WhatsApp нужно разбивать на несколько сообщений (лимит ~4096 символов)
- Никогда не хранить токены и API ключи в коде — использовать переменные окружения в n8n/Make
