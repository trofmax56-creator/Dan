# 🗂️ Паспорт идеи: Локальный LLM-сервер для бизнеса

**Источник:** [What is vLLM? Efficient AI Inference for Large Language Models](https://youtube.com/watch?v=McLdlg5Gc9s) — IBM Technology (2025-05-26)
**Категория:** Инфраструктура
**Заменяет:** —

---

## 💡 Что это даёт бизнесу
Клиент получает собственный сервер для запуска языковых моделей без утечки данных в облако — актуально для юридических, медицинских и финансовых компаний. Снижает стоимость API-запросов при высоких объёмах в 5–10 раз по сравнению с OpenAI. Даёт независимость от зарубежных сервисов и блокировок.

---

## ✅ Плюсы
- Полная конфиденциальность данных — модель работает на сервере клиента, ничего не уходит наружу
- Экономия на API при объёме от 100k токенов/день окупает железо за 2–4 месяца
- Актуально для импортозамещения — можно использовать российские модели (GigaChat, YandexGPT self-hosted)

## ❌ Минусы / Риски
- Целевая аудитория Студии (МСБ 50–500к бюджет) редко имеет GPU-сервер и DevOps — высокий порог входа клиента
- vLLM — сугубо инфраструктурный инструмент, видео не содержит бизнес-кейсов для прямой продажи МСБ

## 🧰 Артефакты для реализации

**Инструменты:** vLLM, Docker, n8n, Nginx (reverse proxy), Selectel / RunPod (GPU хостинг)
**API:** vLLM OpenAI-compatible REST API, HuggingFace Hub API (загрузка моделей)

**Репозитории:**
- https://github.com/vllm-project/vllm

**Документация:**
- https://docs.vllm.ai/en/latest/
- https://huggingface.co/docs

## 🚀 Шаги запуска
1. Шаг 1: Определить профиль клиента — юрист, медицина, финансы с объёмом >50k запросов/мес или требованием конфиденциальности
2. Шаг 2: Развернуть vLLM + выбранную модель (Mistral, LLaMA, GigaChat-open) на GPU-сервере в Docker
3. Шаг 3: Подключить к n8n/Make.com как кастомный HTTP-узел вместо OpenAI — минимальные изменения в пайплайнах
4. Шаг 4: Передать клиенту документацию, настроить мониторинг (Prometheus + Grafana), оформить поддержку
