# 🧠 Skill: CRM + AI Product Factory (v3.0)

Этот скилл превращает материалы из `01_INBOX/Gold` в готовые коммерческие продукты для продажи клиентам МСБ в России.

## 📋 Стандарт описания продукта (9 пунктов)
Каждая сгенерированная идея ДОЛЖНА содержать:

1. **Название:** Формат строго `AI-[Роль]` + расшифровка в скобках.
   — Имя = роль = что заменяет или улучшает. Клиент с первого слова понимает что покупает.
   — Примеры: «AI-Сито» (Умный Квалификатор), «AI-Некромант» (Реаниматор базы), «AI-Ревизор» (Контроль 100% звонков).

2. **Суть:** 1–2 предложения — только механика решения, без воды.

3. **Бизнес-сценарий:** Живая сцена по формату: **ДО → ИИ делает → клиент получает.**
   — Писать конкретными словами, которые говорит клиент и что ИИ отвечает/делает.
   — Клиент должен узнать себя, читая этот абзац.
   — Нельзя: «система помогает автоматизировать процесс квалификации».
   — Можно: «Клиент пишет в WhatsApp: "Сколько стоит?" Менеджер занят. ИИ отвечает: "Привет! Подскажите, какая у вас ниша и сколько сотрудников?" После ответов ИИ сам заполняет карточку в Битриксе.»

4. **Логика:** Путь данных одной строкой через стрелки.
   — Формат: `Trigger → Process → Action`
   — Пример: `Wazzup → n8n Webhook → Claude (Extraction) → Bitrix24 API`

5. **Технический стек:** Список инструментов. Адаптировать под РФ:
   Wazzup (не Twilio), ЮKassa (не Stripe), Dadata (не Clearbit), UIS/МТС Exolve (не Twilio Voice), Тинькофф/Точка (не зарубежные банки).

6. **Алгоритм реализации:** 3–5 шагов для n8n-инженера.
   — Конкретные глаголы действия + числа где применимо.
   — Нельзя: «настроить интеграцию».
   — Можно: «Рассылка по 20 сообщений в час», «Уведомление РОПу при оценке < 3».

7. **Почему купят (ROI):** Формула = **боль + цифра + крючок-инсайт.**
   — Боль: что теряет клиент прямо сейчас без этого продукта.
   — Цифра: рубли, часы, проценты — обязательно конкретно.
   — Крючок: одна фраза, которая запоминается и передаётся дальше.
   — Нельзя: «экономит время и повышает эффективность».
   — Можно: «Заменяет штат ОКК из 2 человек (160 000 руб./мес. ФОТ). Стоимость возврата клиента в 10 раз ниже нового лида.»

8. **Экономика:** Разработка для Студии / инфраструктура клиента в месяц / цена продажи клиенту.

9. **Ссылка на материал:** Прямая ссылка на исходный файл в `Gold`.

## 🔍 Алгоритм отбора Gold → Продукт

### Шаг 1 — Скоринг по Матрице Востребованности
Каждый Gold-файл оценивается по трём критериям:

| Критерий | Вопрос | Диапазон |
|---|---|---|
| **Pain** | Клиент теряет деньги/время прямо сейчас без этого? | 1–10 |
| **Dev** | Сложность реализации (инверсия: 1 = легко, 10 = сложно) | 1–10 |
| **Profit** | Потенциал выручки Студии (10 = продаём за 300к+) | 1–10 |
| **ИТОГ** | Pain + (10 − Dev) + Profit | max 30 |

### Шаг 2 — Роутинг по score
```
ИТОГ > 24  → 05_BIZ_RECIPES/                  (берём в производство)
ИТОГ 20–24 → 08_IDEAS_LAB/08.2_SELECTED/      (следующий спринт)
ИТОГ 18–19 → 08_IDEAS_LAB/08.1_RAW_IDEAS/     (идея, не срочно)
ИТОГ < 18  → пропускаем
```

### Шаг 3 — Фильтры качества (до синтеза)
Отсеиваем файл если:
- нет конкретной боли клиента (абстрактный контент, «вода»)
- решение не реализуемо на n8n + Claude + CRM за 2–4 недели
- нет понятного способа продать за ≥50 000 руб.

### Шаг 4 — Синтез продукта
Заполнить все 9 пунктов стандарта по материалу из Gold-файла.
Добавить тег `#action_needed` если ИТОГ > 24.

## 🛠 Алгоритм работы скилла
1. **Сбор:** Прочитать файл(ы) из `01_INBOX/Gold`.
2. **Скоринг:** Оценить по Матрице (шаг 1–3 выше).
3. **Синтез:** Заполнить все 9 пунктов стандарта.
