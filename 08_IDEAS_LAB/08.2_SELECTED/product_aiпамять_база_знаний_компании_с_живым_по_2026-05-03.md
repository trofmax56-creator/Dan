## 1. Название
**AI-Память** — База знаний компании с живым поиском

## 2. Суть
Корпоративная база знаний на Obsidian + RAG: новый сотрудник или менеджер задаёт вопрос в Telegram, Claude находит ответ в документах компании и отвечает цитатой из регламента — вместо звонка руководителю или часового поиска по папкам.

## 3. Бизнес-сценарий
ДО: Новый менеджер Катя в первую неделю задаёт одни и те же вопросы: «Какая скидка максимальная без согласования?», «Как оформить возврат от юрлица?», «Где шаблон КП для строителей?» Руководитель тратит 40–60 минут в день на ответы. Регламенты лежат в Google Docs, никто их не открывает, они устаревают. → ИИ делает: Все регламенты, скрипты, прайсы, шаблоны загружены в Obsidian-хранилище и проиндексированы. Катя пишет боту: «Максимальная скидка без согласования?» Claude ищет в базе, отвечает: «Согласно регламенту продаж v2.3 от 15 марта: до 7% — на усмотрение менеджера, 7–15% — согласование с РОПом, выше — только директор. Ссылка: [Регламент продаж, раздел 4.2]» → Клиент получает: Руководитель освобождает 40 мин/день. Онбординг нового сотрудника — с 2 недель до 3 дней. База знаний живёт и обновляется.

## 4. Логика
```
Telegram-вопрос → n8n webhook → векторный поиск в базе (Obsidian md-файлы) → Claude генерирует ответ с цитатой → Telegram-ответ сотруднику
```

## 5. Технический стек
- n8n
- claude-sonnet-4-6
- Obsidian
- Qdrant или Supabase pgvector
- Telegram Bot API

## 6. Связки инструментов
```
Telegram Bot → n8n Webhook → Qdrant (vector search по md-файлам) → Claude API (RAG-генерация ответа) → Telegram sendMessage со ссылкой на документ
```

## 7. Алгоритм реализации
1. Собрать все регламенты, скрипты, прайсы компании в Obsidian-хранилище (md-файлы); настроить n8n-пайплайн индексации: при изменении файла — автоматическое обновление векторов в Qdrant
2. Разбить документы на чанки по 500 токенов с overlap 50 токенов, создать embeddings через Claude или text-embedding-3-small, записать в Qdrant с метаданными (имя файла, раздел, дата)
3. На входящий вопрос в Telegram: n8n создаёт embedding вопроса, делает top-5 поиск в Qdrant по косинусному сходству, передаёт релевантные чанки + вопрос в Claude
4. Claude отвечает строго на основе найденных фрагментов, цитирует источник (имя файла + раздел), если ответа нет — явно сообщает «В базе нет информации, обратитесь к [имя ответственного]»
5. Логировать все вопросы без ответа в отдельный Telegram-канал для руководителя — еженедельный дайджест «Чего не хватает в базе знаний»

## 8. Почему купят (ROI)
Боль: руководитель МСБ тратит 30–60 минут в день на ответы одним и тем же вопросам — это 10–20 часов в месяц × средняя стоимость часа директора 3 000 руб. = 30 000–60 000 руб./мес потерянного времени. Онбординг нового сотрудника без базы знаний = 2 недели низкой эффективности = 40 000–80 000 руб. упущенных продаж. Крючок: «Регламент, который никто не читает — не регламент. Регламент, которому можно написать в Telegram — читают все.»
