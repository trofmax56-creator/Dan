## 1. Название
**AI-РОП** — Виртуальный Руководитель Отдела Продаж

## 2. Суть
Claude-агент ежечасно забирает данные сделок из amoCRM/Битрикс24, рассчитывает KPI каждого менеджера, выявляет зависшие сделки и генерирует конкретные рекомендации — кому позвонить, кого мотивировать, где горит план.

## 3. Бизнес-сценарий
ДО: РОП в 9 утра открывает CRM, вручную смотрит воронку каждого из 7 менеджеров, пишет сводку в Excel, проводит планёрку — 2 часа уходит только на подготовку данных. Критичные сделки замечает через 3 дня, когда уже поздно. ИИ делает: каждый час агент забирает все сделки из amoCRM. Claude анализирует: «Сделка #4821 с ООО Гермес — 18 дней без движения на этапе КП. Менеджер Иванов. Рекомендация: срочный звонок сегодня до 17:00, клиент получил КП от конкурента по данным переписки». В 8:55 РОП получает в Telegram готовый дейлик: топ-3 угрозы, топ-3 возможности, кто отстаёт от плана. Клиент получает: планёрка занимает 20 минут вместо 2 часов, ни одна сделка не теряется незаметно.

## 4. Логика
```
Cron (каждый час) → n8n → amoCRM/Bitrix24 API (все сделки за период) → Claude (анализ KPI, выявление аномалий, генерация рекомендаций) → Telegram Bot (дейлик РОПу) + Bitrix24 задачи менеджерам
```

## 5. Технический стек
- n8n

## 6. Связки инструментов
```
n8n Cron (каждый час) → amoCRM REST /api/v4/leads → Claude API (system prompt: аналитик продаж, JSON-ответ с приоритетами) → Telegram sendMessage (РОПу) → Bitrix24 task.add (менеджерам при критичных сделках) → Google Sheets (лог аналитики)
```

## 7. Алгоритм реализации
1. Настроить Cron-триггер в n8n на запуск каждый час; забирать через amoCRM REST /api/v4/leads все сделки с фильтром: статус != закрыто, включая поля менеджер, этап, дата_последнего_изменения, сумма
2. Рассчитать в n8n Code-ноде KPI: дней без движения по каждой сделке, % выполнения плана по менеджеру, средний цикл сделки vs норма — передать структурированный JSON в Claude
3. Claude анализирует по system prompt 'Ты жёсткий РОП': возвращает JSON {alerts: [{deal_id, manager, risk, action}], summary: {top_risks: [], top_opps: [], plan_status: '73%'}}
4. Формировать в n8n Telegram-сообщение с markdown-форматированием: дейлик с 3 блоками (🔴 Угрозы / 🟡 Внимание / 🟢 Возможности), отправлять РОПу в 9:00 и 17:00
5. При обнаружении сделки с просрочкой > 10 дней — автоматически создавать задачу менеджеру в Bitrix24 через task.add с дедлайном сегодня 18:00 и текстом от Claude

## 8. Почему купят (ROI)
РОП тратит 2 часа в день на ручную аналитику воронки — это 40 часов в месяц, которые стоят 60-80 тыс. руб. зарплаты. При этом 23% сделок теряется из-за несвоевременной реакции (сделка зависает > 14 дней без контакта). AI-РОП работает 24/7, замечает проблему в течение часа. Крючок: «Ваш лучший РОП никогда не спит и не пропускает ни одной сделки».
