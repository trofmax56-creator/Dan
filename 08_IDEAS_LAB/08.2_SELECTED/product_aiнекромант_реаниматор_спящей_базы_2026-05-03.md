## 1. Название
**AI-Некромант** — Реаниматор спящей базы

## 2. Суть
Агент выгружает из CRM контакты без активности 90+ дней, формирует персонализированные реактивационные сообщения через Claude на основе истории взаимодействий и запускает цепочку касаний в WhatsApp — без участия менеджера.

## 3. Бизнес-сценарий
ДО: В Bitrix24 лежат 800 контактов, которым последний раз писали полгода назад. «Надо бы позвонить старым клиентам» — говорит РОП каждый квартал. Руки не доходят. Это мёртвый капитал. ИИ делает: n8n каждое воскресенье выгружает из Bitrix24 контакты без активности ≥ 90 дней (до 50 штук в неделю). Claude читает историю взаимодействий каждого контакта и генерирует персональное сообщение: «Дмитрий, в прошлый раз вы интересовались автоматизацией склада — у нас появилось кое-что интересное для вашей ниши». Wazzup рассылает по 20 сообщений в день. Ответившие автоматически переводятся в статус «Реактивирован», менеджеру ставится задача на звонок в течение 2 часов. Клиент получает: ощущение «они помнят про меня», а не массовую рассылку.

## 4. Логика
```
n8n Schedule (воскресенье 10:00) → Bitrix24 REST API (выборка спящих контактов ≥ 90 дней) → Claude (персонализация сообщения на основе истории) → Wazzup API (рассылка 20 шт/день) → Bitrix24 Webhook (входящий ответ) → Bitrix24 API (смена статуса + задача менеджеру)
```

## 5. Технический стек
- n8n
- claude-3-5-sonnet
- Wazzup
- Bitrix24
- Telegram Bot API

## 6. Связки инструментов
```
n8n Cron → Bitrix24 REST API (CRM.CONTACT.LIST с фильтром) → Claude API (генерация персонального текста) → Wazzup API (отправка WhatsApp) → Bitrix24 Webhook (обработка ответа) → Telegram Bot (алерт менеджеру)
```

## 7. Алгоритм реализации
1. Настроить n8n Schedule-триггер каждое воскресенье 10:00: выгрузить из Bitrix24 через crm.contact.list контакты с фильтром LAST_ACTIVITY_DATE < (сегодня - 90 дней), лимит 50 контактов
2. Для каждого контакта получить историю взаимодействий через crm.activity.list: последние 5 активностей (звонки, письма, сделки), извлечь темы и результаты
3. Передать профиль контакта (имя, должность, история активностей, последний интерес) в Claude с промптом: написать реактивационное WhatsApp-сообщение до 200 символов, неформально, с конкретной привязкой к истории, без слова «предлагаем»
4. Добавить сообщения в очередь Wazzup: отправлять по 20 в день с интервалом 15–40 минут между сообщениями (имитация ручной отправки для избежания бана)
5. Через Wazzup Webhook отслеживать входящие ответы: при ответе от контакта из очереди — записать в Bitrix24 статус «Реактивирован», создать задачу менеджеру «Позвонить в течение 2 часов»
6. Отправить менеджеру Telegram-алерт: «♻️ Реактивирован: [Имя], последний контакт [дата]. Ответил: 
7. ['цитата']
8. . Задача в Bitrix24: [ссылка]»
9. Еженедельно формировать отчёт РОПу: отправлено / прочитано / ответили / переведены в работу (данные из Bitrix24 + Wazzup статистика)

## 8. Почему купят (ROI)
Стоимость реактивации одного спящего клиента в 7–10 раз дешевле привлечения нового лида через рекламу. База из 800 спящих контактов при конверсии реактивации 5–8% = 40–64 новых сделки в год, которые просто лежат в CRM. При среднем чеке 100 000 руб. — это 4–6 млн руб. скрытой выручки. Крючок: «Самые дешёвые лиды — те, что уже доверяли вам раньше.»
