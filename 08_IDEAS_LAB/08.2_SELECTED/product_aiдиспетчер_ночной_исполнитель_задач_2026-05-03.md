## 1. Название
**AI-Диспетчер** — Ночной Исполнитель задач

## 2. Суть
Claude/Gemini-агент, развёрнутый на сервере клиента, получает задачи через Telegram и выполняет их по расписанию в нерабочее время — без участия сотрудника и без терминала.

## 3. Бизнес-сценарий
ДО: Руководитель отдела продаж в 23:00 вспоминает, что надо выгрузить отчёт по сделкам за неделю, обновить прайс в Битрикс24 и разослать 40 клиентам напоминание. Говорит себе: 'Утром поставлю задачу менеджеру.' Утром менеджер занят, задача висит до обеда. ИИ делает: Руководитель в 23:00 пишет в Telegram-бот: 'Завтра в 8:00 выгрузи сделки за неделю из Битрикса, обнови цены по прайсу из папки и разошли напоминания клиентам со статусом Переговоры.' Агент подтверждает: 'Принял. Выполню в 8:00. Отчёт пришлю сюда.' В 8:05 руководитель получает сводку: 3 задачи выполнены, 2 клиента не получили сообщение — нет номера телефона. Клиент получает: 3 рутинных задачи закрыты до начала рабочего дня, менеджер с утра занимается продажами, а не копипастом.

## 4. Логика
```
Telegram-сообщение → n8n Webhook → Claude парсит задачу + время → Scheduler ставит задание в очередь → Агент выполняет (Bitrix24 API / Wazzup / файловая система) → Telegram-отчёт руководителю
```

## 5. Технический стек
- n8n
- claude-sonnet-4-5
- Telegram Bot API
- Bitrix24 REST API
- Wazzup
- VPS-сервер (Timeweb/Selectel)

## 6. Связки инструментов
```
Telegram → n8n Webhook → Claude (парсинг задачи + формирование плана) → n8n Scheduler → Bitrix24 REST API + Wazzup API → Telegram (отчёт)
```

## 7. Алгоритм реализации
1. Развернуть n8n на VPS (Timeweb Cloud, от 500 руб./мес), создать Telegram-бота через BotFather, подключить webhook на входящие сообщения
2. Написать Claude-промпт для извлечения из произвольного текста: тип задачи (рассылка / выгрузка / обновление), время выполнения, список объектов (клиенты, документы, поля CRM)
3. Реализовать 3 исполнительных модуля в n8n: выгрузка сделок из Bitrix24 (GET /crm.deal.list), массовая рассылка через Wazzup (POST /messages, батч по 20 сообщений/час), обновление полей карточек (POST /crm.deal.update)
4. Настроить n8n Scheduler-ноду: агент записывает задачу в SQLite-таблицу с полем execute_at, cron каждые 5 минут проверяет очередь и запускает нужный модуль
5. Подключить финальную ноду: Claude формирует текстовый отчёт (выполнено / ошибки / пропущено) и отправляет в Telegram руководителю

## 8. Почему купят (ROI)
Рутинные операции (обновление CRM, рассылки, отчёты) съедают у менеджера 1,5–2 часа ежедневно = 30–40 часов в месяц. При ФОТ менеджера 60 000 руб. — это 15 000–20 000 руб. в месяц на копипаст. Агент выполняет те же задачи ночью, пока все спят. Крючок: 'Пока менеджер спит — агент работает. Утром стол чистый.'
