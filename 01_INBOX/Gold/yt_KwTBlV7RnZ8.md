## Суть
Реальный кейс внедрения аналитики и интеграции МИС-CRM в медицинской клинике для выявления недовыставления услуг врачами, контроля среднего чека и предотвращения финансовых потерь клиники.

## Бизнес-сценарий
Медицинская клиника использует интегрированную систему МИС (медицинская информационная система) и CRM для аналитики работы врачей. Задача: выявить, какие услуги врачи недовыставляют в счета пациентов, проанализировать разницу между оказанными и выставленными услугами, повысить средний чек и контролировать финансовую дисциплину персонала.

## Алгоритм реализации
1. 1. Сбор данных из МИС: экспорт информации о всех оказанных услугах, процедурах и консультациях врачей за определённый период (по врачам, дням, типам услуг)
2. 2. Сбор данных из CRM/биллинга: выгрузка информации о выставленных счётах, проведённых платежах, услугах, зафиксированных в документах
3. 3. Сопоставление данных: автоматизированное сравнение оказанных услуг из МИС с выставленными в счётах из CRM (по врачу, пациенту, дате, типу услуги)
4. 4. Выявление расхождений: определение услуг, которые оказаны в МИС, но не выставлены в счёте (с расчётом недополученной суммы)
5. 5. Создание аналитического отчёта/дашборда: визуализация результатов по врачам с рейтингом недовыставления, суммами потерь, процентом несоответствий
6. 6. Интеграция с CRM/системой управления: автоматическое внесение выявленных расхождений в задачи, уведомления ответственным лицам
7. 7. Настройка мониторинга: регулярная актуализация отчётов (ежедневно/еженедельно) для отслеживания динамики и контроля исправлений

## Технический стек
- МИС (медицинская информационная система) — основной источник данных об оказанных услугах
- CRM/Биллинговая система — источник данных о выставленных счётах и платежах
- n8n или Make.com — платформа для интеграции и автоматизации (при наличии)
- PostgreSQL или аналогичная БД — хранилище обработанных данных
- Metabase, Looker, или встроенные инструменты аналитики — для создания дашбордов
- API МИС и CRM — для автоматического обмена данными
- Google Sheets или Excel — для экспорта промежуточных отчётов
- Python (pandas) — опционально для обработки больших объёмов данных

## Связки инструментов
- МИС → API экспорт → n8n → Нормализация данных → PostgreSQL
- CRM → API экспорт → n8n → Нормализация данных → PostgreSQL
- PostgreSQL → JOIN/сравнение → Выявление расхождений → Metabase/дашборд
- Дашборд → Webhook/Email уведомления → CRM/система задач (создание задач для врачей/администраторов)

## Конфигурация и параметры
- Поле МИС 'Тип услуги' (service_type) — должно соответствовать справочнику CRM
- Поле МИС 'Дата оказания услуги' (service_date) в формате YYYY-MM-DD
- Поле МИС 'ID врача' (doctor_id) — уникальный идентификатор врача
- Поле МИС 'ID пациента' (patient_id) — уникальный идентификатор пациента
- Поле МИС 'Стоимость услуги' (service_cost) — базовая цена услуги из справочника
- Поле CRM 'Выставленные услуги' (billed_services) — список услуг в счёте пациента
- Поле CRM 'Сумма счёта' (invoice_amount) — итоговая сумма выставленного счёта
- Поле CRM 'Дата выставления счёта' (invoice_date) — дата создания счёта в CRM
- Условие сопоставления: service_date (МИС) должна быть в пределах ±1-2 дня от invoice_date (CRM) для одного пациента и врача
- Формула выявления недовыставления: оказанная услуга НЕ найдена в billed_services по врачу, пациенту и дате → расхождение

## Ключевые инсайты
- 1. Типичная проблема: врачи оказывают дополнительные консультации, процедуры, манипуляции, но забывают или намеренно не выставляют их в счёт (потеря 5-20% выручки на врача)
- 2. Критерий выявления: если в МИС записана услуга (с датой, врачом, пациентом), но она отсутствует в CRM-счёте на ту же дату — это недовыставление
- 3. Среднее недовыставление в клиниках: 7-15% от потенциальной выручки из-за административных ошибок и недисциплинированности
- 4. Решение: интегрированный дашборд в реальном времени с рейтингом врачей по степени недовыставления и суммами потерь
- 5. Эффект: после внедрения аналитики средний чек клиники вырос на 10-25%, выручка повысилась без увеличения пациентопотока
- 6. Автоматизация: настройка автоматических уведомлений/задач в CRM при выявлении расхождения (например, в Bitrix24)
- 7. Регулярный мониторинг: еженедельные/ежемесячные отчёты с разбором по врачам и типам услуг для управления клиники
- 8. Данные для анализа: сравнение за одинаковые периоды (неделя, месяц) до внедрения и после — видна прямая корреляция между аналитикой и ростом среднего чека
