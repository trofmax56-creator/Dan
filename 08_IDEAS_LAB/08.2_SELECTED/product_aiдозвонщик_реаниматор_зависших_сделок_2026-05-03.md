## 1. Название
**AI-Дозвонщик** — Реаниматор зависших сделок

## 2. Суть
n8n ежедневно сканирует Bitrix24/amoCRM на сделки без активности более N дней, Claude пишет персональное сообщение под каждый контекст сделки и отправляет через Wazzup в WhatsApp — менеджер видит только те лиды, где клиент ответил.

## 3. Бизнес-сценарий
ДО: В amoCRM 340 сделок в статусе 'думает'. Менеджер Катя смотрит на список и не знает с чего начать — открывает первую сделку, читает историю 3 минуты, пишет шаблонное 'Добрый день, как вы?' Клиент не отвечает. Катя переходит к следующему, через час закрывает CRM с ощущением что ничего не сделала. 60% базы не трогали 2+ месяца. ИИ делает: В 10:00 n8n берёт все сделки в amoCRM без активности 14+ дней, для каждой Claude читает последние 3 сообщения и название продукта, пишет уникальное сообщение: 'Привет, Андрей! Вы интересовались настройкой учёта для вашего магазина — мы как раз запустили новый тариф для розницы, стало дешевле. Хотите гляну что подойдёт?' Wazzup отправляет 20 сообщений в час чтобы не попасть в бан. К обеду Катя видит в CRM только 12 ответивших клиентов с которыми реально работать. Клиент получает: мёртвая база оживает, менеджер фокусируется на горячих.

## 4. Логика
```
Cron 10:00 → amoCRM API (сделки без активности 14+ дней) → Claude (генерация персонального сообщения по контексту сделки) → Wazzup API (очередь 20 сообщ/час) → amoCRM API (обновление последнего контакта)
```

## 5. Технический стек
- n8n
- claude-sonnet-4-5
- Wazzup
- amoCRM или Bitrix24

## 6. Связки инструментов
```
n8n Cron → amoCRM REST API (фильтр last_activity < 14 дней) → Claude API (промпт с контекстом сделки) → Wazzup REST API (отправка с throttle 20/час) → amoCRM API (создание задачи на менеджера если ответ пришёл)
```

## 7. Алгоритм реализации
1. Настроить запрос к amoCRM API с фильтром: статус не 'закрыта/проиграна', дата последнего изменения > 14 дней назад, выгружать батчами по 50 сделок — получить поля: имя контакта, название продукта/услуги, последние 3 примечания, ответственный менеджер
2. Для каждой сделки вызвать Claude с промптом: системная роль — 'опытный менеджер по продажам', задача — написать WhatsApp-сообщение до 100 слов, которое: обращается по имени, ссылается на конкретный продукт из сделки, добавляет 1 актуальный инфоповод (новый тариф / сезонная акция — список инфоповодов обновляется клиентом раз в 2 недели в Google Sheets)
3. Добавить все сообщения в очередь n8n с задержкой 3 минуты между отправками (20 сообщений/час через Wazzup) — это защита от блокировки WhatsApp номера
4. После отправки обновить в amoCRM поле 'дата последнего контакта' и создать задачу на менеджера: 'Реактивация отправлена — ждём ответа до [дата+3 дня]'
5. При ответе клиента в Wazzup (входящий webhook) — автоматически перевести сделку в статус 'Горячий / Перезвонить', уведомить ответственного менеджера в Telegram: 'Андрей Смирнов ответил на реактивацию! Открыть сделку: [ссылка]'

## 8. Почему купят (ROI)
Средняя база МСБ — 300–500 зависших сделок. Если реанимировать хотя бы 5% = 15–25 сделок × средний чек 30 000 руб. = 450 000 – 750 000 руб. выручки из базы, которую уже списали. Стоимость нового лида через рекламу — 2 000–5 000 руб. Стоимость реактивации через AI-Дозвонщик — 50–150 руб. на контакт. Крючок: 'Деньги уже в вашей CRM — их просто не забрали.'
