## 1. Название
**AI-Регистратор** — Автооформление Заявок из WhatsApp в CRM

## 2. Суть
Claude-агент перехватывает входящие сообщения из WhatsApp, извлекает структурированные данные (имя, телефон, задача, бюджет) и мгновенно создаёт карточку клиента + сделку в Битрикс24 без участия менеджера — даже если сообщение написано хаотично.

## 3. Бизнес-сценарий
ДО: Клиент пишет в WhatsApp: «Привет меня зовут Михаил из компании ТехноПром нам нужна интеграция 1с с сайтом срочно бюджет есть позвоните». Менеджер читает, идёт в CRM, вручную создаёт контакт, вводит телефон, заполняет сделку, ставит задачу — 8 минут. При 25 заявках в день — 200 минут в день только на перенос данных. ИИ делает: сообщение пришло → Claude за 5 секунд разбирает: {name: 'Михаил', company: 'ТехноПром', task: 'интеграция 1С с сайтом', urgency: 'срочно', budget: 'есть', phone: '+7...'} → в Битрикс24 создан контакт, сделка на этапе «Новая», задача менеджеру «Перезвонить сегодня». Клиент получает: автоответ «Михаил, приняли вашу заявку! Менеджер свяжется в течение часа». Менеджер получает: уведомление в Telegram с кратким дайджестом и прямой ссылкой на сделку в CRM.

## 4. Логика
```
Wazzup Webhook → n8n → Claude (NLP-парсинг свободного текста в JSON-структуру) → Bitrix24 REST (crm.contact.add + crm.deal.add + task.add) → Wazzup (автоответ клиенту) → Telegram (уведомление менеджеру)
```

## 5. Технический стек
- n8n
- claude-3-5-haiku
- Wazzup
- Bitrix24 REST API
- Telegram Bot API
- Dadata (валидация телефона/ИНН)

## 6. Связки инструментов
```
Wazzup Webhook → n8n → Claude Haiku API (извлечение 8 полей из свободного текста, JSON-ответ) → Dadata (валидация телефона) → Bitrix24 crm.contact.add + crm.deal.add + tasks.task.add → Wazzup sendMessage (автоответ) → Telegram Bot (дайджест менеджеру)
```

## 7. Алгоритм реализации
1. Подключить Wazzup Webhook к n8n; принимать все входящие WhatsApp-сообщения; фильтровать служебные (статусные) сообщения через IF-ноду по типу события
2. Передать текст сообщения в Claude Haiku (дешевле Sonnet для простой задачи) с system prompt: извлечь 8 полей {name, company, inn, phone, email, task, budget, urgency} из свободного текста, вернуть JSON; если поле не найдено — null
3. Валидировать извлечённый телефон через Dadata API suggest/phone; если невалиден — взять phone из реквизитов Wazzup (номер отправителя)
4. Создать в Битрикс24 через REST: 1) crm.contact.add с name, phone, source_id=WA; 2) crm.deal.add с title=задача_клиента, contact_id, stage_id=NEW; 3) tasks.task.add с дедлайном +4 часа и текстом задачи
5. Отправить через Wazzup автоответ клиенту с именем (из Claude): «{name}, заявку приняли! Менеджер свяжется до {время+1час}»; отправить в Telegram менеджеру: имя, компания, задача, ссылка на сделку

## 8. Почему купят (ROI)
При 25 входящих заявках в день менеджер тратит 200 минут = более 3 часов только на ввод данных в CRM. При окладе 60 000 руб. это 12 000 руб./мес чистых потерь на механическую работу. Плюс 15-20% заявок вносятся с ошибками или теряются. AI-Регистратор обрабатывает заявку за 5 секунд, данные всегда корректны. Крючок: «CRM заполняется сама — менеджер только продаёт».
