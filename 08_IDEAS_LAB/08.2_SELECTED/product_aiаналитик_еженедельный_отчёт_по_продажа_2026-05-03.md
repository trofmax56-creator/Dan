## 1. Название
**AI-Аналитик** — Еженедельный отчёт по продажам без Excel

## 2. Суть
Каждую пятницу n8n забирает данные из Bitrix24/amoCRM и 1С, Claude анализирует динамику, сравнивает с прошлой неделей и планом, находит аномалии и формирует читаемый отчёт — директор получает в Telegram готовые выводы и вопросы к команде вместо сырых таблиц.

## 3. Бизнес-сценарий
ДО: Директор Игорь каждую пятницу в 17:00 пишет в общий чат: 'Коля, скинь цифры за неделю'. Коля собирает вручную из 3 систем 40 минут, отправляет Excel. Игорь открывает, видит 200 строк, закрывает, идёт на выходные. В понедельник решения всё равно принимаются по ощущениям. ИИ делает: В пятницу в 16:30 n8n сам забирает из Bitrix24 воронку сделок, из 1С — оплаты, из Google Sheets — план. Claude сравнивает: выручка недели 1 840 000 vs план 2 000 000 (-8%), лидов +23% к прошлой неделе но конверсия упала с 18% до 14% — проблема в этапе 'КП отправлено'. Топ-3 менеджера по выручке. 2 сделки зависли на 21+ дней. В 17:00 Игорь получает в Telegram: '📊 Итоги недели: выручка 1.84 млн (−8% к плану). Внимание: конверсия просела на этапе КП — стоит разобрать на планёрке. Топ: Катя 620к, Дима 480к, Андрей 380к. Зависшие сделки: 2 штуки, ссылки внизу.' Клиент получает: принимает решения на данных, а не ощущениях.

## 4. Логика
```
Cron Пт 16:30 → Bitrix24 API + 1С REST (данные за неделю) → Google Sheets (план) → Claude (анализ + текст отчёта) → Telegram Bot (сообщение директору)
```

## 5. Технический стек
- n8n
- claude-sonnet-4-5

- Google Sheets

## 6. Связки инструментов
```
n8n Cron Пт 16:30 → Bitrix24 API (сделки, воронка, менеджеры) + 1С CSV/REST (оплаты) + Google Sheets (план) → n8n Code (агрегация метрик) → Claude API (анализ, поиск аномалий, текст отчёта) → Telegram sendMessage
```

## 7. Алгоритм реализации
1. Настроить 3 параллельных HTTP-запроса в n8n: Bitrix24 crm.deal.list с фильтром по текущей неделе (поля: сумма, этап, менеджер, дата последнего изменения), 1С через REST-сервис или парсинг CSV-выгрузки (оплаты за неделю), Google Sheets (план недели в именованной ячейке)
2. В Code-ноде n8n агрегировать сырые данные в структурированный JSON: выручка_факт, выручка_план, дельта_%, лиды_неделя, лиды_прошлая, конверсия_по_этапам (массив), топ5_менеджеров, сделки_без_активности_21день (список ID)
3. Передать JSON в Claude с промптом: 'Ты бизнес-аналитик. Напиши отчёт для директора в 150–200 слов. Структура: 1) главная цифра и отклонение от плана, 2) одна ключевая аномалия с объяснением, 3) топ-3 менеджера, 4) action item на следующую неделю. Тон — деловой, без воды, с конкретными числами из данных.'
4. Добавить в Telegram-сообщение ссылки на зависшие сделки (формат ссылки Bitrix24: /crm/deal/details/ID/) — список формируется в Code-ноде из сделок без активности
5. Параллельно отправить в Google Sheets строку с агрегированными метриками недели для накопления исторических данных — через 4 недели Claude начнёт видеть месячный тренд и упоминать его в отчёте

## 8. Почему купят (ROI)
Директор тратит 2–3 часа каждую неделю на сбор и осмысление данных, аналитик/финдир стоит 80 000–120 000 руб./мес — при этом отчёты всё равно приходят поздно и без выводов. AI-Аналитик даёт готовые инсайты каждую пятницу в 17:00 за 6 000 руб./мес инфраструктуры. Первые клиенты обычно говорят: 'Я за первую неделю нашёл дыру, которую не видел полгода.' Крючок: 'Директор наконец управляет компанией, а не гоняется за таблицами.'
