# 🗂️ Паспорт идеи: RAG-агент на базе данных компании

**Источник:** [Как сделать RAG в n8n на Supabase для ИИ Агента](https://youtube.com/watch?v=k4a_G5YFSIs) — Несерьезный айтишник (2025-08-14)
**Категория:** Автоматизация отдела
**Заменяет:** Оператор поддержки / менеджер по работе с внутренними документами

---

## 💡 Что это даёт бизнесу
Бизнес получает ИИ-агента, который мгновенно отвечает на вопросы сотрудников или клиентов, опираясь исключительно на внутренние документы компании (регламенты, прайсы, FAQ, базы знаний). Исключается потребность держать сотрудника, который ищет информацию вручную. Данные хранятся в защищённой Supabase-базе — без утечки в публичные модели.

---

## ✅ Плюсы
- Быстрый старт: n8n + Supabase pgvector поднимаются за 1–2 недели даже без DevOps
- Данные остаются у клиента — Supabase self-hosted или облако в EU/RU, что снимает страх утечки
- Легко масштабируется: добавляешь новые документы без переобучения модели

## ❌ Минусы / Риски
- Качество ответов сильно зависит от качества исходных документов — клиент должен их подготовить
- При большом объёме данных (>10 000 чанков) нужна тонкая настройка эмбеддингов и retrieval, иначе точность падает

## 🧰 Артефакты для реализации

**Инструменты:** n8n (self-hosted или cloud), Supabase (pgvector), OpenAI API / Ollama (локальные модели), LangChain-совместимые эмбеддинги
**API:** OpenAI Embeddings API (text-embedding-3-small), OpenAI Chat Completions API, Supabase REST API

## 🚀 Шаги запуска
1. Шаг 1: Собрать и структурировать документы клиента (PDF, DOCX, Google Docs) — провести аудит базы знаний
2. Шаг 2: Развернуть Supabase (облако или self-hosted), создать таблицу с pgvector, настроить эмбеддинги через n8n-воркфлоу загрузки документов
3. Шаг 3: Собрать n8n-агента с retrieval-цепочкой: вопрос → эмбеддинг → поиск по Supabase → промпт с контекстом → ответ LLM
4. Шаг 4: Подключить интерфейс (Telegram-бот / виджет на сайт / Bitrix24 чат) и провести приёмочное тестирование с клиентом
