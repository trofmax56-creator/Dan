## Суть
Автоматизация обработки лидов в Odoo CRM с использованием n8n и OpenAI для классификации и структурирования данных. Система автоматически распределяет входящие лиды по категориям и полям CRM, избавляя продажи от ручной работы ввода данных.

## Бизнес-сценарий
B2B SaaS компании, отделы продаж и CRM-администраторы которых получают множество неструктурированных лидов из почты, форм и других источников. Автоматизация классифицирует лидов по типам, компаниям, направлениям продаж и автоматически создает контакты в Odoo CRM без ручного ввода.

## Алгоритм реализации
1. 1. Входящий вебхук или триггер в n8n получает данные о новом лиде (email, имя, описание, сообщение)
2. 2. Данные лида отправляются в Claude/OpenAI API с промптом для анализа и классификации
3. 3. AI анализирует текст лида и структурирует информацию: тип лида, название компании, должность, интересы, приоритет
4. 4. Структурированные данные из AI ответа парсятся (извлекаются нужные поля) в формат JSON
5. 5. Сформированный объект отправляется в Odoo CRM через REST API POST запрос для создания нового Lead или Contact
6. 6. После успешного создания отправляется подтверждающее уведомление (опционально: письмо в Slack или email)

## Технический стек
- n8n (платформа оркестрации и автоматизации)
- OpenAI API / Claude API (обработка естественного языка и классификация)
- Odoo CRM REST API (создание и управление лидами)
- Webhook (входящий триггер для n8n)
- JSON парсинг
- HTTP модули n8n для API запросов

## Связки инструментов
- Webhook триггер → n8n (получение данных лида)
- n8n → OpenAI/Claude API (отправка текста и получение классификации)
- OpenAI API ответ → JSON парсер (структурирование)
- JSON → Odoo REST API (создание Lead/Contact в CRM)
- Odoo API → n8n (подтверждение успешного создания)

## Конфигурация и параметры
- Webhook URL в n8n с методом POST для приёма данных лида
- OpenAI API ключ и модель (gpt-4 или gpt-3.5-turbo) в параметрах ноды
- Промпт для AI с инструкциями по классификации лидов и формату ответа JSON
- Одoo API key и base URL (например https://yourodoo.com)
- Параметры для создания Lead в Odoo: name, email, phone, company_name, lead_type, priority, description, tag_ids
- Условие (IF node) для проверки успешности создания записи в Odoo
- Формат парсинга JSON от AI с попытка преобразования строк в правильные типы данных

## Ключевые инсайты
- OpenAI гораздо быстрее парсит неструктурированный текст чем regex и готов работать с нечёткими входными форматами
- Структурирование response от AI в JSON нужно делать через дополнительную ноду парсинга, так как иногда AI выдаёт markdown или текст вместо JSON
- API key для Odoo нужно получить из профиля администратора в самой CRM через Developer Tools
- Рекомендуется добавлять timeout на запросы к OpenAI (обычно 30 сек) чтобы workflow не зависал
- Поле priority в Odoo обычно принимает значения: 0, 1, 2, 3 (где 3 - максимальный приоритет)
- Если лид приходит на новый email - нужно сначала проверить через FILTER или CONDITION не существует ли такой контакт в Odoo, чтобы не создавать дубликаты

## Подводные камни
- OpenAI может иногда возвращать некорректный JSON если промпт недостаточно строгий - всегда добавляй инструкцию 'Return ONLY valid JSON, no markdown'
- Odoo API требует Authentication Header с Bearer token или API key, иначе вернёт 401 ошибку
- Поле company_name в Odoo часто требует отдельного создания Company объекта предварительно через другой API запрос
- Если workflow срабатывает часто (более 100 раз в день) - OpenAI API может быть дорогим, рассмотреть rate limiting
- Некоторые версии Odoo требуют указания team_id для правильного распределения лидов между менеджерами
- Если лид содержит non-ASCII символы (кириллица, иероглифы) - убедитесь что n8n и API кодируют данные в UTF-8
