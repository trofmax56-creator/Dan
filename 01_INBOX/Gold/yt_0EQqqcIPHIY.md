## Суть
Интеграция системы управления CRM (Bitrix24) с платформой обучения и продаж (Геткурс) через платформу Vakas-tools для автоматизации управления контактами и обучающими курсами в медицинской организации. Решение позволяет синхронизировать данные клиентов и автоматически распределять их по учебным программам.

## Бизнес-сценарий
Медицинская организация использует эту интеграцию для автоматического переноса контактов пациентов/клиентов из Bitrix24 в Геткурс, привязки их к соответствующим обучающим программам и отслеживания прогресса прохождения курсов. Это позволяет сократить ручную работу менеджеров и повысить скорость обслуживания клиентов.

## Алгоритм реализации
1. Шаг 1: Регистрация и подключение Vakas-tools платформы через сайт https://vakas-tools.ru/g/y/
2. Шаг 2: Авторизация в Bitrix24 - получение API ключа и права доступа к контактам и сделкам
3. Шаг 3: Авторизация в Геткурс - получение API ключа платформы обучения
4. Шаг 4: Создание правила интеграции в Vakas-tools для синхронизации контактов из Bitrix24 в Геткурс (маппинг полей: ФИО, Email, телефон)
5. Шаг 5: Настройка триггеров - определение условий, при которых контакты передаются в Геткурс (например, при добавлении контакта с определённым тегом или при смене статуса сделки)
6. Шаг 6: Опциональная настройка автоматического зачисления в учебные программы на основе типа контакта или параметров сделки
7. Шаг 7: Тестирование: создание тестового контакта в Bitrix24 и проверка его появления в Геткурс
8. Шаг 8: Запуск синхронизации и мониторинг логов интеграции в Vakas-tools для отслеживания ошибок и успешных передач

## Технический стек
- Vakas-tools - основная платформа интеграции
- Bitrix24 - CRM система управления контактами
- Геткурс - платформа онлайн-обучения и продаж
- REST API Bitrix24
- REST API Геткурс
- Webhook Bitrix24 - для передачи событий в реальном времени
- OAuth 2.0 - протокол авторизации

## Связки инструментов
- Webhook (событие создания контакта в Bitrix24) → Vakas-tools → API Геткурс → зачисление в курс
- Bitrix24 REST API (получение данных контактов) → Vakas-tools (трансформация данных) → Геткурс API (создание пользователя в системе обучения)
- Триггер изменения поля в Bitrix24 → Vakas-tools условный оператор → отправка данных в Геткурс API endpoint

## Конфигурация и параметры
- Endpoint Bitrix24 API: https://[ваш-домен].bitrix24.ru/rest/[ID_пользователя]/
- Endpoint Геткурс API: https://api.getcourse.ru/ (или другой в зависимости от сервера размещения)
- Обязательные параметры Bitrix24 API: auth_token (может быть получен через OAuth или сгенерирован вручную)
- Обязательные параметры Геткурс API: API key или access token
- Маппинг полей Bitrix24 → Геткурс: NAME (ФИО) → имя пользователя, EMAIL → email, PHONE → телефон, ASSIGNED_BY_ID → ответственный менеджер
- Поле условия: Может быть установлено на стадию сделки, тег контакта, тип контакта или пользовательское поле
- Настройка вебхука Bitrix24: метод установки вебхука, URL Vakas-tools для приёма событий, события для отслеживания (onCrmContactAdd, onCrmDealAdd, onCrmContactUpdate)

## Ключевые инсайты
- Vakas-tools позволяет без написания кода интегрировать Bitrix24 и Геткурс через визуальный интерфейс конструктора интеграций
- Критически важно правильно маппировать поля: Email должен совпадать, иначе в Геткурс создастся дублированный пользователь
- Медицинская организация может автоматизировать распределение пациентов по профилактическим и обучающим программам на основе диагноза или типа услуги
- Синхронизация двусторонняя - можно отправлять данные о прогрессе обучения из Геткурса обратно в Bitrix24 для отслеживания KPI менеджерами
- Рекомендуется создать дополнительное поле в Bitrix24 для отслеживания статуса синхронизации и возможных ошибок интеграции
- Геткурс поддерживает групповое зачисление в курсы и создание когорт на основе данных из Bitrix24
- Vakas-tools сохраняет историю всех синхронизаций, что позволяет отследить, когда и какой контакт был перенесён, для аудита и отладки проблем

## Подводные камни
- Ошибка: неправильный формат Email в Bitrix24 приведёт к невозможности создать пользователя в Геткурс - проверяйте валидность Email перед синхронизацией
- Битрикс может иметь разные настройки доступа API в зависимости от тарифа (Free, Basic, Professional, Enterprise) - убедитесь, что REST API включён
- Геткурс может ограничивать количество API запросов в секунду - при синхронизации большого объёма контактов может потребоваться отложенная обработка (batch processing)
- Если контакт уже существует в Геткурс с тем же Email, Vakas-tools должен обновить существующий пользователь, а не создать дубликат - проверьте настройку 'Update or Create' режима
- Webhook Bitrix24 может не срабатывать, если сервер Vakas-tools временно недоступен - есть механизм повторной доставки, но он не бесконечен
- Некоторые пользовательские поля Bitrix24 могут не поддерживаться API - проверьте документацию перед использованием нестандартных полей в условиях
- Геткурс требует наличие в системе курса с определённым ID перед попыткой зачисления контакта - убедитесь, что ID курса указан правильно и курс существует
