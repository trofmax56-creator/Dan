## 1. Название
**AI-Ревизор** — Контроль 100% Звонков и Переписки

## 2. Суть
Многоагентная система на n8n: первый агент ежедневно забирает все диалоги менеджеров из CRM и мессенджеров, второй агент (Claude) оценивает каждый контакт по чеклисту продаж из 12 пунктов, третий формирует персональный отчёт-разбор для каждого менеджера и сводный дашборд для РОПа.

## 3. Бизнес-сценарий
ДО: В компании 6 менеджеров, каждый ведёт 20-30 диалогов в день. ОКК слушает 10% звонков вручную — остальные 90% бесконтрольны. РОП не знает, что менеджер Петров не отрабатывает возражения и теряет горячих клиентов на этапе «дорого». Узнают об этом через месяц по результатам. ИИ делает: каждый вечер в 19:00 агент забирает все переписки из Битрикс24 за день. Claude оценивает каждый диалог: «Приветствие — ✅, Выявление потребности — ✅, Презентация под потребность — ❌ (менеджер не спросил о бюджете), Отработка возражения "дорого" — ❌ (ответил "ну подумайте"), Следующий шаг — ✅». Оценка 6/12. РОП получает в 19:30: «Петров — средний балл 5.2/12, главная проблема: не отрабатывает возражения (8 из 10 диалогов). Рекомендую разбор на планёрке завтра». Клиент получает: объективный контроль 100% контактов без найма сотрудников ОКК.

## 4. Логика
```
Cron (19:00) → Bitrix24 API (все активности дня по менеджерам) → Агент-1 n8n (фильтрация и группировка по менеджерам) → Claude Агент-2 (оценка каждого диалога по 12 критериям, JSON) → Claude Агент-3 (агрегация по менеджеру, рекомендации) → Telegram (персональные отчёты менеджерам + сводный РОПу) + Google Sheets (архив оценок)
```

## 6. Связки инструментов
```
n8n Cron (19:00) → Bitrix24 crm.activity.list (все диалоги дня) + Wazzup API (WhatsApp переписки) → n8n Loop по диалогам → Claude Sonnet (оценка по 12 критериям, JSON {score, fails, wins}) → n8n Aggregate по менеджеру → Claude (персональный отчёт с рекомендациями) → Telegram Bot (менеджеру личный разбор, РОПу сводка) → Google Sheets (append строки)
```

## 7. Алгоритм реализации
1. Настроить Cron 19:00 в n8n; через Bitrix24 crm.activity.list с filter[RESPONSIBLE_ID] забрать все активности (письма, звонки, чаты) за текущий день; через Wazzup API — WhatsApp переписки; объединить через n8n Merge
2. Разбить на массив диалогов; для каждого диалога через Loop-ноду передать в Claude Sonnet с system prompt: чеклист продаж из 12 пунктов (приветствие, выявление потребности, BANT-квалификация, презентация, обработка возражений, фиксация следующего шага и др.); получить JSON {score: 8, fails: ['возражение_дорого'], wins: ['выявление_потребности']}
3. Агрегировать оценки по менеджеру в n8n: средний балл за день, топ-3 ошибки, лучший диалог дня — передать в Claude для генерации персонального текстового разбора на русском языке
4. Отправить каждому менеджеру в Telegram персональный разбор: оценка/12, что сделал хорошо, что улучшить, пример лучшего ответа на типичное возражение — не как упрёк, а как коучинг
5. Отправить РОПу сводный дашборд: рейтинг менеджеров по баллам, кто требует внимания (<6/12), общая динамика vs вчера; сохранить все оценки в Google Sheets для недельной/месячной аналитики

## 8. Почему купят (ROI)
Штат ОКК из 2 сотрудников для контроля 6 менеджеров обходится 120-160 тыс. руб./мес ФОТ — и они всё равно проверяют только 10-15% контактов. AI-Ревизор проверяет 100% диалогов за 800-1200 руб./мес на Claude API. Средний рост выручки отдела после внедрения системы оценки качества — 15-25% за квартал за счёт устранения типовых ошибок. Крючок: «ОКК из 2 человек или ИИ за 1 000 рублей — выбор очевиден».
