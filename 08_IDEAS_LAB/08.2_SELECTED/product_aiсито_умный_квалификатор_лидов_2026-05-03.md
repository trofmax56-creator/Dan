## 1. Название
**AI-Сито** — Умный Квалификатор Лидов

## 2. Суть
ИИ-агент ведёт первичный диалог с входящим лидом в WhatsApp, задаёт квалификационные вопросы по методологии BANT, выставляет балл готовности 1-5 и передаёт менеджеру только «горячих» — остальных дожимает самостоятельно по цепочке касаний.

## 3. Бизнес-сценарий
ДО: Менеджер Катя получает 60 заявок в месяц с сайта. Тратит 30-40 минут на каждую первичную квалификацию по телефону. Итог — 25 часов в месяц на звонки, из которых 70% заявок оказываются нецелевыми («просто интересуюсь», «это для друга», «а сколько стоит минимум»). ИИ делает: клиент оставил заявку на сайте, через 2 минуты получает WhatsApp: «Привет, я Алина из [компания]! Вы оставили заявку на [продукт]. Подскажите, для какого проекта смотрите? Это для вашего бизнеса или личных нужд?» После 3-5 вопросов Claude выставляет BANT-балл. Балл ≥ 4 — Катя получает уведомление «Горячий лид, Андрей, бюджет 200к, решение в этом месяце» с заполненной карточкой. Балл < 3 — агент отправляет полезный контент и ставит задачу вернуться через 2 недели. Катя получает: только тёплые лиды с заполненным профилем.

## 4. Логика
```
Заявка с сайта → n8n Webhook → Wazzup (старт диалога) → Claude (BANT-квалификация, 3-5 вопросов) → скоринг 1-5 → IF балл ≥ 4: Bitrix24 (сделка «Горячий») + Telegram менеджеру / IF балл < 4: цепочка прогрева (D+3, D+7, D+14)
```

## 5. Технический стек
- n8n
- claude-sonnet-4-6
- Wazzup
- Bitrix24
- Telegram Bot API

## 6. Связки инструментов
```
Сайт форма → n8n Webhook → Wazzup REST API (исходящее сообщение) → Wazzup webhook (ответы клиента) → Claude (BANT-скоринг) → Bitrix24 REST API (создание сделки с тегом) → Telegram Bot (алерт менеджеру)
```

## 7. Алгоритм реализации
1. Настроить webhook в n8n для приёма заявок с сайта (или CRM-триггер при создании нового лида), запускать диалог через Wazzup в течение 120 секунд после заявки
2. Написать системный промпт для Claude: провести квалификацию по 4 критериям BANT за ≤ 5 вопросов, хранить контекст диалога в n8n Session Storage, не спрашивать то, что уже известно из заявки
3. После завершения диалога (клиент не отвечает 20 минут или получены все 4 параметра) Claude рассчитывает балл 1-5 и формирует JSON: имя, бюджет, потребность, сроки, балл, краткое резюме
4. При балле ≥ 4: создать сделку в Bitrix24 через REST API с заполненными полями, отправить менеджеру Telegram-карточку с кнопкой «Позвонить сейчас»; при балле < 4: занести в отдельную воронку «Прогрев»
5. Запустить цепочку прогрева для слабых лидов: D+3 (полезный контент), D+7 (кейс клиента), D+14 (персональное предложение) — через Wazzup с лимитом 20 сообщ/час

## 8. Почему купят (ROI)
Менеджер тратит 40 минут на квалификацию одного лида вручную: 60 лидов/мес × 40 мин = 40 часов = полная рабочая неделя уходит только на первичные разговоры. AI-Сито сокращает это до 0 рабочих часов менеджера на нецелевых лидах. При зарплате менеджера 80 000 руб./мес — высвобождается 50% рабочего времени для реальных продаж. Крючок: «Менеджер должен продавать, а не анкетировать.»
