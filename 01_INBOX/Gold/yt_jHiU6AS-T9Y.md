## Суть
Подробное руководство по подключению Zoho CRM к n8n для создания автоматизированных workflows. Интеграция позволяет синхронизировать данные между системами, автоматизировать создание контактов, лидов и других операций в Zoho CRM без необходимости ручного ввода данных.

## Бизнес-сценарий
Компании, использующие Zoho CRM, хотят автоматизировать рутинные операции (создание лидов, обновление контактов, отправка уведомлений) через n8n. На входе — веб-формы, webhook'и, другие системы; на выходе — автоматически созданные/обновленные записи в Zoho CRM, что экономит время сотрудников и снижает количество ошибок ввода данных.

## Алгоритм реализации
1. Шаг 1: Создание API ключа в Zoho CRM — перейти в Zoho CRM → Settings → Developer Tools → API Keys, создать новый API key, скопировать значение для последующей аутентификации в n8n
2. Шаг 2: Создание нового workflow в n8n — открыть n8n → нажать 'New Workflow', добавить триггер (Webhook, Schedule, or Poll для Zoho), задать название workflow
3. Шаг 3: ДобавлениеCredententials (учетные данные) в n8n — в workflow нажать на узел с Zoho CRM → выбрать 'Create New Credential' → выбрать тип аутентификации (API Key или OAuth), вставить API key из Zoho, сохранить
4. Шаг 4: Добавление Zoho CRM ноды в workflow — в n8n перейти на лист workflow → добавить ноду 'Zoho CRM' → выбрать операцию (Create Contact, Create Lead, Update Record и т.д.), настроить параметры (поля для создания, условия фильтрации)
5. Шаг 5: Маппинг полей данных — подключить входные данные к полям Zoho (например, из Webhook подключить body.email → Email field в Zoho CRM), убедиться что все обязательные поля заполнены
6. Шаг 6: Добавление условной логики (опционально) — использовать ноду 'IF' для проверки данных перед отправкой в Zoho (например, проверить что email валиден, только тогда создавать контакт)
7. Шаг 7: Активирование и тестирование workflow — нажать кнопку 'Activate' для включения workflow, отправить тестовые данные через Webhook или используя тестовую функцию, проверить что запись создана в Zoho CRM
8. Шаг 8: Мониторинг выполнения — в разделе 'Executions' n8n проверять логи выполнения workflow, убедиться что нет ошибок аутентификации или проблем с маппингом полей

## Технический стек
- n8n — платформа для workflow автоматизации (самохостиваемая или облачная версия)
- Zoho CRM — облачная CRM система
- Zoho CRM API v2 — REST API для взаимодействия с Zoho CRM
- Webhook — точка входа для передачи данных в workflow n8n
- HTTP Request ноды (опционально) — для прямых API запросов если нужна большая гибкость
- Set ноды — для преобразования и подготовки данных
- IF условные ноды — для логики ветвления
- Switch ноды — для множественных условий и маршрутизации

## Связки инструментов
- Webhook (точка входа) → n8n Zoho CRM ноде → Zoho CRM API v2
- HTTP Request (альтернатива) → POST https://www.zohoapis.com/crm/v2/Contacts (или другой модуль) с параметрами и заголовками Authorization: Zoho-oauthtoken {token}
- n8n Credentials Management → OAuth/API Key для Zoho CRM → Tokens хранятся в n8n vault

## Конфигурация и параметры
- Zoho CRM API Endpoint: https://www.zohoapis.com/crm/v2/ (для создания контактов: /Contacts, для лидов: /Leads и т.д.)
- Методы API: POST (создание), PUT (обновление), GET (получение), DELETE (удаление)
- Обязательные заголовки: Authorization: Zoho-oauthtoken {accessToken}, Content-Type: application/json
- Структура body для создания контакта: { 'data': [{ 'First_Name': 'John', 'Last_Name': 'Doe', 'Email': 'john@example.com', 'Phone': '+1234567890', 'Company': 'Acme Inc' }] }
- Структура body для создания лида: { 'data': [{ 'Last_Name': 'Smith', 'Company': 'Tech Corp', 'Email': 'smith@techcorp.com', 'Phone': '9876543210' }] }
- Пример маппинга в n8n: Webhook data → Set ноде для форматирования → Zoho CRM Create Contact ноде
- Параметры фильтрации: можно использовать criteria для выборки только нужных записей (например, 'Email' like '%@company.com')
- Rate limiting: Zoho CRM API имеет ограничение 20 000 API запросов в день для бесплатного плана, учитывать это при больших объемах данных
- Обработка ошибок: добавить Error Handler ноды для перехвата ошибок аутентификации (401), неверного формата данных (400) и других сбоев
