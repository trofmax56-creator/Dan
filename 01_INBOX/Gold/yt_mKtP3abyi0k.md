## Связки инструментов
- User Input → Claude API Endpoint → Model Processing → JSON Response → Output Display
- Application → Authentication (API Key) → Claude API → Streaming Response → Result Handler
- Agent Loop → Tool Definition → Claude Decision → Tool Execution → Response Parsing

## Конфигурация и параметры
- API Endpoint: https://api.anthropic.com/v1/messages
- Authentication: Header 'x-api-key' с API-ключом
- Model: claude-3-opus-20250219 или claude-3-sonnet (зависит от версии)
- Max tokens: настраивается от 1024 до 4096+ в зависимости от задачи
- Temperature: 0.0-1.0 (0 для детерминированных ответов, 1.0 для творческих)
- Stop sequences: пользовательские стоп-последовательности для контроля вывода
- System prompt: глобальная инструкция для поведения модели
- Messages format: список объектов {role: 'user'/'assistant', content: 'text'}

## Ключевые инсайты
- Claude имеет контекстное окно до 200K токенов (в зависимости от версии), что позволяет работать с большими объёмами информации
- Streaming responses ускоряют восприятие и снижают время ожидания ответа
- System prompts критичны для определения поведения - хорошо написанная система может улучшить качество на 40%+
- Temperature = 0 используется для логики и анализа, temperature = 0.7-1.0 для творческих задач
- Few-shot prompting (примеры в контексте) часто эффективнее, чем fine-tuning для небольших наборов данных
- Cost optimization: использование claude-3-sonnet вместо opus может снизить расходы в 5 раз при сохранении качества для простых задач
- Tool calling позволяет Claude вызывать внешние функции - критично для создания агентов
- Batch API идеален для обработки большого количества запросов с задержкой (экономит до 50% стоимости)
- Vision capabilities позволяют анализировать изображения в одном запросе с текстом
- Error handling и retry-логика обязательны - API может отвечать 429 (rate limit) или 500 (server error)

## Подводные камни
- API-ключ не должен храниться в коде - использовать environment переменные или .env файлы
- Context window конечен - длинные диалоги требуют управления историей или суммаризации
- Стоимость считается за input и output токены отдельно - output дороже
- Claude не всегда точен в подсчётах и сложной математике - использовать инструменты для вычислений
- Модель может 'галлюцинировать' информацию - всегда проверять критичные данные
- Rate limits зависят от плана подписки - free tier имеет жёсткие ограничения
- Streaming может затруднить обработку полного ответа - нужна буферизация
- Tools/функции должны возвращать JSON - неправильный формат вызовет ошибку
- Многоязычность работает, но на английском качество выше
- Версия модели часто меняется - обновляйте endpoints в коде или используйте алиасы
