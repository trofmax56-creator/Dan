## 1. Название
**AI-Синхрон** — Двусторонний мост 1С↔CRM

## 2. Суть
AI-агент на n8n автоматически синхронизирует данные контрагентов, заказов и остатков между 1С и Bitrix24/amoCRM в режиме реального времени, исключая ручной перенос и двойные записи.

## 3. Бизнес-сценарий
ДО: Менеджер закрыл сделку в amoCRM — бухгалтер сидит и вручную перебивает реквизиты в 1С. «Ты уже внёс ООО Ромашка? А ИНН правильный?» Ошибки, дубли, потерянные часы. ИИ делает: как только в amoCRM сделка переходит в статус «Выиграна» — n8n перехватывает вебхук, Claude проверяет и нормализует реквизиты через Dadata (ИНН, КПП, адрес), создаёт контрагента в 1С через REST API и возвращает номер контрагента обратно в карточку CRM. Клиент получает: бухгалтер открывает 1С — контрагент уже есть, реквизиты выверены, можно выставлять счёт.

## 4. Логика
```
amoCRM Webhook (сделка выиграна) → n8n → Dadata (валидация ИНН/реквизитов) → Claude (нормализация полей) → 1С REST API (создание/обновление контрагента) → amoCRM REST API (запись ID контрагента 1С в сделку)
```

## 6. Связки инструментов
```
amoCRM Webhook → n8n → Dadata API (проверка ИНН) → Claude (маппинг полей) → 1С HTTP-сервис → amoCRM API (обратная запись)
```

## 7. Алгоритм реализации
1. Настроить webhook в amoCRM на событие смены этапа сделки на «Выиграна» (POST на n8n endpoint)
2. Извлечь из payload 8 полей контрагента: название, ИНН, КПП, адрес, телефон, email, ответственный менеджер, сумма сделки
3. Отправить ИНН в Dadata API для обогащения: получить юридический адрес, ОГРН, статус компании, руководителя
4. Передать обогащённые данные в Claude с промптом нормализации: привести поля к формату 1С (длина наименования ≤ 100 символов, адрес в структуру ФИАС)
5. Создать HTTP POST-запрос к 1С REST API (endpoint /hs/crm/v1/counterparties) с нормализованными данными; при дубле по ИНН — выполнить PATCH вместо POST
6. Записать возвращённый ID контрагента 1С в кастомное поле сделки amoCRM через PATCH /api/v4/leads/{id}
7. Отправить уведомление в Telegram ответственному менеджеру: «Контрагент [название] создан в 1С, счёт можно выставлять»

## 8. Почему купят (ROI)
Каждая выигранная сделка = 25–40 минут ручного переноса данных между системами. При 30 сделках в месяц — это 20 часов бухгалтера (≈ 15 000 руб. ФОТ) плюс ошибки реквизитов в каждом третьем счёте, которые задерживают оплату на 3–7 дней. Крючок: «Пока бухгалтер перебивает ИНН — деньги стоят на месте.»
