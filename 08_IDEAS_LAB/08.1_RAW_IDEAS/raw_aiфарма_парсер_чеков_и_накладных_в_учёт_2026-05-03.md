## 1. Название
**AI-Фарма** — Парсер чеков и накладных в учёт

## 2. Суть
Агент мониторит почту и Telegram компании, извлекает данные из чеков, накладных и актов через Claude Vision, категоризирует расходы и автоматически создаёт записи в 1С или МойСклад — без ручного ввода первичной документации.

## 3. Бизнес-сценарий
ДО: Менеджер по закупкам получает 40 накладных в месяц на почту в PDF и фото. Бухгалтер просит прислать все в конце месяца. Итог: хаос в папке «Скачанные», 3 дня ручного ввода в 1С в последних числах, постоянные ошибки в суммах и пропущенные документы. «Где накладная от Иванова за 15-е?» — звонок раз в неделю. ИИ делает: поставщик присылает PDF на почту → n8n перехватывает письмо → Claude Vision читает документ: извлекает поставщика, дату, список позиций с ценами, итоговую сумму, НДС → создаёт приходную накладную в МойСклад через API → бухгалтер видит документ уже разнесённым, проверяет одним кликом. Клиент получает: закрытие месяца за 2 часа вместо 3 дней.

## 4. Логика
```
Gmail/IMAP Webhook (новое письмо с вложением) → n8n → Claude Vision API (парсинг PDF/изображения накладной) → МойСклад REST API (создание приходной накладной) → Telegram Bot (уведомление бухгалтеру о новом документе)
```

## 5. Технический стек
- n8n
- claude-3-5-sonnet
- Gmail API / IMAP
- МойСклад API
- 1С REST API
- Telegram Bot API

## 6. Связки инструментов
```
Gmail IMAP Trigger → n8n → Claude Vision API (извлечение данных из PDF/фото) → МойСклад REST API (приходная накладная) → Telegram уведомление бухгалтеру
```

## 7. Алгоритм реализации
1. Настроить n8n IMAP-триггер на почтовый ящик бухгалтерии: мониторить входящие письма с вложениями .pdf, .jpg, .png каждые 5 минут
2. Извлечь вложение, конвертировать PDF в base64, передать в Claude Vision API с промптом: «Извлеки из документа в JSON: тип документа, поставщик (название+ИНН), дата, список позиций (название/артикул/количество/цена/сумма), итого, НДС, номер документа»
3. Валидировать извлечённый JSON: проверить наличие обязательных полей (поставщик, дата, итого); при неполных данных — отправить документ в Telegram бухгалтеру с пометкой «требует ручной проверки»
4. Поиск контрагента в МойСклад по ИНН через GET /entity/counterparty?filter=inn={inn}; если не найден — создать через POST /entity/counterparty с данными из документа
5. Создать приходную накладную в МойСклад через POST /entity/supply с позициями, суммой, привязкой к контрагенту и загрузкой оригинального PDF как вложения
6. Отправить бухгалтеру Telegram-уведомление: «✅ Накладная №[X] от [поставщик] на сумму [Y] руб. создана в МойСклад. [ссылка]»
7. Настроить еженедельный Schedule-триггер: формировать реестр обработанных документов за неделю и отправлять главному бухгалтеру

## 8. Почему купят (ROI)
Ручной ввод одной накладной — 8–12 минут. При 40 документах в месяц — 7 часов бухгалтера (≈ 5 000 руб. ФОТ) только на ввод, плюс ошибки в каждом пятом документе приводят к расхождениям при сверке. Реальная боль — конец месяца, когда всё валится одновременно. AI-Фарма обрабатывает документ за 40 секунд после получения. Крючок: «Накладная пришла — она уже в учёте. Бухгалтер занимается анализом, а не перепечаткой.»
