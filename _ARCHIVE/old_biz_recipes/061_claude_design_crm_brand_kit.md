### 1. Описание идеи

Claude Design умеет запоминать фирменный стиль компании один раз и дальше генерировать любые материалы — лендинги, КП, презентации, инфографики — автоматически в нужных цветах и шрифтах. Экспорт в HTML, PDF, PPTX, Canva. Для CRM-команды это означает: менеджер описывает сделку текстом, агент выдаёт готовое коммерческое предложение в брендбуке — без дизайнера и без Word-шаблонов.

### 2. Технический стек

- AI: Claude Design (claude.ai/design, под капотом Opus 4.7)
- CRM: Bitrix24 (данные сделки через REST API) / amoCRM
- Middleware: n8n (триггер → сборка данных → запрос к Claude Design API → возврат файла)
- Форматы вывода: HTML → PDF (через Puppeteer), PPTX, Canva

### 3. Пошаговый план внедрения

**Шаг 1 — Настройка Brand Kit в Claude Design**
1. Открыть https://claude.ai/design
2. Загрузить логотип компании + скриншот главной страницы сайта
3. Claude извлекает цвета, шрифты, стиль — сохраняется как «бренд» в сессии
4. Проверить: запросить «создай визитку» — должен применить фирменный стиль автоматически

**Шаг 2 — Шаблоны для типовых КП**
Создать в Claude Design 3 базовых шаблона:
- КП на продукт (1 страница)
- Презентация для ЛПР (5–7 слайдов)
- Инфографика «почему мы» (для рассылки)

Сохранить как HTML-шаблоны с переменными: `{{client_name}}`, `{{product}}`, `{{price}}`, `{{manager_name}}`

**Шаг 3 — Автоматизация через n8n + Bitrix24**
```
Триггер: смена стадии сделки в Bitrix24 → "КП отправить"
  ↓
n8n: GET /crm.deal.get → собрать данные сделки
  ↓
n8n: HTTP Request → Claude API с промптом:
  "Создай КП для {client_name}. Продукт: {product}. Цена: {price} руб.
   Используй фирменный стиль. Формат: HTML."
  ↓
n8n: Puppeteer → HTML → PDF
  ↓
n8n: Bitrix24 API → прикрепить PDF к сделке + отправить клиенту через email/WhatsApp
```

**Шаг 4 — Промпт для генерации КП**
```
Ты — дизайнер компании [НАЗВАНИЕ]. Создай коммерческое предложение.

Клиент: {{client_name}}, компания {{client_company}}
Продукт/услуга: {{product_description}}
Стоимость: {{price}} руб. / {{payment_terms}}
Менеджер: {{manager_name}}, {{manager_phone}}

Требования:
- Фирменный стиль (цвета и шрифты из загруженного брендбука)
- Одна страница A4
- Секции: суть предложения / что получает клиент / цена / следующий шаг / контакты
- Язык: деловой, без клише
- Формат вывода: HTML, готовый к печати
```

### 4. Код интеграции

```python
# n8n Function Node: сборка данных сделки из Bitrix24
deal = items[0]['json']
return [{
    'json': {
        'client_name':    deal.get('CONTACT_NAME', ''),
        'client_company': deal.get('COMPANY_TITLE', ''),
        'product':        deal.get('COMMENTS', ''),
        'price':          deal.get('OPPORTUNITY', ''),
        'manager_name':   deal.get('ASSIGNED_BY_NAME', ''),
        'deal_id':        deal.get('ID', '')
    }
}]
```

```javascript
// n8n HTTP Request body → Claude API
{
  "model": "claude-opus-4-7",
  "max_tokens": 4096,
  "messages": [{
    "role": "user",
    "content": "Создай КП для {{client_name}} ({{client_company}}).\nПродукт: {{product}}\nЦена: {{price}} руб.\n\nВыведи готовый HTML-документ с инлайн CSS, без внешних зависимостей."
  }]
}
```

```bash
# Puppeteer: HTML → PDF (Node.js)
npx puppeteer-cli print kp.html kp.pdf --format A4 --margin-top 15 --margin-bottom 15
```

### 5. Ограничения и нюансы

- Claude Design жрёт ~60–70% недельного Pro-лимита ($20) за один сложный проект. Для серийной генерации нужен Max ($100/мес) или Teams
- Новый токенизатор Opus 4.7: те же промпты могут стоить на 1–35% больше токенов — пересчитайте бюджет
- Память бренда живёт в рамках сессии/проекта — при новой сессии нужно заново загружать Brand Kit (пока нет постоянного хранилища)
- Экспорт в Canva требует ручного шага (Canva API не открыт для автоматического импорта)

### 6. Оценка эффективности

- Сложность: 3/5
- ROI: устраняет 30–60 минут дизайнера на каждое КП; менеджер получает файл автоматически при смене стадии сделки; конверсия в отправленные КП растёт за счёт скорости
