## Суть
Видео демонстрирует использование системных промптов и файлов контекста для Claude, чтобы избежать повторного объяснения одних и тех же инструкций. Показывается, как один файл с четкими инструкциями решает множество однотипных задач без необходимости переполнять каждый запрос подробными объяснениями.

## Бизнес-сценарий
Разработчики, автоматизаторы и специалисты по AI используют эту технику для масштабирования работы с Claude API. Вместо того чтобы писать полные инструкции к каждому запросу, один раз подготавливается файл с контекстом, который загружается ко всем последующим запросам — экономится время и снижается стоимость API.

## Алгоритм реализации
1. Подготовить системный промпт или базовый файл инструкций с полным описанием правил обработки данных
2. Сохранить файл в формате .txt или загрузить его в контекстное хранилище (Files API)
3. При каждом новом запросе добавлять файл в качестве контекста через Files API или встраивать содержимое в system prompt
4. Формировать минимальные user-запросы, ссылаясь только на конкретные данные, которые нужно обработать
5. Claude автоматически применяет правила из файла контекста ко всем поступающим запросам
6. Масштабировать процесс: один файл инструкций × множество задач = постоянная логика без повторений

## Технический стек
- Claude API (Claude 3 Opus / Sonnet / Haiku)
- Files API (для загрузки и переиспользования контекстных файлов)
- System prompts
- Python или другой язык программирования для интеграции
- HTTP-клиент для отправки запросов к API
- Хранилище файлов (локальное или облачное)

## Связки инструментов
- User запрос → Files API → Claude API
- Контекстный файл → System prompt → Обработка задач
- n8n/Make → Files API → Claude → Результат

## Конфигурация и параметры
- Использование Files API endpoint: POST /files для загрузки файла контекста
- Указание file_id в запросе к Claude API при отправке задач
- Параметр system: содержит основные инструкции для обработки
- Параметр messages: содержит только минимальный запрос с конкретными данными
- Поддерживаемые форматы файлов: .txt, .pdf, .json, .csv и другие текстовые форматы
- Максимальный размер файла зависит от плана (обычно до 10-20 MB)
- Параметр model: claude-3-opus-20240229 или claude-3-5-sonnet-20241022

## Ключевые инсайты
- Один раз написанный системный промпт с правилами экономит десятки повторных объяснений и часы разработчика
- Files API позволяет переиспользовать одну инструкцию для сотен запросов без увеличения стоимости на дублирование текста
- Контекстные файлы особенно эффективны для повторяющихся операций: парсинг, трансформация данных, форматирование
- При использовании файлов можно снизить длину индивидуальных промптов на 70-80%, оставляя только переменные данные
- Система работает лучше всего, когда правила четкие и однозначные — Claude точно следует инструкциям из файла
- Файл контекста можно версионировать: v1.0, v1.1 и т.д. для отслеживания изменений логики обработки
- Интеграция с n8n/Make: в webhook → загрузить файл один раз в Files API → во все последующие цепочки автоматизации
- Экономия токенов: если инструкции весят 2000 токенов, и их писать к каждому из 100 запросов — экономия 200k токенов в месяц

## Подводные камни
- Если изменить файл контекста, нужно обновить file_id во всех запросах, иначе будут использоваться старые инструкции
- Files API требует явного указания file_id в каждом запросе — система не запоминает файл автоматически для всех будущих запросов
- Большие файлы (> 5 MB) могут замедлить обработку запроса и увеличить latency
- Если в файле содержатся конфиденциальные данные, они будут видны в контексте всем запросам — нужна осторожность с токенами API
- Claude может игнорировать части инструкций из файла, если они противоречат его базовым принципам — нужна четкая формулировка
- При интеграции с автоматизаторами важно кэшировать file_id, чтобы не загружать один файл 100 раз
- Старые версии Claude API могут не поддерживать Files API — нужна актуальная версия клиента
