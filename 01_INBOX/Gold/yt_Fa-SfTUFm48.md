## Суть
Реализация двусторонней интеграции между Битрикс24 и Диадок (СБИС) для автоматизации электронного документооборота. Система синхронизирует счёта-фактуры, акты выполненных работ и другие документы в обе стороны, исключая ручную работу с документами и ошибки данных.

## Бизнес-сценарий
Компании, ведущие коммерческую деятельность, используют интеграцию для автоматического создания договорных документов в Диадок из заказов в Битрикс24, получение подписанных документов обратно и синхронизации статусов. Обрабатываются счета-фактуры, акты выполненных работ, товаротранспортные накладные.

## Алгоритм реализации
1. 1. Настройка API-ключей и доступа между Битрикс24 и Диадок: генерация токенов в обоих сервисах, проверка прав доступа к организациям-контрагентам
2. 2. Создание вебхука в Битрикс24, который срабатывает при изменении статуса сделки (создание заказа) — вебхук направляет данные сделки в промежуточный сервис обработки
3. 3. Получение данных заказа из CRM: извлечение ID сделки, наименование контрагента, сумма, список товаров/услуг, реквизиты отправителя (компания) и получателя (клиент)
4. 4. Трансформация данных в формат Диадок API: маппинг полей Битрикс24 на структуру документа Диадок (тип документа, реквизиты участников, детали товаров), создание XML/JSON с метаданными документа
5. 5. Отправка счёта-фактуры или акта в Диадок через REST API: вызов метода SendDocument с параметрами RecipientBoxId (ящик получателя), DocumentType, Content, подписание документа электронной подписью отправителя
6. 6. Получение уведомлений от Диадок: вебхук Диадок отправляет уведомления о принятии документа (DeliveryConfirmation), подписи контрагента (DocflowStatusChanged), отправления
7. 7. Обработка входящих уведомлений: парсинг статуса документа, извлечение подписанного документа из Диадок (GetDocument API), сохранение подписанного файла в Битрикс24 (файлы сделки)
8. 8. Обновление статуса в Битрикс24: изменение статуса сделки на 'Подписано', добавление комментария с информацией из Диадок, создание файла счёта-фактуры в области документов сделки
9. 9. Настройка двусторонней синхронизации: обработка входящих документов от контрагентов (их счета, акты), создание противоположных заказов или проверочных документов в Битрикс24
10. 10. Тестирование и мониторинг: проверка логов API, использование вебхук-тестировщика Диадок, настройка уведомлений об ошибках, создание отчётов по статусам документов в CRM

## Технический стек
- Битрикс24 REST API
- Диадок API (REST)
- Вебхуки Битрикс24
- Вебхуки Диадок
- JSON/XML для обмена данными
- cURL или библиотеки HTTP-клиента (Node.js, Python requests)
- Электронная подпись (УЭП) — интеграция с провайдером ЭП
- Система управления промежуточными серверами (можно n8n, Make, Zapier или собственный сервер на Node.js/Python)

## Связки инструментов
- Вебхук Битрикс24 (создание заказа) → HTTP-сервер обработки → Диадок REST API (SendDocument)
- Диадок Вебхук (DeliveryConfirmation, DocflowStatusChanged) → HTTP-сервер → Битрикс24 REST API (обновление сделки, загрузка файла)

## Конфигурация и параметры
- Тип вебхука Битрикс24: ONCRMDEALUPDATE (при изменении сделки)
- Трига Диадок: DocumentReceived, DocumentSigned, DeliveryConfirmed
- API Диадок endpoint: https://api-1.diadoc.ru/v5/SendDocument (основной эндпоинт отправки)
- API Битрикс24 endpoint: https://[ВАША-СУБДОМЕН].bitrix24.ru/rest/1/[REST-КЛЮЧ]/crm.deal.update
- Поле для маппинга: Битрикс24 (UF_COUNTERPARTY) → Диадок (RecipientBoxId)
- Реквизиты отправителя: сохраняются в параметрах интеграции (ИНН, КПП, наименование, ящик Диадок)
- Электронная подпись: используется сертификат компании, хранится на сервере обработки или в УЭП-сервисе
- Формат документа в Диадок: СЧЕТФАКТУРА (TaxInvoice), АКТ (AcceptanceCertificate), ТОВАРТРАНСПОРТНАЯНАЛАДНАЯ (Waybill)
- Маппинг статусов: Битрикс24 'На подпись' → Диадок 'DocumentCreated', 'Подписано' → Диадок 'DocumentSigned и DeliveryConfirmed'
