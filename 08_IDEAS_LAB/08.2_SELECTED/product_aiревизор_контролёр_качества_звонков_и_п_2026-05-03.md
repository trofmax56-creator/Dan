## 1. Название
**AI-Ревизор** — Контролёр качества звонков и переписки

## 2. Суть
Все входящие звонки и переписка в WhatsApp автоматически транскрибируются и оцениваются Claude по чеклисту компании — РОП получает ежедневный отчёт с оценками, цитатами нарушений и рейтингом менеджеров без прослушивания записей вручную.

## 3. Бизнес-сценарий
ДО: РОП Максим раз в неделю садится слушать звонки — успевает проверить 8–10 из 200. Остальные 190 никто не слышит. Менеджер Дима продолжает говорить 'не знаю' вместо 'уточню и перезвоню', обещает скидки без согласования, не проговаривает следующий шаг. Максим узнаёт об этом случайно, когда клиент уже ушёл. ИИ делает: Звонок заканчивается → UIS отправляет запись в n8n → Whisper транскрибирует за 2 минуты → Claude оценивает по 8 критериям чеклиста (приветствие, выявление потребности, презентация, работа с возражениями, следующий шаг, запрещённые фразы, обещания, тон) → каждый критерий получает 0 или 1 балл → итоговая оценка 6/8. Максим в 19:00 получает в Telegram сводку: 'Дима — 3 звонка сегодня, средний балл 4.5/8. Нарушения: в 2 звонках не назначил следующий шаг. Цитата: [текст]. Алина — 5 звонков, средний балл 7.2/8'. Клиент получает: 100% охват звонков, объективная оценка без субъективности РОПа, данные для KPI.

## 4. Логика
```
UIS/МТС Exolve (запись звонка) → n8n Webhook → Whisper API (транскрипция) → Claude (оценка по чеклисту) → Google Sheets (лог) → Telegram (ежедневный отчёт РОПу в 19:00)
```

## 6. Связки инструментов
```
UIS Webhook (запись MP3) → n8n → Whisper API (транскрипция) → Claude API (оценка 8 критериев → JSON) → Google Sheets (append строки) → Cron 19:00 → Claude (агрегация дня) → Telegram Bot (отчёт РОПу)
```

## 7. Алгоритм реализации
1. Подключить UIS или МТС Exolve к n8n через HTTP webhook на событие 'звонок завершён', получить URL записи MP3 и метаданные (менеджер, длительность, номер клиента)
2. Скачать MP3 через HTTP-узел n8n, отправить в Whisper API (или локальный faster-whisper на VPS), получить транскрипт — для звонков до 10 минут время обработки 60–90 секунд
3. Передать транскрипт в Claude с системным промптом: чеклист из 8 критериев (согласовать с клиентом на старте), инструкция вернуть JSON {criteria: [{name, score, quote}], total, summary, red_flags} — поле red_flags заполняется при наличии запрещённых фраз
4. Записать строку в Google Sheets: дата, менеджер, длительность, оценка, ссылка на запись, флаги нарушений — таблица становится базой данных для ежемесячных KPI
5. Cron в 19:00 агрегирует строки за день, Claude формирует читаемый отчёт в формате 'топ-3 менеджера / 3 нарушения дня / 1 лучший момент', отправляет в Telegram РОПу; если у менеджера оценка < 4/8 — немедленный алерт без ожидания вечернего отчёта

## 8. Почему купят (ROI)
ОКК из 2 человек = 160 000 руб./мес ФОТ, при этом они физически успевают проверить 15–20% звонков. AI-Ревизор проверяет 100% за 34 000 руб./мес (разработка + инфраструктура). За первый месяц обычно находят 2–3 системных нарушения, которые стоили потери 10–15% конверсии. Крючок: 'РОП наконец управляет данными, а не своей памятью о звонках.'
