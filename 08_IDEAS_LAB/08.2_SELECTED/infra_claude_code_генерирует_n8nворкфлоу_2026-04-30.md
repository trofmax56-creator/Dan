# 🗂️ Паспорт идеи: Claude Code генерирует n8n-воркфлоу

**Источник:** [Stop Building N8N Automations, Claude Code Builds them INSTANTLY](https://youtube.com/watch?v=0T6q4v8Rb_s) — Ben AI (2026-02-09)
**Категория:** Инфраструктура
**Заменяет:** n8n-разработчик / автоматизатор

---

## 💡 Что это даёт бизнесу
Claude Code по текстовому описанию задачи генерирует готовые JSON-воркфлоу для n8n, которые можно сразу импортировать — без ручного drag-and-drop. Клиент получает автоматизацию в разы быстрее, студия снижает трудозатраты на прототипирование. Повторяющиеся задачи (лидогенерация, уведомления, интеграции CRM) закрываются за часы, а не дни.

---

## ✅ Плюсы
- Скорость: прототип воркфлоу за 5–15 минут вместо 2–8 часов ручной сборки
- Масштабируемость: один промпт → десятки однотипных воркфлоу для разных клиентов (шаблонизация)
- Снижение порога входа: junior-специалист студии закрывает задачи уровня middle

## ❌ Минусы / Риски
- Сгенерированные воркфлоу требуют ручной проверки и тестирования — Claude допускает ошибки в нодах с кастомными API
- Зависимость от качества промптинга: без чёткого ТЗ результат нестабилен, нужна внутренняя библиотека промптов

## 🧰 Артефакты для реализации

**Инструменты:** Claude Code (claude.ai/code), n8n (self-hosted или n8n.cloud), VS Code + Claude extension
**API:** Anthropic Claude API (claude-3-5-sonnet / claude-opus-4), n8n REST API (импорт/экспорт воркфлоу)

**Репозитории:**
- https://github.com/n8n-io/n8n — официальный репо n8n для изучения схем нод

**Документация:**
- https://docs.n8n.io/api/
- https://docs.anthropic.com/en/docs/claude-code

## 🚀 Шаги запуска
1. Шаг 1: Собрать библиотеку из 20–30 типовых задач клиентов (CRM-интеграции, уведомления, парсинг лидов) и написать под каждую системный промпт для Claude Code
2. Шаг 2: Протестировать генерацию воркфлоу на реальных клиентских кейсах, зафиксировать частые ошибки и добавить валидационный чеклист
3. Шаг 3: Внедрить процесс внутри студии — Claude генерирует черновик, junior проверяет по чеклисту, senior принимает финально
4. Шаг 4: Упаковать как внутренний инструмент-акселератор и опционально — как платный мини-курс / шаблонный пакет для продажи клиентам
