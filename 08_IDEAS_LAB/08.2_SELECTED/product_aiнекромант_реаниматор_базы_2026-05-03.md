## 1. Название
**AI-Некромант** — Реаниматор Базы

## 2. Суть
ИИ-агент автоматически проходится по базе «спящих» клиентов (не было касания 60+ дней) в CRM, генерирует персонализированное сообщение для каждого на основе истории сделок и отправляет в WhatsApp дозированными батчами — реанимирует базу без участия менеджера.

## 3. Бизнес-сценарий
ДО: в Битрикс24 лежит 1 200 контактов, у которых последнее касание было 3–12 месяцев назад. РОП знает, что «там деньги», но менеджеры заняты текущими сделками и никто не берётся за реактивацию — «некогда, их слишком много». База гниёт. ИИ делает: n8n ежедневно в 10:00 выгружает из Битрикс24 100 контактов без касания 60+ дней → Claude для каждого генерирует персональное сообщение на основе истории: «Привет, Михаил! Помните, мы делали вам монтаж в ноябре? У нас сейчас акция на обслуживание — хотели бы записаться до конца месяца?» → Wazzup отправляет по 20 сообщений в час (лимит безопасной рассылки) → ответы автоматически создают новые задачи менеджерам. Клиент получает: через месяц 8–15% базы снова в воронке, без единого часа ручной работы менеджера.

## 4. Логика
```
n8n Cron (10:00 ежедневно) → Bitrix24 crm.contact.list (фильтр: последнее касание > 60 дней) → Claude (генерация персонального текста по истории) → Wazzup sendMessage (20 сообщений/час батчинг) → входящий ответ → Bitrix24 crm.activity.add (новая задача менеджеру)
```

## 5. Технический стек
- n8n
- claude-sonnet-4-5

- Bitrix24

## 6. Связки инструментов
```
n8n Cron → Bitrix24 REST API crm.contact.list → Claude API (персонализация) → Wazzup API (батчинг 20/час) → Wazzup Webhook (входящий ответ) → Bitrix24 crm.activity.add
```

## 7. Алгоритм реализации
1. Настроить n8n Cron на 10:00 МСК, запрос к Bitrix24 crm.contact.list с фильтрами: LAST_ACTIVITY_DATE < NOW()-60days, STATUS_ID != 'CONVERTED', лимит 100 записей в день
2. Для каждого контакта запросить историю через crm.activity.list — последние 3 касания, суммы сделок, продукты; передать в Claude как контекст для генерации персонального текста (1–3 предложения)
3. Написать Claude-промпт с запретом на шаблонность: «Напиши живое WhatsApp-сообщение от имени менеджера компании X, учти что клиент покупал Y в дату Z, не используй слово "предложение" и восклицательные знаки»
4. Реализовать батчинг через Split In Batches нода: 20 сообщений в час через Wazzup с задержкой 3 минуты между каждым — защита от блокировки WhatsApp-аккаунта
5. Настроить Wazzup webhook на входящие ответы: автоматически создавать задачу в Битрикс24 на ответственного менеджера с текстом ответа и ссылкой на историю контакта

## 8. Почему купят (ROI)
База из 1 000 «спящих» клиентов при конверсии реактивации 10% и среднем чеке 50 000 руб. = 5 000 000 руб. потенциальной выручки, которая просто лежит в CRM. Менеджеры не реактивируют базу не потому что не хотят — у них нет времени писать 1 000 персональных сообщений. AI-Некромант пишет их за ночь. Крючок: «Ваши лучшие лиды уже заплатили вам однажды».
