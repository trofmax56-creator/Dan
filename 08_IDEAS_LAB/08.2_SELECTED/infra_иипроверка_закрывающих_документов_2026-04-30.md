# 🗂️ Паспорт идеи: ИИ-проверка закрывающих документов

**Источник:** [Как ФНС проверяет бизнес на упрощёнке? Скрывать ЭТО ОПАСНО для предпринимателя!](https://youtube.com/watch?v=XAjk-1aUsJU) — О налогах с Екатериной Арсентьевой (2025-07-23)
**Категория:** Автоматизация отдела
**Заменяет:** Бухгалтер / документальный контролёр

---

## 💡 Что это даёт бизнесу
Система автоматически проверяет входящие акты, счета-фактуры и накладные на соответствие требованиям ФНС и внутренним правилам компании. Снижает риск налоговых претензий из-за ошибок в первичке. Освобождает бухгалтера от рутинной проверки сотен документов вручную.

---

## ✅ Плюсы
- Острая боль: ошибки в первичке — одна из топ-причин доначислений при проверках УСН
- Массовый рынок: каждый МСБ работает с закрывающими документами ежемесячно
- Легко оцифровать ценность: 1 доначисление ФНС = десятки тысяч рублей убытка клиента

## ❌ Минусы / Риски
- Видео не содержит технической базы — идея генерируется по запросу, не по контенту видео
- Требуется интеграция с 1С или buhgalteria.ru, что усложняет внедрение у каждого клиента

## 🧰 Артефакты для реализации

**Инструменты:** n8n, Claude API (claude-3-5-sonnet), Tesseract OCR / pdf2image, 1С-коннектор или API ЭДО (Диадок, СБИС)
**API:** Anthropic Claude API, Диадок API (SKB Kontur), СБИС API, Google Drive API (для хранения скан-копий)

**Репозитории:**
- n8n-io/n8n
- tesseract-ocr/tesseract

## 🚀 Шаги запуска
1. Шаг 1: Собрать типовой чеклист ФНС-требований к первичке (реквизиты, подписи, соответствие договору) — основа промпта
2. Шаг 2: Настроить пайплайн загрузки документов: email / ЭДО / Диск → OCR → структурированный JSON
3. Шаг 3: Написать Claude-промпт для анализа документа по чеклисту + генерации отчёта об ошибках
4. Шаг 4: Подключить уведомления в Bitrix24/amoCRM или Telegram с указанием конкретных нарушений по каждому документу
