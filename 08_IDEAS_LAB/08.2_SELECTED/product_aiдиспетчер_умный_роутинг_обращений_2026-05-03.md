## 1. Название
**AI-Диспетчер** — Умный роутинг обращений

## 2. Суть
ИИ-агент принимает все входящие обращения из WhatsApp, Telegram и форм сайта, определяет тип запроса (продажа / сервис / рекламация / спам) и автоматически маршрутизирует нужному специалисту с готовым контекстом — менеджер получает не сообщение, а задачу.

## 3. Бизнес-сценарий
ДО: В сервисном центре по ремонту оборудования все сообщения валятся в один общий WhatsApp. Менеджер Оля читает каждое: «Добрый день, у меня сломался принтер Canon» — это сервис. «Хочу купить 10 картриджей» — это продажи. «Почему мне не вернули деньги за ремонт» — это рекламация. Оля тратит 2–3 часа в день просто на сортировку и пересылку коллегам. Часть сообщений теряется. ИИ делает: клиент пишет «Привет, у нас перестал работать плоттер HP, нужна срочная диагностика». AI-Диспетчер за 8 секунд определяет: тип — «сервисная заявка», срочность — «высокая» (слово «срочная»), оборудование — «HP плоттер». Создаёт задачу в Bitrix24 на сервисного инженера Алексея, отправляет клиенту: «Приняли! Алексей свяжется в течение 30 минут». Клиент получает: мгновенный ответ и уверенность, что его не потеряли.

## 4. Логика
```
Wazzup / Telegram Webhook → n8n → Claude (классификация типа + срочности + тематики) → Bitrix24 (создание задачи нужному отделу) → автоответ клиенту + уведомление ответственному
```

## 5. Технический стек
- n8n
- claude-haiku
- Wazzup
- Bitrix24
- Telegram Bot API

## 6. Связки инструментов
```
Wazzup Webhook + Telegram Webhook → n8n Router → Claude API (классификация по 4 категориям) → Bitrix24 REST API (задача на отдел) → Wazzup/Telegram (автоответ клиенту)
```

## 7. Алгоритм реализации
1. Настроить единый n8n Webhook для Wazzup (WhatsApp) и Telegram Bot, нормализовать входящие сообщения в единый JSON-формат
2. Передать текст сообщения в Claude Haiku с промптом: определить категорию (продажа / сервис / рекламация / спам / другое), срочность (высокая / средняя / низкая), ключевое оборудование или продукт
3. По результату классификации через n8n Switch создать задачу в Bitrix24 на нужный отдел: ответственный, дедлайн (высокая — 30 мин, средняя — 2 часа, низкая — 24 часа), текст обращения как описание
4. Отправить клиенту автоответ через Wazzup/Telegram: персонализированное подтверждение с именем ответственного и временем ответа согласно срочности
5. Раз в день в 09:00 отправлять руководителю в Telegram статистику: количество обращений по категориям, процент ответов в дедлайн, топ-3 темы дня

## 8. Почему купят (ROI)
Компания из 10 человек теряет в среднем 2–3 обращения в неделю из-за хаоса в одном общем мессенджере. При среднем чеке 15 000 руб. это 30 000–45 000 руб. потерянной выручки ежемесячно. Плюс один менеджер тратит 2 часа в день на ручную сортировку = 40 часов в месяц = 10 000 руб. при ФОТ 50 000. AI-Диспетчер окупается за 1 месяц. Крючок: «Клиент не знает кто ответственный — он просто уходит к конкуренту, который ответил быстрее.»
