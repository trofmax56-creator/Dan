# 🔗 1С УНФ + Bitrix24: синхронизация контрагентов

> Автоматическая передача данных контрагентов из 1С УНФ в Bitrix24 без ручного ввода. ИНН как уникальный идентификатор — нет дублей.

## Боль клиента

- Данные в 1С и Bitrix24 расходятся — реквизиты обновляют в одной системе, забывают в другой
- Менеджеры тратят время на ручной ввод/обновление контрагентов
- При выставлении счёта CRM-менеджер звонит бухгалтеру за реквизитами

## Решение

Интеграция через приложение «Полезные реквизиты контрагентов» из маркетплейса Bitrix24 + REST API:
- Синхронизация по расписанию (раз в час)
- ИНН = уникальный ключ (нет дублей)
- Одностороннее (1С → Bitrix24) или двустороннее

## Технический стек

```
1С УНФ (HTTP-интеграция / REST API 1С)
  → n8n Schedule Trigger (cron каждый час)
  → HTTP Request: выгрузить справочник контрагентов из 1С
  → Code Node: маппинг полей (ИНН, название, телефон, email, реквизиты)
  → HTTP Request: Bitrix24 crm.company.list (поиск по ИНН)
    → EXISTS → crm.company.update
    → NOT EXISTS → crm.company.add
  → Лог: Google Sheets (дата, кол-во обновлённых, ошибки)
```

**Альтернатива:** приложение «Полезные реквизиты контрагентов» из Bitrix24 маркетплейс (no-code, готовое).
