## Суть
Интеграция Битрикс24 и СБИС для синхронизации данных и автоматизации документооборота. Позволяет исключить ручной ввод информации между системами и сократить временные затраты на обработку документов.

## Бизнес-сценарий
Компании, использующие СБИС для документооборота и Битрикс24 для управления клиентами и бизнес-процессами. Задача: синхронизировать контрагентов, счета, акты и другие документы из СБИС в Битрикс24 без ручного переввода, автоматизировать создание сделок и задач на основе данных из СБИС.

## Алгоритм реализации
1. Шаг 1: Подготовка Битрикс24. Перейти в Администратор → Интеграции → Создать новое приложение REST API. Установить разрешения для работы с контрагентами (crm.contact.list, crm.contact.add), сделками (crm.deal.list, crm.deal.add), документами. Получить API-ключ и REST endpoint (обычно https://your-bitrix24.bitrix24.ru/rest/).
2. Шаг 2: Получение доступа к API СБИС. В личном кабинете СБИС сгенерировать AppID и AppSecret для подключения через API. Получить информацию об endpoint СБИС API (https://api.sbis.ru/). Убедиться, что авторизация настроена через OAuth2 или Bearer token.
3. Шаг 3: Создание промежуточного сервиса интеграции (n8n, Make.com или Python-скрипт). Если используется n8n: создать новый workflow, добавить триггер по расписанию (например, каждый час) или webhook для получения новых данных из СБИС.
4. Шаг 4: Синхронизация контрагентов. В интеграции настроить запрос к СБИС API (GET-запрос к endpoint контрагентов) для получения списка. Парсить JSON-ответ: извлечь ИНН, название компании, ФИО контактного лица, телефон, email. Через Битрикс24 REST API создавать контакты (crm.contact.add) или обновлять существующие (crm.contact.update) на основе ИНН как уникального идентификатора.
5. Шаг 5: Синхронизация финансовых документов. Запрашивать из СБИС счета, акты, накладные через API (обычно endpoints вроде /api/v1/documents или /api/v1/invoices). Для каждого документа проверять статус (черновик, отправлен, оплачен). Создавать в Битрикс24 сделку (crm.deal.add) с суммой счёта, датой, привязкой к контрагенту. Сохранять номер документа СБИС в кастомном поле Битрикс24 для отслеживания и избежания дубликатов.
6. Шаг 6: Настройка обработки ошибок и логирования. Добавить try-catch блоки в коде интеграции. Логировать ошибки запросов (неверный API-ключ, таймауты, невалидные данные) с отправкой уведомлений в Telegram или email. Реализовать повторные попытки (retry logic) с экспоненциальной задержкой.
7. Шаг 7: Тестирование и мониторинг. Провести тестовый запуск с небольшим объёмом данных. Проверить корректность маппинга полей между системами. Настроить мониторинг статуса workflow — отслеживать успешные синхронизации и ошибки через логи интеграции.

## Технический стек
- Битрикс24 REST API
- СБИС API (v1)
- n8n (если используется no-code платформа)
- Make.com (альтернатива n8n)
- Python 3.x с библиотеками: requests, json, datetime
- OAuth2 / Bearer Token для аутентификации
- Webhook (для автоматических триггеров)
- JSON парсер
- Telegram Bot API или SMTP (для уведомлений об ошибках)

## Связки инструментов
- Webhook (СБИС API) → n8n/Make.com → Битрикс24 REST API
- СБИС API (GET /contacts) → Парсинг JSON → Битрикс24 CRM (crm.contact.add/update)
- СБИС API (GET /invoices) → Фильтрация по статусу → Битрикс24 CRM (crm.deal.add) → Сохранение ID в кастомное поле
- Trigger (расписание или webhook) → Проверка дубликатов (по ИНН/номеру счёта) → Создание/обновление записей → Логирование результата

## Конфигурация и параметры
- Битрикс24: REST API endpoint формата https://[ваш-портал].bitrix24.ru/rest/[ID_приложения]/[ACCESS_TOKEN]/
- СБИС: AppID и AppSecret генерируются в личном кабинете, используются для получения Bearer token
- Маппинг полей: СБИС field_name (ИНН, ООО_название) → Битрикс24 UF_CRM_field (кастомные поля должны быть созданы предварительно)
- Уникальные идентификаторы: ИНН для контрагентов, номер счёта + дата для документов (для избежания дубликатов)
- Периодичность синхронизации: рекомендуется каждый час или по мере поступления нового документа в СБИС
- Кастомные поля в Битрикс24: создать поля для хранения ID из СБИС, статус синхронизации, дату последней синхронизации
- API rate limits: СБИС обычно ограничивает 100 запросов в минуту, Битрикс24 — зависит от тарифа (15-100 запросов в секунду)
