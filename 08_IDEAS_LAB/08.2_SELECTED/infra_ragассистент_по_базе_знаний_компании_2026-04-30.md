# 🗂️ Паспорт идеи: RAG-ассистент по базе знаний компании

**Источник:** [RAG | САМОЕ ПОНЯТНОЕ ОБЪЯСНЕНИЕ!](https://youtube.com/watch?v=22tkx79icy4) — AI RANEZ (2025-05-11)
**Категория:** Замена должности
**Заменяет:** Оператор поддержки / менеджер по внутренним регламентам

---

## 💡 Что это даёт бизнесу
Сотрудники и клиенты получают мгновенные ответы на основе внутренних документов компании (регламенты, прайсы, договоры, FAQ) без участия человека. Руководитель перестаёт отвечать на одни и те же вопросы — система делает это 24/7. Снижается нагрузка на поддержку на 40–70%, новые сотрудники адаптируются в 2–3 раза быстрее.

---

## ✅ Плюсы
- Массовый спрос: каждая компания с 5+ сотрудниками имеет неструктурированную базу знаний
- Легко упаковать в тиражируемый продукт с шаблонами под отрасли (юристы, медклиники, дилеры)
- Recurring-выручка: поддержка, обновление базы, хостинг векторной БД — каждый месяц

## ❌ Минусы / Риски
- Качество ответов сильно зависит от качества исходных документов клиента — придётся разгребать хаос
- Клиенты в СМБ недооценивают ценность и торгуются: нужна чёткая демонстрация ROI на их данных

## 🧰 Артефакты для реализации

**Инструменты:** n8n (оркестрация пайплайна RAG), Qdrant / Weaviate (векторная база данных), LangChain / LlamaIndex (RAG-фреймворк), Telegram Bot API / Bitrix24 Open Channels (интерфейс)
**API:** OpenAI API (GPT-4o / Embeddings), Anthropic Claude API, YandexGPT API (для клиентов с требованием локализации), Bitrix24 REST API

**Репозитории:**

## 🚀 Шаги запуска
1. Шаг 1: Собрать документы клиента (PDF, DOCX, Google Docs, Notion) → разбить на чанки → сгенерировать эмбеддинги → загрузить в Qdrant
2. Шаг 2: Настроить RAG-пайплайн в n8n или LangChain: запрос пользователя → поиск по векторной БД → формирование контекста → ответ LLM
3. Шаг 3: Подключить интерфейс (Telegram-бот, виджет на сайт или чат Bitrix24) с логированием диалогов
4. Шаг 4: Передать клиенту инструкцию по пополнению базы, настроить алерты на «не знаю» ответы, запустить в прод
