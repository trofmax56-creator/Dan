# 🗂️ Паспорт идеи: ИИ-команда агентов для автоматизации кода

**Источник:** [AI агенты в 2026: всё что работает прямо сейчас (Claude Code, n8n, RAG, OpenClaw, Agent Teams)](https://youtube.com/watch?v=kgSy4NPWZ_4) — Олег Стефанов (2026-03-02)
**Категория:** Замена должности
**Заменяет:** Junior/Middle разработчик, технический аналитик

---

## 💡 Что это даёт бизнесу
Клиент получает автономную команду ИИ-агентов (Claude Code + n8n), которая пишет, тестирует и деплоит код по задачам без участия штатного разработчика. Снижение затрат на разработку на 40–70% за счёт замены рутинных задач агентами. RAG-база знаний позволяет агентам работать в контексте конкретного бизнеса и его процессов.

---

## ✅ Плюсы
- Claude Code уже сейчас закрывает реальные задачи: написание скриптов, интеграции, автотесты — без найма разработчика
- n8n как оркестратор позволяет собрать мультиагентную цепочку без глубокого программирования — Студия умеет с ним работать
- RAG даёт агентам контекст клиента, что резко снижает галлюцинации и повышает качество выхода

## ❌ Минусы / Риски
- Claude Code требует чёткого промпт-инжиниринга и настройки — без этого агент будет делать мусор; высокий порог качества внедрения
- Юридические и репутационные риски: если агент напишет баг в продакшн-код клиента, ответственность ложится на Студию

## 🧰 Артефакты для реализации

**Инструменты:** Claude Code (Anthropic), n8n (оркестрация агентов), Qdrant / Weaviate (векторная БД для RAG), Make.com (альтернатива n8n для простых цепочек)
**API:** Anthropic Claude API (claude-3-5-sonnet / claude-opus-4), OpenAI API (резервная модель), GitHub API (для работы с репозиториями клиента)

**Репозитории:**
- anthropics/claude-code (официальный CLI)
- n8n-io/n8n

**Документация:**
- https://docs.anthropic.com/claude-code
- https://docs.n8n.io/advanced-ai/
- https://qdrant.tech/documentation/

## 🚀 Шаги запуска
1. Шаг 1: Собрать пилот на одном клиенте — настроить Claude Code + n8n для автоматизации 3–5 повторяющихся задач (генерация отчётов, написание интеграций, авто-тесты)
2. Шаг 2: Построить RAG-базу под клиента — загрузить документацию, API-схемы, примеры кода в Qdrant; настроить retrieval-цепочку в n8n
3. Шаг 3: Настроить мультиагентную команду: агент-постановщик задач → агент-разработчик (Claude Code) → агент-тестировщик → агент-деплоер
4. Шаг 4: Упаковать в типовой продукт со SLA, обучить клиента работать с агентами через интерфейс (Telegram-бот или веб-форма), подключить мониторинг ошибок
