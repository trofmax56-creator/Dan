## 1. Название
**AI-Диспетчер** — Умный Роутер Входящих

## 2. Суть
ИИ-агент на n8n перехватывает все входящие сообщения из WhatsApp/Telegram, классифицирует тип обращения (заявка, жалоба, вопрос цены, повторный клиент) и мгновенно маршрутизирует их нужному менеджеру с заполненной карточкой в CRM — без ручной сортировки.

## 3. Бизнес-сценарий
ДО: в WhatsApp компании за ночь накопилось 12 сообщений. Утром менеджер Катя час читает каждое, решает кому переслать, вручную создаёт задачи. Три сообщения теряются в переписке. ИИ делает: клиент пишет «Хочу узнать цену на монтаж» — n8n через Wazzup получает сообщение, Claude за 2 секунды определяет тип «горячая заявка», извлекает контакт и запрос, создаёт сделку в Битрикс24 и отправляет уведомление менеджеру по монтажу: «Новая заявка от Ивана, +7-912-xxx, интересует монтаж, ответить до 10:00». Клиент получает: автоответ «Привет, Иван! Уже передал ваш запрос специалисту, он свяжется в течение 30 минут» — через 8 секунд после сообщения.

## 4. Логика
```
Wazzup Webhook → n8n → Claude (классификация + извлечение данных) → IF/Switch по типу → Bitrix24 REST API (создать сделку) + Wazzup (отправить автоответ) + уведомление менеджеру
```

## 5. Технический стек
- n8n
- claude-sonnet-4-5
- Wazzup
- Bitrix24

## 6. Связки инструментов
```
Wazzup Webhook → n8n Trigger → Claude API (классификация обращения, 5 типов) → Switch-нода → Bitrix24 crm.deal.add + Wazzup sendMessage
```

## 7. Алгоритм реализации
1. Подключить Wazzup к n8n через webhook, настроить приём входящих из WhatsApp и Telegram одним потоком
2. Написать Claude-промпт для классификации: 5 типов обращений (заявка/цена/жалоба/повтор/спам) + извлечение имени, телефона, сути запроса в JSON
3. Создать Switch-ноду на 5 веток: для заявки — crm.deal.add в Битрикс24 + назначить ответственного по матрице (тип продукта → менеджер)
4. Настроить автоответ через Wazzup в течение 15 секунд с персональным обращением по имени
5. Отправлять уведомление в Telegram ответственного менеджера с кратким резюме и ссылкой на сделку в Битрикс24

## 8. Почему купят (ROI)
Менеджер тратит 1,5–2 часа ежедневно на ручную сортировку и распределение входящих — при ставке 60 000 руб./мес это 15 000 руб. в месяц на чистую сортировку. Плюс компания теряет до 20% лидов из-за медленной реакции: если ответ приходит через 2+ часа, конверсия падает в 4 раза. Крючок: «Первый ответ за 15 секунд — это не скорость, это конверсия».
