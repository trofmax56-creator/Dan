## Суть
Автоматизация в n8n для поиска и идентификации высокоценных клиентов (lead generation). Система автоматически анализирует данные потенциальных клиентов и фильтрует их по критериям высокой стоимости для бизнеса.

## Бизнес-сценарий
Lead generation и квалификация лидов: автоматическое выявление и фильтрация потенциальных клиентов с высокой покупательной способностью для B2B компаний, продажных команд и маркетологов.

## Алгоритм реализации
1. Шаг 1: Получение списка потенциальных клиентов через вебхук или интеграцию с CRM (например, из базы контактов, LinkedIn, форм обратной связи)
2. Шаг 2: Нормализация и обогащение данных о клиенте (извлечение компании, должности, бюджета, размера компании, отрасли)
3. Шаг 3: Применение фильтров высокоценности (компании с оборотом >N млн, сотрудники в нужных позициях, релевантная отрасль, бюджет >=N)
4. Шаг 4: Получение дополнительной информации о компании через API (например, через Apollo.io, Hunter.io, или подобные сервисы обогащения данных)
5. Шаг 5: Применение условной логики для определения квалификации лида (MQL, SQL, готовность к контакту)
6. Шаг 6: Отправка квалифицированных лидов в CRM (Bitrix24, amoCRM) или в систему для дальнейшей обработки и контакта

## Технический стек
- n8n (основная платформа автоматизации)
- Webhook (для приёма входящих данных о лидах)
- HTTP Request (для обогащения данных через внешние API)
- API обогащения контактов (Apollo.io, Hunter.io, Clearbit, RocketReach или подобные)
- Условная логика и фильтры (Function Node для JavaScript логики)
- CRM интеграции (Bitrix24 API, amoCRM API, HubSpot API)
- JSON парсинг и трансформация данных
- Переменные окружения для API ключей

## Связки инструментов
- Webhook → n8n (входящие данные о потенциальных клиентах)
- n8n HTTP Request → API обогащения данных (Apollo.io, Hunter.io)
- n8n Function Node → применение фильтров и логики квалификации
- n8n → CRM (Bitrix24/amoCRM API для записи квалифицированных лидов)
- n8n → Уведомление (отправка напоминания менеджеру о новом перспективном клиенте)

## Конфигурация и параметры
- Webhook URL: эндпоинт n8n для приёма POST запросов с данными о контактах
- API ключи: переменные окружения для сервисов обогащения (APOLLO_API_KEY, HUNTER_API_KEY и т.д.)
- Фильтры компаний: минимальный бюджет (>50000 USD), размер компании (>50 сотрудников), релевантные отрасли (SaaS, Tech, Finance и т.д.)
- Поля квалификации: revenue компании, number of employees, industry, job title соответствие, location, company website
- Условия логики: IF (revenue > 1M AND employees > 50 AND industry IN relevant_list) THEN qualify_as_high_value
- Параметры обогащения: запрос к Apollo.io/Hunter.io с email или названием компании, получение полного профиля
- Поля CRM: маппинг полученных данных на поля в целевой CRM (custom fields для хранения score квалификации)
- Сегментация лидов: разделение на категории (Enterprise, Mid-market, SMB) на основе выявленных критериев

## Ключевые инсайты
- Основной хак: использование комбинации данных (компания + должность + размер) для автоматической квалификации вместо ручной проверки каждого лида
- Обогащение данных критично: сырые контакты часто содержат минимум информации, поэтому необходимо использовать APIs для получения полного профиля компании
- Score-based система: присвоение числовых баллов каждому критерию (5 баллов за revenue >5M, 3 балла за размер >100 сотр., 2 балла за релевантную отрасль) и отсеивание по минимальному порогу
- Холодный email не работает без фильтрации: массовый спам неэффективен, нужна точная квалификация перед контактом
- Автоматизация экономит время: вместо ручного скрининга 1000 лидов за неделю система обрабатывает их за минуты
- Данные со временем устаревают: необходимо периодически переверифицировать информацию о компаниях (бюджеты меняются, люди увольняются)
- Интеграция с CRM обязательна: найденные лиды должны сразу попадать в воронку продаж для менеджеров, а не в отдельный список
- API лимиты: большинство сервисов обогащения имеют лимиты запросов (100-1000/месяц на фри плане), нужно оптимизировать количество запросов
