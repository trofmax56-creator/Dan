### 1. Описание идеи

Практический кейс: как внедрить первого AI-агента в компанию на базе n8n. Агент получает входящие заявки (письмо, форма, CRM), анализирует их через Claude/GPT и автоматически выполняет рутинные действия — заполняет CRM, назначает ответственного, отправляет первичный ответ клиенту.

Это «нулевой шаг» — первый агент, который доказывает ROI и открывает бюджет на дальнейшую автоматизацию.

### 2. Технический стек

- n8n (cloud или self-hosted)
- Claude API — `claude-haiku-4-5-20251001` (дёшево, быстро)
- Webhook / Email Trigger (IMAP) / CRM Webhook
- Bitrix24 API / amoCRM API
- Telegram Bot API (уведомления команде)
- Google Sheets (логирование + метрики агента)

### 3. Пошаговый план внедрения

**Шаг 1 — Выбрать ONE процесс для первого агента**
```
Критерии хорошего первого процесса:
✓ Повторяется > 20 раз/день
✓ Занимает 5–15 минут ручного труда
✓ Имеет чёткий входной формат (письмо, форма, сообщение)
✓ Результат можно проверить

Примеры: обработка входящих заявок, квалификация лидов,
         первичный ответ клиенту, распределение задач
```

**Шаг 2 — Схема агента**
```
ТРИГГЕР (Webhook / Email / Schedule)
  → ОБОГАЩЕНИЕ (получить доп. данные из CRM / БД)
  → AI АНАЛИЗ (Claude: классификация, извлечение, решение)
  → ДЕЙСТВИЕ (записать в CRM, отправить письмо, создать задачу)
  → ЛОГИРОВАНИЕ (Google Sheets: дата, вход, выход, статус)
  → УВЕДОМЛЕНИЕ (Telegram: только нестандартные кейсы)
```

**Шаг 3 — Системный промпт (критично)**
```
Ты — менеджер по обработке входящих заявок компании [Название].

Получаешь данные о заявке и должен:
1. Определить тип заявки (консультация / заказ / рекламация / спам)
2. Извлечь: имя клиента, телефон, email, суть запроса, бюджет
3. Оценить приоритет: HIGH (срочно, крупный бюджет) / MEDIUM / LOW
4. Предложить первичный ответ клиенту (2–3 предложения, дружелюбно)

Верни только JSON:
{
  "type": "...",
  "client": {"name": "...", "phone": "...", "email": "..."},
  "priority": "HIGH|MEDIUM|LOW",
  "summary": "...",
  "reply": "..."
}
```

**Шаг 4 — Действие по результату**
```javascript
// Switch нода по priority
HIGH:   → amoCRM создать сделку + задачу менеджеру + Telegram алерт
MEDIUM: → amoCRM создать контакт + поставить в очередь
LOW:    → только Google Sheets лог
SPAM:   → игнорировать

// Отправить reply клиенту:
// Email нода / Telegram нода / HTTP Request → API мессенджера
```

**Шаг 5 — Метрики агента**
```
Google Sheets колонки:
timestamp | input_text | type | priority | client_name | action_taken | processing_time_ms
```
Через неделю видно: точность классификации, % случаев когда нужна ручная правка.

### 4. Сценарии использования

| Входящий канал | Тип заявки | Агент делает |
|---|---|---|
| Email inbox | Запрос на консультацию | Создаёт сделку в CRM + отвечает |
| Форма сайта | Заказ товара | Проверяет наличие + подтверждает |
| Telegram бот | Вопрос о цене | Отвечает из прайса + квалифицирует |
| WhatsApp | Рекламация | Создаёт тикет + уведомляет менеджера |

### 5. Нюансы

- Системный промпт — 80% качества работы агента. Составить совместно с клиентом на реальных примерах заявок
- Логировать ВСЁ: через 2 недели клиент захочет доказательств ROI — данные должны быть
- Не автоматизировать сразу 5 процессов. Один агент → доказали ROI → следующий
- Claude Haiku дешевле GPT-3.5 в 3–5 раз при сопоставимом качестве для извлечения данных

### 6. Оценка эффективности

- Сложность: 2/5
- Время внедрения: 3–5 дней
- ROI: агент экономит 3–5 часов/день рутины. Клиент видит результат за первую неделю
