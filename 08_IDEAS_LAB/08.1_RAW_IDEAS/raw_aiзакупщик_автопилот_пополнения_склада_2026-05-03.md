## 1. Название
**AI-Закупщик** — Автопилот Пополнения Склада

## 2. Суть
Агент мониторит остатки в МоёмСкладе, при достижении порогового уровня по каждой позиции автоматически формирует заявку поставщику по email/WhatsApp с нужным количеством и ценой из последнего заказа — кладовщик перестаёт быть диспетчером между складом и поставщиком.

## 3. Бизнес-сценарий
ДО: Кладовщик Виталий каждое утро открывает МойСклад, смотрит остатки, сравнивает с нормой запаса в Excel, потом вручную пишет поставщику в WhatsApp: «Добрый день! Нам нужно: арматура 12мм — 500 кг, цемент М400 — 20 мешков...» На это уходит 45-60 минут ежедневно. Раз в месяц забывает — и стройка встаёт. ИИ делает: каждое утро в 8:00 агент забирает остатки из МоёСклада, сравнивает с настроенными минимальными порогами, формирует список дефицита, Claude составляет вежливое сообщение поставщику с конкретным перечнем и объёмами, отправляет через WhatsApp или email. Виталий получает: уведомление «Заявка поставщику отправлена, 12 позиций» — и идёт заниматься складом. ИИ получает: подтверждение от поставщика и логирует в МоёмСкладе.

## 4. Логика
```
Cron (ежедневно 08:00) → МойСклад API (остатки < порог) → n8n → Claude (формирование текста заявки) → Wazzup (WhatsApp поставщику) / Email SMTP → МойСклад (создание заказа поставщику) → Telegram (уведомление кладовщику)
```

## 6. Связки инструментов
```
Cron → n8n → МойСклад REST API (GET остатки) → n8n Filter (остаток < min_stock) → Claude (текст заявки) → Wazzup REST API (WhatsApp поставщику) → МойСклад REST API (POST заказ поставщику) → Telegram Bot (отчёт кладовщику)
```

## 7. Алгоритм реализации
1. Подключить МойСклад API в n8n, настроить GET-запрос к /entity/stock с группировкой по складу; создать в МоёмСкладе кастомное поле «Минимальный остаток» для каждой номенклатуры
2. Фильтровать позиции где текущий остаток ≤ поле «Минимальный остаток», для каждой вычислять количество к заказу = (целевой запас 14 дней × средний расход/день) − текущий остаток
3. Передать список дефицита в Claude: сгенерировать текст заявки в формате для конкретного поставщика (обращение по имени, перечень позиций, единицы измерения, цены из последнего заказа)
4. Отправить заявку через Wazzup на WhatsApp поставщика и дублировать на email через SMTP; одновременно создать «Заказ поставщику» в МоёмСкладе со статусом «Ожидает подтверждения»
5. При входящем подтверждении от поставщика в WhatsApp — Claude распознаёт факт согласования, обновляет статус заказа в МоёмСкладе на «Подтверждён» и отправляет итоговый отчёт кладовщику в Telegram

## 8. Почему купят (ROI)
Кладовщик тратит 45 минут ежедневно на мониторинг остатков и переписку с поставщиками = 15 часов в месяц. При ставке 50 000 руб./мес это 9 000 руб./мес чистых потерь только на этой задаче. Но главная боль не в деньгах — один пропущенный заказ у строительной компании = простой бригады на 1 день = 50 000-150 000 руб. потерь. Крючок: «Склад никогда не закончится, если за ним следит тот, кто не забывает.»
