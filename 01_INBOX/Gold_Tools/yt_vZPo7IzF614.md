## Суть
Пошаговое руководство по установке Cursor AI (IDE на базе VS Code с AI-помощником) на Windows 11, включая загрузку инсталлятора, базовую конфигурацию и первичную настройку для начала работы

## Бизнес-сценарий
Разработчики на Windows 11, которые хотят установить и начать использовать Cursor AI — расширенный код-редактор с встроенной AI-помощницей для написания и отладки кода

## Алгоритм реализации
1. Шаг 1: Переход на официальный сайт Cursor (cursor.sh) через браузер
2. Шаг 2: Нажатие кнопки 'Download' на главной странице для скачивания инсталлятора для Windows 11
3. Шаг 3: Ожидание завершения загрузки .exe файла инсталлятора (примерно 200-300 МБ в зависимости от версии)
4. Шаг 4: Запуск скачанного .exe файла двойным кликом или через 'Открыть' из папки загрузок
5. Шаг 5: Принятие лицензионного соглашения (License Agreement) в окне установщика
6. Шаг 6: Выбор директории установки (по умолчанию C:\Users\[имя_пользователя]\AppData\Local\Programs\Cursor или оставить по умолчанию)
7. Шаг 7: Выбор дополнительных опций (создание ярлыка на рабочем столе, добавление в контекстное меню 'Open with Cursor')
8. Шаг 8: Нажатие кнопки 'Install' и ожидание завершения установки (обычно 2-5 минут)
9. Шаг 9: Запуск Cursor AI после завершения установки (автоматическое открытие окна приложения)
10. Шаг 10: Первичная конфигурация — вход в аккаунт (Sign in with GitHub или Email) для активации функций AI
11. Шаг 11: Выбор плана подписки (Free/Pro/Business) и привязка платёжных данных если выбран платный план
12. Шаг 12: Синхронизация настроек VS Code (если уже установлен) через синхронизацию Settings Sync
13. Шаг 13: Установка расширений и тем оформления по предпочтениям
14. Шаг 14: Проверка работоспособности AI-помощника через простую команду (например, Ctrl+K для быстрой генерации кода)

## Конфигурация и параметры
- Официальный сайт: cursor.sh
- Кнопка загрузки: 'Download for Windows'
- Путь установки по умолчанию: %APPDATA%\Local\Programs\Cursor
- Опция контекстного меню: 'Open with Cursor' (рекомендуется включить)
- Файл конфигурации: %APPDATA%\Cursor\User\settings.json
- Директория расширений: %APPDATA%\Cursor\extensions
- Переменная окружения PATH: добавить путь к Cursor.exe для команды 'cursor' в терминале
- API ключ для Pro-плана: хранится в %APPDATA%\Cursor\User\state.vscdb
- Кэш моделей AI: %APPDATA%\Cursor\models
- Логи приложения: %APPDATA%\Cursor\logs

## Ключевые инсайты
- Cursor AI требует подключение к интернету для работы AI-функций, даже если редактор кода работает оффлайн
- При первом запуске автоматически скачиваются модели AI (Claude, GPT-4 и т.д.), что может занять 5-10 минут и 1-2 ГБ дискового пространства
- Если выбран план Pro, первые 500 запросов к Claude 3.5 включены в месячную подписку, затем начисляются токены
- Free-план предоставляет 100 запросов в месяц к AI-помощнику с ограничением размера контекста до 2000 токенов
- Синхронизация с GitHub позволяет автоматически загружать и синхронизировать ваши расширения, настройки и ключи доступа
- Горячие клавиши по умолчанию: Ctrl+K (генерация кода), Ctrl+L (чат с AI), Ctrl+/ (комментарии AI), Cmd+Shift+L (встроить в контекст)
- Настройка proxy требуется для корпоративных сетей — проверить File → Preferences → Settings → HTTP: Proxy
- Если приложение не запускается после установки, очистить %APPDATA%\Cursor\Cache и перезагрузить компьютер
- Для отключения телеметрии: перейти в Settings, найти 'Telemetry' и установить значение 'off'
- Возможна интеграция с локальными моделями через LLM Proxy (например, Ollama) с редактированием файла settings.json

## Подводные камни
- Инсталлятор требует права администратора (UAC-запрос) — без них установка может завершиться с ошибкой
- Windows Defender может выдать предупреждение на .exe файл при первом запуске — это нормально, нужно выбрать 'Run anyway'
- Если на ПК уже установлен VS Code, Cursor может перехватить ассоциацию файлов .code — осторожно при множественных IDE
- Первая загрузка может быть медленной (10-30 секунд) из-за инициализации и загрузки моделей AI
- Если AI не работает, проверить наличие интернета и брандмауэра — Cursor использует порты 443 и 8080
- Обновления Cursor устанавливаются автоматически, могут выключить приложение без предупреждения — сохранять работу регулярно
- При переносе на другой ПК убедиться, что сохранены ключи API и состояние аутентификации из %APPDATA%\Cursor
- Если использовать локальные модели (Ollama), требуется выделить достаточно RAM (минимум 8 ГБ для моделей типа Mistral или Llama 2)
