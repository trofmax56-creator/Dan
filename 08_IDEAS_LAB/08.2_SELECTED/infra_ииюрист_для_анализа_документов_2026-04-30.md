# 🗂️ Паспорт идеи: ИИ-юрист для анализа документов

**Источник:** [Нейросети для юристов. Революция в работе!](https://youtube.com/watch?v=mRfUp1P5SNw) — Дебри Права (Дебри права) (2025-12-16)
**Категория:** Замена должности
**Заменяет:** Помощник юриста / младший юрист

---

## 💡 Что это даёт бизнесу
Система автоматически анализирует договоры, исковые заявления и нормативные акты, выявляет риски и формирует правовые заключения за минуты вместо часов. Малый бизнес получает юридическую проверку документов без штатного юриста или дорогого аутсорса. Снижение расходов на юридическое сопровождение на 40–70% при сохранении качества рутинных проверок.

---

## ✅ Плюсы
- Массовый спрос: каждый МСБ подписывает договоры и боится рисков, боль реальная и дорогостоящая
- Понятный ROI для клиента: один сохранённый судебный спор окупает систему многократно
- Продукт легко упаковать в SaaS или типовое решение под нишу (e-commerce, строительство, аренда)

## ❌ Минусы / Риски
- Юридическая ответственность: ИИ может ошибиться, клиент может предъявить претензии — нужен чёткий дисклеймер и согласование с юристом
- Высокая конкуренция со стороны готовых LegalTech-сервисов (Legium.io, Caselook, ChatGPT-обёртки)

## 🧰 Артефакты для реализации

**Инструменты:** n8n / Make.com — оркестрация пайплайна обработки документов, Claude API (Anthropic) — анализ длинных юридических текстов (200k контекст), Bitrix24 / amoCRM — хранение результатов и уведомления, LangChain — RAG по нормативной базе (ГК РФ, ТК РФ и т.д.)
**API:** Anthropic Claude API (claude-3-5-sonnet), OpenAI GPT-4o API, КонсультантПлюс API (если доступен), Google Drive API / Dropbox API — загрузка документов

**Репозитории:**
- github.com/langchain-ai/langchain
- github.com/run-llama/llama_index

**Документация:**

## 🚀 Шаги запуска
1. Шаг 1: Определить нишу (e-commerce, стройка, аренда) и собрать 30–50 типовых договоров для обучения промптов
2. Шаг 2: Создать RAG-базу из ГК РФ, профильных законов и судебной практики через LlamaIndex; настроить Claude API для анализа рисков по чек-листу
3. Шаг 3: Собрать пайплайн в n8n: загрузка PDF → извлечение текста → анализ ИИ → генерация отчёта в PDF/Notion → уведомление в CRM
4. Шаг 4: Упаковать в готовый продукт с интерфейсом (Telegram-бот или веб-форма), провести пилот с 2–3 клиентами, получить кейс и запустить продажи
