## 1. Название
**AI-Лидогон** — Машина Холодных Продаж

## 2. Суть
n8n-агент ежедневно берёт список контактов из Google Sheets, генерирует через Claude персонализированное письмо под каждого получателя и отправляет через корпоративную почту; при положительном ответе автоматически создаёт сделку в amoCRM и уведомляет менеджера.

## 3. Бизнес-сценарий
ДО: Менеджер по развитию тратит 3 часа в день на написание холодных писем: изучает сайт компании, придумывает тему, пишет текст, вручную копирует в почту. Отправляет 10-15 писем в день. Конверсия 1-2%. ИИ делает: в 8 утра агент берёт 50 контактов из Google Sheets, для каждого Claude пишет письмо: анализирует название компании и сферу, подбирает релевантный кейс из базы, формулирует тему и 4 абзаца тела письма под конкретную боль отрасли. К 8:15 50 персонализированных писем отправлены со скоростью 20 штук в час (чтобы не попасть в спам). ООО «Металлопрофиль» отвечает: «Интересно, можете рассказать подробнее?» — через 2 минуты в amoCRM новая сделка «Металлопрофиль», менеджер получает в Telegram: «Горячий ответ! Позвони до обеда». Клиент получает: 50 касаний в день вместо 15, менеджер звонит только тем, кто ответил.

## 4. Логика
```
Cron (09:00 ежедневно) → Google Sheets (список контактов без статуса 'отправлено') → Claude (генерация письма под сферу компании) → SMTP/Gmail (рассылка 20 писем/час) → Webhook (входящие ответы) → Claude (анализ тональности ответа) → amoCRM (создание сделки при positive) → Telegram (уведомление менеджеру)
```

## 6. Связки инструментов
```
n8n Cron → Google Sheets (фильтр: status=new, лимит 50) → Dadata API (ИНН → сфера деятельности) → Claude API (генерация письма) → Gmail SMTP (отправка с задержкой 180 сек между письмами) → Gmail Webhook (входящие ответы) → Claude (POSITIVE/NEGATIVE/NEUTRAL) → amoCRM /api/v4/leads (при POSITIVE) → Telegram Bot
```

## 7. Алгоритм реализации
1. Создать Cron-триггер в n8n на 09:00 по МСК; забирать из Google Sheets до 50 строк с filter: status = 'new'; после обработки менять статус на 'sent' + timestamp
2. Обогатить каждый контакт через Dadata API: по ИНН получить ОКВЭД, размер компании, регион — передать в Claude как контекст для персонализации
3. Claude генерирует письмо по шаблону: тема (упоминание сферы клиента), 4 абзаца (боль отрасли → наш кейс → конкретный результат в цифрах → CTA), подпись; temperature = 0.8 для уникальности
4. Отправлять через Gmail API с задержкой 180 секунд между письмами (20 писем/час); логировать message_id для отслеживания ответов в Google Sheets
5. Настроить Gmail Webhook на входящие; при получении ответа Claude анализирует тональность, возвращает {sentiment: 'POSITIVE', confidence: 85}; при confidence > 70 и POSITIVE — создать лид в amoCRM + уведомление в Telegram

## 8. Почему купят (ROI)
Менеджер пишет 15 писем в день по 12 минут каждое — 3 часа ежедневно, 60 часов в месяц = 40 000 руб. рабочего времени на механический труд. AI-Лидогон отправляет 50 персонализированных писем за 15 минут. При конверсии холодных писем 2% и среднем чеке 150 тыс. — каждые 50 писем приносят 1 сделку = 150 тыс. руб. потенциальной выручки. Крючок: «Пока конкурент пишет одно письмо, вы отправляете пятьдесят».
