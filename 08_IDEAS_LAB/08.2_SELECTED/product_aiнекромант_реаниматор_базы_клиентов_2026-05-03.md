## 1. Название
**AI-Некромант** — Реаниматор базы клиентов

## 2. Суть
ИИ-агент сегментирует «мёртвую» базу в CRM по давности и поведению, пишет персональные реактивационные сообщения для каждого сегмента и автоматически рассылает их через WhatsApp с отслеживанием ответов и передачей горячих лидов менеджеру.

## 3. Бизнес-сценарий
ДО: РОП говорит: «У нас 3 000 клиентов в amoCRM, которые ничего не купили больше года. Хотим написать им, но вручную — нереально, а шаблонная рассылка спамит и банит номер». ИИ делает: n8n выгружает базу, Claude сегментирует на 4 группы (1–3 мес / 3–6 мес / 6–12 мес / >12 мес), для каждой пишет уникальное сообщение с упоминанием последней покупки: «Александр, вы брали у нас насос в марте — как он работает? У нас сейчас акция на расходники». Рассылка идёт по 20 сообщений в час через Wazzup. Клиент получает: из 3 000 «мёртвых» — 180–300 ответов, 40–60 сделок.

## 4. Логика
```
amoCRM (выгрузка базы >90 дней без активности) → n8n → Claude (сегментация + генерация текста) → Wazzup (рассылка 20 msg/час) → входящий ответ → Claude (классификация: горячий/тёплый/отказ) → amoCRM (создание задачи менеджеру)
```

## 5. Технический стек
- n8n
- claude-sonnet-4-7
- amoCRM REST API
- Wazzup API
- Google Sheets (лог рассылки)

## 6. Связки инструментов
```
amoCRM API (GET contacts, фильтр last_activity > 90 дней) → n8n → Claude (сегментация + персональный текст) → Wazzup API sendMessage (rate: 20/час) → Wazzup Webhook (входящие) → Claude (intent detection) → amoCRM API (POST task для менеджера при статусе 'горячий')
```

## 7. Алгоритм реализации
1. Выгрузить из amoCRM через REST API все контакты с last_activity_date > 90 дней, исключить статусы 'Отказ' и 'Черный список'; батчировать по 100 контактов
2. Для каждого контакта передать в Claude контекст: имя, дата последнего контакта, название последней сделки/продукта — Claude возвращает JSON с полями: segment (1–4), message_text (персональный текст 50–80 слов), priority (1–3)
3. Настроить n8n Rate Limit: отправлять через Wazzup API по 20 сообщений в час (1 сообщение каждые 3 минуты), логировать статус отправки (sent/failed) в Google Sheets
4. Настроить Wazzup Webhook на входящие сообщения: n8n ловит ответы, передаёт в Claude с промптом 'Определи намерение: горячий (хочет купить), тёплый (интересуется), нейтральный (просто ответил), отказ' — возвращает JSON с intent
5. При intent = 'горячий': создать задачу в amoCRM через POST /tasks ('Позвонить в течение 2 часов'), уведомить менеджера в Telegram с текстом переписки; при intent = 'отказ': поставить тег 'Реактивация_отказ' и исключить из будущих рассылок

## 8. Почему купят (ROI)
Стоимость нового лида в МСБ — 800–3 000 руб. Реактивация «мёртвого» клиента через AI-Некромант — 15–40 руб./контакт (стоимость API + рассылки). База в 3 000 контактов: при конверсии 2% = 60 сделок по средней цене 15 000 руб. = 900 000 руб. выручки из «мусорного» раздела CRM. Крючок: «Золото не в новых лидах — оно в CRM, которую вы перестали открывать».
